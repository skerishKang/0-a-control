# Part 2: 보이스 클로닝 심화

## 2.2 보이스 클로닝 모델

본 장에서는 보이스 클로닝 기술의 핵심인 음성 합성 모델에 대해 심층적으로 다룬다. 특히, Zero-shot TTS 모델을 중심으로 모델의 핵심 차별화 포인트, 기술적 차이, 응용 예시, 모델 구조, 성능 비교 및 분석 등을 상세히 설명한다. 또한, 각 모델의 구현 및 학습 과정을 실습을 통해 직접 경험하고, 모델 성능을 평가하고 개선하는 방법을 제시한다.

### 2.2.1 Zero-shot TTS 모델

#### Zero-shot TTS 핵심 기술 및 응용 예시 요약

Zero-shot TTS(Text-to-Speech)는 학습 데이터에 등장하지 않은 새로운 화자의 음성 스타일로 텍스트를 합성하는 기술이다. 전통적인 음성 합성 방법은 특정 화자의 음성 데이터를 사용하여 모델을 학습시키기 때문에, 새로운 화자의 음성을 합성하려면 해당 화자의 데이터를 추가적으로 학습해야 한다. 반면, Zero-shot TTS는 소량의 음성 데이터만으로도 새로운 화자의 음성을 합성할 수 있어, 데이터 부족 문제를 해결하고, 음성 합성 시스템의 유연성을 향상시킬 수 있다.

Zero-shot TTS의 핵심 기술은 다음과 같다.

1. **화자 임베딩 (Speaker Embedding):** 음성 신호에서 화자의 고유한 특징을 추출하여 저차원의 벡터 공간에 표현한다. 화자 임베딩은 음성 합성 모델에 화자 정보를 제공하는 역할을 한다.
   - **기술:**
     - **d-vector:** 음성 신호의 짧은 구간에서 화자의 특징을 추출하여 평균 풀링하는 방식
     - **x-vector:** TDNN(Time Delay Neural Network)과 통계 풀링을 사용하여 긴 발화에서도 안정적인 특징을 추출하는 방식
     - **i-vector:** Joint Factor Analysis(JFA) 기반의 통계적 모델링 방식
     - **Adaptive Speaker Verification (ASV) Embedding:** 사전 학습된 화자 인식 모델로부터 특징을 추출하는 방식
   - **수학적 표현:**
     - **d-vector 예시:**
       $$
       d = \frac{1}{T} \sum_{t=1}^{T} DNN(f_t)
       $$
       여기서 \(d\)는 d-vector, \(T\)는 프레임 수, \(f_t\)는 t번째 프레임의 특징 벡터, DNN은 심층 신경망을 나타낸다.
   - **성능:** 화자 식별 정확도, EER(Equal Error Rate) 등의 지표를 사용하여 평가

2. **조건부 생성 (Conditional Generation):** 화자 임베딩을 조건으로 하여 음성을 생성하는 방법이다. 조건부 생성 모델은 화자 임베딩을 입력으로 받아, 해당 화자의 음성 스타일을 반영한 음성을 생성한다.
   - **기술:**
     - **어텐션 메커니즘:** 인코더(텍스트)와 디코더(음성) 간의 관계를 모델링하여 문맥 정보를 활용한다.
     - **정규화 기법:** 화자 임베딩을 신경망 레이어에 통합하기 위해 조건부 배치 정규화(Conditional Batch Normalization), 적응적 레이어 정규화(Adaptive Layer Normalization) 등을 사용한다.
   - **수학적 표현:**
     - **조건부 생성 모델의 일반적인 형태:**
       $$
       p(y|x, c)
       $$
       여기서 \(y\)는 생성된 음성, \(x\)는 텍스트, \(c\)는 조건 정보(화자 임베딩)를 나타낸다.
   - **성능:** MOS (Mean Opinion Score), ABX test 등의 지표를 사용하여 평가

3. **메타 학습 (Meta Learning):** 다양한 화자의 음성 데이터를 사용하여, 새로운 화자에 대한 적응 능력을 학습하는 방법이다. 메타 학습은 few-shot learning, transfer learning 등 다양한 기법을 사용할 수 있다.
   - **기술:**
     - **Model-Agnostic Meta-Learning (MAML):** 모델의 초기 상태를 학습하여, 새로운 작업에 빠르게 적응할 수 있도록 한다.
     - **Reptile:** MAML과 유사하지만, 계산 효율성을 높인 방법
   - **수학적 표현:**
     - **MAML의 업데이트 규칙:**
       $$
       \theta' = \theta - \alpha \nabla_{\theta} \mathcal{L}_{task_i}(\theta)
       $$
       $$
       \theta \leftarrow \theta - \beta \nabla_{\theta} \sum_{task_i} \mathcal{L}_{task_i}(\theta')
       $$
       여기서 \(\theta\)는 모델 파라미터, \(\alpha\)는 inner loop 학습률, \(\beta\)는 outer loop 학습률, \(\mathcal{L}_{task_i}\)는 i번째 task에 대한 손실 함수를 나타낸다.
   - **성능:** Few-shot adaptation 성능, 새로운 화자에 대한 MOS 등을 사용하여 평가

4. **어텐션 메커니즘 (Attention Mechanism):** 텍스트 정보와 음성 특징 간의 관계를 모델링하는 데 사용되며, Zero-shot TTS 모델에서 문맥 정보를 효과적으로 활용할 수 있도록 도와준다.
   - **기술:**
     - **Self-Attention:** 입력 시퀀스 내의 요소 간의 관계를 모델링
     - **Cross-Attention:** 텍스트 인코딩과 음성 디코딩 간의 관계를 모델링
   - **수학적 표현:**
     - 어텐션 가중치 계산:
       $$
       \alpha_i = \frac{\exp(e_i)}{\sum_{j=1}^{T} \exp(e_j)}
       $$
       $$
       e_i = a(q, k_i)
       $$
       여기서 \(q\)는 query, \(k_i\)는 i번째 key, \(a\)는 어텐션 스코어를 계산하는 함수를 나타낸다.
   - **성능:** 음성 품질, 문장 이해도 등을 사용하여 평가

#### Zero-shot TTS의 응용 예시

1. **개인화된 음성 비서:** 사용자의 음성 스타일을 모방한 음성 비서를 구축할 수 있다. 사용자의 감정 상태에 따라 음성 스타일을 조절할 수 있다.
2. **오디오북 제작:** 다양한 화자의 음성으로 오디오북을 제작할 수 있다. 책의 등장인물에 맞는 음색을 가진 화자를 선택하여 몰입감을 높일 수 있다.
3. **게임 캐릭터 음성:** 게임 캐릭터의 음성을 실시간으로 생성할 수 있다. 캐릭터의 성격, 감정, 상황에 맞는 음성 스타일을 적용할 수 있다.
4. **음성 더빙:** 외국 영화나 드라마의 음성을 실시간으로 더빙할 수 있다. 배우의 입모양과 싱크를 맞추면서도 자연스러운 더빙 음성을 생성할 수 있다.
5. **음성 복원:** 손상된 음성 신호를 복원할 때 화자의 특징을 유지하면서 음성 품질을 향상시킬 수 있다. 과거 인물의 음성을 복원하여 역사적 기록물을 보존할 수 있다.
6. **저작권 보호:** 보이스 클로닝 기술을 사용하여 생성된 음성에 워터마크를 삽입하여 저작권을 보호할 수 있다.

#### Zero-shot 모델 기술적 차이 및 구체적인 사례 간략히 소개

Zero-shot TTS 모델은 다양한 기술적 특징을 가지며, 각 모델은 서로 다른 장단점을 가진다. 대표적인 Zero-shot TTS 모델과 그 특징은 다음과 같다.

1. **Tacotron 2 + WaveGlow:**
   - **구조:** Attention 기반의 sequence-to-sequence 모델인 Tacotron 2와 IAE(Invertible Autoregressive Flow) 기반의 보코더인 WaveGlow를 결합한 모델이다.
   - **기술적 특징:**
     - Tacotron 2는 텍스트에서 멜 스펙트로그램을 생성하고, WaveGlow는 멜 스펙트로그램에서 음성 파형을 생성한다.
     - 화자 임베딩은 Tacotron 2의 인코더 또는 디코더에 적용되어, 생성되는 멜 스펙트로그램의 화자 스타일을 제어한다.
     - WaveGlow는 IAE 기반이므로, 빠른 생성 속도를 제공한다.
   - **장점:** 비교적 간단한 구조로 구현할 수 있으며, 준수한 음성 품질을 제공한다.
   - **단점:** WaveGlow는 IAE 기반 모델이므로, 학습 및 추론 속도가 느리다는 단점을 가진다. 또한 WaveGlow는 음성 신호의 고주파 성분을 정확하게 생성하지 못할 수 있다.
   - **사례:** LJSpeech 데이터셋을 사용하여 학습된 Tacotron 2 + WaveGlow 모델은 자연스러운 음성 품질을 제공하지만, 새로운 화자에 대한 적응력이 부족하다는 평가를 받는다.

2. **Transformer TTS:**
   - **구조:** Self-Attention 기반의 Transformer 네트워크를 사용하여 텍스트에서 음성을 직접 생성하는 모델이다.
   - **기술적 특징:**
     - Transformer는 인코더-디코더 구조를 가지며, self-attention 메커니즘을 사용하여 장거리 의존성을 효과적으로 모델링한다.
     - 화자 임베딩은 Transformer 인코더 또는 디코더에 적용되어, 생성되는 음성의 화자 스타일을 제어한다.
   - **장점:** Transformer의 강력한 표현 능력을 활용하여 고품질의 음성을 생성할 수 있다.
   - **단점:** 모델 크기가 크고, 학습에 많은 데이터가 필요하다. 또한 텍스트와 오디오 간의 길이 차이로 인해 학습이 어려울 수 있다.
   - **사례:** VCTK 데이터셋을 사용하여 학습된 Transformer TTS 모델은 Tacotron 2 + WaveGlow 모델보다 더 자연스럽고 풍부한 음성을 생성하지만, 학습 시간이 더 오래 걸린다는 단점이 있다.

3. **FastSpeech:**
   - **구조:** Transformer 기반의 비 자기 회귀 모델로, 텍스트에서 음성을 매우 빠르게 생성할 수 있다.
   - **기술적 특징:**
     - FastSpeech는 텍스트 인코더, 길이 예측기, 음성 디코더로 구성된다.
     - 길이 예측기는 입력 텍스트 시퀀스의 각 음소에 대한 길이(duration)를 예측하고, 예측된 길이 정보를 사용하여 텍스트 인코더의 출력을 확장한다.
     - 확장된 텍스트 인코더의 출력을 음성 디코더에 입력하여 음성 특징(예: 멜 스펙트로그램)을 병렬적으로 생성한다.
     - 화자 임베딩은 FastSpeech의 Transformer 인코더 또는 디코더에 적용되어, 생성되는 음성의 화자 스타일을 제어한다.
   - **장점:** 빠른 생성 속도를 제공하며, 비교적 작은 모델 크기로도 좋은 성능을 낼 수 있다. 비 자기 회귀 모델이므로, 병렬 처리가 가능하다.
   - **단점:** Transformer TTS에 비해 음성 품질이 다소 떨어질 수 있다. 길이 예측의 정확도가 음성 품질에 큰 영향을 미친다.
   - **사례:** LJSpeech 데이터셋을 사용하여 학습된 FastSpeech 모델은 Transformer TTS 모델보다 훨씬 빠른 속도로 음성을 생성하지만, 자연스러움이나 감정 표현력에서 다소 부족하다는 평가를 받는다.

4. **GAN-TTS:**
   - **구조:** 생성 모델(Generator)과 판별 모델(Discriminator)로 구성된 생성적 적대 신경망 (GAN) 기반 모델이다. 생성 모델은 음성 특징 파라미터 (예: 멜 스펙트로그램)를 입력으로 받아 음성 파형을 생성하고, 판별 모델은 생성된 음성 파형과 실제 음성 파형을 구별한다.
   - **기술적 특징:**
     - GAN은 생성 모델과 판별 모델이 경쟁적으로 학습하는 과정에서 음성 품질을 향상시킨다.
     - WGAN, Spectral Normalization 등 다양한 기법을 사용하여 GAN의 학습 안정성을 높인다.
   - **장점:** 고품질의 음성을 생성할 수 있으며, 특히 음성 신호의 디테일을 살리는 데 강점을 가진다.
   - **단점:** 학습이 불안정하고, 모드 붕괴 문제가 발생할 수 있다.
   - **사례:** VCTK 데이터셋을 사용하여 학습된 GAN-TTS 모델은 매우 자연스럽고 현실감 있는 음성을 생성하지만, 학습 과정이 까다롭고 모델 튜닝에 많은 시간이 소요된다는 단점이 있다.

5. **Diff-TTS:**
   - **구조:** Diffusion 모델을 기반으로 하며, 텍스트 정보를 입력으로 받아 점진적인 디노이징 과정을 통해 음성 신호를 생성한다.
   - **기술적 특징:**
     - Forward diffusion process: 점진적으로 잡음을 추가
     - Reverse diffusion process: 신경망을 통해 잡음을 제거하며 음성을 복원
   - **장점:** 고품질 및 다양한 음성 생성, 학습 안정성이 높음
   - **단점:** 샘플링 과정 느림, 계산 비용 높음
   - **사례:** LJ Speech 데이터셋으로 훈련된 Diff-TTS는 다른 모델에 비해 높은 품질의 음성을 생성하지만, 샘플링 속도가 느려 실시간 응용에는 제약이 있다.

6. **F5-TTS (Flow-based Feature Fusion for Text-to-Speech)**
   - **구조:**
     - 텍스트 인코더 (Text Encoder): 텍스트 시퀀스를 입력으로 받아 hidden representation을 생성한다.
     - 음성 인코더 (Speech Encoder): 음성 신호에서 화자 임베딩을 추출하고, 음성 특징 (예: 멜 스펙트로그램)을 생성한다.
     - Feature Fusion Module: 텍스트 인코더의 출력과 음성 인코더의 출력을 flow-based 모델을 사용하여 융합한다.
     - 디코더 (Decoder): 융합된 정보를 기반으로 음성 신호를 생성한다.
   - **작동 원리:** F5-TTS는 텍스트 정보와 음성 정보를 각각 인코딩하고, flow-based 모델을 사용하여 두 정보 간의 복잡한 관계를 모델링한다. 모델은 텍스트 정보와 음성 정보를 융합하여, 텍스트에 해당하는 음성을 생성하면서도, 음성 스타일을 유지할 수 있다.
   - **수학적 표현:**
     - F5-TTS의 핵심은 flow-based feature fusion에 있으며, 다음과 같이 표현될 수 있다.
       - 텍스트 인코더 출력: \(h_t = \text{Encoder}(x)\)
       - 음성 인코더 출력: \(h_s = \text{Encoder}(s)\)
       - Fusion: \(h_f = \text{Flow}(h_t, h_s)\)
       여기서 \(x\)는 텍스트, \(s\)는 음성, \(h_t\)는 텍스트 인코더의 출력, \(h_s\)는 음성 인코더의 출력, \(h_f\)는 flow-based 모델을 통해 융합된 특징을 나타낸다.
   - **장점:**
     1. **높은 음성 품질:** 텍스트 정보와 음성 정보를 효과적으로 융합하여 자연스럽고 고품질의 음성을 생성할 수 있다.
     2. **뛰어난 화자 적응 능력:** 새로운 화자의 음성을 소량의 데이터만으로도 효과적으로 합성할 수 있다.
     3. **다양한 스타일 표현:** 다양한 스타일의 음성 데이터를 학습하여, 사용자가 원하는 스타일로 음성을 생성할 수 있다.
   - **단점:** 모델 구조가 복잡하고, 학습에 많은 계산 비용이 소요된다.
   - **적용 예시:**
     1. **개인 맞춤형 음성 합성:** 사용자의 음성 스타일을 모방하여 개인화된 음성을 생성하는 데 사용될 수 있다.
     2. **다국어 음성 합성:** 다양한 언어에 대한 음성 합성 기술을 개발하는 데 사용될 수 있다.
     3. **감정 표현 음성 합성:** 감정 정보를 음성 합성에 반영하여 더욱 실감나는 음성을 생성하는 데 사용될 수 있다.

### 2.2.2 딥러닝 기반 보코더 

#### 2.2.2.3 Normalizing Flow 기반 보코더

- **Normalizing Flow 모델의 특징 및 작동 원리**

Normalizing Flow 모델은 복잡한 데이터 분포를 단순한 분포(예: 가우시안 분포)로 변환하는 가역적인(invertible) 변환을 학습하는 생성 모델입니다. 이러한 모델은 데이터의 확률 밀도를 명시적으로 계산할 수 있다는 장점이 있으며, 음성 합성 분야에서 고품질의 음성을 생성하는 데 사용됩니다.

Normalizing Flow 모델은 다음과 같은 특징을 가집니다.

1. **가역적 변환 (Invertible Transformation):** Normalizing Flow 모델은 입력 데이터와 출력 데이터 간의 관계가 가역적이므로, 입력 데이터를 출력 데이터로 변환할 수 있을 뿐만 아니라, 출력 데이터를 다시 입력 데이터로 정확하게 복원할 수 있습니다.
2. **확률 밀도 계산 (Probability Density Calculation):** Normalizing Flow 모델은 데이터의 확률 밀도를 명시적으로 계산할 수 있습니다. 이는 모델이 생성한 데이터의 품질을 평가하고, 모델을 최적화하는 데 유용합니다.
3. **다양한 변환 함수 (Various Transformation Functions):** Normalizing Flow 모델은 다양한 형태의 가역적 변환 함수(예: Affine Coupling Layer, Invertible Convolution)를 사용할 수 있습니다. 이러한 변환 함수를 조합하여 복잡한 데이터 분포를 효과적으로 모델링할 수 있습니다.

Normalizing Flow 모델의 작동 원리는 다음과 같습니다.

1. **기저 분포 선택:** 단순한 형태의 기저 분포(base distribution) \(p_z(z)\)를 선택합니다 (예: 가우시안 분포).
2. **가역적 변환 학습:** 입력 데이터 \(x\)를 기저 분포 \(z\)로 변환하는 가역적 변환 함수 \(f\)를 학습합니다. 이 때, \(f\)는 여러 개의 가역적 레이어로 구성될 수 있습니다.
3. **데이터 생성:** 기저 분포에서 샘플링된 \(z\)를 역변환 함수 \(f^{-1}\)를 사용하여 데이터 영역으로 변환하여 데이터를 생성합니다.

수학적으로 Normalizing Flow 모델은 다음과 같이 표현될 수 있습니다.

$$
x = f^{-1}(z)
$$

여기서 \(x\)는 생성된 데이터, \(z\)는 기저 분포에서 샘플링된 값, \(f^{-1}\)는 역변환 함수를 나타낸다.

데이터의 확률 밀도는 다음과 같이 계산될 수 있습니다.

$$
p_x(x) = p_z(f(x)) \left| \det \frac{\partial f(x)}{\partial x} \right|
$$

여기서 \(p_x(x)\)는 데이터 \(x\)의 확률 밀도, \(p_z(f(x))\)는 변환된 데이터 \(f(x)\)의 기저 분포에서의 확률 밀도, \(\left| \det \frac{\partial f(x)}{\partial x} \right|\)는 변환 함수 \(f\)의 야코비안 행렬(Jacobian matrix)의 절댓값을 나타낸다. 야코비안 행렬은 변환 함수의 복잡도를 나타내는 지표이며, 학습 안정성에 영향을 미칩니다.

- **WaveGlow, Flowavenet 등 모델 구조 상세 분석**

WaveGlow와 Flowavenet은 Normalizing Flow 모델을 기반으로 하는 대표적인 오디오 생성 모델입니다. 이 두 모델은 가역적인 변환을 사용하여 음성 파형을 생성하며, 고품질의 음성을 생성할 수 있다는 장점을 가집니다.

1. **WaveGlow:**
   - WaveGlow는 Glow 모델을 기반으로 하며, Affine Coupling Layer를 사용하여 가역적 변환을 수행합니다. WaveGlow는 멜 스펙트로그램을 입력으로 받아, 음성 파형을 생성합니다.
   - **구조:**
     - 입력: 멜 스펙트로그램
     - Flow 스텝: 여러 개의 Affine Coupling Layer로 구성
     - 출력: 음성 파형
   - **Affine Coupling Layer:**
     - Affine Coupling Layer는 다음과 같이 정의됩니다.
       $$
       y_{1:d} = x_{1:d} \\
       y_{d+1:D} = x_{d+1:D} \odot \exp(s(x_{1:d})) + t(x_{1:d})
       $$
       여기서 \(x\)는 입력, \(y\)는 출력, \(s\)는 스케일 함수, \(t\)는 변환 함수를 나타냅니다. 스케일 함수와 변환 함수는 신경망으로 구현되며, 입력의 일부를 사용하여 나머지 부분을 변환합니다.
   - **장점:** WaveGlow는 계산 효율성이 높고, 빠른 속도로 음성 신호를 생성할 수 있습니다.
   - **단점:** WaveGlow는 학습 데이터에 의존적이며, 학습 데이터에 없는 음성 스타일을 생성하기 어렵다는 단점을 가집니다.

2. **Flowavenet:**
   - Flowavenet은 WaveGlow와 유사하지만, multi-scale architecture를 사용하여 음성 신호의 장기 의존성을 효과적으로 모델링합니다. Flowavenet은 또한 squeeze operation이라는 기법을 사용하여 계산 복잡도를 줄입니다.
   - **구조:**
     - 입력: 멜 스펙트로그램
     - Multi-scale architecture: 여러 개의 Flow 스텝으로 구성, 각 스텝은 서로 다른 스케일에서 특징을 추출
     - Squeeze operation: 채널 수를 늘리고, 시간 해상도를 줄이는 연산
     - 출력: 음성 파형
   - **장점:** Flowavenet은 WaveGlow보다 더 높은 품질의 음성을 생성할 수 있으며, 특히 장기 의존성이 중요한 음성 신호에서 좋은 성능을 보입니다.
   - **단점:** Flowavenet은 WaveGlow보다 모델 크기가 크고, 학습에 더 많은 데이터가 필요합니다.

| 모델        | 특징                                                                 | 장점                                               | 단점                                                                  |
|-------------|----------------------------------------------------------------------|---------------------------------------------------|-----------------------------------------------------------------------|
| WaveGlow    | Affine Coupling Layer 기반, Normalizing Flow 모델                                     | 계산 효율성 높음, 빠른 생성 속도                              | 데이터 의존성, 새로운 음성 스타일 생성 어려움                                                                  |
| Flowavenet  | Multi-Scale Architecture, Squeeze Operation                          | WaveGlow보다 높은 품질, 장기 의존성 모델링                                    | 모델 크기 큼, 학습 데이터 많이 필요                                                       |

- **합성 과정에서 파라미터 역할 및 영향 분석**

WaveGlow와 Flowavenet 모두 학습 과정에서 다양한 파라미터들이 사용되며, 이러한 파라미터들은 모델의 성능에 큰 영향을 미칩니다. 주요 파라미터는 다음과 같습니다.

1. **학습률 (Learning Rate):**
   - 학습률은 모델이 학습 데이터에 얼마나 빠르게 적응하는지를 결정하는 파라미터입니다. 학습률이 너무 크면 학습이 불안정해지고, 너무 작으면 학습 속도가 느려질 수 있습니다.
   - 최적의 학습률은 데이터셋, 모델 구조, optimizer 등에 따라 달라지며, 실험적으로 결정해야 합니다.

2. **배치 크기 (Batch Size):**
   - 배치 크기는 한 번에 학습하는 데이터 샘플의 개수를 의미합니다. 배치 크기가 크면 학습이 안정화되지만, 메모리 사용량이 증가하고, 일반화 성능이 저하될 수 있습니다. 배치 크기가 작으면 학습이 불안정해지지만, 메모리 사용량이 줄어들고, 일반화 성능이 향상될 수 있습니다.

3. **가중치 초기화 (Weight Initialization):**
   - 가중치 초기화는 모델의 초기 가중치 값을 설정하는 방법입니다. 가중치 초기화는 모델의 학습 속도와 최종 성능에 큰 영향을 미치며, Xavier Initialization, He Initialization 등 다양한 방법이 사용될 수 있습니다.

4. **Optimizer:**
   - Adam, RMSprop 등 다양한 최적화 알고리즘이 있으며, 각각 수렴 속도와 안정성 측면에서 장단점이 존재합니다. 적절한 Optimizer 선택은 학습의 효율성을 높이는 데 중요합니다.

5. **Loss Function:**
   - 모델의 학습 목표를 정의하는 손실 함수를 설정합니다. Normalizing Flow 모델에서는 Negative Log-Likelihood(NLL) Loss를 주로 사용하며, 목적에 따라 추가적인 손실 함수를 결합할 수 있습니다.

실험을 통해 각 파라미터가 모델의 성능에 미치는 영향을 분석하고, 최적의 파라미터 조합을 찾아야 합니다.

### 2.2.3 딥러닝 기반 보코더 (계속)

#### GAN (Generative Adversarial Network) 기반 보코더
