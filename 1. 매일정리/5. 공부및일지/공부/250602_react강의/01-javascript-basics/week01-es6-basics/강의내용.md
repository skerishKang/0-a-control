## 🔧 2. 변수 선언: let, const

### 실습 1:  변수 선언 - var vs let/const

### 기본 원칙: 변수 선언이 먼저!

**올바른 순서:**
```javascript
// ✅ 선언 먼저, 사용 나중
var myVar = "값";
console.log(myVar); // "값" 출력

let myLet = "값";  
console.log(myLet); // "값" 출력

const myConst = "값";
console.log(myConst); // "값" 출력
```

**잘못된 순서:**
```javascript
// ❌ 사용 먼저, 선언 나중
console.log(myVar); // undefined (var의 문제점)
var myVar = "값";

console.log(myLet); // ReferenceError! (let/const는 에러)
let myLet = "값";
```

### var의 진짜 문제점

**문제의 핵심: 선언 전 사용해도 에러가 안 남**

```javascript
// var: 잘못된 순서인데도 에러 안 남 (위험!)
console.log(userName); // undefined
var userName = "김철수";
```

**왜 위험한가?**
- 변수 선언을 깜빡했는지, 아니면 다른 곳에 있는지 알기 어려움
- undefined가 나와도 "정상"으로 보임
- 디버깅할 때 문제 찾기 어려움

### let/const의 장점

**올바른 순서 강제: 선언 전 사용하면 명확한 에러**

```javascript
// let/const: 잘못된 순서면 바로 에러 (안전!)
console.log(userName); // ReferenceError!
let userName = "김철수";
```

**장점:**
- "아, 변수를 먼저 선언해야겠구나!" 바로 알 수 있음
- 실수를 즉시 발견 가능
- 코드가 예측 가능해짐

### 추가 문제들

맞습니다! 잘못된 예만 있으면 안 되죠. 올바른 예도 함께 보여드릴게요.

### 추가 문제들

**문제 1: 블록 스코프 무시**

```javascript
// ❌ var: 블록 스코프 무시 (문제!)
if (true) {
    var x = 1;
}
console.log(x); // 1 (블록 밖에서도 접근됨)

// ✅ let/const: 블록 스코프 지킴 (올바름)
if (true) {
    let y = 1;
    const z = 2;
}
console.log(y); // ReferenceError! (블록 밖 접근 불가)
console.log(z); // ReferenceError! (블록 밖 접근 불가)
```

**문제 2: 반복문 클로저 문제**

```javascript
// ❌ var: 모든 setTimeout이 같은 변수 공유 (문제!)
for (var i = 0; i < 3; i++) {
    setTimeout(() => console.log("var:", i), 100); // 3, 3, 3
}

// ✅ let: 각 반복마다 새로운 변수 생성 (올바름)
for (let j = 0; j < 3; j++) {
    setTimeout(() => console.log("let:", j), 100); // 0, 1, 2
}
```

**문제 3: 재선언 허용**

```javascript
// ❌ var: 같은 이름으로 재선언 가능 (문제!)
var userName = "김철수";
var userName = "이영희"; // 오류 안 남
console.log(userName); // "이영희"

// ✅ let/const: 재선언 방지 (올바름)
let userAge = 20;
let userAge = 30; // SyntaxError! (안전함)

const userCity = "서울";
const userCity = "부산"; // SyntaxError! (안전함)
```

**핵심: var는 예상과 다르게 동작하고, let/const는 예상대로 동작합니다!**

### let vs const 구분
- **const**: 값이 안 바뀔 때 (기본 사용)
- **let**: 값이 바뀔 때만

### 결론

**1. 변수는 반드시 선언부터 하자!**
**2. var 대신 let/const를 써서 실수를 방지하자!**

**핵심: "선언 → 사용" 순서를 지키고, 실수했을 때 바로 알 수 있게 let/const를 쓰자!**

## 실습 2: let vs const 올바른 사용법 설명

### const 사용 예시 분석

```javascript
const API_URL = "https://api.example.com";  // 상수
const users = [];  // 배열 자체는 변경 안 함
```

**왜 const를 썼을까?**
- `API_URL`: 서버 주소는 한 번 정하면 바뀌지 않음
- `users`: 배열 변수 자체를 다른 배열로 바꿀 일이 없음

**중요한 포인트:**
```javascript
users.push({ name: "김철수" }); // ✅ 가능!
```
**const는 "변수 자체"를 바꾸는 걸 막는 거지, "내용물"을 바꾸는 건 허용합니다.**

### let 사용 예시 분석

```javascript
let currentUser = null;  // 값이 변경될 예정
let isLoading = true;    // 상태가 바뀜
```

**왜 let을 썼을까?**
- `currentUser`: null → "김철수" → "이영희" 처럼 계속 바뀜
- `isLoading`: true → false 로 상태 변경됨

### 나쁜 예시 설명

```javascript
// ❌ 나쁜 예시
const currentUser = null;
currentUser = "김철수"; // TypeError!
```

**왜 에러날까?** const로 선언한 변수에 새로운 값을 대입하려고 해서!

### 실제 상황 예시

```javascript
// 웹 애플리케이션에서 흔한 패턴
const config = {
    apiUrl: "https://api.example.com",
    timeout: 5000
};

let user = null;           // 로그인 전에는 null
let loginStatus = "로그아웃"; // 상태 변경됨

// 로그인 후
user = { name: "김철수", id: 123 };
loginStatus = "로그인";

// 설정 변경은 안 됨
// config = {}; // TypeError!

// 하지만 설정 내용 변경은 가능
config.timeout = 10000; // ✅ 가능
```

### 백엔드 개발자 관점 설명

```javascript
// 데이터베이스 연결 설정 (한 번 정하면 안 바뀜)
const dbConfig = {
    host: "localhost",
    port: 5432,
    database: "myapp"
};

// 현재 요청 상태 (계속 바뀜)
let requestCount = 0;
let currentRequest = null;

// 요청 처리할 때마다
requestCount++;                    // 숫자 증가
currentRequest = "사용자 정보 조회"; // 요청 내용 변경
```

### 판단 기준 요약

**const 쓸 때:**
- 설정값, 상수
- 배열이나 객체 (내용물 변경은 OK, 통째로 바꾸지 않을 때)
- 함수

**let 쓸 때:**
- 카운터, 상태값
- 반복문 변수
- 조건에 따라 다른 값이 들어갈 변수

**기본 원칙: 일단 const로 시작하고, 값을 바꿔야 할 때만 let으로 변경!**

## ⚡ 3. 화살표 함수

### 실습 3: 함수 선언 방식 비교
```javascript
// 전통적인 함수들
function calculateTotal(items) {
    return items.reduce(function(sum, item) {
        return sum + item.price;
    }, 0);
}

// 화살표 함수로 변환
const calculateTotal = (items) => {
    return items.reduce((sum, item) => sum + item.price, 0);
};

// 더 간결하게
const calculateTotal = items => 
    items.reduce((sum, item) => sum + item.price, 0);
```

### 실습 4: 배열 메서드와 화살표 함수
```javascript
const products = [
    { name: "노트북", price: 1000000, category: "전자제품" },
    { name: "마우스", price: 50000, category: "전자제품" },
    { name: "책", price: 20000, category: "도서" }
];

// 전자제품만 필터링
const electronics = products.filter(product => product.category === "전자제품");

// 가격 10% 할인
const discounted = products.map(product => ({
    ...product,
    price: product.price * 0.9
}));

// 총 가격 계산
const total = products.reduce((sum, product) => sum + product.price, 0);

console.log("전자제품:", electronics);
console.log("할인된 상품들:", discounted);
console.log("총 가격:", total);
```

## 🎯 5. 구조분해할당

### 실습 7: API 응답 데이터 처리
```javascript
// 일반적인 API 응답 구조
const apiResponse = {
    status: 200,
    message: "성공",
    data: {
        user: {
            id: 123,
            profile: {
                name: "박백엔드",
                email: "park@backend.com",
                role: "developer"
            },
            preferences: {
                theme: "dark",
                language: "ko"
            }
        },
        permissions: ["read", "write"]
    }
};

// 기존 방식 (번거로움)
const userId = apiResponse.data.user.id;
const userName = apiResponse.data.user.profile.name;
const userEmail = apiResponse.data.user.profile.email;

// 구조분해할당 (깔끔함)
const {
    data: {
        user: {
            id: userId,
            profile: { name: userName, email: userEmail, role },
            preferences: { theme, language }
        },
        permissions
    }
} = apiResponse;

console.log(`사용자: ${userName} (${role})`);
console.log(`설정: ${theme} 테마, ${language} 언어`);
console.log(`권한: ${permissions.join(", ")}`);
```

## 🧪 6. 종합 실습

### 미니 프로젝트: 사용자 관리 시스템
```javascript
// 사용자 데이터
const users = [
    { id: 1, name: "김철수", email: "kim@test.com", role: "admin", isActive: true },
    { id: 2, name: "이영희", email: "lee@test.com", role: "user", isActive: false },
    { id: 3, name: "박민수", email: "park@test.com", role: "user", isActive: true }
];

// 1. 활성 사용자만 필터링
const activeUsers = users.filter(({ isActive }) => isActive);

// 2. 사용자 정보 포맷팅
const formatUser = ({ name, email, role }) => 
    `${name} (${email}) - ${role}`;

// 3. 활성 사용자 목록 생성
const activeUserList = activeUsers
    .map(formatUser)
    .join('\n');

// 4. 결과 출력
const report = `
=== 활성 사용자 리포트 ===
총 사용자: ${users.length}명
활성 사용자: ${activeUsers.length}명

활성 사용자 목록:
${activeUserList}
`;

console.log(report);
```
