## 2장 요약: 정규 표현식, 텍스트 정규화, 편집 거리

2장은 텍스트 데이터를 처리하기 위한 기본적인 도구와 기술을 소개합니다. 주요 내용은 다음과 같습니다.

**1. 정규 표현식 (Regular Expressions):**

*   텍스트 패턴을 정의하고 검색하는 강력한 도구입니다.
*   특정 문자열, 문자 클래스, 반복, 선택, 앵커 등을 사용하여 복잡한 검색 조건을 표현할 수 있습니다.
*   정규 표현식을 통해 텍스트에서 원하는 정보 (예: 가격, 특정 단어)를 추출하거나 특정 패턴을 가진 문자열을 찾을 수 있습니다.

**2. 텍스트 정규화 (Text Normalization):**

*   텍스트를 일관성 있고 분석하기 쉬운 형태로 변환하는 과정입니다.
*   주요 단계는 다음과 같습니다:

    *   **토큰화 (Tokenization):** 텍스트를 단어, 구두점 등의 의미 있는 단위 (토큰)로 분리합니다. 언어 및 목적에 따라 다양한 토큰화 규칙이 적용될 수 있습니다.
    *   **단어 정규화 (Word Normalization):** 대소문자 통일, 불필요한 문자 제거 등을 통해 단어의 형태를 표준화합니다.
    *   **어간 추출 (Stemming) 및 표제어 추출 (Lemmatization):** 단어의 기본 형태 (어간 또는 표제어)를 찾아 단어의 의미적 유사성을 높입니다.
    *   **문장 분리 (Sentence Segmentation):** 텍스트를 문장 단위로 분리합니다.

**3. 최소 편집 거리 (Minimum Edit Distance):**

*   두 문자열이 얼마나 유사한지를 측정하는 방법입니다.
*   한 문자열을 다른 문자열로 변환하는 데 필요한 최소한의 편집 작업 (삽입, 삭제, 대체) 횟수를 계산합니다.
*   최소 편집 거리 알고리즘은 동적 프로그래밍을 사용하여 효율적으로 계산됩니다.
*   철자 교정, 개체명 일치, 음성 인식 등 다양한 NLP 작업에 활용됩니다.

**핵심 내용:**

*   **정규 표현식:** 패턴 매칭을 위한 강력한 도구
*   **텍스트 정규화:** 텍스트 데이터의 전처리 과정
*   **최소 편집 거리:** 문자열 간 유사성 측정

이 장에서 소개된 기술들은 자연어 처리의 기본적인 구성 요소이며, 다양한 응용 분야에서 활용됩니다.

2장의 핵심 내용 중 상당 부분이 텍스트 전처리에 관한 내용입니다. 

*   **텍스트 정규화** 자체가 텍스트 데이터를 분석 가능한 형태로 만들기 위한 전처리 과정입니다. 토큰화, 단어 정규화, 어간 추출/표제어 추출, 문장 분리 모두 전처리에 포함됩니다.

*   **정규 표현식**은 텍스트 전처리 과정에서 특정 패턴을 찾거나 불필요한 부분을 제거하는 데 사용될 수 있습니다. 예를 들어, HTML 태그 제거, 특수 문자 삭제, 이메일 주소 추출 등에 활용됩니다.

따라서, 2장은 자연어 처리 모델을 구축하기 전에 수행해야 하는 필수적인 전처리 단계들을 설명하고 있다고 할 수 있습니다.

## 3장 요약: N-gram 언어 모델

3장은 자연어 처리에서 핵심적인 개념인 언어 모델(Language Model, LM)과 가장 기본적인 형태인 N-gram 언어 모델을 소개합니다. 주요 내용은 다음과 같습니다.

**1. 언어 모델 (Language Model, LM):**

*   문장 또는 단어 시퀀스에 확률을 할당하는 모델입니다.
*   주어진 단어 시퀀스 다음에 등장할 다음 단어를 예측하는 데 사용될 수 있습니다.
*   언어 모델은 음성 인식, 기계 번역, 철자 교정 등 다양한 NLP 작업에 필수적입니다.

**2. N-gram 언어 모델:**

*   n-1개의 이전 단어를 사용하여 다음 단어를 예측하는 간단한 통계적 언어 모델입니다.
*   n-gram 모델은 Markov 가정을 따르며, 이는 특정 단어의 확률이 이전 n-1개의 단어에만 의존한다는 가정입니다.
*   n-gram 모델의 파라미터는 코퍼스에서 n-gram 빈도를 계산하고 정규화하여 추정됩니다 (MLE, Maximum Likelihood Estimation).

**3. N-gram 모델의 평가:**

*   언어 모델의 성능은 Extrinsic 평가 (특정 작업에서의 성능 향상 측정) 또는 Intrinsic 평가 (Perplexity 측정)를 통해 평가할 수 있습니다.
*   Perplexity는 언어 모델이 테스트 데이터를 얼마나 잘 예측하는지를 나타내는 지표입니다. Perplexity가 낮을수록 모델의 성능이 좋습니다.

**4. 희소 문제와 Smoothing:**

*   n-gram 모델은 코퍼스에 나타나지 않은 n-gram에 대해 0 확률을 할당하는 희소(sparsity) 문제를 겪습니다.
*   Smoothing은 이러한 희소 문제를 해결하기 위해 확률값을 조정하는 기술입니다. 주요 Smoothing 방법은 다음과 같습니다:

    *   Laplace Smoothing (Add-one Smoothing): 모든 n-gram 빈도에 1을 더합니다.
    *   Add-k Smoothing: 모든 n-gram 빈도에 k (0 < k < 1)를 더합니다.
    *   Backoff and Interpolation: 높은 순서의 n-gram이 없는 경우 낮은 순서의 n-gram을 사용합니다.
    *   Kneser-Ney Smoothing: 단어의 "새로운 등장" 확률을 고려하여 보다 정교한 Smoothing을 수행합니다.

**5. 대규모 언어 모델과 Stupid Backoff:**

*   웹 데이터와 같은 방대한 데이터셋을 사용하여 매우 큰 규모의 언어 모델을 구축할 수 있습니다.
*   Stupid Backoff는 실제 확률 분포를 만들려고 시도하지 않고, 하위 n-gram에 고정된 가중치를 적용하는 간단한 backoff 방법입니다.

**6. 고급 주제:**

*   Perplexity와 Entropy의 관계
*   cross-entropy
*   Absolute Discounting

**핵심 내용:**

*   N-gram 모델은 간단하지만 효과적인 언어 모델링 방법입니다.
*   Smoothing은 희소 문제를 해결하고 모델의 일반화 성능을 향상시키는 데 필수적입니다.
*   Perplexity는 언어 모델의 성능을 평가하는 데 사용되는 중요한 지표입니다.

이 장에서는 N-gram 언어 모델의 기본 개념과 다양한 Smoothing 기술을 학습하여, 실제 자연어 처리 문제에 적용할 수 있도록 도와줍니다.

3장에서 다루는 내용은 **모델링**에 해당합니다.

*   **전처리**는 텍스트 데이터를 정제하고 분석 가능한 형태로 만드는 단계입니다. (2장에서 다룬 내용)
*   **모델링**은 전처리된 데이터를 기반으로 특정 작업을 수행하기 위한 모델을 구축하는 단계입니다. 3장에서는 언어 모델이라는 특정 유형의 모델을 만드는 방법을 다룹니다.
*   **평가**는 구축된 모델의 성능을 측정하고 개선하는 단계입니다. 3장에서는 Perplexity를 사용하여 언어 모델을 평가하는 방법을 다룹니다.

따라서, 3장의 N-gram 언어 모델은 전처리된 텍스트 데이터를 사용하여 문장의 확률을 예측하거나 다음 단어를 예측하는 모델을 만드는 데 사용됩니다. 이후 9장에서 뉴럴 네트워크 기반 언어 모델을 배우면서 조금 더 고차원적인 모델링을 학습하게 될 것입니다.

## 4장 요약: 나이브 베이즈와 감성 분석

4장은 텍스트 분류를 위한 기본적인 확률적 모델인 나이브 베이즈 (Naive Bayes) 분류기를 소개하고, 특히 감성 분석(Sentiment Analysis)에 적용하는 방법을 자세히 설명합니다. 

**1. 분류 (Classification):**

*   입력 데이터(텍스트)를 미리 정의된 범주(클래스) 중 하나로 할당하는 작업입니다. 
*   텍스트 분류에는 감성 분석, 스팸 감지, 언어 식별, 저자 판별 등 다양한 유형이 있습니다.

**2. 지도 학습 (Supervised Learning):**

*   입력 데이터와 해당 정답 레이블이 주어졌을 때, 모델이 새로운 입력에 대한 정확한 출력을 예측하는 방법을 학습합니다.
*   나이브 베이즈 분류기는 지도 학습 알고리즘입니다.

**3. 나이브 베이즈 분류기 (Naive Bayes Classifier):**

*   베이즈 정리를 기반으로 하며, 각 특징(feature)이 클래스 레이블에 조건부로 독립적이라는 "나이브" 가정을 합니다.
*   텍스트 분류에 적용하기 위해 텍스트를 "Bag of Words" 모델로 표현합니다 (단어의 순서 무시, 빈도만 고려).
*   나이브 베이즈 모델은 간단하고 효율적이지만, 조건부 독립 가정 때문에 성능에 제한이 있을 수 있습니다.

**4. 나이브 베이즈 훈련 (Training):**

*   훈련 데이터에서 클래스 사전 확률 P(c)와 특징 조건부 확률 P(f|c)을 최대 우도 추정 (MLE) 방식으로 계산합니다.
*   MLE 추정 시 0 확률 문제를 해결하기 위해 Smoothing (예: Add-one Smoothing)을 적용합니다.

**5. 나이브 베이즈 테스트 (Testing):**

*   새로운 문서에 대해 각 클래스의 사후 확률 P(c|d)을 계산하고, 가장 높은 확률을 가진 클래스를 선택합니다.
*   실제 계산 시에는 계산 효율성과 언더플로우 방지를 위해 로그 공간에서 계산합니다.

**6. 감성 분석 (Sentiment Analysis) 최적화:**

*   텍스트 분류의 특성을 고려한 나이브 베이즈 분류기의 성능 개선 방법:
    *   이진 나이브 베이즈 (Binary Naive Bayes): 단어 빈도 대신 존재 여부만 사용합니다.
    *   부정 처리 (Negation Handling): NOT_단어 형태를 추가하여 부정 표현을 고려합니다.
    *   감성 어휘 목록 (Sentiment Lexicons): 미리 정의된 감성 단어 목록을 특징으로 사용합니다.

**7. 평가 (Evaluation):**

*   분류기 성능 평가를 위한 지표: 정밀도 (Precision), 재현율 (Recall), F-측도 (F-measure)
*   다중 클래스 분류에서의 평가 방법: Macro-averaging, Micro-averaging
*   Test sets and Cross-validation: 모델 성능을 평가하기 위한 데이터 분할 전략.
*   Statistical Significance Testing: 두 모델의 성능 차이가 통계적으로 유의미한지 확인하는 방법 (Approximate Randomization, Paired Bootstrap Test).

**8. 분류에서의 Harm 방지:**

*   모델로 인해 사회적 그룹이 폄하되는 것을 방지하기 위한 노력:
    *   표현적 위해 (Representational Harms)

**9.핵심 내용:**

*   나이브 베이즈 분류기는 텍스트 분류를 위한 간단하고 효율적인 모델입니다.
*   Bag of Words 모델은 텍스트를 간략하게 표현하는 데 효과적입니다.
*   Smoothing은 0 확률 문제를 해결하고 모델의 일반화 성능을 향상시키는 데 필수적입니다.
*   정밀도, 재현율, F-측도는 분류기 성능을 평가하는 데 사용되는 중요한 지표입니다.

이 장에서는 나이브 베이즈 분류기의 기본 원리와 감성 분석에서의 활용 방법을 학습하여, 텍스트 데이터 분류 문제를 해결할 수 있도록 도와줍니다. 또한 실제 시스템에서 발생할 수 있는 Harm을 방지하기 위해 노력해야 합니다.

## 5장 요약: 로지스틱 회귀

5장은 로지스틱 회귀 (Logistic Regression)를 소개하며, 텍스트 분류에 적용하는 방법을 자세히 설명합니다. 로지스틱 회귀는 나이브 베이즈 분류기와 함께 많이 사용되는 중요한 분류 모델입니다.

**1. 로지스틱 회귀의 기본 개념:**

*   나이브 베이즈와 달리 데이터가 생성되는 방식에 대한 가정을 하지 않는 **판별 모델(discriminative model)** 입니다. 
*   입력 특징과 클래스 레이블 간의 관계를 직접적으로 학습합니다.
*   주어진 입력에 대해 특정 클래스에 속할 확률을 예측하는 확률적 모델입니다.
*   분류하고자 하는 특정 데이터에 맞춰 모델을 훈련하는 방법을 배웁니다.

**2. 주요 구성 요소:**

*   **특징 표현 (Feature Representation):** 텍스트를 수치 벡터로 표현합니다.
*   **분류 함수 (Classification Function):** 시그모이드 함수 (Sigmoid Function)를 사용하여 확률값을 계산합니다.
*   **목적 함수 (Objective Function):** Cross-entropy loss를 최소화하는 방향으로 모델을 학습합니다.
*   **최적화 알고리즘 (Optimization Algorithm):** Stochastic gradient descent (SGD) 알고리즘을 사용하여 목적 함수를 최적화합니다.

**3. 시그모이드 함수 (Sigmoid Function):**

*   실수 값을 입력받아 0과 1 사이의 확률 값으로 변환하는 함수입니다.
*   로지스틱 회귀 모델의 출력층에서 활성화 함수로 사용됩니다.

**4. Cross-entropy 손실 함수 (Cross-entropy Loss Function):**

*   모델의 예측 확률과 실제 레이블 간의 차이를 측정하는 데 사용됩니다.
*   손실 함수 값을 최소화하는 방향으로 모델을 학습합니다.

**5. 확률적 경사 하강법 (Stochastic Gradient Descent, SGD):**

*   손실 함수를 최소화하기 위해 모델의 파라미터를 반복적으로 업데이트하는 최적화 알고리즘입니다.
*   각 훈련 샘플에 대해 기울기를 계산하고 파라미터를 업데이트합니다.

**6. 규제화 (Regularization):**

*   과적합 (Overfitting)을 방지하기 위해 모델의 복잡도를 제한하는 기술입니다.
*   L1 규제 (Lasso Regression)와 L2 규제 (Ridge Regression)가 있습니다.

**7. 다항 로지스틱 회귀 (Multinomial Logistic Regression):**

*   클래스가 두 개 이상인 경우 (다중 클래스 분류)에 사용됩니다.
*   소프트맥스 함수 (Softmax Function)를 사용하여 각 클래스에 대한 확률을 계산합니다.

**8. 모델 해석 (Interpreting Models):**

*   모델의 해석 가능성은 모델의 의사 결정 과정을 이해하는 데 중요합니다.
*   로지스틱 회귀의 계수를 분석하여 각 특징이 분류 결정에 미치는 영향을 파악할 수 있습니다.

**핵심 내용:**

*   로지스틱 회귀는 텍스트 분류를 위한 강력하고 널리 사용되는 알고리즘입니다.
*   시그모이드 함수, Cross-entropy 손실 함수, SGD 등의 핵심 개념을 이해하는 것이 중요합니다.
*   규제화는 과적합을 방지하고 모델의 일반화 성능을 향상시키는 데 도움이 됩니다.
*   로지스틱 회귀의 계수를 분석하여 모델의 의사 결정 과정을 이해할 수 있습니다.

이 장에서는 로지스틱 회귀의 기본 원리와 다양한 최적화 및 규제 기술을 학습하여, 텍스트 데이터 분류 문제를 효과적으로 해결할 수 있도록 도와줍니다.

## 6장 요약: 벡터 의미론과 임베딩

6장은 단어의 의미를 표현하는 다양한 방법들을 제시하고, 특히 벡터 의미론 (Vector Semantics)과 단어 임베딩 (Word Embeddings)의 개념을 자세히 설명합니다. 

**1. 어휘 의미론 (Lexical Semantics):**

*   단어의 의미를 연구하는 언어학 분야입니다.
*   동의어 (Synonym), 다의어 (Polysemy), 의미장 (Semantic Field)과 같은 어휘 관계를 다룹니다.
*   어휘 의미론적 지식은 단어 간의 의미적 관계를 이해하고, 텍스트의 의미를 해석하는 데 도움이 됩니다.

**2. 벡터 의미론 (Vector Semantics):**

*   단어를 고차원 공간의 벡터 (Vector)로 표현하여 단어 간의 의미적 유사성을 측정하는 방법입니다.
*   분포 가설 (Distributional Hypothesis)에 기반하여, 비슷한 문맥에서 사용되는 단어는 비슷한 의미를 가진다는 아이디어를 활용합니다.
*   대표적인 벡터 의미론 모델은 다음과 같습니다.
    *   **Term-Document Matrix:** 단어-문서 행렬을 사용하여 단어의 의미를 표현합니다.
    *   **Term-Term Matrix (Word-Context Matrix):** 단어-단어 (문맥) 행렬을 사용하여 단어의 의미를 표현합니다.

**3. Term Frequency-Inverse Document Frequency (TF-IDF):**

*   Term-Document Matrix에서 단어의 중요도를 계산하는 데 사용되는 가중치 부여 방식입니다.
*   자주 등장하는 단어 (the, a 등)의 중요도를 낮추고, 특정 문서에만 자주 등장하는 단어의 중요도를 높입니다.

**4. Pointwise Mutual Information (PMI):**

*   Term-Term Matrix에서 단어 간의 연관성을 측정하는 데 사용되는 지표입니다.
*   두 단어가 우연히 함께 나타나는 것보다 얼마나 더 자주 함께 나타나는지를 측정합니다.
*   Positive PMI (PPMI)는 negative 값을 0으로 대체하여 결과를 개선합니다.

**5. 단어 임베딩 (Word Embeddings):**

*   단어를 저차원 (50-1000 차원)의 밀집 벡터 (Dense Vector)로 표현하는 방법입니다.
*   고차원의 희소 벡터 (Sparse Vector)보다 효율적이고, 단어 간의 의미적 관계를 더 잘 포착합니다.
*   대표적인 단어 임베딩 모델은 다음과 같습니다.
    *   **Word2vec (Skip-gram with Negative Sampling):** 주변 단어를 예측하는 작업을 통해 단어 임베딩을 학습합니다.
    *   **Fasttext:** Subword 정보를 활용하여 미등록 단어 (Out-of-Vocabulary, OOV) 문제에 대한 대처 능력을 향상시킵니다.
    *   **GloVe (Global Vectors):** 전역 코퍼스 통계 정보를 활용하여 단어 임베딩을 학습합니다.

**6. 임베딩 시각화 (Visualizing Embeddings):**

*   고차원의 임베딩 벡터를 2차원 또는 3차원으로 차원 축소하여 시각화합니다.
*   t-SNE (t-distributed Stochastic Neighbor Embedding)와 같은 차원 축소 기법이 사용됩니다.

**7. 임베딩의 의미적 속성 (Semantic Properties of Embeddings):**

*   단어 임베딩은 다양한 의미적 속성을 포착할 수 있습니다.
    *   **유사성 (Similarity):** 의미적으로 유사한 단어는 임베딩 공간에서 가깝게 위치합니다.
    *   **유추 (Analogy):** 임베딩 공간에서 벡터 연산을 통해 단어 간의 관계를 추론할 수 있습니다.
    *   **편향 (Bias):** 훈련 데이터에 존재하는 편향이 임베딩에 반영될 수 있으며, 이는 사회적 불평등을 심화시킬 수 있습니다.

**핵심 내용:**

*   벡터 의미론은 단어의 의미를 수치적으로 표현하고, 단어 간의 유사성을 측정하는 강력한 도구입니다.
*   단어 임베딩은 텍스트 데이터를 효과적으로 표현하고, 다양한 NLP 작업의 성능을 향상시키는 데 기여합니다.
*   단어 임베딩은 모델의 편향 문제를 야기할 수 있으며, 이를 해결하기 위한 연구가 진행 중입니다.

이 장에서는 벡터 의미론과 단어 임베딩의 기본 개념과 다양한 모델을 학습하고, 실제 자연어 처리 문제에 적용할 수 있도록 도와줍니다.

6장은 주로 **단어 표현 학습**에 대한 내용을 담고 있으며, 이는 모델 학습의 중요한 부분입니다. 구체적으로는 단어를 벡터 형태로 표현하는 방법론을 배우고, 이렇게 학습된 단어 벡터 (임베딩)가 실제 자연어 처리 모델에 어떻게 적용되는지 설명합니다.

**6장에서 학습한 내용이 모델 학습에 어떻게 적용되는지 구체적으로 설명해 드리겠습니다.**

1.  **모델 입력 (Feature Engineering):**
    *   대부분의 머신러닝 모델은 수치 데이터를 입력으로 받습니다. 따라서 텍스트 데이터를 모델에 넣기 위해서는 적절한 수치 형태로 변환해야 합니다.
    *   6장에서 배운 벡터 의미론과 임베딩은 단어를 수치 벡터로 변환하는 효과적인 방법을 제공합니다. 예를 들어, 텍스트 분류 모델 (감성 분석, 스팸 감지 등)을 만들 때 각 단어를 해당 단어의 임베딩 벡터로 대체할 수 있습니다.

2.  **모델 성능 향상:**
    *   단어 임베딩은 단어 간의 의미적 관계를 포착하므로, 모델이 텍스트의 의미를 더 잘 이해하도록 도와줍니다.
    *   특히, 희소 데이터 (Sparse Data) 환경에서 단어 임베딩은 모델의 일반화 (Generalization) 성능을 향상시키는 데 큰 도움이 됩니다.

3.  **모델 구성 요소:**
    *   단어 임베딩은 모델 자체의 구성 요소로 사용될 수도 있습니다. 예를 들어, 최근에 많이 사용되는 Transformer 기반 모델 (BERT, GPT 등)은 단어 임베딩을 필수적인 구성 요소로 사용합니다.

**예시:**
감성 분석 모델을 구축한다고 가정해 보겠습니다.

*   **과거 (단순 모델):** 나이브 베이즈 분류기를 사용하여 단어 빈도를 특징으로 사용합니다.
*   **현재 (고급 모델):** 각 단어를 Word2vec 또는 GloVe로 학습된 임베딩 벡터로 변환하고, 이 임베딩 벡터들을 평균하거나 연결하여 문장 전체를 표현합니다. 그런 다음 이 문장 벡터를 입력으로 하여 로지스틱 회귀 모델이나 신경망 모델을 학습합니다.

결론적으로, 6장에서 배운 벡터 의미론과 단어 임베딩은 텍스트 데이터를 효과적으로 표현하고, 다양한 자연어 처리 모델의 입력으로 사용하여 모델의 성능을 향상시키는 데 중요한 역할을 합니다. 7장부터는 본격적으로 이 임베딩을 기반으로 조금 더 복잡한 모델링 방법을 학습하게 될 것입니다.

## 7장 요약: 신경망과 신경 언어 모델

7장은 현대 자연어 처리에서 핵심적인 딥러닝 모델의 기반이 되는 신경망 (Neural Networks)의 기본 개념과 구조를 소개하고, 신경망을 활용한 언어 모델링 (Language Modeling)에 대해 자세히 설명합니다.

**1. 신경망의 기본 단위: 유닛 (Unit):**

*   신경망은 작고 간단한 연산 단위인 유닛 (Unit)으로 구성됩니다.
*   각 유닛은 여러 개의 입력을 받아 가중합 (Weighted Sum)을 계산하고, 비선형 활성화 함수 (Activation Function)를 적용하여 결과를 출력합니다.
*   대표적인 활성화 함수: 시그모이드 (Sigmoid), tanh, ReLU (Rectified Linear Unit)

**2. 피드포워드 신경망 (Feedforward Neural Network):**

*   다층 퍼셉트론 (Multi-Layer Perceptron, MLP)이라고도 불립니다.
*   입력층, 은닉층 (Hidden Layer), 출력층으로 구성된 다층 네트워크입니다.
*   각 층의 유닛은 이전 층의 모든 유닛과 연결되어 있습니다 (Fully-Connected).
*   순환 구조 (Cycle)가 없는 단방향 네트워크입니다.

**3. 층 (Layer):**

*   **입력층 (Input Layer):** 외부로부터 입력을 받는 층입니다.
*   **은닉층 (Hidden Layer):** 입력층과 출력층 사이에 위치하며, 비선형 변환을 통해 특징을 추출합니다.
*   **출력층 (Output Layer):** 최종 결과를 출력하는 층입니다.

**4. 모델 학습 (Training):**

*   신경망의 학습은 손실 함수 (Loss Function)를 최소화하는 방향으로 파라미터 (가중치, 편향)를 조정하는 과정입니다.
*   Cross-entropy 손실 함수는 분류 문제에서 널리 사용됩니다.
*   경사 하강법 (Gradient Descent) 알고리즘을 사용하여 손실 함수를 최적화합니다.
*   역전파 알고리즘 (Backpropagation)을 사용하여 각 파라미터에 대한 기울기를 계산합니다.
    *   이때 순방향 계산 그래프와 역방향 계산 그래프를 통해 효과적으로 계산합니다.

**5. 활성화 함수의 역할:**

*   비선형 활성화 함수는 신경망이 비선형적인 관계를 학습할 수 있도록 합니다.
*   ReLU는 경사 소실 (Vanishing Gradient) 문제를 완화하는 데 효과적입니다.

**6. 언어 모델링 (Language Modeling):**

*   신경망을 사용하여 단어 시퀀스의 확률을 예측하거나, 주어진 단어 다음에 등장할 단어를 예측합니다.
*   단어 임베딩 (Word Embedding)은 단어를 저차원 벡터로 표현하여 모델의 성능을 향상시키는 데 사용됩니다.
    *   이때 임베딩 벡터를 처음부터 학습하거나, 사전 훈련된 임베딩 벡터 (Word2vec, GloVe 등)를 사용할 수 있습니다.

**7. 주요 내용:**

*   신경망은 다양한 NLP 문제에 적용 가능한 강력한 모델입니다.
*   피드포워드 신경망은 기본적인 신경망 구조이며, 역전파 알고리즘을 통해 학습할 수 있습니다.
*   단어 임베딩은 신경망의 입력으로 사용되어 모델의 성능을 향상시킬 수 있습니다.
*   합성곱 신경망(CNN), 순환 신경망(RNN)을 사용하여 모델의 성능을 향상시킬 수 있습니다.

이 장에서는 신경망의 기본 개념과 언어 모델링에서의 활용 방법을 학습하여, 딥러닝 기반의 자연어 처리 모델을 구축할 수 있도록 도와줍니다.

## 8장 요약: 품사 태깅과 개체명 인식

8장은 텍스트 시퀀스에 대한 레이블링을 수행하는 기본적인 자연어 처리 작업인 품사 태깅 (Part-of-Speech Tagging)과 개체명 인식 (Named Entity Recognition, NER)을 소개합니다.

**1. 품사 (Parts of Speech, POS):**

*   단어의 문법적인 역할과 기능을 나타내는 범주입니다.
*   명사 (Noun), 동사 (Verb), 형용사 (Adjective), 부사 (Adverb), 전치사 (Preposition) 등이 있습니다.
*   품사 정보는 문장의 구조와 의미를 이해하는 데 중요한 역할을 합니다.

**2. 품사 태깅 (POS Tagging):**

*   텍스트 시퀀스에 있는 각 단어에 적절한 품사 태그를 할당하는 작업입니다.
*   품사 태깅은 구문 분석 (Parsing), 의미 분석 (Semantic Analysis) 등 다양한 NLP 작업의 전처리 단계로 활용됩니다.

**3. 개체명 (Named Entity):**

*   고유한 이름을 가진 개체 (사람, 장소, 조직 등)를 지칭하는 단어 또는 구 (Phrase)입니다.
*   개체명 정보는 텍스트에서 중요한 정보를 추출하고, 관계 추출 (Relation Extraction)과 같은 다운스트림 작업에 활용됩니다.

**4. 개체명 인식 (NER):**

*   텍스트 시퀀스에서 개체명을 식별하고, 해당 유형 (PERSON, LOCATION, ORGANIZATION 등)을 분류하는 작업입니다.
*   Named Entity Recognition (NER)의 어려움
    *   단어 자체만으로는 파악하기 어렵다.
    *   줄임말이나 약어를 사용하기도 한다.

**5. 시퀀스 레이블링 (Sequence Labeling):**

*   입력 시퀀스의 각 요소에 레이블을 할당하는 작업입니다.
*   품사 태깅과 개체명 인식은 대표적인 시퀀스 레이블링 작업입니다.
*   IOB, BIOES 등의 태깅 스키마를 사용하여 개체명의 경계를 표현할 수 있습니다.

**6. Hidden Markov Model (HMM):**

*   생성 모델 (Generative Model) 기반의 시퀀스 레이블링 알고리즘입니다.
*   Markov 가정에 기반하여, 현재 상태는 이전 상태에만 의존한다고 가정합니다.
*   Viterbi 알고리즘을 사용하여 가장 가능성 있는 태그 시퀀스를 찾습니다.

**7. Conditional Random Field (CRF):**

*   판별 모델 (Discriminative Model) 기반의 시퀀스 레이블링 알고리즘입니다.
*   HMM보다 더 유연하게 다양한 특징 (Feature)을 사용할 수 있습니다.
*   Viterbi 알고리즘을 사용하여 가장 가능성 있는 태그 시퀀스를 찾습니다.

**8. 평가:**

*   품사 태깅은 정확도 (Accuracy)를 사용하여 평가합니다.
*   개체명 인식은 정밀도 (Precision), 재현율 (Recall), F1-측도 (F1-measure)를 사용하여 평가합니다.

**9. 핵심 내용:**

*   품사 태깅과 개체명 인식은 텍스트 분석의 핵심적인 요소입니다.
*   HMM과 CRF는 시퀀스 레이블링 작업을 위한 대표적인 알고리즘입니다.
*   모델의 성능을 향상시키기 위해서는 다양한 특징을 활용하고, 적절한 평가 지표를 선택해야 합니다.

이 장에서는 품사 태깅과 개체명 인식의 기본 개념과 다양한 알고리즘을 학습하여, 텍스트 시퀀스 데이터를 분석하고 의미 있는 정보를 추출할 수 있도록 도와줍니다.

8장은 모델링 기법 중에서도 특히 **시퀀스 레이블링 (Sequence Labeling)**이라는 특정 유형의 모델링에 초점을 맞춥니다. 하지만 전처리와도 어느 정도 관련이 있습니다. 

**8장의 내용을 좀 더 자세히 살펴보면:**

*   **주요 내용:** 특정 유형의 모델링 (시퀀스 레이블링) 기법을 배우고, 이를 품사 태깅과 개체명 인식이라는 구체적인 문제에 적용합니다.

*   **전처리와의 연관성:** 시퀀스 레이블링 모델을 학습시키기 위해서는 전처리가 선행되어야 합니다.
    *   **토큰화 (Tokenization):** 텍스트를 단어 단위로 분리해야 각 단어에 품사 태그를 할당할 수 있습니다.
    *   **단어 정규화 (Word Normalization):** 대소문자 통일, 불용어 제거 등을 통해 모델의 성능을 향상시킬 수 있습니다.

*   **모델링과의 연관성:** 6장에서 배운 단어 임베딩은 8장의 모델 (HMM, CRF)에서 특징으로 사용될 수 있습니다. 또한, 7장에서 배운 신경망 모델은 순환 신경망 (RNN)과 같은 형태로 시퀀스 레이블링에 적용될 수 있습니다.

*   **결론적으로, 8장은 전처리된 데이터를 사용하여 시퀀스 데이터의 패턴을 학습하고, 각 요소에 레이블을 할당하는 모델을 구축하는 방법을 배우는 내용입니다.** 이전 장에서 배운 단어 임베딩이나 신경망 모델을 활용하여 더 복잡한 시퀀스 레이블링 모델을 만들 수도 있습니다. 또한 이후 9장에서는 Transformer나 BERT를 활용한 더 강력한 시퀀스 레이블링 모델을 학습하게 될 것입니다.

따라서, 8장은 다시 전처리로 돌아가는 것이 아니라, 모델링 기법을 확장하고 심화하는 과정이라고 볼 수 있습니다.

## 9장 요약: 시퀀스 처리를 위한 딥러닝 아키텍처

9장은 자연어 처리에서 순차적인 정보를 처리하는 데 효과적인 딥러닝 아키텍처인 순환 신경망 (Recurrent Neural Networks, RNNs)과 트랜스포머 (Transformers)를 소개합니다.

**1. 순환 신경망 (RNNs):**

*   **기본 개념:** 이전 시점의 정보를 현재 시점의 입력과 함께 사용하여 순차적인 데이터를 처리하는 신경망입니다. 내부 순환 연결 (Recurrent Connection)을 통해 과거 정보를 기억하고 활용할 수 있습니다.
*   **작동 방식:** 각 시점에서 입력과 이전 은닉 상태를 받아 현재 은닉 상태를 계산하고, 이를 바탕으로 출력을 예측합니다.
*   **장점:** 가변 길이의 시퀀스를 처리할 수 있으며, 과거 정보를 활용하여 문맥을 고려한 예측이 가능합니다.
*   **단점:** 기울기 소실 (Vanishing Gradient) 문제로 인해 장기 의존성을 학습하기 어렵습니다.

**2. Long Short-Term Memory (LSTM):**

*   RNN의 장기 의존성 문제를 해결하기 위해 고안된 구조입니다.
*   **게이트 (Gate)**라는 메커니즘을 사용하여 정보를 선택적으로 기억하고 잊음으로써 장기적인 문맥 정보를 효율적으로 관리합니다.
*   LSTM은 RNN보다 복잡하지만, 더 강력한 성능을 보여줍니다.

**3. Transformers:**

*   RNN의 순차적인 처리 방식에서 벗어나, 모든 입력에 동시에 접근하여 정보를 처리하는 새로운 아키텍처입니다.
*   **Self-Attention:** 핵심적인 구성 요소이며, 입력 시퀀스 내의 단어들 간의 관계를 파악하는 데 사용됩니다.
*   **Positional Encoding:** 단어의 위치 정보를 모델에 제공하기 위해 사용됩니다.
*   **병렬 처리:** RNN과 달리 모든 계산을 병렬로 처리할 수 있어 학습 속도가 빠릅니다.
*   RNN의 장점인 sequential한 처리에 따른 문맥적 정보 유지와 Transformer의 병렬 처리의 장점을 섞어쓰기도 한다.
*   RNN 기반의 모델보다 압도적으로 뛰어난 성능을 보인다.

**4. 다양한 응용:**

*   언어 모델링 (Language Modeling)
*   시퀀스 분류 (Sequence Classification, 감성 분석 등)
*   텍스트 생성 (Text Generation)
*   기계 번역 (Machine Translation)
*   텍스트 요약 (Text Summarization)
     *   Autoregressive generation 방식을 사용.

**5. 핵심 내용:**

*   RNN과 Transformer는 순차적인 데이터를 처리하는 데 효과적인 딥러닝 아키텍처입니다.
*   LSTM은 RNN의 장기 의존성 문제를 해결하기 위한 효과적인 구조입니다.
*   Transformer는 Self-Attention 메커니즘을 통해 장거리 의존성을 효율적으로 포착합니다.
*   RNN과 Transformer는 다양한 자연어 처리 작업에 널리 사용됩니다.

이 장에서는 순환 신경망과 트랜스포머의 기본 개념과 다양한 변형 모델들을 학습하고, 실제 자연어 처리 문제에 적용할 수 있도록 도와줍니다.

## 11장 요약: 기계 번역과 인코더-디코더 모델

11장은 한 언어에서 다른 언어로의 자동 번역을 수행하는 기계 번역 (Machine Translation, MT)의 기본 원리와 핵심 기술인 인코더-디코더 모델 (Encoder-Decoder Model)을 소개합니다.

**1. 기계 번역 (MT):**

*   컴퓨터를 사용하여 한 언어 (Source Language)에서 다른 언어 (Target Language)로 텍스트를 자동 번역하는 기술입니다.
*   정보 접근성 향상, 번역 작업 지원, 실시간 의사소통 등 다양한 분야에서 활용됩니다.

**2. 언어 간의 차이점 (Language Divergences):**

*   번역의 어려움을 야기하는 언어 간의 다양한 차이점을 이해하는 것이 중요합니다.
    *   **구조적 차이 (Word Order):** 어순 (Subject-Verb-Object)의 차이, 전치사/후치사의 사용 차이 등이 있습니다.
    *   **어휘적 차이 (Lexical Divergences):** 단어의 의미 범위가 다르거나, 특정 단어에 해당하는 단어가 없는 경우 (Lexical Gap)가 있습니다.
    *   **형태론적 차이 (Morphological Differences):** 언어의 형태소 구조의 복잡성 정도에 차이가 있습니다 (교착어, 굴절어 등).
    *   **지시대상 밀도(Referential density)** 영어를 사용할 때, 지시 대상과 1번만 명시하면 되는 것을, 계속 명시해야 하는 언어의 특성.

**3. 인코더-디코더 모델 (Encoder-Decoder Model):**

*   입력 시퀀스를 처리하여 고정된 길이의 Context Vector를 생성하는 인코더 (Encoder)와, Context Vector를 기반으로 출력 시퀀스를 생성하는 디코더 (Decoder)로 구성됩니다.
*   RNN 또는 Transformer 등의 다양한 아키텍처를 사용하여 구현할 수 있습니다.

**4. 어텐션 메커니즘 (Attention Mechanism):**

*   디코더가 각 출력 단어를 생성할 때, 인코더의 모든 은닉 상태 (Hidden State)를 참조하여 입력 시퀀스의 어떤 부분에 집중해야 할지를 결정합니다.
*   Context Vector가 모든 정보를 담아야 하는 Bottleneck 문제를 해결하고, 더 정확한 번역을 가능하게 합니다.

**5. 디코딩 (Decoding):**

*   모델이 학습된 후, 새로운 입력에 대한 번역을 생성하는 과정입니다.
*   탐욕적 디코딩 (Greedy Decoding)은 각 단계에서 가장 확률이 높은 단어를 선택하는 방식입니다.
*   빔 서치 (Beam Search)는 각 단계에서 확률이 높은 k개의 후보를 유지하면서 탐색하여 더 나은 번역 결과를 얻도록 합니다.
*   최적의 디코딩을 위한 다양한 방법 (Monte Carlo Search)

**6. 학습 데이터 (Training Data):**

*   인코더-디코더 모델은 병렬 코퍼스 (Parallel Corpus, Bitext)를 사용하여 학습합니다.
*   병렬 코퍼스는 원문 (Source Text)과 번역문 (Target Text)으로 구성된 데이터셋입니다.

**7. 역번역 (Backtranslation):**

*   병렬 코퍼스 데이터가 부족한 경우, 단일 언어 코퍼스 (Monolingual Corpus)를 활용하여 데이터를 증강하는 기법입니다.
*   중간 번역 모델을 사용하여 단일 언어 코퍼스를 번역하고, 이를 학습 데이터로 활용합니다.

**8. 평가 (Evaluation):**

*   기계 번역의 성능은 다음 두 가지 측면에서 평가합니다.
    *   **적절성 (Adequacy):** 번역문이 원문의 의미를 얼마나 정확하게 전달하는가.
    *   **유창성 (Fluency):** 번역문이 얼마나 자연스럽고 문법적으로 올바른가.
*   대표적인 자동 평가 지표: BLEU (Bilingual Evaluation Understudy)
*   BLEU (bilingual evaluation understudy) 는 한 번역이 얼마나 많은 단어와 구를 인간 번역과 공유하는가를 본다.
*   사람이 평가하는게 좋지만, 자동 평가 지표도 많이 사용한다

**9. 핵심 내용:**

*   인코더-디코더 모델은 기계 번역의 핵심 기술이며, 다양한 시퀀스 생성 작업에 활용될 수 있습니다.
*   어텐션 메커니즘은 모델이 입력 시퀀스의 중요한 부분에 집중하도록 하여 성능을 향상시킵니다.
*   빔 서치는 더 나은 번역 결과를 얻기 위해 탐색 공간을 확장하는 데 사용됩니다.

이 장에서는 기계 번역의 기본 원리와 인코더-디코더 모델의 다양한 구성 요소를 학습하고, 실제 자연어 처리 문제에 적용할 수 있도록 도와줍니다.

## 12장 요약: 구성 성분 문법

12장은 문장 구조를 분석하는 데 사용되는 주요 문법 이론인 구성 성분 문법 (Constituency Grammar)을 소개합니다.

**1. 구문적 구성 성분 (Syntactic Constituency):**

*   단어들이 함께 묶여 하나의 단위처럼 행동할 수 있다는 개념입니다.
*   명사구 (Noun Phrase), 동사구 (Verb Phrase) 등 다양한 유형의 구성 성분이 있습니다.
*   구성 성분은 문장 내에서 다른 위치로 이동하거나, 생략되거나, 대치될 수 있습니다.

**2. 문맥 자유 문법 (Context-Free Grammars, CFGs):**

*   문장 구조를 형식적으로 모델링하는 데 사용되는 규칙 기반 시스템입니다.
*   단말 기호 (Terminal Symbol): 단어와 같이 더 이상 분해할 수 없는 기본 요소입니다.
*   비단말 기호 (Non-Terminal Symbol): 구, 절 등과 같이 다른 규칙으로 확장될 수 있는 요소입니다.
*   규칙 (Rule): 비단말 기호가 어떤 단말 기호 또는 다른 비단말 기호로 구성될 수 있는지를 정의합니다.
*   시작 기호 (Start Symbol): 문장의 시작을 나타내는 특별한 비단말 기호입니다.

**3. 파싱 (Parsing):**

*   CFG를 사용하여 문장의 구문 구조를 분석하고, Parse Tree를 생성하는 과정입니다.
*   Parse Tree는 문장의 구성 성분과 그들의 계층적 관계를 보여줍니다.

**4. 주요 개념:**

*   **Dominate:** 상위 노드가 하위 노드를 포함한다.
*   **프리포즈 (Preposed):** 문장 앞으로 이동.
*   **포스트포즈 (Postposed):** 문장 뒤로 이동.

**5. 트리뱅크 (Treebanks):**

*   문장과 해당 구문 분석 트리가 함께 저장된 코퍼스입니다.
*    Penn Treebank는 영어 구문 분석 연구에 널리 사용되는 데이터셋입니다.
*   구문 분석 모델 학습 및 평가에 사용됩니다.

**6. 어휘화 문법 (Lexicalized Grammars):**

*   구 구조 규칙뿐만 아니라, 어휘 (Lexicon) 정보도 함께 활용하는 문법입니다.
*   Head-Driven Phrase Structure Grammar (HPSG), Combinatory Categorial Grammar (CCG) 등이 있습니다.

**7. Chomsky Normal Form (CNF):**

*   모든 CFG를 변환할 수 있는 표준 형식입니다.
*   이진 분기 구조를 가지므로 파싱 알고리즘의 효율성을 높입니다.
*   "모든 규칙의 오른쪽에 두 개의 비단말 기호 또는 하나의 단말 기호만 존재한다." 라는 제약 조건을 갖는다.

**핵심 내용:**

*   구성 성분 문법은 문장 구조를 형식적으로 표현하고 분석하는 데 사용됩니다.
*   CFG는 자연어 구문 분석의 기본 모델이며, 다양한 파싱 알고리즘의 기반이 됩니다.
*   트리뱅크는 구문 분석 모델 학습 및 평가를 위한 중요한 데이터 소스입니다.

이 장에서는 구문 분석의 기본 원리와 다양한 문법 모델을 학습하여, 문장 구조를 이해하고 자동 구문 분석 시스템을 개발할 수 있도록 도와줍니다.

## 13장 요약: 구성 성분 파싱

13장은 텍스트에 구문 구조를 할당하는 작업인 구성 성분 파싱(Constituency Parsing)을 다룹니다. 문맥 자유 문법(CFG)을 기반으로 한 구문 분석 알고리즘을 소개하고, 평가 방법 및 최근 딥러닝 기반 파싱 모델을 살펴봅니다.

**1. 구문 분석 (Syntactic Parsing):**

*   문장의 구문 구조를 분석하고, 이를 트리 형태로 표현하는 과정입니다.
*   구문 분석은 문장의 의미를 이해하고, 다양한 NLP 작업의 성능을 향상시키는 데 도움이 됩니다.

**2. 구조적 모호성 (Structural Ambiguity):**

*   하나의 문장이 둘 이상의 구문 분석 트리를 가질 수 있는 경우입니다.
*   PP-attachment, coordination ambiguity, noun-phrase bracketing ambiguity등이 있습니다.

**3. CKY 파싱 (CKY Parsing):**

*   Context-Free Grammar를 위한 동적 프로그래밍 기반의 파싱 알고리즘입니다.
*   문법을 Chomsky Normal Form (CNF)으로 변환해야 합니다.
*   각 셀에 가능한 모든 구성 성분을 저장하여 효율적으로 구문 분석 트리를 생성합니다.
*   셀을 채우는 방법 (상향식)
*   이 알고리즘은 parser일 뿐이고, 최적의 parse 트리를 선택하진 않는다.

**4. Span-Based Neural Constituency Parsing:**

*   CFG없이 neural classifier를 사용해 score를 준다.
*   기존의 CKY알고리즘을 수정해서 조합한다.
*   span기반 구문 분석 알고리즘은 CKY 알고리즘을 활용하여 가장 높은 점수를 가진 구문 분석 트리를 찾는 방법을 제시합니다.

**5. 평가 (Evaluation):**

*   구문 분석의 성능은 PARSEVAL 메트릭을 사용하여 평가합니다.
    *   Labeled recall, labeled precision, cross-brackets 등의 지표를 사용합니다.

**6. 제한 사항:**

*   완전한 구문 분석이 필요하지 않은 작업의 경우 부분 구문 분석 (Partial Parsing) 또는 청킹 (Chunking)을 사용할 수 있습니다.

**7. 고급 주제:**

*   CCG Parsing(조합 범주 문법)
*   A* Algorithm (휴리스틱 탐색)

**핵심 내용:**

*   구문 분석은 문장 구조를 이해하는 데 필수적인 단계입니다.
*   CKY 알고리즘은 효율적인 구문 분석을 위한 중요한 도구입니다.
*   다양한 평가 지표를 사용하여 구문 분석 모델의 성능을 측정할 수 있습니다.

이 장에서는 구문 분석의 기본 개념과 다양한 알고리즘을 학습하여, 복잡한 문장의 구조를 이해하고 자동 구문 분석 시스템을 개발할 수 있도록 도와줍니다.

## 14장 요약: 의존 구문 분석

14장은 문장 구조를 표현하는 또 다른 방법인 의존 구문 분석 (Dependency Parsing)을 소개합니다. 구성 성분 문법과 달리, 의존 구문 분석은 문장 내 단어 간의 의존 관계에 초점을 맞춥니다.

**1. 의존 문법 (Dependency Grammar):**

*   문장의 구문 구조를 단어 간의 이항 관계 (Binary Relation)로 표현합니다.
*   문장 내 모든 단어는 하나의 헤드 (Head)를 가지며, 헤드는 다른 단어에 의존 (Dependent)합니다.
*   문법 관계 (Grammatical Relation)를 사용하여 헤드와 의존어 간의 관계 유형을 명시합니다.
*   구조는 constituent를 갖지 않고, head와 dependent만 갖는다.

**2. 의존 관계 (Dependency Relations):**

*   주어 (Subject), 목적어 (Object), 보어 (Complement), 수식어 (Modifier) 등 다양한 유형의 문법 관계가 존재합니다.
*   Universal Dependencies (UD)는 여러 언어에서 일관성 있게 사용할 수 있도록 정의된 의존 관계 표준입니다.
*   UD dependency relation은 클라우살 인자 관계와 명칭 인자 관계로 나뉜다

**3. 투사성 (Projectivity):**

*   의존 트리의 모든 arc는 projective 하다
*   투사성은 헤드와 의존어 사이에 다른 단어가 존재할 수 있는지 여부를 나타냅니다.
*   많은 언어는 Non-projective한 경우가 많다

**4. 의존 구문 분석 알고리즘 (Dependency Parsing Algorithms):**

*   **Transition-Based Parsing:** 스택 (Stack)과 버퍼 (Buffer)를 사용하여 탐욕적으로 의존 트리를 생성합니다.
*   **Graph-Based Parsing:** 문장을 완전 연결 그래프로 표현하고, 최대 신장 트리 (Maximum Spanning Tree)를 찾아 의존 트리를 생성합니다.

**5. 학습 (Learning):**

*   의존 구문 분석 모델은 지도 학습 (Supervised Learning) 방식으로 학습됩니다.
*   트리뱅크 (Treebank)로부터 학습 데이터를 추출하여 모델을 학습시킵니다.

**6. 평가 (Evaluation):**

*   의존 구문 분석의 성능은 다음 지표를 사용하여 평가합니다.
    *   Labeled Attachment Score (LAS): 헤드와 의존 관계 모두 정확하게 예측한 비율.
    *   Unlabeled Attachment Score (UAS): 헤드만 정확하게 예측한 비율.

**핵심 내용:**

*   의존 구문 분석은 문장 구조를 표현하는 또 다른 방법이며, 의미 분석 및 정보 추출에 유용합니다.
*   Transition-based 및 Graph-based 파싱은 의존 구문 분석을 위한 대표적인 알고리즘입니다.
*   트리뱅크는 의존 구문 분석 모델 학습 및 평가를 위한 중요한 데이터 소스입니다.

이 장에서는 의존 구문 분석의 기본 개념과 다양한 알고리즘을 학습하여, 문장 구조를 이해하고 자동 의존 구문 분석 시스템을 개발할 수 있도록 도와줍니다.

## 15장 요약: 문장 의미의 논리적 표현

15장은 언어적 표현의 의미를 형식적인 구조로 포착하는 아이디어를 소개하며, 특히 자연어 처리에서 널리 사용되는 의미 표현 언어인 First-Order Logic (FOL) 및 Description Logics (DLs)를 중심으로 다룹니다.

**1. 의미 표현 (Meaning Representations):**

*   언어적 요소와 현실 세계에 대한 비언어적 지식을 연결하는 데 필요한 구조입니다.
*   질문 답변, 메뉴에서 음식 선택, 레시피 따르기 등 다양한 의미 처리 작업에 필수적입니다.

**2. 의미 표현 언어 (Meaning Representation Languages):**

*   의미를 형식적으로 표현하기 위한 구문 및 의미 체계를 정의합니다.
*   다양한 언어가 존재하며, 주요 목표는 언어적 입력을 컴퓨터가 이해할 수 있도록 변환하는 것입니다.

**3. 의미 표현의 요구 사항:**

*   **검증 가능성 (Verifiability):** 진위 여부를 판단할 수 있어야 합니다.
*   **명확성 (Unambiguous):** 하나의 입력에 하나의 의미 표현이 존재해야 합니다.
*   **표현력 (Expressiveness):** 다양한 주제를 다룰 수 있을 만큼 표현력이 풍부해야 합니다.
*   **변수 지원:** 명시적으로 언급하지 않은 대상을 지칭할 수 있어야 합니다.
*   **추론 지원:** 새로운 정보를 도출할 수 있는 추론 능력을 제공해야 합니다.

**4. First-Order Logic (FOL):**

*   유연하고 잘 이해되며 계산적으로 다루기 쉬운 의미 표현 언어입니다.
*   객체 (Objects), 속성 (Properties), 관계 (Relations)를 표현하는 데 사용됩니다.
*   수량화 (Quantification)를 통해 변수의 범위를 지정할 수 있습니다 (∀: 모든, ∃: 존재).

**5. 술어 (Predicate):**

*   객체의 속성 또는 객체 간의 관계를 나타내는 데 사용됩니다.
*   단항 술어 (Unary Predicate): 객체의 속성을 나타냅니다 (예: Restaurant(Maharani)).
*   이항 술어 (Binary Predicate): 객체 간의 관계를 나타냅니다 (예: Serves(Maharani,VegetarianFood)).

**6. 논리 연산자 (Logical Connectives):**

*   FOL Formula를 더 크게 만든다.
*   ∧ (AND): 두 formula 모두 만족.
*   ∨ (OR): 두 formula 중 하나 만족.
*   ⇒ (implicates): a implies b

**7. 람다 표기법 (Lambda Notation):**

*   함수 추상화를 위한 도구입니다.
*   수식에서 변수를 정의하고, 그 변수를 사용하는 함수를 정의하는 데 사용됩니다 (λx.P(x)).

**8. 모델 기반 의미론 (Model-Theoretic Semantics):**

*   의미 표현을 현실 세계의 객체, 속성, 관계로 매핑하는 데 사용됩니다.
*   모델은 특정 상황 또는 세계에 대한 공식적인 표현입니다.

**9. 추론 (Inference):**

*   새로운 정보를 도출하거나, 기존 정보를 검증하는 과정입니다.
*   Modus Ponens, Forward Chaining, Backward Chaining, Resolution 등의 추론 규칙이 있습니다.

**10. 사건 및 상태 표현 (Event and State Representations):**

*   사건 (Event): 상태의 변화를 나타냅니다.
*   상태 (State): 시간이 지나도 변하지 않는 속성 또는 조건입니다.
*   Neo-Davidsonian 표현은 사건 변수를 사용하여 사건과 관련된 정보를 표현합니다.

**11. Description Logics (DLs):**

*   FOL의 하위 집합으로, 지식 표현 및 추론을 위한 형식적인 기반을 제공합니다.
*   TBox (Terminology): 용어 (개념) 간의 관계를 정의합니다.
*   ABox (Assertion Box): 개별 객체에 대한 사실을 진술합니다.

**핵심 내용:**

*   의미 표현은 자연어 처리에서 핵심적인 개념이며, 지능적인 시스템을 구축하는 데 필수적입니다.
*   FOL은 강력하고 널리 사용되는 의미 표현 언어입니다.
*   DLs는 특정 추론 작업에 효율적인 의미 표현 언어입니다.

이 장에서는 의미 표현의 기본 개념과 다양한 표현 언어를 학습하여, 자연어 처리 시스템이 문장의 의미를 이해하고 추론할 수 있도록 도와줍니다.

## 17장 요약: 정보 추출 (Information Extraction)

17장은 비정형 텍스트에서 의미 있는 정보를 추출하여 구조화된 데이터로 변환하는 정보 추출 (Information Extraction, IE) 기술을 소개합니다.

**1. 정보 추출 (IE):**

*   텍스트에 내재된 비정형 정보를 추출하여 관계형 데이터베이스, 지식 그래프 등의 구조화된 형태로 변환하는 과정입니다.

**2. 관계 추출 (Relation Extraction):**

*   텍스트에서 엔티티 (Named Entity) 간의 의미적 관계를 식별하고 분류하는 작업입니다.
*   부분-전체, 인과 관계, 직업 관계 등 다양한 유형의 관계를 추출할 수 있습니다.
*   Knowledge Graphs를 활용하여 Search Engine에서 사용자에게 정보를 제공합니다.

**3. Event Extraction:**

*   주어진 text로부터 event를 찾는 task. event에 참가하는 Entity와 함께,
*   문장에서 Event를 추출하고 Event type을 분류하는 task.

**4. 템플릿 채우기 (Template Filling):**

*   정해진 slot에 넣어 템플릿을 완성
*   핵심 내용, 액터, 액션, 태그 등을 찾기

**5. 17장에서 소개하는 IE 접근 방법:**

*   **패턴 기반 추출 (Pattern-based Extraction):** 사람이 직접 정의한 규칙 또는 패턴 (Hearst Pattern)을 사용하여 관계를 추출합니다.
    *   장점: 높은 정확도 (Precision)
    *   단점: 낮은 재현율 (Recall), 도메인 의존성
*   **지도 학습 (Supervised Learning):** 손으로 어노테이션된 data를 사용해서 추출
*   **준지도 학습 (Semi-Supervised Learning):**소수의 seed pattern을 사용해 bootstrapping으로 새로운 패턴을 찾고, 텍스트와 문장을 추출
    *   Distant supervision: Large DB를 활용해 많은 seed example을 얻고, text에서 많은 pattern 특징 추출
        *   장점: 수많은 양질의 data로 학습, 사람이 label하지 않아도 됨
        *   단점: 노이즈가 많음
*   **비지도 학습 (Unsupervised Learning):**
    *   Open information Extraction (Open IE): 아무것도 없어도 웹에서 관계 추출

**핵심 내용:**

*   정보 추출은 텍스트 데이터를 활용하여 의미 있는 정보를 얻는 데 필수적인 기술입니다.
*   다양한 IE 접근 방식은 각각 장단점을 가지며, 적절한 방법을 선택하는 것이 중요합니다.

이 장에서는 정보 추출의 기본 개념과 다양한 기술들을 학습하여, 텍스트 데이터에서 유용한 정보를 추출하고 활용할 수 있도록 도와줍니다.

## 18장 요약: 단어 의미와 WordNet

18장은 단어의 의미를 다루는 방법을 소개하고, 특히 단어 의미의 표현과 관련된 지식 베이스인 WordNet을 자세히 설명합니다. 또한, 문맥에서 단어의 올바른 의미를 식별하는 작업인 단어 의미 중의성 해소 (Word Sense Disambiguation, WSD)에 대해 다룹니다.

**1. 단어 의미 (Word Sense):**

*   단어의 의미는 종종 여러 가지로 해석될 수 있습니다 (다의성, Polysemy).
*   Word Sense는 단어 의미의 특정 측면을 나타내는 이산적인 표현입니다.

**2. 사전과 시소러스 (Dictionaries and Thesauruses):**

*   단어의 의미를 정의하는 방법으로 Glosses를 사용합니다 (설명).
*   WordNet과 같은 리소스는 단어 간의 관계 (동의어, 반의어, 상위어, 하위어 등)를 명시적으로 나타냅니다.

**3. 동의성 (Synonymy)와 반의성 (Antonymy):**

*   동의어는 의미가 동일하거나 거의 동일한 단어입니다.
*   반의어는 반대되는 의미를 가진 단어입니다.
*   Taxonomic Relations: Hypernymy (상위) vs Hyponymy (하위)

**4. WordNet:**

*   단어의 의미 관계를 표현하는 데 널리 사용되는 어휘 데이터베이스입니다.
*   Synset (동의어 집합), Gloss (정의), Lexicographic Category (의미장) 등의 정보를 제공합니다.

**5. 단어 의미 중의성 해소 (Word Sense Disambiguation, WSD):**

*   문맥에서 단어의 올바른 의미 (Word Sense)를 식별하는 작업입니다.
*   WSD는 Lexical Sample task, All-Words task 등으로 분류할 수 있습니다.

**6. WSD 알고리즘:**

*   **지식 기반 방법 (Knowledge-based Methods):** WordNet과 같은 어휘 자원을 활용합니다 (예: Lesk Algorithm).
*   **지도 학습 방법 (Supervised Learning Methods):** Sense-labeled corpus (SemCor)를 사용하여 모델을 학습합니다.
    *   contextual Embeddings를 사용하는 것이 가장 성능이 좋다.

**7. 주요 내용:**

*   단어의 의미는 고정된 것이 아니라 문맥에 따라 달라질 수 있습니다.
*   WordNet은 단어 간의 의미 관계를 체계적으로 표현하는 데 유용한 리소스입니다.
*   WSD는 문맥 정보를 활용하여 단어의 올바른 의미를 식별하는 중요한 작업입니다.

이 장에서는 단어의 의미와 관련된 다양한 개념과 지식을 학습하고, WordNet과 WSD 기술을 활용하여 텍스트 분석의 정확도를 높일 수 있도록 도와줍니다.

## 19장 요약: 의미 역할 레이블링 (Semantic Role Labeling)

19장은 문장 내에서 술어 (Predicate)와 그 인자 (Argument) 간의 의미적 관계를 식별하는 작업인 의미 역할 레이블링 (Semantic Role Labeling, SRL)을 소개합니다.

**1. 의미 역할 (Semantic Roles):**

*   문장 내에서 술어 (주로 동사)와 그 인자들이 참여하는 방식에 대한 추상적인 모델을 제공합니다.
*   **에이전트 (Agent):** 행동의 주체
*   **테마 (Theme):** 행동의 영향을 받는 대상
*   **목표 (Goal):** 행동의 목적지

**2. Thematic roles:**

*   대체로 volitional causer를 뜻하는 agent, direct affect를 받는 theme등, 몇가지 정해진 rule로 semantic role의 종류를 표현.

**3. 지식 자원:**

*   **PropBank:** Penn Treebank 문장에 대해 의미 역할을 주석 처리한 데이터셋입니다.
    *   Core argument (ARG0, ARG1, ARG2...)와 adjunct modifier (ARGM-LOC, ARGM-TMP, ARGM-ADV,...)등으로 구성
*   **FrameNet:** 단어에 의해 evoked되는 Frame을 기반으로 하는 semantic role labeling.

**4. diathese alternation**

*   동사에서, 같은 뜻이여도 어순에 따라 semantic role이 달라지는 현상

**5. 주요 내용:**

*   의미 역할 레이블링은 문장의 의미 구조를 이해하는 데 중요한 단계입니다.
*   PropBank 및 FrameNet과 같은 자원을 활용하여 다양한 의미 역할을 정의하고 학습할 수 있습니다.

**6. 접근 방법:**

*   수작업 규칙을 사용하여 명시
*   Feature기반으로 통계 모델 구성
    *   장점: 쉽게 구현 가능
    *   단점: 노동량이 높음, 낮은 recall
*   Neural Netwrok 기반
    *   더 나은 결과를 보임

이 장에서는 의미 역할 레이블링의 기본 개념과 다양한 접근 방식을 학습하여, 텍스트 데이터에서 술어와 그 인자 간의 의미적 관계를 파악하고 활용할 수 있도록 도와줍니다.

## 20장 요약: 감정, 정서, 함축적 의미를 위한 어휘 (Lexicons for Sentiment, Affect, and Connotation)

20장은 텍스트에 담긴 감정, 정서, 함축적 의미를 분석하는 데 사용되는 어휘 자원에 대해 소개합니다. 감정 분석 (Sentiment Analysis)에서 더 나아가, 텍스트의 더 풍부하고 미묘한 감정적 측면을 이해하는 방법을 다룹니다.

**1. 감정 (Emotion):**

*   외부 또는 내부 사건에 대한 평가에 대한 비교적 짧은 반응. (joy, sadness, anger, fear)

**2. 정서 (Affect):**

*   감정, 기분, 태도, 성격 특성 등 다양한 정서 상태를 포괄하는 넓은 개념입니다.

**3. 감정 어휘 (Affect Lexicons):**

*   단어와 구문에 대한 감정적 정보를 포함하는 목록입니다.
*   감정 어휘는 감정 분석, 기분 감지, 대인 관계 분석 등 다양한 NLP 작업에 활용됩니다.
    *   General Inquirer, LIWC, EmoLex 등의 예시가 있다

**4. 감정 어휘 구축 방법:**

*   **수동 레이블링 (Human Labeling):** 사람이 직접 단어의 감정을 레이블링합니다 (Crowd-sourcing).
    *   장점: 정확도가 높음
    *   단점: 비용과 시간이 많이 소요
*   **준지도 학습 (Semi-Supervised Learning):** Seed Words를 기반으로 bootstrapping을 사용하여 감정 어휘를 확장합니다.
    *   특정 단어의 근처 단어를 본다
        *   Turney and Littman
        *   Hamilton
*   **지도 학습 (Supervised Learning):** 리뷰 점수와 같은 supervision을 쓴다

**5. 팁:**

*   점수를 매기는 방법
*   검증하는 방법
*   Stop words (the, is, are)을 사용하지 않는다

**6. 함축적 의미 프레임 (Connotation Frame):**

*   동사가 인자에 대해 전달하는 의미적 정보를 표현합니다.
*   저자 (작가, 편집자)가 단어/구, 대화 상대, 다른 항목과 연결된 감정에 대해 정보를 준다
*   동일한 syntactic pattern 이여도, 어감의 차이를 표현가능

**핵심 내용:**

*   감정 어휘는 텍스트에 담긴 감정, 정서, 함축적 의미를 분석하는 데 필수적인 자원입니다.
*   다양한 감정 어휘 구축 방법을 이해하고, 적절한 방법을 선택하는 것이 중요합니다.

이 장에서는 다양한 어휘 자원을 활용하여 텍스트 데이터의 감정적 측면을 더 깊이 이해할 수 있도록 도와줍니다.

## 21장 요약: 지시어 해소 (Coreference Resolution)

21장은 텍스트 내에서 동일한 개체 (Entity)를 지칭하는 표현 (Mention)들을 연결하는 작업인 지시어 해소 (Coreference Resolution)에 대해 다룹니다.

**1. 지시어 해소 (Coreference Resolution):**

*   텍스트 내에서 동일한 개체 (사람, 장소, 사물 등)를 지칭하는 명사구 (Noun Phrase), 대명사 (Pronoun), 고유 명사 (Name) 등을 식별하고 연결하는 작업입니다.
*   문맥적 정보 연결, 지시대상 이해
*   이전 대화나 문맥에서 언급된 내용인지, 새롭게 언급하는 내용인지 확인
*   어떠한 관계를 가지는지 파악

**2. 주요 용어:**

*   **Mention:** 텍스트 내에서 개체를 지칭하는 표현입니다.
*   **Referent:** Mention이 실제로 가리키는 대상 (Discourse Entity)입니다.
*   **Anaphora:** 이전에 언급된 대상을 지칭하는 Mention.
*   **Antecedent:** Anaphora가 가리키는 대상
*   **Cataphora:** 뒤에 나올 Mention을 먼저 언급하는 경우.
*   **Coreference Chain/Cluster:** 동일한 개체를 지칭하는 Mention들의 집합입니다.

**3. 유형:**

*   **Entity Linking**: Discourse entity를 real-world individual에 Mapping
*   **Zero Anaphor** zero pronoun을 활용
*   **non-referring Expressions**: 가짜 주어
*   **정보 상태 (Information Status):**
    *   new: 기존에 없던 Entities를 introduce
    *   unused: Hearer는 알지만 Discourse엔 없는 Entities introduce
    *   old: Discourse에도 있고 Hearer도 아는 Entities access

**4. 지시어 해소의 어려움:**

*   단어 자체는 파악하기 쉽지만 문맥이 중요
*   언급 내용 중 일부만 지시하고 있다.
*   복잡한 구조적, semantic 제약조건

**5. 제약 조건 (Constraints):**

*   **Number Agreement:** 수 일치
*   **Gender or Noun Class Agreement:** 성 일치
*   **Binding Theory Constraints:** 문법적 제약
*   **Recency:** 최근에 언급된 개체에 대한 선호
*   **Grammatical Role:** 주어에 대한 선호
*   등등

**6. 핵심 내용:**

*   Mention Detection, Mention간의 관계 분석, Coreference 결정을 진행
*   지시어 해소는 텍스트 이해의 핵심적인 부분이며, 다양한 NLP 작업의 성능을 향상시키는 데 기여합니다.

이 장에서는 지시어 해소의 기본 개념과 다양한 접근 방식 (규칙 기반, 기계 학습 기반)을 학습하여, 텍스트 데이터에서 개체 간의 관계를 파악하고 활용할 수 있도록 도와줍니다.

## 22장 요약: 담화 일관성 (Discourse Coherence)

22장은 텍스트의 연속된 문장들이 어떻게 연결되어 의미 있는 전체를 구성하는지, 즉 담화 일관성 (Discourse Coherence)에 대해 다룹니다.

**1. 담화 (Discourse):**

*   고려중인 모든 문장 단위
*   단순히 나열된 문장들의 집합이 아닌, 의미적으로 연결된 문장들의 집합입니다.
*   뉴스 기사, 대화, 소설 등 다양한 형태가 있습니다.

**2. 응집성 (Coherence):**

*   담화를 구성하는 문장들이 서로 연결되어 의미적으로 일관성을 유지하는 것을 의미합니다.
*   응집성을 통해 독자는 텍스트를 더 쉽게 이해하고 기억할 수 있습니다.
*   유창하게 쓴다는 건, 흐름을 잘 파악한다는 것과 같다

**3. 응집성의 종류:**

*   **Coherence relations**: 인과 관계, 이유, 대조와 같은 문장끼리의 관계
*   **Entity-based coherence**: 
        - Centering Theory (Grosz) : 어떤 Entity를 주목할 것인지
        - Entity Grid model : 어떤 Entity가 attention을 받는지 아닌지를 파악해 통계적으로 분석
*   **Topical coherence**: 문맥과 어휘가 어떤 관계를 갖는지 분석

**4. 평가:**

*   인간 평가 (Human Evaluation)
*   자동 평가 (Automatic Evaluation)
*   구체적인 수치를 기반한 객관적 평가도 가능

**핵심 내용:**

*   담화 일관성은 텍스트 이해에 중요한 역할을 합니다.
*   다양한 모델을 사용하여 담화 일관성을 측정하고 분석할 수 있습니다.
*   담화 일관성 분석은 텍스트 요약, 질의 응답 등 다양한 NLP 작업에 활용될 수 있습니다.

이 장에서는 담화 일관성의 기본 개념과 다양한 모델을 학습하여, 텍스트의 구조와 의미를 더 깊이 이해할 수 있도록 도와줍니다.

## 23장 요약: 질문 응답 (Question Answering)

23장은 자연어 질문에 대해 자동으로 답변을 생성하는 시스템을 구축하는 방법을 다룹니다. 텍스트 데이터에서 획득하거나 지식 베이스(Knowledge Base)에 존재하는 정형 데이터로 활용하여 질의응답에 사용하는 방법을 다룬다.

**1. 질문 응답 (Question Answering, QA):**

*   사용자의 질문에 대해 관련 정보 또는 답변을 제공하는 시스템입니다.
*   가상 어시스턴트, 검색 엔진, 데이터베이스 쿼리 등 다양한 인터페이스를 통해 구현될 수 있습니다.

**2. 질문 응답의 종류:**

*   **Factoid Question Answering:** 짧은 사실 정보를 요구하는 질문에 답변합니다.
*   **Long-Form Question Answering:** 설명이나 추론을 필요로 하는 복잡한 질문에 답변합니다.
*   **Community Question Answering:** Quora, Stack Overflow와 같은 커뮤니티 데이터를 활용하여 답변합니다.

**3. IR-기반 질문 응답 (IR-based QA):**

*   **검색 (Retrieval):** 질문과 관련된 문서를 검색합니다 (Information retrieval, IR).
*   **독해 (Reading Comprehension):** 검색된 문서에서 답변 범위를 추출합니다.
*   어디서 찾을지 알면 간단한 문제

**4. 지식 기반 질문 응답 (Knowledge-Based QA):**

*   **지식 베이스 (Knowledge Base):** 사실 (Fact)과 규칙 (Rule)을 저장하는 구조화된 데이터베이스입니다.
*   **의미 분석 (Semantic Parsing):** 질문을 논리적 형태로 변환합니다.
*   **추론 (Inference):** 논리적 형태를 사용하여 지식 베이스를 쿼리하고 답변을 생성합니다.
*   구조화된 DB를 활용해야하기 때문에 복잡한 작업에는 어렵다

**5. 검색 기법 (retrieval):**

*   Bag of words에서 term-frequency weight (TF-IDF)를 사용
*   Vector space model로 변경

**6. 답변 분석 (Answer analysis):**

*   구문으로 분석하여 answer type을 결정
*   지문에서 구함

**7. 퓨샷 방법 (Few shot method):**

*   사전학습된 언어 모델 (GPT-3)가 충분한 지식을 갖고 있다면, 프롬프트 (프롬프팅)을 통해 결과를 얻는다.

**핵심 내용:**

*   질문 응답은 자연어 처리의 중요한 응용 분야입니다.
*   IR-based QA와 Knowledge-Based QA는 질문에 답변하는 서로 다른 접근 방식입니다.
*   단어 임베딩 및 신경망 모델은 질문 응답 시스템의 성능을 향상시키는 데 기여합니다.

이 장에서는 질문 응답 시스템의 기본 원리와 다양한 접근 방식을 학습하여, 사용자의 질문에 대해 정확하고 관련성 높은 답변을 제공할 수 있도록 도와줍니다.

## 24장 요약: 챗봇 및 대화 시스템

24장은 인간과의 자연스러운 대화를 모방하거나 특정 작업을 지원하도록 설계된 시스템인 챗봇 및 대화 시스템 (Dialogue Systems)의 기본 원리와 아키텍처를 소개합니다.

**1. 대화의 속성 (Properties of Human Conversation):**

*   순서 (Turn): 대화는 발화의 순서로 구성되며, 각 발화는 한 사람이 기여합니다.
*   대화 행위 (Speech act): 대화에서 하는 것
*   공통 기반 (Common Ground): 대화 참여자가 서로 이해하고 동의하는 공유된 지식.
*   근거 (Grounding): 상대방이 이해했는지 확인하는 것.

**2. 챗봇 (Chatbots):**

*   비구조화된 대화(채팅)를 모방하는 시스템입니다.
*   룰 기반, 정보 검색 기반, 인코더-디코더 생성기의 세 가지 아키텍처가 있습니다.
    *   Rule-based : ELIZA
    *   Corpus-based: Retrieval, Generation

**3. 과제 지향적 대화 (Task-Oriented Dialogue):**

*   사용자가 특정 작업을 완료하도록 지원하는 시스템입니다. (예: 항공편 예약, 식당 추천)
*   프레임 기반 아키텍처 (Frame-based Architecture)를 사용합니다.
*   Modern 시스템은 dialogue state archtecture 를 사용
*   NLU (Natural Language Understanding)
*   Dialogue act detection
*   Dialogue policy
*   Natural language Generation (NLG)

**4. 대화 정책 (Dialogue Policy):**

*   다음을 위해 시스템이 취해야 할 action을 결정한다.
*   사용자 의도 파악
*   다음 action을 제안

**5. 대화 시스템 디자인 (Dialogue System Design):**

*   사용자 인터뷰를 통해 수집
*   테스트를 반복 진행

**6. 윤리적 고려 사항:**

*   악용을 막기 위해, 잠재적인 피해를 최소화 해야 함
*   무슨 일이 일어날지 예상하고 책임감을 가져야함

**핵심 내용:**

*   대화 시스템은 인간과의 자연스러운 상호 작용을 가능하게 하는 기술입니다.
*   챗봇은 룰 기반, 코퍼스 기반, neural로 디자인 할 수 있다.
*   대화 시스템 설계는 사용자 중심적이어야 하며, 윤리적인 측면도 고려해야 합니다.

이 장에서는 챗봇과 대화 시스템의 기본 개념과 다양한 아키텍처 및 디자인 원칙을 학습하여, 실제 사용자에게 유용한 대화형 인터페이스를 구축할 수 있도록 도와줍니다.

## 25장 요약: 음성학 (Phonetics)

25장은 언어의 소리, 즉 음성 (Phone)에 대한 연구인 음성학 (Phonetics)의 기본 개념과 원리를 소개합니다. 언어의 소리에 대한 특징, 생성, 인식에 대해 다룬다.

**1. 음성 (Phone):**

*   언어에서 사용되는 개별적인 소리 단위입니다.
*   문자 (Letter)와 음성 (Phone)는 반드시 일대일 대응되지 않습니다.
*   IPA (International Phonetic Alphabet) 또는 ARPAbet과 같은 음성 기호를 사용하여 표현합니다.
*   Vowel, consonant, nasal

**2. 조음 음성학 (Articulatory Phonetics):**

*   입, 목, 코 등의 조음 기관 (Articulator)을 사용하여 소리가 어떻게 생성되는지를 연구합니다.
*   Consonants는 airflow를 제한하는 방법을 통해 구분
*   Vowels는 혀의 위치에 따라 구분

**3. 음향 음성학 (Acoustic Phonetics):**

*   음파의 물리적 특성을 분석하여 소리를 연구합니다.
*   Frequency (Hz), amplitude, spectrogram 등의 개념을 사용합니다.
*   스펙트럼 (spectrum)
*   포먼트 (formant)

**4. 초분절 음운론 (Suprasegmental Phonology):**

*   강세 (Accent), 운율 (Tune), 억양 (Intonation) 등과 같이 단어 또는 음절 이상의 운율적 특징을 다룹니다.
*   언어의 운율적 특징을 분석하여 의미를 파악합니다.

**5. 음성 자원 (Phonetic Resources):**

*   온라인 발음 사전, 음성 코퍼스 (TIMIT, Switchboard 등)와 같은 다양한 음성학 자원을 활용할 수 있습니다.

**핵심 내용:**

*   음성학은 음성 인식 및 합성 시스템 개발에 필수적인 지식입니다.
*   조음 음성학은 소리의 생성 과정을 이해하는 데 도움이 됩니다.
*   음향 음성학은 소리의 물리적 특성을 분석하고, 음성 인식에 필요한 정보를 추출하는 데 사용됩니다.

이 장에서는 음성학의 기본 개념과 다양한 음성 자원을 학습하여, 음성 인식 및 합성 시스템 개발에 필요한 지식을 습득할 수 있도록 도와줍니다.

## 26장 요약: 자동 음성 인식 (ASR) 및 텍스트 음성 변환 (TTS)

26장은 음성 신호를 텍스트로 변환하는 자동 음성 인식 (Automatic Speech Recognition, ASR)과 텍스트를 음성 신호로 변환하는 텍스트 음성 변환 (Text-To-Speech, TTS) 기술을 소개합니다.

**1. 자동 음성 인식 (ASR):**

*   음성 파형 (Waveform)을 텍스트 시퀀스로 변환하는 작업입니다.
*   **음성 특징 추출 (Feature Extraction):** 음성 파형에서 Mel spectrum vector를 추출합니다.
*   음성 data -> log mel spectrum vector -> 알파벳
*   **인코더-디코더 (Encoder-Decoder) 모델:** 음향 특징과 텍스트 간의 관계를 학습합니다. Attention기법을 활용

**2. 음성 인식의 어려움:**

*   화자 변동 (Speaker Variation), 발음 변동, 잡음 환경 등

**3. 연결주의적 시간적 분류 (Connectionist Temporal Classification, CTC):**

*   디코딩할 때 유용한 정보를 제공하는 알고리즘

**4. 텍스트 음성 변환 (TTS):**

*   텍스트 시퀀스를 음성 파형으로 변환하는 작업입니다.
*   **인코더-디코더 (Encoder-Decoder) 모델:** 텍스트를 입력으로 받아 mel spectrogram을 생성, mel 스펙트로그램을 다시 음성 신호로 변환하는 Vocoder를 사용합니다.
*   WaveNet - autoregressive 모델을 사용해 고품질 음성을 생성.
*   Tacotron2 - mel 스펙트로그램을 활용해 더 사실적인 음성을 생성.

**5. 음성 합성에 대한 어려움:**

*   운율 (Prosody), 강세 (Stress), 억양 (Intonation) 등을 모델링하기 어렵습니다.
*   자연스러운 음성 생성을 위해서는 운율에 대한 제어가 중요합니다.

**6. 텍스트 정규화 (Text Normalization):**

*   TTS 시스템의 전처리 단계로, 숫자, 날짜, 약어 등 비표준 단어 (Non-Standard Word)를 표준적인 형태로 변환합니다.

**7. 평가:**

*   **ASR:** 단어 오류율 (Word Error Rate, WER)을 사용합니다.
*   **TTS:** 평균 의견 점수 (Mean Opinion Score, MOS)를 사용합니다. 청취자를 활용해 평가

**핵심 내용:**

*   ASR과 TTS는 음성과 언어 간의 변환을 가능하게 하는 핵심 기술입니다.
*   인코더-디코더 모델은 ASR과 TTS 모두에 효과적으로 사용될 수 있습니다.
*   합성곱 신경망(CNN), 순환 신경망(RNN), 어텐션, 트랜스포머
*   log-mel 스펙트럼, waveform을 활용해 고품질 오디오를 생성

이 장에서는 음성 인식과 텍스트 음성 변환의 기본 개념과 다양한 기술들을 학습하여, 인간과 컴퓨터 간의 음성 기반 의사소통을 구현할 수 있도록 도와줍니다.
