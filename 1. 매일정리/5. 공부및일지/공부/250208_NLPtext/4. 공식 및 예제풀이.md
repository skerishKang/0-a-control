# $\hat{c} = \underset{c \in C}{\operatorname{argmax}} P(c|d) $ (4.1)
## 공식 (4.1) 설명:

이 공식은 주어진 문서(데이터) `d`가 주어졌을 때, 가장 확률이 높은 카테고리(클래스) `c`를 찾는 최대 사후 확률 (Maximum A Posteriori, MAP) 추정 방법을 나타냅니다.

**각 부분별 의미:**

*   **$\hat{c}$**:  추정된 최적의 카테고리 (예측값). "c hat"이라고 읽으며, 추정값을 나타낼 때 자주 사용되는 기호입니다.
*   **$c$**:  특정 카테고리 (예: "스포츠", "정치", "경제" 등).
*   **$C$**:  가능한 모든 카테고리의 집합.
*   **$\operatorname{argmax}$**:  함수를 최대로 만드는 변수(argument)를 반환하는 연산자. 즉, 주어진 조건에서 P(c|d)를 최대로 만드는 c를 찾습니다.
*   **$P(c|d)$**:  사후 확률 (Posterior Probability). 문서 `d`가 주어졌을 때, 이 문서가 카테고리 `c`에 속할 확률.
*   **$d$**: 주어진 문서(데이터).

**전체적인 의미 해석:**

"주어진 문서 d에 대해, 가능한 모든 카테고리 c (c ∈ C) 중에서 사후 확률 P(c|d)를 최대로 만드는 카테고리 c를 찾아 $\hat{c}$ (예측된 카테고리)로 결정한다."

**쉽게 풀어 설명하면:**

새로운 문서(d)가 들어왔을 때, 이 문서가 어떤 카테고리(C)에 속할지 예측하는 상황입니다.  여러 카테고리 중에서, 이 문서가 각 카테고리에 속할 확률을 모두 계산해보고, 그중 가장 확률이 높은 카테고리를 선택하는 방법입니다.

**베이즈 정리 (Bayes' Theorem) 와의 관계:**

사후 확률 P(c|d)는 직접 구하기 어려운 경우가 많습니다. 그래서 베이즈 정리를 이용하여 다음과 같이 변형하여 계산하는 것이 일반적입니다.

$P(c|d) = \frac{P(d|c)P(c)}{P(d)}$

*   $P(d|c)$:  우도 (Likelihood). 카테고리 `c`가 주어졌을 때, 문서 `d`가 나타날 확률.
*   $P(c)$:  사전 확률 (Prior Probability).  문서와 관계없이 카테고리 `c`가 나타날 확률 (일반적인 분포).
*   $P(d)$:  증거 (Evidence).  문서 `d`가 나타날 확률 (일반적인 분포).

P(d)는 모든 c에 대해 동일하므로 argmax를 계산할 때는 생략 가능합니다. 따라서 실제로는:

$\hat{c} = \underset{c \in C}{\operatorname{argmax}} P(d|c)P(c)$ 를 사용합니다.

## 예상 문제

**문제 1:**

다음 용어들을 설명하세요:

*   사후 확률 (Posterior Probability)
*   $\operatorname{argmax}$
*   최대 사후 확률 (MAP) 추정

**답안 1:**

*   **사후 확률 (Posterior Probability)**: 어떤 데이터(증거)가 주어졌을 때, 특정 사건(가설)이 발생할 확률. 베이즈 정리를 통해 계산되며, 사전 확률과 우도를 이용하여 업데이트된 확률을 의미합니다.
*   **$\operatorname{argmax}$**:  함수의 값을 최대로 만드는 입력값(argument)을 반환하는 연산자.
*   **최대 사후 확률 (MAP) 추정**: 주어진 데이터를 바탕으로, 사후 확률을 최대로 만드는 가설(카테고리, 클래스 등)을 선택하는 추정 방법.

**문제 2:**

문서 분류 문제에서 MAP 추정 방법을 사용하는 이유를 간략하게 설명하고, 베이즈 정리와의 관계를 설명하세요.

**답안 2:**

MAP 추정 방법을 사용하는 이유는 주어진 문서에 대해 가장 가능성이 높은 카테고리를 선택함으로써 분류 정확도를 높일 수 있기 때문입니다. 사후 확률은 베이즈 정리를 통해 사전 확률과 우도의 곱으로 표현될 수 있으며 P(d)는 모든 카테고리에서 같기 때문에, 우도와 사전확률을 최대화하는 카테고리를 찾는 문제로 변환하여 계산을 단순화 할 수 있습니다.

**문제 3:**

만약 스팸 메일 필터링 시스템을 만든다고 가정할 때, 공식 (4.1)의 각 요소(c, C, d, P(c|d))는 무엇을 의미하는지 구체적인 예시를 들어 설명하세요.

**답안 3:**

*   **c**:  "스팸" 또는 "스팸 아님(햄)" 과 같은 메일의 카테고리.
*   **C**:  {"스팸", "스팸 아님"} 과 같이 가능한 모든 카테고리의 집합.
*   **d**:  새로 들어온 이메일의 내용 (텍스트, 제목, 발신자 등).
*   **P(c|d)**:  주어진 이메일(d)이 "스팸" 카테고리(c)에 속할 확률 (또는 "스팸 아님" 카테고리에 속할 확률).  베이즈 정리를 사용하여 계산합니다. 예를들어 "광고", "무료" 같은 단어가 포함된 메일은 스팸일 확률이 높고, "회의", "보고서" 같은 단어가 포함된 메일은 스팸이 아닐 확률이 높을 것입니다.

## 계산 문제 5가지 (MAP 추정)

**문제 1:**

어떤 문서 분류 모델에서, 문서 d가 주어졌을 때 카테고리 A에 속할 사후 확률 P(A|d) = 0.6 이고, 카테고리 B에 속할 사후 확률 P(B|d) = 0.4 라고 한다. MAP 추정 방법에 따라 문서 d는 어떤 카테고리로 분류되어야 하는가?

**답안 1:**

$\hat{c} = \underset{c \in \{A, B\}}{\operatorname{argmax}} P(c|d)$

P(A|d) = 0.6, P(B|d) = 0.4 이므로, P(A|d) > P(B|d) 입니다. 따라서 문서 d는 카테고리 A로 분류됩니다.

**문제 2:**

두 개의 카테고리 C1, C2 가 존재하고, 사전 확률은 P(C1) = 0.7, P(C2) = 0.3 이다. 문서 d가 주어졌을 때, 각 카테고리에서의 우도는 P(d|C1) = 0.2, P(d|C2) = 0.6 이라고 한다. 베이즈 정리를 이용하여 사후 확률을 계산하고, MAP 추정에 따라 문서 d를 분류하세요. (P(d)는 계산하지 않아도 됩니다.)

**답안 2:**

베이즈 정리에 따라 사후 확률은 우도와 사전 확률의 곱에 비례합니다.

*   P(C1|d) ∝ P(d|C1)P(C1) = 0.2 * 0.7 = 0.14
*   P(C2|d) ∝ P(d|C2)P(C2) = 0.6 * 0.3 = 0.18

P(C2|d) > P(C1|d) 이므로, 문서 d는 카테고리 C2로 분류됩니다.

**문제 3:**

세 개의 카테고리 X, Y, Z 가 있고, 사전 확률은 P(X) = 0.5, P(Y) = 0.3, P(Z) = 0.2 이다. 문서 d에 대한 우도는 다음과 같다:

*   P(d|X) = 0.1
*   P(d|Y) = 0.4
*   P(d|Z) = 0.3

MAP 추정 방법을 사용하여 문서 d를 분류하세요.

**답안 3:**

*   P(X|d) ∝ P(d|X)P(X) = 0.1 * 0.5 = 0.05
*   P(Y|d) ∝ P(d|Y)P(Y) = 0.4 * 0.3 = 0.12
*   P(Z|d) ∝ P(d|Z)P(Z) = 0.3 * 0.2 = 0.06

P(Y|d)가 가장 크므로, 문서 d는 카테고리 Y로 분류됩니다.

네, 문제 4의 보충 설명 부분만 따로 분리하고, 마크다운을 사용하여 가독성을 높이겠습니다.

**문제 4:**

동전 던지기 실험에서 앞면(H)이 나올 확률을 추정하려고 한다. 사전 분포로 Beta(2, 2) 분포를 사용하고, 10번의 동전 던지기 실험 결과 7번 앞면(H)과 3번 뒷면(T)이 나왔다. MAP 추정을 통해 앞면이 나올 확률을 추정하세요. (Beta 분포의 최빈값(mode) 공식을 활용: mode = (α-1)/(α+β-2), α: 앞면이 나온 횟수+사전분포의 첫번째 파라미터, β: 뒷면이 나온 횟수+사전분포의 두번째 파라미터)

**답안 4:**

사전 분포 Beta(2, 2)에서 α = 2, β = 2 입니다.
10번의 실험 결과, 앞면(H) 7번, 뒷면(T) 3번이 나왔으므로, 사후 분포는 Beta(2+7, 2+3) = Beta(9, 5)가 됩니다.

Beta 분포의 최빈값(mode) 공식에 따라:

mode = (α-1)/(α+β-2) = (9-1)/(9+5-2) = 8/12 = 2/3

따라서 MAP 추정에 따른 앞면이 나올 확률은 2/3 입니다.

> **💡 보충 설명: 베이즈 추론, 베타 분포, 최빈값, 베르누이 시행, 이항 분포**
>
> 이 문제는 베이즈 추론, 베타 분포, 최빈값(mode), 베르누이 시행, 이항 분포에 대한 사전 지식이 필요합니다.
>
>**베르누이 시행 (Bernoulli Trial):**
>
>결과가 두 가지 중 하나로만 나오는 실험을 베르누이 시행이라고 합니다. 예를 들어, 동전 던지기(앞면/뒷면), 시험 합격/불합격 등이 베르누이 시행에 해당합니다. 각 시행은 독립적이며, 성공 확률(예: 동전 앞면이 나올 확률)은 θ로 동일합니다.
>
>**이항 분포 (Binomial Distribution):**
>
>동일한 베르누이 시행을 *n*번 독립적으로 반복했을 때, 성공 횟수의 분포를 이항 분포라고 합니다. 즉, 이항 분포는 *n*번의 베르누이 시행에서 성공 횟수가 *k*번일 확률을 나타냅니다.
>
> *   **베이즈 추론 (Bayesian Inference):** 베이즈 정리를 이용하여 사전 분포(prior distribution)와 관측 데이터(likelihood, 여기서는 이항 분포)를 결합하여 사후 분포(posterior distribution)를 얻는 통계적 추론 방법입니다.
> *   **베타 분포 (Beta Distribution):** 0과 1 사이의 값을 가지는 연속 확률 분포로, 두 개의 매개변수 α와 β로 정의됩니다. α는 성공(예: 앞면) 횟수와 관련 있고, β는 실패(예: 뒷면) 횟수와 관련 있습니다. 베르누이 시행(또는 이항 분포)의 성공 확률에 대한 사전 분포나 사후 분포를 나타내는 데 자주 사용됩니다.
>
>     Beta 분포의 확률 밀도 함수는 다음과 같습니다:
>
>     $f(\theta; \alpha, \beta) = \frac{\theta^{\alpha-1}(1-\theta)^{\beta-1}}{B(\alpha, \beta)}$
>
>     여기서 $B(\alpha, \beta)$는 베타 함수로, 확률 밀도 함수의 적분이 1이 되도록 하는 정규화 상수입니다.
> *   **최빈값 (Mode):** 확률 분포에서 확률 밀도 함수(probability density function, PDF) 또는 확률 질량 함수(probability mass function, PMF)가 최대가 되는 값. 즉, 가장 발생 가능성이 높은 값을 의미합니다. Beta 분포의 최빈값(mode, 가장 확률 밀도가 높은 값)은 다음과 같이 계산됩니다:
>
>     $mode = \frac{\alpha - 1}{\alpha + \beta - 2}$ (단, α > 1, β > 1)
> *   **켤레사전분포(Conjugate Prior Distribution):** 어떤 likelihood 함수(여기서는 이항분포)가 주어졌을 때, 켤레사전분포를 사용하면, prior distribution과 posterior distribution이 같은 분포 family에 속하게 됩니다. 예를 들어, 이항분포 likelihood에 대해서는 Beta distribution이 켤레사전분포입니다.

**문제 4 (최종 수정):**

동전 던지기 실험에서 앞면(H)이 나올 확률 θ를 추정하려고 합니다. θ에 대한 사전 믿음을 나타내는 사전 분포로 Beta(α, β) 분포를 사용합니다. Beta 분포는 0과 1 사이의 값을 가지는 확률 변수에 대한 분포로, 동전 던지기처럼 베르누이 시행의 확률을 모델링하는 데 적합합니다.

**베르누이 시행 (Bernoulli Trial):**

결과가 두 가지 중 하나로만 나오는 실험을 베르누이 시행이라고 합니다. 예를 들어, 동전 던지기(앞면/뒷면), 시험 합격/불합격 등이 베르누이 시행에 해당합니다. 각 시행은 독립적이며, 성공 확률(예: 동전 앞면이 나올 확률)은 θ로 동일합니다.

**이항 분포 (Binomial Distribution):**

동일한 베르누이 시행을 *n*번 독립적으로 반복했을 때, 성공 횟수의 분포를 이항 분포라고 합니다. 즉, 이항 분포는 *n*번의 베르누이 시행에서 성공 횟수가 *k*번일 확률을 나타냅니다.

Beta 분포의 확률 밀도 함수는 다음과 같습니다:

$f(\theta; \alpha, \beta) = \frac{\theta^{\alpha-1}(1-\theta)^{\beta-1}}{B(\alpha, \beta)}$

여기서 $B(\alpha, \beta)$는 베타 함수로, 확률 밀도 함수의 적분이 1이 되도록 하는 정규화 상수입니다.

Beta 분포의 최빈값(mode, 가장 확률 밀도가 높은 값)은 다음과 같이 계산됩니다:

$mode = \frac{\alpha - 1}{\alpha + \beta - 2}$ (단, α > 1, β > 1)

10번의 동전 던지기 실험(즉, 10번의 베르누이 시행) 결과 7번 앞면(H)과 3번 뒷면(T)이 나왔습니다. 사전 분포로 Beta(2, 2)를 사용했을 때, MAP(Maximum A Posteriori) 추정을 통해 앞면이 나올 확률 θ를 추정하세요.

**배경 설명:**

*   **베이즈 추론 (Bayesian Inference):** 베이즈 정리를 이용하여 사전 분포(prior distribution)와 관측 데이터(likelihood, 여기서는 이항 분포)를 결합하여 사후 분포(posterior distribution)를 얻는 통계적 추론 방법입니다.
*   **베타 분포 (Beta Distribution):** 0과 1 사이의 값을 가지는 연속 확률 분포로, 두 개의 매개변수 α와 β로 정의됩니다. α는 성공(예: 앞면) 횟수와 관련 있고, β는 실패(예: 뒷면) 횟수와 관련 있습니다. 베르누이 시행(또는 이항 분포)의 성공 확률에 대한 사전 분포나 사후 분포를 나타내는 데 자주 사용됩니다.
*   **최빈값 (Mode):** 확률 분포에서 확률 밀도 함수(probability density function, PDF) 또는 확률 질량 함수(probability mass function, PMF)가 최대가 되는 값. 즉, 가장 발생 가능성이 높은 값을 의미합니다.
* **켤레사전분포(Conjugate Prior Distribution):** 어떤 likelihood 함수(여기서는 이항분포)가 주어졌을 때, 켤레사전분포를 사용하면, prior distribution과 posterior distribution이 같은 분포 family에 속하게 됩니다. 예를 들어, 이항분포 likelihood에 대해서는 Beta distribution이 켤레사전분포입니다.

**풀이 (상세):**

1.  **사전 분포 설정:** 문제에서 사전 분포는 Beta(2, 2)로 주어졌습니다. 이는 θ에 대한 초기 믿음을 나타냅니다. α = 2, β = 2 이므로, 초기에는 앞면과 뒷면이 나올 확률이 비슷하다고 가정하는 것입니다.

2.  **데이터 관측 (우도):** 10번의 동전 던지기에서 7번 앞면(H), 3번 뒷면(T)이 나왔습니다. 이는 성공 확률이 θ인 베르누이 시행을 10번 반복한 결과이며, 이항 분포를 따릅니다.

3.  **사후 분포 계산:** 베이즈 정리에 따르면, 사후 분포는 사전 분포와 우도(likelihood)의 곱에 비례합니다. 베타 분포는 이항 분포의 켤레 사전 분포이므로, 사후 분포도 베타 분포를 따릅니다.

    사후 분포의 매개변수는 다음과 같이 업데이트됩니다:

    *   α<sub>posterior</sub> = α<sub>prior</sub> + number of heads = 2 + 7 = 9
    *   β<sub>posterior</sub> = β<sub>prior</sub> + number of tails = 2 + 3 = 5

    따라서 사후 분포는 Beta(9, 5)가 됩니다.

4.  **MAP 추정:** MAP 추정은 사후 분포를 최대로 만드는 θ 값을 찾는 것입니다. 베타 분포의 경우, 최빈값(mode)이 MAP 추정값이 됩니다.

    최빈값 공식을 사용하여 MAP 추정값을 계산합니다:

    $mode = \frac{\alpha_{posterior} - 1}{\alpha_{posterior} + \beta_{posterior} - 2} = \frac{9 - 1}{9 + 5 - 2} = \frac{8}{12} = \frac{2}{3}$

**결론:**

MAP 추정을 통해 얻은 앞면이 나올 확률 θ의 추정값은 2/3입니다. 이는 사전 믿음(Beta(2, 2))과 관측 데이터(7번 앞면, 3번 뒷면)를 종합적으로 고려한 결과입니다. 초기에는 앞면과 뒷면이 나올 확률이 비슷하다고 가정했지만, 데이터를 관측한 후에는 앞면이 나올 확률이 더 높다고 추정하게 된 것입니다.

**문제 5:**

스팸 메일 필터링 시스템에서, "무료" 라는 단어가 포함된 메일 중 80%가 스팸 메일이고, "무료" 라는 단어가 포함되지 않은 메일 중 10%가 스팸 메일이라고 한다.  전체 메일 중 스팸 메일의 비율은 20%라고 가정한다. 새로 수신된 메일에 "무료" 라는 단어가 포함되어 있다면, 이 메일은 스팸일 확률과 스팸이 아닐 확률 중 어느쪽이 더 높은가?

**답안 5:**

*   c1 = 스팸, c2 = 스팸아님
*   d = "무료" 라는 단어 포함

*   P(c1) = 0.2 (스팸 메일의 사전 확률)
*   P(c2) = 0.8 (스팸이 아닌 메일의 사전 확률)
*   P(d|c1) = 0.8 ("무료" 단어가 포함된 메일 중 스팸 메일의 비율)
*   P(d|c2) = 0.2 (1 - 0.8, "무료"가 포함된 메일중 스팸이 아닌 메일의 비율, "무료"가 포함된 메일 중 스팸이 아닌 메일은 전체 스팸이 아닌 메일 중 20% 이므로)
*   P(¬d|c2) = 0.9, ¬d="무료"라는 단어 미포함

*   P(c1|d) ∝ P(d|c1)P(c1) = 0.8 * 0.2 = 0.16
*   P(c2|d) ∝ P(d|c2)P(c2) = 0.2 * 0.8 = 0.16
*   값이 같으므로, "무료"라는 단어가 포함된 경우 스팸일 확률과 스팸이 아닐 확률이 동일하다.

# $P(x|y) = \frac{P(y|x)P(x)}{P(y)} $ (4.2)
## 공식 (4.2) 설명:

이 공식은 조건부 확률 (Conditional Probability) 간의 관계를 나타내는 **베이즈 정리 (Bayes' theorem)** 입니다. 베이즈 정리는 확률 이론에서 가장 중요한 정리 중 하나이며, 데이터 분석, 머신 러닝, 통계학 등 다양한 분야에서 널리 활용됩니다.

**각 부분별 의미:**

*   **$P(x|y)$**:  사후 확률 (Posterior Probability). 사건 *y*가 주어졌을 때, 사건 *x*가 발생할 확률.
*   **$P(y|x)$**:  우도 (Likelihood). 사건 *x*가 주어졌을 때, 사건 *y*가 발생할 확률.
*   **$P(x)$**:  사전 확률 (Prior Probability). 사건 *y*와 관계없이 사건 *x*가 발생할 확률 (일반적인 믿음).
*   **$P(y)$**:  증거 (Evidence).  사건 *x*와 관계없이 사건 *y*가 발생할 확률.

**전체적인 의미 해석:**

"사건 *y*가 발생했다는 정보가 주어졌을 때, 사건 *x*가 발생할 확률(사후 확률)은, 사건 *x*가 발생했을 때 사건 *y*가 발생할 확률(우도)과 사건 *x*가 발생할 사전 확률의 곱을 사건 *y*가 발생할 확률(증거)로 나눈 값과 같다."

**쉽게 풀어 설명하면:**

이미 발생한 사건 *y*를 관찰한 후에, 어떤 사건 *x*가 발생했을 가능성을 업데이트하는 방법입니다.  베이즈 정리는 기존의 믿음(사전 확률)에 새로운 증거(우도)를 반영하여, 더 정확한 믿음(사후 확률)을 얻을 수 있게 해줍니다.

**일상 생활에서의 예시:**

*   **의학 진단:**  특정 질병(x)을 가진 환자가 특정 증상(y)을 보일 확률을 알고 있고, 전체 인구에서 해당 질병의 유병률(P(x))을 알고 있다면, 어떤 사람이 특정 증상을 보였을 때(y) 실제로 그 질병을 가지고 있을 확률(P(x|y))을 베이즈 정리를 통해 계산할 수 있습니다.
* **스팸 메일:** "무료"라는 단어가 나타나는 확률과 스팸메일의 확률로 부터, "무료"라는 단어가 나타났을때의 스팸메일일 확률을 알 수 있다.

## 예상 문제 및 답안:

**문제 1:**

베이즈 정리의 각 항(사후 확률, 우도, 사전 확률, 증거)에 대해 설명하고, 각 항이 의미하는 바를 간단한 예시와 함께 설명하세요.

**답안 1:**

*   **사후 확률 (Posterior Probability):** 어떤 사건이 발생했다는 조건 하에 다른 사건이 발생할 확률. 예) 특정 증상이 나타난 환자가 실제로 특정 질병을 가지고 있을 확률.
*   **우도 (Likelihood):** 어떤 사건이 발생했다는 가정 하에, 관측된 데이터(증거)가 나타날 확률. 예) 특정 질병을 가진 환자가 특정 증상을 보일 확률.
*   **사전 확률 (Prior Probability):** 관측된 데이터(증거)와 관계없이 어떤 사건이 발생할 확률에 대한 초기 믿음. 예) 전체 인구에서 특정 질병의 유병률.
*   **증거 (Evidence):** 관측된 데이터(증거)가 나타날 확률. 예) 전체 인구에서 특정 증상이 나타날 확률.

**문제 2:**

어떤 질병 A의 유병률(P(A))은 1%라고 알려져 있습니다. 이 질병에 대한 검사 방법이 있는데, 질병이 있는 사람에게 양성 반응(P(Pos|A))을 보일 확률은 99%이고, 질병이 없는 사람에게 음성 반응(P(Neg|¬A))을 보일 확률은 95%입니다. 어떤 사람이 이 검사에서 양성 반응(Pos)을 보였다면, 실제로 이 사람이 질병 A를 가지고 있을 확률(P(A|Pos))은 얼마인가요?

**답안 2:**

베이즈 정리를 사용하여 P(A|Pos)를 계산합니다.

*   P(A) = 0.01 (질병 A의 유병률 - 사전 확률)
*   P(¬A) = 0.99 (질병 A가 없을 확률)
*   P(Pos|A) = 0.99 (질병이 있는 사람에게 양성 반응 확률 - 우도)
*   P(Neg|¬A) = 0.95 (질병이 없는 사람에게 음성 반응 확률)
*   P(Pos|¬A) = 1 - P(Neg|¬A) = 0.05 (질병이 없는 사람에게 양성 반응 확률)

먼저, P(Pos) (증거)를 계산해야 합니다:

P(Pos) = P(Pos|A)P(A) + P(Pos|¬A)P(¬A) = (0.99 * 0.01) + (0.05 * 0.99) = 0.0099 + 0.0495 = 0.0594

이제 베이즈 정리를 적용합니다:

P(A|Pos) =  [P(Pos|A)P(A)] / P(Pos) = (0.99 * 0.01) / 0.0594 = 0.0099 / 0.0594 ≈ 0.1667

따라서 검사에서 양성 반응을 보인 사람이 실제로 질병 A를 가지고 있을 확률은 약 16.67%입니다.  검사의 정확도가 높아 보이지만, 질병의 유병률이 매우 낮기 때문에 양성 반응이 나왔더라도 실제로 질병이 있을 확률은 상대적으로 낮습니다.

**문제 3:**

어떤 공장에서 생산되는 제품의 불량률은 5%입니다. 불량품을 검사하는 기계가 있는데, 불량품을 불량품으로 판정할 확률(P(Defective|Defective))은 90%이고, 정상 제품을 정상 제품으로 판정할 확률(P(Good|Good))은 80%입니다. 이 기계가 어떤 제품을 불량품으로 판정했다면, 실제로 이 제품이 불량품일 확률은 얼마인가요?

**답안 3:**

*   D: 불량품, G: 정상 제품
*   P(D) = 0.05 (불량률 - 사전 확률)
*   P(G) = 0.95 (정상 제품일 확률)
*   P(Defective|D) = 0.90 (불량품을 불량품으로 판정할 확률 - 우도)
*   P(Good|G) = 0.80 (정상 제품을 정상 제품으로 판정할 확률)
*   P(Defective|G) = 1- P(Good|G) = 0.2 (정상 제품인데 불량품으로 판정)

P(Defective) = P(Defective|D)P(D) + P(Defective|G)P(G)
=(0.9 * 0.05) + (0.2 * 0.95) = 0.045 + 0.19 = 0.235

> **💡 보충 설명: 전체 확률의 법칙**
>
> **P(Defective) = P(Defective|D) + P(Defective|G) 처럼 계산하면 안 되는 이유**는, 이 식이 *전체 확률의 법칙 (Law of Total Probability)* 을 올바르게 적용하지 않았기 때문입니다.
>
> **전체 확률의 법칙**은 어떤 사건의 확률을, 그 사건이 발생할 수 있는 모든 가능한 경우들을 나누어 계산하는 방법입니다. 이 문제에서는 "제품이 불량품으로 판정되는 사건 (Defective)"이 발생할 수 있는 경우는 두 가지입니다.
>
> 1.  실제로 불량품(D)이고, 기계가 불량품으로 판정(Defective)하는 경우
> 2.  실제로 정상 제품(G)인데, 기계가 불량품으로 판정(Defective)하는 경우
>
> 이 두 경우는 서로 배타적(mutually exclusive)이며, 모든 가능성을 포함합니다(exhaustive). 따라서 전체 확률의 법칙에 따라 P(Defective)를 계산하려면, 각 경우의 확률을 더해야 합니다.
>
> **그런데 각 경우의 확률은 단순한 조건부 확률 P(Defective|D)나 P(Defective|G)가 아닙니다.** 왜냐하면:
>
> *   **P(Defective|D):** "불량품이라는 조건 하에서 불량품으로 판정될 확률"입니다. 즉, 이미 불량품으로 *가정*하고 시작하는 확률입니다.
> *   **P(Defective|G):** "정상 제품이라는 조건 하에서 불량품으로 판정될 확률"입니다. 즉, 이미 정상 제품으로 *가정*하고 시작하는 확률입니다.
>
> 우리가 구해야 하는 것은 *가정 없이* "불량품으로 판정될 확률"입니다. 따라서 각 경우의 확률을 계산하려면, 조건부 확률뿐만 아니라, *그 조건이 실제로 발생할 확률* (즉, 불량품일 확률 P(D) 또는 정상 제품일 확률 P(G))을 함께 고려해야 합니다.
>
> **따라서 올바른 식은 다음과 같습니다:**
>
> P(Defective) = P(Defective|D)P(D) + P(Defective|G)P(G)
>
> *   **P(Defective|D)P(D):**  "불량품이면서, 기계가 불량품으로 판정할 확률"입니다. 불량품일 확률(P(D))과 불량품을 불량품으로 판정할 확률(P(Defective|D))을 곱하여 계산합니다.
> *    **P(Defective|G)P(G):** "정상 제품인데, 기계가 불량품으로 판정할 확률"입니다. 정상 제품일 확률(P(G))과 정상 제품을 불량품으로 판정할 확률(P(Defective|G))을 곱하여 계산합니다.
>
> **요약:**
>
> 전체 확률의 법칙을 사용할 때는, 각 조건부 확률에 그 조건이 발생할 확률(사전 확률)을 반드시 곱해야 합니다. 그렇지 않으면, 각 경우가 전체 확률에 기여하는 정도를 올바르게 반영할 수 없습니다.

P(D|Defective) = [P(Defective|D) * P(D)]/ P(Defective)
= (0.9 * 0.05) / 0.235 = 0.045 / 0.235 ≈ 0.1915

따라서 기계가 불량품으로 판정한 제품이 실제로 불량품일 확률은 약 19.15%입니다.

## 베이즈 정리 계산 문제 5가지

**문제 1:**

어떤 도시의 택시는 85%가 파란색이고 15%가 녹색입니다. 밤에 택시와 관련된 뺑소니 사고가 발생했고, 목격자는 택시가 녹색이었다고 증언했습니다. 법원은 목격자가 야간에 택시 색깔을 정확하게 식별할 확률이 80%라고 판단했습니다. 뺑소니 사고를 낸 택시가 실제로 녹색이었을 확률은 얼마일까요?

**답안 1:**

*   G: 택시가 녹색, B: 택시가 파란색
*   W_G: 목격자가 녹색이라고 증언, W_B: 목격자가 파란색이라고 증언

*   P(G) = 0.15 (녹색 택시 비율 - 사전 확률)
*   P(B) = 0.85 (파란색 택시 비율)
*   P(W_G|G) = 0.80 (녹색 택시를 녹색이라고 증언할 확률 - 우도)
*   P(W_B|B) = 0.80 (파란색 택시를 파란색이라고 증언할 확률)
*   P(W_G|B) = 0.20 (파란색 택시를 녹색이라고 증언할 확률)

P(W_G) = P(W_G|G)P(G) + P(W_G|B)P(B) = (0.80 * 0.15) + (0.20 * 0.85) = 0.12 + 0.17 = 0.29

P(G|W_G) = [P(W_G|G)P(G)] / P(W_G) = (0.80 * 0.15) / 0.29 = 0.12 / 0.29 ≈ 0.4138

따라서 뺑소니 택시가 실제로 녹색이었을 확률은 약 41.38%입니다.

**문제 2:**

어떤 희귀 질병의 유병률은 0.1%입니다. 이 질병을 진단하는 검사는 질병이 있는 사람에게 99%의 확률로 양성 반응을 보이고, 질병이 없는 사람에게 98%의 확률로 음성 반응을 보입니다. 어떤 사람이 이 검사에서 양성 반응을 보였다면, 이 사람이 실제로 이 질병을 앓고 있을 확률은 얼마일까요?

**답안 2:**

*   D: 질병 있음, ¬D: 질병 없음
*   Pos: 양성 반응, Neg: 음성 반응

*   P(D) = 0.001 (유병률)
*   P(¬D) = 0.999
*   P(Pos|D) = 0.99
*   P(Neg|¬D) = 0.98
*   P(Pos|¬D) = 1 - P(Neg|¬D) = 0.02

P(Pos) = P(Pos|D)P(D) + P(Pos|¬D)P(¬D) = (0.99 * 0.001) + (0.02 * 0.999) = 0.00099 + 0.01998 = 0.02097

P(D|Pos) = [P(Pos|D)P(D)] / P(Pos) = (0.99 * 0.001) / 0.02097 = 0.00099 / 0.02097 ≈ 0.0472

따라서 검사에서 양성 반응을 보인 사람이 실제로 질병을 앓고 있을 확률은 약 4.72%입니다.

**문제 3:**

두 개의 주머니 A와 B가 있습니다. 주머니 A에는 빨간 공 3개와 파란 공 2개가 들어있고, 주머니 B에는 빨간 공 2개와 파란 공 5개가 들어있습니다. 임의로 주머니 하나를 선택하고, 그 주머니에서 임의로 공 하나를 꺼냈더니 빨간 공이었습니다. 이 공이 주머니 A에서 나왔을 확률은 얼마일까요?

**답안 3:**

*   A: 주머니 A를 선택, B: 주머니 B를 선택
*   R: 빨간 공, Bl: 파란 공

*   P(A) = 0.5 (주머니 A를 선택할 확률)
*   P(B) = 0.5 (주머니 B를 선택할 확률)
*   P(R|A) = 3/5 (주머니 A에서 빨간 공을 꺼낼 확률)
*   P(R|B) = 2/7 (주머니 B에서 빨간 공을 꺼낼 확률)

P(R) = P(R|A)P(A) + P(R|B)P(B) = (3/5 * 0.5) + (2/7 * 0.5) = 0.3 + 0.1429 = 0.4429

P(A|R) = [P(R|A)P(A)] / P(R) = (3/5 * 0.5) / 0.4429 = 0.3 / 0.4429 ≈ 0.6774

따라서 빨간 공이 주머니 A에서 나왔을 확률은 약 67.74%입니다.

**문제 4:**

세 개의 상자 A, B, C가 있습니다. 각 상자에는 두 개의 서랍이 있습니다. 상자 A의 각 서랍에는 금화가 하나씩 들어있고, 상자 B의 각 서랍에는 은화가 하나씩 들어있고, 상자 C에는 한 서랍에 금화, 다른 서랍에 은화가 들어있습니다. 임의로 상자 하나를 선택하고, 그 상자의 서랍 하나를 열었더니 금화가 나왔습니다. 이 상자의 다른 서랍에도 금화가 들어있을 확률은 얼마일까요?

**답안 4:**

* A, B, C: 각 상자를 선택하는 사건
* G: 금화, S: 은화

* P(A) = P(B) = P(C) = 1/3
* P(G|A) = 1 (A 상자에서 금화가 나올 확률)
* P(G|B) = 0 (B 상자에서 금화가 나올 확률)
* P(G|C) = 1/2 (C 상자에서 금화가 나올 확률)

P(G) = P(G|A)P(A) + P(G|B)P(B) + P(G|C)P(C)
      = (1 * 1/3) + (0 * 1/3) + (1/2 * 1/3) = 1/3 + 0 + 1/6 = 1/2

P(A|G) = [P(G|A)P(A)]/P(G) = (1 * 1/3) / (1/2) = (1/3) / (1/2) = 2/3

금화가 든 서랍을 열었을때, 다른 서랍에도 금화가 있는 경우는 선택한 상자가 A인 경우이다.
따라서, 이 상자의 다른 서랍에도 금화가 들어있을 확률은 P(A|G) 와 같으므로 2/3 입니다.

**문제 5:**

어떤 이메일 서비스 사용자의 60%가 스팸 필터를 사용하고, 40%는 사용하지 않습니다. 스팸 필터를 사용하는 사용자 중 90%가 스팸 메일을 스팸 폴더로 분류하고, 스팸 필터를 사용하지 않는 사용자 중 20%가 스팸 메일을 스팸 폴더로 분류한다고 합니다(수동으로 분류). 전체 스팸 메일 중 스팸 폴더로 분류된 스팸 메일의 비율은 얼마일까요? (단, 사용자가 받는 메일 중 스팸 메일과 정상 메일의 비율은 알 수 없습니다.)

**답안 5:**
이 문제는 베이즈 정리를 직접 사용하는 문제는 아니지만, 조건부 확률과 전체 확률의 법칙을 사용하여 풀 수 있습니다.

*   F: 스팸 필터 사용, ¬F: 스팸 필터 미사용
*   S: 스팸 메일, ¬S: 정상 메일
*   Spam_folder: 스팸 폴더로 분류

*   P(F) = 0.6
*   P(¬F) = 0.4
*   P(Spam_folder|F, S) = 0.9 (필터 사용자가 스팸 메일을 스팸 폴더로 분류할 확률)
*   P(Spam_folder|¬F, S) = 0.2 (필터 미사용자가 스팸 메일을 스팸 폴더로 분류할 확률)

전체 스팸 메일 중 스팸 폴더로 분류된 스팸 메일의 비율을 구하는 것이므로, P(Spam_folder|S)를 구해야 합니다. 전체 확률의 법칙(Law of Total Probability)에 따라 다음과 같이 나타낼 수 있습니다.

P(Spam_folder|S) = P(Spam_folder|F, S)P(F) + P(Spam_folder|¬F, S)P(¬F)
                   = (0.9 * 0.6) + (0.2 * 0.4)
                   = 0.54 + 0.08
                   = 0.62

따라서 전체 스팸 메일 중 스팸 폴더로 분류된 스팸 메일의 비율은 62%입니다.

# $\hat{c} = \underset{c \in C}{\operatorname{argmax}} P(c|d) = \underset{c \in C}{\operatorname{argmax}} \frac{P(d|c)P(c)}{P(d)} $ (4.3)

## 공식 (4.3) 설명:

이 공식은 주어진 문서(데이터) *d*가 주어졌을 때, 가장 확률이 높은 카테고리(클래스) *c*를 찾는 최대 사후 확률 (Maximum A Posteriori, MAP) 추정 방법을 나타냅니다. 공식 (4.1)에서 베이즈 정리를 적용하여 사후 확률 P(c|d)를 우도, 사전확률, 증거로 나타낸 형태입니다.

**각 부분별 의미:**

*   **$\hat{c}$**:  추정된 최적의 카테고리 (예측값).
*   **$c$**:  특정 카테고리.
*   **$C$**:  가능한 모든 카테고리의 집합.
*   **$\operatorname{argmax}$**:  함수를 최대로 만드는 변수(argument)를 반환하는 연산자.
*   **$P(c|d)$**:  사후 확률 (Posterior Probability). 문서 *d*가 주어졌을 때, 이 문서가 카테고리 *c*에 속할 확률.
*   **$P(d|c)$**:  우도 (Likelihood). 카테고리 *c*가 주어졌을 때, 문서 *d*가 나타날 확률.
*   **$P(c)$**:  사전 확률 (Prior Probability).  문서와 관계없이 카테고리 *c*가 나타날 확률 (일반적인 분포).
*   **$P(d)$**:  증거 (Evidence).  문서 *d*가 나타날 확률.
*   **$d$**: 주어진 문서(데이터).

**전체적인 의미 해석:**

"주어진 문서 *d*에 대해, 가능한 모든 카테고리 *c* 중에서 사후 확률 P(c|d)를 최대로 만드는 카테고리 *c*를 찾는다. 사후확률은 베이즈 정리에 의해 $\frac{P(d|c)P(c)}{P(d)}$ 와 같으므로, 이를 최대로 만드는 *c*를 찾아 $\hat{c}$ (예측된 카테고리)로 결정한다."

**P(d)가 생략 가능한 이유:**

$\operatorname{argmax}$ 연산은 P(c|d)를 최대로 만드는 *c*를 찾는 것이 목적입니다. 그런데 P(d)는 주어진 문서 *d*에 대한 확률로, 카테고리 *c*와는 무관한 상수입니다. 즉, 모든 *c*에 대해 P(d)는 동일한 값으로 나누어지므로, 최댓값을 결정하는 데 영향을 주지 않습니다. 따라서 argmax 연산을 할 때는 P(d)를 생략하고, P(d|c)P(c)를 최대로 만드는 *c*를 찾아도 결과는 같습니다.

> **핵심 이유: P(d)는 모든 카테고리 c에 대해 동일한 상수**
>
> $\operatorname{argmax}_c P(c|d)$ 를 계산하는 것은 주어진 문서 *d*가 주어졌을 때 확률을 최대로 만드는 카테고리 *c*를 찾는 것입니다. 베이즈 정리에 의해 $P(c|d) = \frac{P(d|c)P(c)}{P(d)}$ 입니다.  여기서 분모의 P(d)는 '문서 d가 나타날 확률'인데, 이는 우리가 어떤 카테고리 *c*를 선택하든 변하지 않는 값입니다.  *c*에 대해 상수로 취급될 수 있습니다.
>
> **비유를 통한 설명**
>
> 예를 들어, 100미터 달리기 시합에서 누가 1등을 할지 예측하는 상황을 생각해 봅시다.
>
> *   **목표:** 가장 빨리 결승선에 도착하는 선수(최단 시간)를 찾는 것
> *   **P(c|d):** 각 선수(*c*)가 특정 출발 신호(*d*)를 들었을 때 1등을 할 확률
> *   **P(d|c):** 각 선수(*c*)가 1등을 했을 때, 그 출발 신호(*d*)가 발생했을 확률 (선수가 1등하는 능력)
> *   **P(c):** 각 선수(*c*)가 애초에 1등을 할 확률 (선수의 기본 실력)
> *   **P(d):** 출발 신호(*d*)가 울릴 확률 (모든 선수에게 동일)
>
> 여기서 출발 신호는 모든 선수에게 동일하게 주어집니다. 즉, P(d)는 모든 선수에게 동일한 상수입니다. 누가 1등을 할지는 각 선수의 능력(P(d|c)와 P(c))에 따라 결정되지, 출발 신호 자체(P(d))와는 무관합니다. 따라서 P(d)를 고려하지 않고, P(d|c)P(c)만 비교해도 1등을 예측할 수 있습니다.
>
> **계산 예시**
>
> 두 개의 카테고리(c1, c2)와 문서 d가 있다고 가정하고, 다음과 같은 확률 값을 갖는다고 해봅시다.
>
> *   P(d|c1) = 0.05
> *   P(c1) = 0.6
> *   P(d|c2) = 0.1
> *   P(c2) = 0.4
> *   P(d) = 0.05 (계산 편의상 주어졌다고 가정, 실제로는 몰라도 됩니다.)
>
> 이제 P(c|d)를 직접 계산해 보겠습니다.
>
> *   **c1에 대해서:**
>     *   P(c1|d) = (P(d|c1) * P(c1)) / P(d) = (0.05 * 0.6) / 0.05 = 0.6
> *   **c2에 대해서:**
>     *   P(c2|d) = (P(d|c2) * P(c2)) / P(d) = (0.1 * 0.4) / 0.05 = 0.8
>
> $\operatorname{argmax}_c P(c|d)$ 는 c2입니다.
>
> 이제 P(d)를 생략하고 P(d|c)P(c)만 계산해 보겠습니다.
>
> *   **c1에 대해서:**
>     *   P(d|c1) * P(c1) = 0.05 * 0.6 = 0.03
> *   **c2에 대해서:**
>     *   P(d|c2) * P(c2) = 0.1 * 0.4 = 0.04
>
> $\operatorname{argmax}_c P(d|c)P(c)$ 역시 c2입니다.
>
> 보시다시피 P(d)를 포함하여 계산하나, 생략하고 P(d|c)P(c)만 계산하나 $\operatorname{argmax}$ 결과는 같습니다. P(d)는 모든 카테고리에 대해 같은 값으로 나누어주는 역할을 할 뿐, 최댓값을 결정하는 데는 아무런 영향을 주지 않기 때문입니다. 따라서 계산의 효율성을 위해 P(d)를 생략하는 것입니다.

**실제 계산:**

실제로는 다음과 같이 P(d)를 생략한 형태를 사용합니다.

$\hat{c} = \underset{c \in C}{\operatorname{argmax}} P(d|c)P(c)$

즉, 우도와 사전 확률의 곱을 최대로 만드는 카테고리를 찾습니다.

## 예상 문제 및 답안:

**문제 1:**

공식 (4.3)에서 P(d)를 생략하고 계산해도 되는 이유를 설명하세요.

**답안 1:**

P(d)는 주어진 문서 d에 대한 확률로, 카테고리 c와는 무관한 상수입니다.  $\operatorname{argmax}$ 연산은 P(c|d)를 최대로 만드는 c를 찾는 것이 목적인데, 모든 c에 대해 P(d)는 동일한 값으로 나누어지므로, 최댓값을 만드는 c의 값을 바꾸지 않습니다. 따라서 P(d)를 생략하고 P(d|c)P(c)를 최대로 만드는 c를 찾아도 결과는 같습니다.

**문제 2:**

MAP 추정 방법에서 우도(P(d|c))와 사전 확률(P(c))이 하는 역할을 설명하고, 이 두 요소가 어떻게 상호작용하여 최종 결정($\hat{c}$)에 영향을 미치는지 설명하세요.

**답안 2:**

*   **우도 (P(d|c)):**  특정 카테고리 c가 주어졌을 때, 관측된 문서 d가 나타날 확률입니다.  문서 d의 내용이 카테고리 c와 얼마나 관련성이 높은지를 나타냅니다.
*   **사전 확률 (P(c)):**  문서 d와 관계없이, 카테고리 c가 나타날 확률입니다.  일반적으로 카테고리 c가 얼마나 흔한지를 나타냅니다.

MAP 추정에서는 우도와 사전 확률의 곱(P(d|c)P(c))을 최대로 만드는 카테고리 c를 선택합니다.  우도가 높으면 해당 카테고리가 문서 d의 내용을 잘 설명한다는 의미이고, 사전 확률이 높으면 해당 카테고리가 일반적으로 더 흔하다는 의미입니다.  이 두 요소가 서로 상호작용하여 최종 결정에 영향을 미칩니다. 예를 들어, 어떤 카테고리의 우도가 매우 높더라도 사전 확률이 매우 낮다면, 다른 카테고리가 선택될 수 있습니다. 반대로, 사전 확률이 높더라도 우도가 너무 낮으면 다른 카테고리가 선택될 수 있습니다.

**문제 3:**

스팸 메일 분류 문제를 예로 들어, 공식 (4.3)의 각 요소(c, C, d, P(d|c), P(c), P(d))가 무엇을 의미하는지 구체적으로 설명하세요.

**답안 3:**

*   **c:** "스팸" 또는 "정상(햄)"과 같은 메일의 카테고리.
*   **C:** {"스팸", "정상"}과 같이 가능한 모든 카테고리의 집합.
*   **d:** 새로 들어온 이메일의 내용 (텍스트, 제목, 발신자 등).
*   **P(d|c):** 특정 카테고리(스팸 또는 정상)가 주어졌을 때, 해당 이메일 내용(d)이 나타날 확률. 예를 들어, "스팸" 메일에서 "광고", "무료" 등의 단어가 나타날 확률이 높고, "정상" 메일에서는 "회의", "보고서" 등의 단어가 나타날 확률이 높을 것입니다.
*   **P(c):**  각 카테고리(스팸 또는 정상)의 일반적인 비율. 예를 들어, 전체 메일 중 20%가 스팸이고 80%가 정상이라면, P(스팸) = 0.2, P(정상) = 0.8입니다.
*   **P(d):**  특정 이메일 내용(d)이 나타날 확률.  스팸/정상 여부와 관계없이 계산되며, 베이즈 정리에서 정규화 상수 역할을 합니다. 실제 MAP 추정에서는 계산하지 않아도 됩니다.

**문제 4:**

두 개의 카테고리 A와 B가 있고, P(A) = 0.6, P(B) = 0.4 입니다. 문서 d가 주어졌을 때, P(d|A) = 0.2, P(d|B) = 0.3 이라고 합니다. MAP 추정을 사용하여 문서 d를 분류하세요.

**답안 4:**

MAP 추정에서는 P(d|c)P(c)를 최대로 만드는 카테고리 c를 선택합니다.

*   P(d|A)P(A) = 0.2 * 0.6 = 0.12
*   P(d|B)P(B) = 0.3 * 0.4 = 0.12

P(d|A)P(A) = P(d|B)P(B) 이므로, 두 카테고리의 사후 확률이 같습니다. 이 경우, 일반적으로는 사전 확률이 더 높은 카테고리를 선택하거나(이 문제에서는 두 사후확률이 같으므로 어떤 것을 선택해도 무방), 무작위로 선택하거나, 다른 기준을 적용할 수 있습니다.

**문제 5:**

세 개의 카테고리 X, Y, Z가 있고, 사전 확률은 P(X) = 0.3, P(Y) = 0.5, P(Z) = 0.2 입니다. 문서 d에 대한 우도는 다음과 같습니다:

*   P(d|X) = 0.4
*   P(d|Y) = 0.2
*   P(d|Z) = 0.6

MAP 추정 방법을 사용하여 문서 d를 분류하세요.

**답안 5:**

*   P(d|X)P(X) = 0.4 * 0.3 = 0.12
*   P(d|Y)P(Y) = 0.2 * 0.5 = 0.10
*   P(d|Z)P(Z) = 0.6 * 0.2 = 0.12

P(d|X)P(X)와 P(d|Z)P(Z)가 0.12로 가장 크고 P(d|Y)P(Y) 보다 크므로, 문서 d는 카테고리 X 또는 Z로 분류될 수 있습니다.
___
$\hat{c} = \underset{c \in C}{\operatorname{argmax}} P(c|d) = \underset{c \in C}{\operatorname{argmax}} P(d|c)P(c) \qquad (4.4)$
___
### 공식 (4.4) 설명:

이 공식은 주어진 문서 d가 주어졌을 때, 최적의 카테고리(클래스) $\hat{c}$ 를 찾는 방법을 나타냅니다.  두 개의 $\operatorname{argmax}$ 표현은 동일한 결과를 낳습니다.

*   **$\hat{c}$:**  "c hat"이라고 읽으며, 예측된(추정된) 최적의 카테고리를 의미합니다.
*   **$\underset{c \in C}{\operatorname{argmax}}$:**  집합 C에 속하는 모든 가능한 카테고리 c 중에서,  함수의 값을 최대로 만드는 c를 찾으라는 의미입니다.
*   **P(c|d):**  문서 d가 주어졌을 때, 이 문서가 카테고리 c에 속할 확률 (사후 확률)입니다.
*   **P(d|c):** 카테고리 c가 주어졌을 때, 문서 d가 나타날 확률 (가능도, likelihood)입니다.
*   **P(c):**  카테고리 c가 나타날 확률 (사전 확률)입니다.

**첫 번째 표현 ($\underset{c \in C}{\operatorname{argmax}} P(c|d)$):**

이 표현은 주어진 문서 d에 대해, 각 카테고리 c의 사후 확률 P(c|d)를 계산하고, 그 확률을 최대로 만드는 카테고리 c를 선택하는 것이 최적의 예측 ($\hat{c}$)이라는 것을 의미합니다.  직관적으로 가장 이해하기 쉬운 표현입니다.

**두 번째 표현 ($\underset{c \in C}{\operatorname{argmax}} P(d|c)P(c)$):**

첫 번째 표현에서 P(c|d)는 베이즈 정리에 의해 $\frac{P(d|c)P(c)}{P(d)}$로 나타낼 수 있습니다.  그런데 $\operatorname{argmax}$ 연산에서 P(d)는 모든 c에 대해 동일한 상수이므로 (앞서 설명한 것처럼) 최댓값을 찾는 데 영향을 주지 않습니다.  따라서 P(d)를 생략하고 P(d|c)P(c)를 최대로 만드는 c를 찾아도 같은 결과를 얻습니다.  이것이 두 번째 표현입니다.

**핵심:**  실제 계산에서는 P(c|d)를 직접 구하기 어려운 경우가 많기 때문에, 베이즈 정리를 이용하여 P(d|c)와 P(c)를 통해 간접적으로 최적의 카테고리를 찾습니다. P(d)는 계산할 필요가 없으므로 효율적입니다.

### 예상 문제 및 답안

**예상 문제 1:**

어떤 스팸 메일 필터링 시스템에서, 다음과 같은 확률들이 주어졌다고 가정합니다.

*   P("스팸"| "광고") = 0.9
*   P("스팸" | "정상") = 0.01
*   P("광고") = 0.2
*   P("정상") = 0.8
*    P("무료") = 0.4
* P("무료"| "스팸") = 0.7
* P("무료"| "정상")=0.01

새로운 메일 "무료 광고"가 도착했을 때, 이 메일이 스팸일 확률과 정상 메일일 확률 중 어떤 것이 더 높은가? 그리고 그 이유는 무엇인가?

**답안 1:**

공식 (4.4)를 적용하여 "무료 광고" 메일이 스팸(c1)인지 정상(c2)인지 판별합니다.  "무료 광고"를 d라고 표현하겠습니다.

1.  **P(d|c)P(c) 계산 (P(d) 생략):**

    *   스팸 (c1 = "스팸")일 경우:  "무료"와 "광고" 단어가 독립이라고 가정하면:
        P(d|c1)P(c1) = P("무료"|"스팸") * P("광고"|"스팸") * P("스팸")
        여기서,P("스팸") = P("스팸"| "광고")* P("광고") + P("스팸" | "정상") * P("정상") = (0.9 * 0.2) + (0.01 * 0.8) = 0.188

        P(d|c1)P(c1) = 0.7 * 0.9 * 0.188 = 0.11844

    *   정상 (c2 = "정상")일 경우:
        P(d|c2)P(c2) = P("무료"|"정상") * P("광고"|"정상") * P("정상")
         P("정상")은 이미 주어졌고, P("광고"|"정상")을 구해야 함.
        P("광고") = P("광고"| "스팸") * P("스팸") + P("광고"| "정상") * P("정상") 에서,

       P("광고"| "스팸")는 P("스팸"| "광고") = 0.9 = P("스팸", "광고")/P("광고") => P("스팸", "광고")=0.9 * 0.2 = 0.18
        P("스팸", "광고") = P("광고"| "스팸") * P("스팸") => P("광고"| "스팸")  = 0.18 / 0.188 = 0.957

         P("광고") = 0.957 * 0.188 + P("광고"| "정상") * 0.8
        0.2 = 0.1799 +  P("광고"| "정상") * 0.8
        P("광고"| "정상") = 0.025

        P(d|c2)P(c2) =  0.01 * 0.025 * 0.8= 0.0002

2.  **argmax 비교:**

    P(d|"스팸")P("스팸") (0.11844) > P(d|"정상")P("정상") (0.0002)

3.  **결론:**

    "무료 광고" 메일은 정상 메일일 확률보다 스팸일 확률이 훨씬 더 높습니다.  이는 스팸 메일에서 "무료"와 "광고"라는 단어가 나타날 확률(가능도)과 스팸 메일의 사전 확률의 곱이, 정상 메일에서 이 단어들이 나타날 확률과 정상 메일의 사전 확률의 곱보다 훨씬 크기 때문입니다.

**예상 문제 2:**

(문제 1과 동일한 조건에서) P(d)를 계산하고, P(c|d)를 계산하여, argmax P(c|d) 와 argmax P(d|c)P(c)의 값이 같은지 확인하시오.

**답안 2:**

1. **P(d) 계산:**
P(d) = P("무료 광고") = P("무료 광고", "스팸") + P("무료 광고","정상")
                     = P("무료 광고"|"스팸")P("스팸") + P("무료 광고"|"정상")P("정상")
                     = (0.7 * 0.9 * 0.188) + (0.01 * 0.025 * 0.8)
                     =  0.11844 + 0.0002
                     = 0.11864

2. **P(c|d) 계산:**

    *   P("스팸"|"무료 광고") = P("무료 광고"|"스팸")P("스팸") / P("무료 광고")  = 0.11844 / 0.11864 = 0.9983
    *    P("정상"|"무료 광고") =  P("무료 광고"|"정상")P("정상") /  P("무료 광고") = 0.0002 / 0.11864 = 0.00168

3. **argmax 비교:**

     argmax P(c|d) = "스팸"
     argmax P(d|c)P(c) = "스팸" (문제 1에서 계산)
     두 값이 같음을 확인

**핵심:** 문제 2는 P(d)를 명시적으로 계산하여 P(c|d)를 구해도, P(d)를 생략하고 P(d|c)P(c)만으로 구한 결과와 동일함을 보여줍니다.

*   **답:** "육류" 카테고리일 확률이 가장 높습니다.

$\hat{c} = \underset{c \in C}{\operatorname{argmax}} \overset{\text{likelihood}}{{P}(d|c)} \overset{\text{prior}}{{P}(c)} \qquad (4.5)$
___
### 공식 (4.5) 설명:

이 공식은 (4.4)와 본질적으로 동일하며, 나이브 베이즈 분류에서 최적의 카테고리 $\hat{c}$를 찾는 방법을 더 명확하게 표현한 것입니다.  주어진 문서 *d*에 대해, 각 카테고리 *c*에 대한 가능도(likelihood)와 사전 확률(prior)의 곱을 최대화하는 카테고리 *c*를 선택합니다.

*   **$\hat{c}$:** 예측된(추정된) 최적의 카테고리.
*   **$\underset{c \in C}{\operatorname{argmax}}$:** 집합 C (모든 가능한 카테고리 집합)에 속하는 모든 카테고리 c 중에서, 뒤에 오는 표현식(P(d|c)P(c))을 최대로 만드는 c를 찾으라는 의미.
*   **$\overset{\text{likelihood}}{{P}(d|c)}$:** 가능도(Likelihood). 카테고리 c가 주어졌을 때, 문서 d가 관측될 확률.  즉, 특정 카테고리 c에서 문서 d의 단어들이 나타날 확률.
*   **$\overset{\text{prior}}{{P}(c)}$:** 사전 확률(Prior Probability).  문서 d와는 상관없이, 카테고리 c가 나타날 확률.  일반적으로 훈련 데이터에서 각 카테고리의 빈도를 통해 추정.

**핵심:**

1.  **가능도 (Likelihood):** 특정 카테고리에서 주어진 문서의 단어들이 얼마나 자주 나타나는지를 나타냅니다.  예를 들어, "스포츠" 카테고리에서 "축구", "골" 같은 단어들이 많이 나타난다면, 이러한 단어들을 포함하는 문서에 대한 "스포츠" 카테고리의 가능도는 높아집니다.

2.  **사전 확률 (Prior):**  특정 카테고리가 애초에 얼마나 흔한지를 나타냅니다.  예를 들어, 전체 문서 중 "스포츠" 관련 문서가 많다면, "스포츠" 카테고리의 사전 확률은 높아집니다.

3.  **argmax:** 나이브 베이즈 분류기는 가능도와 사전 확률의 곱을 최대화하는 카테고리를 선택합니다.  이는 주어진 문서에 대해, 해당 카테고리가 나타날 확률(사전 확률)과 그 카테고리에서 이 문서의 단어들이 나타날 확률(가능도)을 모두 고려하여 가장 적합한 카테고리를 찾는다는 의미입니다.

### 예상 문제 및 답안

**예상 문제 1:**

두 개의 카테고리 "긍정" (pos)과 "부정" (neg)으로 영화 리뷰를 분류하는 나이브 베이즈 분류기가 있습니다. 다음 확률들이 주어졌습니다.

*   P("재밌다" | "pos") = 0.8
*   P("별로다" | "pos") = 0.1
*   P("재밌다" | "neg") = 0.2
*   P("별로다" | "neg") = 0.9
*   P("pos") = 0.6  (긍정 리뷰의 사전 확률)
*   P("neg") = 0.4  (부정 리뷰의 사전 확률)

새로운 리뷰에 "재밌다"와 "별로다"라는 단어가 모두 포함되어 있다면, 이 리뷰는 "긍정"과 "부정" 중 어느 카테고리로 분류되어야 합니까? ("재밌다"와 "별로다"는 독립이라고 가정)

**답안 1:**

공식 (4.5)를 적용하여 "재밌다 별로다" 리뷰가 긍정(c1)인지 부정(c2)인지 판별합니다. "재밌다 별로다"를 d라고 표현하겠습니다.

1.  **P(d|c)P(c) 계산:**

    *   긍정 (c1 = "pos")일 경우:
        P(d|c1)P(c1) = P("재밌다" | "pos") * P("별로다" | "pos") * P("pos") = 0.8 * 0.1 * 0.6 = 0.048

    *   부정 (c2 = "neg")일 경우:
        P(d|c2)P(c2) = P("재밌다" | "neg") * P("별로다" | "neg") * P("neg") = 0.2 * 0.9 * 0.4 = 0.072

2.  **argmax 비교:**

     P(d|"neg")P("neg") (0.072) > P(d|"pos")P("pos") (0.048)

3.  **결론:**

     "재밌다 별로다" 리뷰는 긍정 리뷰일 확률보다 부정 리뷰일 확률이 더 높으므로, 부정 카테고리로 분류됩니다.

**예상 문제 2:**

(문제 1과 동일한 조건) 만약 새로운 리뷰가 "재밌다 재밌다 재밌다" 라면 이 리뷰는 어느 카테고리로 분류되어야 하겠는가?

**답안 2:**

1.  **P(d|c)P(c) 계산:**

    *   긍정 (c1 = "pos")일 경우:
        P(d|c1)P(c1) = P("재밌다" | "pos") * P("재밌다" | "pos") * P("재밌다" | "pos") * P("pos") = 0.8 * 0.8 * 0.8 * 0.6 = 0.3072

    *   부정 (c2 = "neg")일 경우:
        P(d|c2)P(c2) = P("재밌다" | "neg") * P("재밌다" | "neg") * P("재밌다" | "neg") * P("neg") = 0.2 * 0.2 * 0.2 * 0.4 = 0.0032

2.  **argmax 비교:**

      P(d|"pos")P("pos") (0.3072) > P(d|"neg")P("neg") (0.0032)

3.  **결론:**
     "재밌다 재밌다 재밌다" 리뷰는 긍정 리뷰일 확률이 압도적으로 높으므로, 긍정 카테고리로 분류됩니다.

**예상 문제 3:**

세 개의 카테고리 A, B, C가 있고, 각 카테고리에 대한 사전 확률은 다음과 같습니다.
* P(A) = 0.3
* P(B) = 0.5
* P(C) = 0.2

문서 d에 대한 각 카테고리의 가능도(likelihood)는 다음과 같습니다.
* P(d|A) = 0.02
* P(d|B) = 0.01
* P(d|C) = 0.04

이 문서 d는 어떤 카테고리로 분류되어야 합니까?

**답안 3:**

각 카테고리에 대해 P(d|c)P(c)를 계산합니다.

*   A: 0.02 * 0.3 = 0.006
*   B: 0.01 * 0.5 = 0.005
*   C: 0.04 * 0.2 = 0.008

argmax 비교: 0.  008 (C) > 0.006 (A) > 0.005 (B)

결론: 문서 d는 카테고리 C로 분류됩니다.

$\hat{c} = \underset{c \in C}{\operatorname{argmax}}\overset{\text{likelihood}}{ P(f_1,f_2,...,f_n|c)}\overset{\text{prior}}{ P(c)}$ (4.6)
___
### 공식 (4.6) 설명:

이 공식은 나이브 베이즈 분류기의 핵심 공식을 좀 더 일반화된 형태로 나타낸 것입니다. 이전 공식들에서는 문서 *d*를 사용했지만, 여기서는 문서 *d*를 구성하는 특징(feature)들 $f_1, f_2, ..., f_n$을 명시적으로 사용합니다.

*   **$\hat{c}$:** 예측된(추정된) 최적의 카테고리.
*   **$\underset{c \in C}{\operatorname{argmax}}$:** 집합 C (모든 가능한 카테고리 집합)에 속하는 모든 카테고리 c 중에서, 뒤에 오는 표현식(P(f\_1, f\_2, ..., f\_n|c)P(c))을 최대로 만드는 c를 찾으라는 의미.
*   **$\overset{\text{likelihood}}{P(f_1, f_2, ..., f_n|c)}$:** 가능도(Likelihood). 카테고리 c가 주어졌을 때, 특징 벡터 $(f_1, f_2, ..., f_n)$이 관측될 확률. 즉, *특정 카테고리 c에서 주어진 특징들이 나타날 확률* 입니다. 나이브 베이즈 분류에서는 이 특징들이 서로 독립이라고 가정합니다.
*    **나이브 베이즈 가정**: $P(f_1, f_2, ..., f_n|c) = P(f_1|c)P(f_2|c)...P(f_n|c) = \prod_{i=1}^{n} P(f_i|c)$
*   **$\overset{\text{prior}}{P(c)}$:** 사전 확률(Prior Probability). 문서의 특징들과는 상관없이, 카테고리 c가 나타날 확률.

**핵심:**

1.  **특징 (Features):**  문서 분류 문제에서 특징은 주로 단어(word)가 됩니다.  하지만 단어뿐만 아니라, 단어의 빈도, 특정 구(phrase), 문서의 길이, 작성자 등 다양한 정보를 특징으로 사용할 수 있습니다.  예를 들어, 스팸 메일 분류에서는 "광고", "무료", "당첨" 등의 단어, 이메일 제목의 길이, 보낸 사람의 주소 등이 특징이 될 수 있습니다.
2.  **나이브 베이즈 가정 (독립성 가정):**  나이브 베이즈 분류기는 *주어진 카테고리 내에서* 특징들이 서로 독립이라고 가정합니다.  즉, 한 특징의 존재/부재가 다른 특징의 존재/부재에 영향을 주지 않는다고 가정합니다. 이 가정은 현실에서는 완전히 성립하지 않을 수 있지만 (예: "축구"와 "골"은 연관성이 높음), 계산을 매우 단순화하고, 많은 경우에 실제로 잘 작동합니다.
3.  **가능도 계산 (독립성 가정 하):**  $P(f_1, f_2, ..., f_n|c) = P(f_1|c) * P(f_2|c) * ... * P(f_n|c) = \prod_{i=1}^{n} P(f_i|c)$. 각 특징이 나타날 확률을 모두 곱합니다.

### 예상 문제 및 답안

**예상 문제 1:**

두 개의 카테고리 "과일" (fruit)과 "채소" (vegetable)가 있습니다.  문서의 특징으로 "빨간색", "둥근모양", "단맛" 세 가지를 사용합니다. 다음 확률들이 주어졌습니다.

*   P("빨간색" | "fruit") = 0.9

### 공식 (4.7) 설명:

$P(f_1, f_2, ..., f_n|c) = P(f_1|c) \cdot P(f_2|c) \cdot ... \cdot P(f_n|c)$ (4.7)

이 공식은 나이브 베이즈 분류기의 핵심 가정인 **조건부 독립(conditional independence)**을 나타내는 매우 중요한 식입니다.  이전 공식들에서 $P(f_1, f_2, ..., f_n|c)$ (카테고리 c가 주어졌을 때 특징 벡터 $(f_1, f_2, ..., f_n)$가 나타날 확률, 즉 가능도)를 어떻게 계산하는지를 구체적으로 보여줍니다.

*   **$P(f_1, f_2, ..., f_n|c)$:** 카테고리 *c*가 주어졌을 때, 특징 벡터 $(f_1, f_2, ..., f_n)$가 나타날 확률 (가능도).
*   **$P(f_i|c)$:** 카테고리 *c*가 주어졌을 때, *i*번째 특징 $f_i$가 나타날 확률.
*   **$\cdot$ (곱하기):**  각 특징별 확률을 모두 곱한다는 의미.

**핵심: 조건부 독립 가정 (Conditional Independence Assumption)**

나이브 베이즈 분류기는 "카테고리가 주어졌을 때, 특징들은 서로 *독립*이다"라고 가정합니다.  이것이 바로 조건부 독립 가정입니다.  즉, 카테고리 *c*를 알고 있다면, 한 특징 $f_i$의 값은 다른 특징 $f_j$ (i ≠ j)의 값에 대한 정보를 주지 않는다고 가정하는 것입니다.

*   **현실과의 괴리:**  실제 데이터에서는 특징들이 완전히 독립적이지 않은 경우가 많습니다.  예를 들어, "축구"와 "골"이라는 단어는 서로 연관성이 높습니다.
*   **그럼에도 사용 이유:**
    1.  **계산 단순화:**  조건부 독립 가정 덕분에 가능도 계산이 매우 쉬워집니다.  $P(f_1, f_2, ..., f_n|c)$를 직접 추정하려면 엄청나게 많은 데이터가 필요하고 계산량이 많아지지만, 이 가정을 사용하면 각 특징별 확률 $P(f_i|c)$만 추정하면 되므로 훨씬 효율적입니다.
    2.  **실제 성능:**  많은 경우에 이 가정이 현실과 다름에도 불구하고, 나이브 베이즈 분류기는 실제로 잘 작동합니다.

**공식의 의미:**

카테고리 *c*가 주어졌을 때, 특징 벡터 $(f_1, f_2, ..., f_n)$가 나타날 확률은, 각 특징 $f_i$가 카테고리 *c*에서 나타날 확률들을 모두 곱한 것과 같다는 의미입니다.

*   P("무료"=1 | "정상") = 0.05
*   P("당첨"=1 | "정상") = 0.01

새로운 메일에 "광고"와 "무료"는 포함되어 있고, "당첨"은 포함되어 있지 않다면 (f1="광고"=1, f2="무료"=1, f3="당첨"=0), 다음을 계산하세요.

$c_{NB} = \underset{c \in C}{\operatorname{argmax}} P(c) \underset{f \in F}{\prod} P(f|c) \qquad (4.8)$
### 공식 (4.8) 설명:

$c_{NB} = \underset{c \in C}{\operatorname{argmax}} P(c) \prod_{f \in F} P(f|c)$ (4.8)

이 공식은 나이브 베이즈 분류기(Naive Bayes classifier)의 최종 결정 규칙을 나타냅니다. 주어진 특징 집합 *F*를 갖는 문서(또는 데이터)에 대해, 가장 확률이 높은 카테고리 $c_{NB}$를 선택하는 방법을 보여줍니다.

*   **$c_{NB}$:** 나이브 베이즈 분류기가 선택하는 최적의 카테고리 (NB는 Naive Bayes를 의미).
*   **$\underset{c \in C}{\operatorname{argmax}}$:** 집합 *C* (모든 가능한 카테고리 집합)에 속하는 모든 카테고리 *c* 중에서, 뒤에 오는 표현식($P(c) \prod_{f \in F} P(f|c)$)을 최대로 만드는 *c*를 찾으라는 의미.
*   **$P(c)$:** 카테고리 *c*의 사전 확률 (Prior probability). 문서의 특징과 관계없이, 카테고리 *c*가 나타날 확률.
*   **$\prod_{f \in F} P(f|c)$:**  가능도(Likelihood)를 나타내는 부분입니다.
    *   **$\prod$ (파이, product):** 곱셈 기호. 집합 *F*에 속하는 모든 특징 *f*에 대해 $P(f|c)$를 곱한다는 의미.
    *   **$f \in F$:** 특징 집합 *F*에 속하는 하나의 특징 *f*.
    *   **$P(f|c)$:** 카테고리 *c*가 주어졌을 때 특징 *f*가 나타날 확률 (조건부 확률).

**핵심:**

1.  **사전 확률과 가능도의 곱:** 나이브 베이즈 분류기는 카테고리의 사전 확률 $P(c)$와 가능도 $\prod_{f \in F} P(f|c)$의 곱을 최대화하는 카테고리를 선택합니다.
2.  **가능도 계산 (조건부 독립 가정):**  $\prod_{f \in F} P(f|c)$는 카테고리 *c*가 주어졌을 때 모든 특징들이 나타날 확률을 의미하는데, 나이브 베이즈의 핵심 가정인 *조건부 독립*에 의해 각 특징별 확률의 곱으로 계산됩니다. (이것이 공식 4.7에서 설명된 내용)
3.  **최적 카테고리 선택:**  모든 가능한 카테고리 *c*에 대해 $P(c) \prod_{f \in F} P(f|c)$를 계산하고, 이 값이 가장 큰 카테고리를 선택합니다.

**간단한 예시:**

"스포츠"와 "경제" 두 카테고리가 있고, 문서의 특징으로 "축구", "주가" 두 단어를 사용한다고 가정해 봅시다.

*   P("스포츠") = 0.6 (스포츠 관련 기사가 더 많다고 가정)
*   P("경제") = 0.4
*   P("축구" | "스포츠") = 0.8
*   P("주가" | "스포츠") = 0.1
*   P("축구" | "경제") = 0.05
*   P("주가" | "경제") = 0.9

새로운 문서에 "축구"라는 단어만 있다면 (다른 특징은 없음),

*   "스포츠" 카테고리에 대한 값:  P("스포츠") * P("축구" | "스포츠") = 0.6 * 0.8 = 0.48
*   "경제" 카테고리에 대한 값: P("경제") * P("축구" | "경제") = 0.4 * 0.05 = 0.02

0.  48 > 0.02 이므로, 나이브 베이즈 분류기는 이 문서를 "스포츠" 카테고리로 분류합니다.

**예상 문제 1:**

두 개의 카테고리 "긍정" (pos)과 "부정" (neg)으로 영화 리뷰를 분류합니다. 리뷰의 특징으로 "재밌다", "지루하다" 두 단어의 포함 여부를 사용합니다 (포함: 1, 미포함: 0). 다음 확률들이 주어졌습니다.

*   P("긍정") = 0.7
*   P("부정") = 0.3
*   P("재밌다"=1 | "긍정") = 0.9
*   P("재밌다"=1 | "부정") = 0.2
*   P("지루하다"=1 | "긍정") = 0.1
*   P("지루하다"=1 | "부정") = 0.8

새로운 리뷰에 "재밌다"는 포함, "지루하다"는 미포함이라면, 나이브 베이즈 분류기는 이 리뷰를 어느 카테고리로 분류하겠습니까?

**답안 1:**

*   "긍정" 카테고리에 대한 값:
    P("긍정") * P("재밌다"=1 | "긍정") * P("지루하다"=0 | "긍정") = 0.7 * 0.9 * (1 - 0.1) = 0.7 * 0.9 * 0.9 = 0.567

*   "부정" 카테고리에 대한 값:
    P("부정") * P("재밌다"=1 | "부정") * P("지루하다"=0 | "부정") = 0.3 * 0.2 * (1 - 0.8) = 0.3 * 0.2 * 0.2 = 0.012

0.  567 > 0.012 이므로, 나이브 베이즈 분류기는 이 리뷰를 "긍정"으로 분류합니다.

___
### 공식 (4.9) 설명:

이 공식은 나이브 베이즈 분류기(Naive Bayes classifier)의 결정 규칙을 나타내는 또 다른 형태입니다. 공식 (4.8)과 거의 동일하지만, 특징 *f* 대신 문서 내 위치(position) *i*에 있는 단어 *w<sub>i</sub>* 를 사용한다는 점이 다릅니다. 즉, 문서 내 *단어의 순서*를 고려하는 것처럼 보이지만, *나이브 베이즈 분류에서는 각 위치가 서로 독립적이라고 가정*합니다.

*   **$c_{NB}$:** 나이브 베이즈 분류기가 선택하는 최적의 카테고리.
*   **$\underset{c \in C}{\operatorname{argmax}}$:** 집합 *C* (모든 가능한 카테고리 집합)에 속하는 모든 카테고리 *c* 중에서, 뒤에 오는 표현식($P(c) \prod_{i \in positions} P(w_i|c)$)을 최대로 만드는 *c*를 찾으라는 의미.
*   **$P(c)$:** 카테고리 *c*의 사전 확률 (Prior probability).
*   **$P(c) \underset{i \in positions}{\prod} P(f|c)$:** 가능도(Likelihood)를 나타내는 부분.
    *   **$\prod$ (파이, product):** 곱셈 기호. 문서 내의 모든 위치 *i*에 대해 $P(w_i|c)$를 곱한다는 의미.
    *   **$i \in positions$:** 문서 내의 위치(position) *i*.  예를 들어, 문서가 "I love this movie"라면, positions = {1, 2, 3, 4}가 됩니다.
    *   **$w_i$:** 문서 내 *i*번째 위치에 있는 단어.  위의 예시에서 $w_1$="I", $w_2$="love", $w_3$="this", $w_4$="movie"가 됩니다.
    *    **$P(w_i|c)$:** 카테고리 *c*가 주어졌을 때, *i*번째 위치에 단어 *w<sub>i</sub>* 가 나타날 확률.

**핵심:**

1.  **단어 위치:** 공식 (4.9)는 문서 내 단어의 위치(*i*)를 명시적으로 고려하는 것처럼 보입니다.
2.  **나이브 베이즈 가정 (위치 독립):** 하지만 나이브 베이즈 분류기는 *주어진 카테고리 내에서* 각 위치가 서로 독립적이라고 가정합니다. 즉, *i*번째 위치에 어떤 단어가 오는지는 *j*번째 위치 *(i ≠ j)* 에 어떤 단어가 오는지에 영향을 받지 않는다고 가정합니다.
    * 이 독립성 가정은 이전 공식들의 특징들 간의 독립성 가정과 동일한 맥락입니다.
    * 결국 $P(w_1, w_2, ..., w_n | c) = P(w_1|c)P(w_2|c)...P(w_n|c)$ 가 됩니다.
3.  **같은 단어 반복:** 같은 단어가 여러 위치에 나타날 수 있습니다. 예를 들어 "movie movie movie"라는 문서에서 $P(w_1|c)$, $P(w_2|c)$, $P(w_3|c)$는 모두 $P("movie"|c)$로 동일한 값을 가집니다.
4. **Bag of Words와 유사:** 이 공식은 사실상 Bag of Words 모델과 동일한 가정을 사용합니다. Bag of Words 모델은 단어의 순서를 무시하고, 단지 어떤 단어가 문서에 *존재하는지* (또는 얼마나 *자주* 등장하는지)만 고려합니다.

**예시:**

"긍정"과 "부정" 두 카테고리가 있고, 문서가 "I love this movie"라고 가정해 봅시다.

*   P("긍정") = 0.6
*   P("부정") = 0.4
*   P("I" | "긍정") = 0.1
*   P("love" | "긍정") = 0.3
*   P("this" | "긍정") = 0.05
*   P("movie" | "긍정") = 0.2
*   P("I" | "부정") = 0.05
*   P("love" | "부정") = 0.01
*   P("this" | "부정") = 0.02
*   P("movie" | "부정") = 0.1

*   "긍정"에 대한 값: 0.6 * 0.1 * 0.3 * 0.05 * 0.2 = 0.00018
*   "부정"에 대한 값: 0.4 * 0.05 * 0.01 * 0.02 * 0.1 = 0.0000004

0.  00018 > 0.0000004 이므로, 이 문서는 "긍정"으로 분류됩니다.

$c_{NB} = \underset{c \in C}{\operatorname{argmax}} \log P(c) + \sum\limits_{i \in positions} \log P(w_i|c) \qquad$(4.10)

### 공식 (4.10) 설명:

이 공식은 나이브 베이즈 분류기의 결정 규칙을 나타내는 또 다른 형태입니다. 이전 공식 (4.9)와 거의 동일하지만, 중요한 차이점은 확률 값에 로그(log)를 취한다는 것입니다. 로그를 취함으로써 곱셈 연산이 덧셈 연산으로 바뀌어 계산상의 이점을 얻을 수 있습니다.

*   **$c_{NB}$:** 나이브 베이즈 분류기가 선택하는 최적의 카테고리.
*   **$\underset{c \in C}{\operatorname{argmax}}$:** 집합 *C* (모든 가능한 카테고리 집합)에 속하는 모든 카테고리 *c* 중에서, 뒤에 오는 표현식($\log P(c) + \sum_{i \in positions} \log P(w_i|c)$)을 최대로 만드는 *c*를 찾으라는 의미.
*   **$\log P(c)$:** 카테고리 *c*의 사전 확률 $P(c)$에 자연로그(ln) 또는 밑이 2인 로그(log₂) 등을 취한 값.
*   **$\sum\limits_{i \in positions} \log P(w_i|c)$:**
    *   **$\sum$ (시그마, summation):** 덧셈 기호. 문서 내의 모든 위치 *i*에 대해 $\log P(w_i|c)$를 더한다는 의미.
    *   **$i \in positions$:** 문서 내의 위치(position) *i*.
    *   **$w_i$:** 문서 내 *i*번째 위치에 있는 단어.
    *   **$\log P(w_i|c)$:** 카테고리 *c*가 주어졌을 때, *i*번째 위치에 단어 *w<sub>i</sub>*가 나타날 확률 $P(w_i|c)$에 로그를 취한 값.

**핵심: 로그를 사용하는 이유**

1.  **Underflow 방지:** 확률 값은 0과 1 사이의 작은 값입니다.  많은 특징(단어)에 대해 확률을 계속 곱하다 보면, 컴퓨터가 표현할 수 있는 가장 작은 수보다 더 작아져서 0으로 처리되는 언더플로우(underflow) 문제가 발생할 수 있습니다. 로그를 취하면 곱셈이 덧셈으로 바뀌고, 로그 값은 원래 값보다 훨씬 큰 범위(음수 포함)를 가지므로 언더플로우 문제를 방지할 수 있습니다.

2.  **계산 효율성:** 곱셈 연산보다 덧셈 연산이 컴퓨터에서 더 빠르고 효율적입니다.

3.  **수학적 편리성:** 로그 함수는 단조 증가(monotonically increasing) 함수입니다. 즉, *x* > *y* 이면 log(*x*) > log(*y*)입니다. 따라서, $P(c) \prod P(w_i|c)$를 최대로 만드는 *c*를 찾는 것이나, $\log P(c) + \sum \log P(w_i|c)$를 최대로 만드는 *c*를 찾는 것이나 결과는 같습니다. argmax의 결과에 영향을 주지 않습니다.

**로그의 밑(base):**

자연로그(ln, 밑이 *e* )를 사용해도 되고, 밑이 2인 로그(log₂)를 사용해도 됩니다. 어떤 로그를 사용하든 argmax의 결과는 동일합니다. 다만, 사용하는 라이브러리나 구현에 따라 기본 설정이 다를 수 있으므로 확인이 필요합니다.

**예시:**

이전 예시를 로그를 사용하여 다시 계산해 보겠습니다. "긍정"과 "부정" 두 카테고리가 있고, 문서가 "I love this movie"라고 가정합니다.

*   P("긍정") = 0.6

*   P("I" | "긍정") = 0.1
*   P("love" | "긍정") = 0.3

### 공식 (4.11) 설명:

이 공식은 나이브 베이즈 분류에서 카테고리 *c*의 사전 확률(prior probability) $\hat{P}(c)$를 추정하는 방법을 나타냅니다.  주어진 훈련 데이터에서 전체 문서 수($N_{doc}$) 대비 특정 카테고리 *c*에 속하는 문서 수($N_c$)의 비율로 계산합니다.

*   **$\hat{P}(c)$:** 카테고리 *c*의 사전 확률 추정치.  "P hat of c"라고 읽으며, 실제 확률 $P(c)$를 훈련 데이터를 통해 추정한 값이라는 의미입니다.
*   **$N_c$:** 훈련 데이터에서 카테고리 *c*에 속하는 문서의 수.
*   **$N_{doc}$:** 훈련 데이터에 있는 전체 문서의 수.

**핵심:**

1.  **빈도 기반 추정 (Frequency-based estimation):**  $\hat{P}(c)$는 훈련 데이터에서 카테고리 *c*가 얼마나 자주 나타나는지를 나타내는 빈도 기반 추정치입니다.  예를 들어, 전체 100개의 문서 중 60개가 "스포츠" 카테고리에 속한다면, $\hat{P}("스포츠") = 60/100 = 0.6$이 됩니다.
2.  **최대 우도 추정 (Maximum Likelihood Estimation, MLE):** 이 공식은 최대 우도 추정(MLE)의 한 예입니다. MLE는 주어진 데이터를 가장 잘 설명하는 확률 분포의 모수(parameter)를 찾는 방법입니다.  여기서는 카테고리 *c*의 사전 확률이 모수가 되고, 주어진 훈련 데이터(문서 집합)를 가장 잘 설명하는 사전 확률은 각 카테고리의 빈도 비율이 됩니다.
3. **나이브 베이즈에서 사용:** 나이브베이즈는 $c_{NB} = \underset{c \in C}{\operatorname{argmax}} P(c) \underset{f \in F}{\prod} P(f|c)$ 와 같은 공식을 사용하는데, 이때 P(c)의 값을 추정하기 위해 해당 공식을 사용하는 것입니다.

**예시:**

1000개의 이메일로 구성된 훈련 데이터가 있고, 그 중 300개가 스팸 메일(카테고리 "spam")이고, 700개가 정상 메일(카테고리 "ham")이라고 가정해 봅시다.

*   $N_{spam}$ = 300
*   $N_{ham}$ = 700
*   $N_{doc}$ = 1000

그러면,

*   $\hat{P}("spam") = \frac{N_{spam}}{N_{doc}} = \frac{300}{1000} = 0.3$
*   $\hat{P}("ham") = \frac{N_{ham}}{N_{doc}} = \frac{700}{1000} = 0.7$
