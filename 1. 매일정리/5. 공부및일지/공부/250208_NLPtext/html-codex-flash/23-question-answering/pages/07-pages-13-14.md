- source_text: The answer extraction task is commonly modeled by **span labeling**: identifying in the passage a **span** (a continuous string of text) that constitutes an answer. ## Block 7
- type: paragraph
- source_text: If each span a starts at position a_s and ends at position a_e, we make the simplifying assumption that this probability can be estimated as P(a|q, p) = P_{start}(a_s | q, p)P_{end}(a_e | q, p). - korean_translation: 각 구간 a가 $a_s$ 위치에서 시작하여 $a_e$ 위치에서 끝난다면, 우리는 이 확률을 $P(a|q, p) = P_{start}(a_s | q, p)P_{end}(a_e | q, p)$로 추정할 수 있다는 단순화된 가정을 합니다. 따라서 구절 내의 각 토큰 $p_i$에 대해 우리는 두 가지 확률을 계산합니다. $p_i$가 정답 구간의 시작일 확률 $P_{start}(i)$와 $p_i$가 정답 구간의 끝일 확률 $P_{end}(i)$입니다.

## Block 8
- type: paragraph
- source_text: For many datasets the annotated documents/passages are longer than the maximum 512 input tokens BERT allows, such as Natural Questions whose gold passages are full Wikipedia pages. In such cases, following Alberti et al. (2019), we can create multiple pseudo-passage observations from the labeled Wikipedia page. Each observation is formed by concatenating [CLS], the question, [SEP], and tokens from the document. We walk through the document, sliding a window of size 512 (or rather, 512 minus the question length n minus special tokens) and packing the window of tokens into each next pseudo-passage. The answer span for the observation is either labeled [CLS] (= no answer in this particular window) or the gold-labeled span is marked. The same process can be used for inference, breaking up each retrieved document into separate observation passages and labeling each observation. The answer can be chosen as the span with the highest probability (or nil if no span is more probable than [CLS]).
- korean_translation: 많은 데이터셋에서 주석이 달린 문서나 구절은 BERT가 허용하는 최대 512개의 입력 토큰보다 깁니다. 정답 구절이 위키피디아 전체 페이지인 Natural Questions가 그 예입니다. 이런 경우 Alberti et al.
