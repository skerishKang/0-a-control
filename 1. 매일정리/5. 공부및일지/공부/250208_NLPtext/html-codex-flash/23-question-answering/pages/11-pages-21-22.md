## Block 5
- type: paragraph
- source_text: An alternative approach to doing QA is to query a pretrained language model, forcing a model to answer a question solely from information stored in its parameters. For example Roberts et al. (2020) use the T5 language model, which is an encoder-decoder architecture pretrained to fill in masked spans of text. Fig. 23.16 shows the architecture; the deleted spans are marked by <M>, and the system is trained to have the decoder generating the missing spans (separated by <M>).
- korean_translation: QA를 수행하는 또 다른 방법은 사전 학습된 언어 모델에 쿼리를 던져, 모델이 파라미터에 저장된 정보만으로 질문에 답하게 하는 것입니다. 예를 들어 Roberts 등(2020)은 마스킹된 텍스트 범위를 채우도록 사전 학습된 인코더-디코더 구조인 T5 언어 모델을 사용합니다. 그림 23.16은 그 구조를 보여줍니다. 삭제된 범위는 <M>으로 표시되며, 시스템은 디코더가 누락된 범위를 (<M>으로 구분하여) 생성하도록 학습됩니다.
- study_note: Closed-book QA using the T5 model.

## Block 6
- type: paragraph
- source_text: Roberts et al. (2020) then finetune the T5 system to the question answering task, by giving it a question, and training it to output the answer text in the decoder. Using the largest 11-billion-parameter T5 model does competitively, although not quite as well as systems designed specifically for question answering.
- korean_translation: Roberts 등(2020)은 질문을 입력하고 디코더에서 답변 텍스트를 출력하도록 학습시킴으로써 T5 시스템을 질문 답변 작업에 미세 조정(fine-tune)합니다. 가장 큰 110억 개의 파라미터를 가진 T5 모델을 사용하면 질문 답변 전용으로 설계된 시스템만큼은 아니지만 경쟁력 있는 성능을 보여줍니다.
