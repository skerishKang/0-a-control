More recent entity linking models are based on biencoders, encoding a candidate mention span, encoding an entity, and computing the dot product between the encodings. This allows embeddings for all the entities in the knowledge base to be precomputed and cached (Wu et al., 2019). Let’s sketch the ELQ linking algorithm of Li et al. (2020), which is given a question q and a set of candidate entities from Wikipedia with associated Wikipedia text, and outputs tuples (e, ms, me) of entity id, mention start, and mention end. As Fig. 23.13 shows, it does this by encoding each Wikipedia entity using text from Wikipedia, encoding each mention span using text from the question, and computing their similarity, as we describe below.
Korean Translation
보다 최근의 개체 연결 모델은 후보 언급 범위(mention span)를 인코딩하고, 개체를 인코딩한 다음, 두 인코딩 간의 내적(dot product)을 계산하는 바이엔코더(biencoders)를 기반으로 합니다. 이를 통해 지식 베이스의 모든 개체에 대한 임베딩을 미리 계산하여 캐싱할 수 있습니다(Wu et al., 2019). Li 등(2020)의 ELQ 연결 알고리즘을 개략적으로 살펴보면, 이 알고리즘은 질문 q와 관련 위키피디아 텍스트가 있는 위키피디아의 후보 개체 집합을 입력받아 개체 id, 언급 시작, 언급 종료로 구성된 튜플 (e, ms, me)을 출력합니다. 그림 23.13에서 보듯이, 이 알고리즘은 위키피디아의 텍스트를 사용하여 각 위키피디아 개체를 인코딩하고, 질문의 텍스트를 사용하여 각 언급 범위를 인코딩한 후, 아래에서 설명할 것처럼 그 유사도를 계산함으로써 이를 수행합니다.
