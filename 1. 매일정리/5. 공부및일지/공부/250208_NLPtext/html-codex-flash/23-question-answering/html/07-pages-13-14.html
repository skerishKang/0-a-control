Source Text
The answer extraction task is commonly modeled by **span labeling**: identifying in the passage a **span** (a continuous string of text) that constitutes an answer. Neural algorithms for reading comprehension are given a question q of n tokens q_1, ..., q_n and a passage p of m tokens p_1, ..., p_m. Their goal is thus to compute the probability P(a|q, p) that each possible span a is the answer.
Korean Translation
정답 추출 작업은 일반적으로 **구간 라벨링(span labeling)**으로 모델링됩니다. 즉, 구절 내에서 답변이 되는 **구간**(연속된 텍스트 문자열)을 식별하는 것입니다. 기계 독해를 위한 신경망 알고리즘에는 n개의 토큰 q_1, ..., q_n으로 이루어진 질문 q와 m개의 토큰 p_1, ..., p_m으로 이루어진 구절 p가 주어집니다. 따라서 이들의 목표는 가능한 모든 구간 a에 대해 그것이 정답일 확률 P(a|q, p)를 계산하는 것입니다.
Study Note
정답 추출을 수학적으로 정규화하여 각 구간의 확률을 구하는 과정으로 설명합니다.

If each span a starts at position a_s and ends at position a_e, we make the simplifying assumption that this probability can be estimated as P(a|q, p) = P_{start}(a_s | q, p)P_{end}(a_e | q, p). Thus for each token p_i in the passage we’ll compute two probabilities: P_{start}(i) that p_i is the start of the answer span, and P_{end}(i) that p_i is the end of the answer span.
Korean Translation
각 구간 a가 $a_s$ 위치에서 시작하여 $a_e$ 위치에서 끝난다면, 우리는 이 확률을 $P(a|q, p) = P_{start}(a_s | q, p)P_{end}(a_e | q, p)$로 추정할 수 있다는 단순화된 가정을 합니다. 따라서 구절 내의 각 토큰 $p_i$에 대해 우리는 두 가지 확률을 계산합니다. $p_i$가 정답 구간의 시작일 확률 $P_{start}(i)$와 $p_i$가 정답 구간의 끝일 확률 $P_{end}(i)$입니다.
Study Note
전체 구간의 확률을 시작점 확률과 끝점 확률의 곱으로 분해하여 계산하는 기법을 설명합니다.

Source Text
For many datasets the annotated documents/passages are longer than the maximum 512 input tokens BERT allows, such as Natural Questions whose gold passages are full Wikipedia pages. In such cases, following Alberti et al. (2019), we can create multiple pseudo-passage observations from the labeled Wikipedia page. Each observation is formed by concatenating [CLS], the question, [SEP], and tokens from the document. We walk through the document, sliding a window of size 512 (or rather, 512 minus the question length n minus special tokens) and packing the window of tokens into each next pseudo-passage. The answer span for the observation is either labeled [CLS] (= no answer in this particular window) or the gold-labeled span is marked. The same process can be used for inference, breaking up each retrieved document into separate observation passages and labeling each observation. The answer can be chosen as the span with the highest probability (or nil if no span is more probable than [CLS]).
Korean Translation
많은 데이터셋에서 주석이 달린 문서나 구절은 BERT가 허용하는 최대 512개의 입력 토큰보다 깁니다. 정답 구절이 위키피디아 전체 페이지인 Natural Questions가 그 예입니다. 이런 경우 Alberti et al. (2019)을 따라 라벨링된 위키피디아 페이지로부터 여러 개의 의사 구절(pseudo-passage) 관측치를 만들 수 있습니다. 각 관측치는 [CLS], 질문, [SEP], 그리고 문서의 토큰들을 연결하여 형성됩니다. 우리는 문서를 훑으면서 512 크기의 윈도우(정확히는 512에서 질문 길이 n과 특수 토큰을 뺀 크기)를 슬라이딩하여 각 다음 의사 구절에 토큰 윈도우를 채워 넣습니다. 관측치에 대한 정답 구간은 [CLS](= 이 특정 윈도우 내에는 답이 없음)로 라벨링되거나 정답으로 표기된 구간이 표시됩니다. 동일한 과정을 추론 시에도 사용할 수 있는데, 검색된 각 문서를 별개의 관측 구절들로 나누고 각 관측치에 라벨을 붙입니다. 답변은 가장 높은 확률을 가진 구간으로 선택될 수 있습니다(어떤 구간도 [CLS]보다 확률이 높지 않으면 정답 없음).
