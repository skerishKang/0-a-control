In this chapter we describe the two major paradigms for factoid question answering. Information-retrieval (IR) based QA, sometimes called open domain QA, relies on the vast amount of text on the web or in collections of scientific papers like PubMed. Given a user question, information retrieval is used to find relevant passages. Then neural reading comprehension algorithms read these retrieved passages and draw an answer directly from spans of text.
Korean Translation
이 장에서 우리는 팩토이드 질의응답의 두 가지 주요 패러다임을 설명합니다. 정보 검색(IR) 기반 QA는 때로 오픈 도메인 QA라고도 불리며, 웹이나 PubMed와 같은 과학 논문 컬렉션의 방대한 텍스트 양에 의존합니다. 사용자 질문이 주어지면 정보 검색을 통해 관련 구절을 찾습니다. 그런 다음 신경망 독해(reading comprehension) 알고리즘이 추출된 구절을 읽고 텍스트 구간(spans)에서 직접 정답을 이끌어냅니다.
Study Note

Korean Translation
두 번째 패러다임인 지식 기반 질의응답(knowledge-based question answering)에서 시스템은 쿼리의 의미적 표현(semantic representation)을 구축합니다. 예를 들어 "텍사스와 접한 주는 어디인가요?"를 논리적 표현인 λx.state(x) ∧ borders(x, texas)로 매핑하거나, "에이다 러블레이스는 언제 태어났나요?"를 간격 관계인 birth-year (Ada Lovelace, ?x)로 매핑합니다. 이러한 의미 표현은 사실 데이터베이스를 쿼리하는 데 사용됩니다.
Study Note

The IR task we consider is called ad hoc retrieval, in which a user poses a query to a retrieval system, which then returns an ordered set of documents from some collection. A document refers to whatever unit of text the system indexes and retrieves (web pages, scientific papers, news articles, or even shorter passages like paragraphs). A collection refers to a set of documents being used to satisfy user requests. A term refers to a word in a collection, but it may also include phrases. Finally, a query represents a user’s information need expressed as a set of terms. The high-level architecture of an ad hoc retrieval engine is shown in Fig. 23.1.
Korean Translation
우리가 고려하는 IR 작업은 애드혹(ad hoc) 검색이라고 불리며, 사용자가 검색 시스템에 쿼리를 던지면 시스템이 어떤 컬렉션에서 정렬된 문서 집합을 반환하는 방식입니다. 문서(document)는 시스템이 색인하고 검색하는 모든 텍스트 단위(웹 페이지, 과학 논문, 뉴스 기사, 또는 단락과 같은 짧은 구절)를 의미합니다. 컬렉션(collection)은 사용자 요청을 충족하기 위해 사용되는 문서들의 집합을 의미합니다. 용어(term)는 컬렉션에 포함된 단어를 의미하지만 구(phrase)를 포함할 수도 있습니다. 마지막으로 쿼리(query)는 일련의 용어들로 표현된 사용자의 정보 요구를 나타냅니다. 애드혹 검색 엔진의 상위 수준 아키텍처는 그림 23.1에 나와 있습니다.
