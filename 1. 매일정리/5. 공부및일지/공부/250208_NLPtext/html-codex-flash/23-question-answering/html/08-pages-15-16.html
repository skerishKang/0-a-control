As a simple baseline we introduce the TAGME linker (Ferragina and Scaiella, 2011) for Wikipedia, which itself draws on earlier algorithms (Mihalcea and Csomai 2007, Cucerzan 2007, Milne and Witten 2008). Wikification algorithms define the set of entities as the set of Wikipedia pages, so we’ll refer to each Wikipedia page as a unique entity e. TAGME first creates a catalog of all entities (i.e. all Wikipedia pages, removing some disambiguation and other meta-pages) and indexes them in a standard IR engine like Lucene. For each page e, the algorithm computes an in-link count in(e): the total number of in-links from other Wikipedia pages that point to e. These counts can be derived from Wikipedia dumps.
Korean Translation
간단한 베이스라인으로 위키피디아를 위한 TAGME 링커(Ferragina and Scaiella, 2011)를 소개합니다. 이는 이전의 알고리즘들(Mihalcea and Csomai 2007, Cucerzan 2007, Milne and Witten 2008)을 기반으로 합니다. 위키피디아 연결(Wikification) 알고리즘은 개체 집합을 위키피디아 페이지 집합으로 정의하므로, 각 위키피디아 페이지를 고유한 개체 e로 지칭할 것입니다. TAGME는 먼저 모든 개체(즉, 일부 모호성 해소 및 기타 메타 페이지를 제거한 모든 위키피디아 페이지)의 카탈로그를 만들고 이를 Lucene과 같은 표준 IR 엔진에 색인합니다. 각 페이지 e에 대해, 알고리즘은 해당 페이지를 가리키는 다른 위키피디아 페이지의 총 인링크(in-link) 수인 in(e)를 계산합니다. 이 횟수는 위키피디아 덤프에서 도출할 수 있습니다.

We compute a Wikipedia anchor dictionary by including, for each Wikipedia page e, e’s title as well as all the anchor texts from all Wikipedia pages that point to e. For each anchor string a we’ll also compute its total frequency freq(a) in Wikipedia (including non-anchor uses), the number of times a occurs as a link (which we’ll call link(a)), and its link probability linkprob(a) = link(a)/freq(a). Some cleanup of the final anchor dictionary is required, for example removing anchor strings composed only of numbers or single characters, that are very rare, or that are very unlikely to be useful entities because they have a very low linkprob.
Korean Translation
각 위키피디아 페이지 e에 대해, e의 제목과 e를 가리키는 모든 위키피디아 페이지의 모든 앵커 텍스트를 포함하여 위키피디아 앵커 사전을 계산합니다. 각 앵커 문자열 a에 대해 위키피디아에서의 총 빈도 freq(a)(앵커가 아닌 사용 포함), a가 링크로 나타나는 횟수(link(a)라 함), 그리고 링크 확률 linkprob(a) = link(a)/freq(a)도 계산합니다. 최종 앵커 사전의 일부 정리가 필요한데, 예를 들어 숫자나 단일 문자로만 구성된 앵커 문자열, 매우 드물게 나타나는 문자열, 또는 linkprob가 매우 낮아 유용한 개체가 될 가능성이 매우 낮은 문자열을 제거하는 작업입니다.

Source Text
If a mention span is unambiguous (points to only one entity/Wikipedia page), we are done with entity linking! However, many spans are ambiguous, matching anchors for multiple Wikipedia entities/pages. The TAGME algorithm uses two factors for disambiguating ambiguous spans, which have been referred to as prior probability and relatedness/coherence. The first factor is p(e|a), the probability with which the span refers to a particular entity. For each page e ∈ E(a), the probability p(e|a) that anchor a points to e, is the ratio of the number of links into e with anchor text a to the total number of occurrences of a as an anchor:
Korean Translation
언급 범위가 모호하지 않다면(단 하나의 개체/위키피디아 페이지를 가리키는 경우), 개체 연결이 완료됩니다! 그러나 많은 범위가 모호하며, 여러 위키피디아 개체/페이지에 대한 앵커와 일치합니다. TAGME 알고리즘은 모호한 범위를 해소하기 위해 사전 확률(prior probability)과 관련성/일관성(relatedness/coherence)이라고 불리는 두 가지 요소를 사용합니다. 첫 번째 요소는 p(e|a)로, 해당 범위가 특정 개체를 참조할 확률입니다. 각 페이지 e ∈ E(a)에 대해, 앵커 a가 e를 가리킬 확률 p(e|a)는 앵커 텍스트 a를 사용하여 e로 들어오는 링크 수를 앵커로서의 a의 총 발생 횟수로 나눈 비율입니다.
Study Note
