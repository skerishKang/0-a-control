We would like to know if δ(x) > 0, meaning that our logistic regression classifier has a higher F1 than our naive Bayes classifier on X. δ(x) is called the effect size; a bigger δ means that A seems to be way better than B; a small δ means A seems to be only a little better. Why don't we just check if δ(x) is positive? Suppose we do, and we find that the F1 score of A is higher than B's by .04. Can we be certain that A is better? We cannot! That's because A might just be accidentally better than B on this particular x. We need something more: we want to know if A's superiority over B is likely to hold again if we checked another test set x′, or under some other set of circumstances.

The hypothesis H0, called the null hypothesis, supposes that δ(x) is actually negative or zero, meaning that A is not better than B. We would like to know if we can confidently rule out this hypothesis, and instead support H1, that A is better. We do this by creating a random variable X ranging over all test sets. Now we ask how likely is it, if the null hypothesis H0 was correct, that among these test sets we would encounter the value of δ(x) that we found. We formalize this likelihood as the p-value: the probability, assuming the null hypothesis H0 is true, of seeing the δ(x) that we saw or one even greater
Korean Translation
귀무 가설(null hypothesis)이라 불리는 가설 $H_0$은 $\delta(x)$가 실제로는 음수이거나 0이라고 가정합니다. 즉, A가 B보다 낫지 않다는 뜻입니다. 우리는 이 가설을 확신을 가지고 배제(rule out)할 수 있는지, 그리고 대신 A가 더 낫다는 $H_1$을 지지할 수 있는지 알고 싶습니다. 이를 위해 가능한 모든 테스트 세트들을 범위로 하는 랜덤 변수 $X$를 만듭니다. 그런 다음, 만약 귀무 가설 $H_0$이 맞다면, 이러한 테스트 세트들 사이에서 우리가 발견한 $\delta(x)$ 값이 나타날 가능성은 얼마나 되는지 묻습니다. 우리는 이 가능성을 P-값(p-value)으로 공식화합니다. 즉, 귀무 가설 $H_0$이 참이라고 가정할 때, 우리가 관찰한 $\delta(x)$ 혹은 그보다 더 큰 값을 보게 될 확률입니다.

A very small p-value means that the difference we observed is very unlikely under the null hypothesis, and we can reject the null hypothesis. What counts as very small? It is common to use values like .05 or .01 as the thresholds. A value of .01 means that if the p-value (the probability of observing the δ we saw assuming H0 is true) is less than .01, we reject the null hypothesis and assume that A is indeed better than B. We say that a result (e.g., "A is better than B") is statistically significant if the δ we saw has a probability that is below the threshold and we therefore reject this null hypothesis.
Korean Translation
매우 작은 P-값은 우리가 관찰한 차이가 귀무 가설 하에서 발생할 가능성이 매우 낮음을 의미하며, 우리는 귀무 가설을 기각할 수 있습니다. '매우 작다'는 기준은 무엇일까요? 일반적으로 0.05나 0.01 같은 값들을 문턱값(thresholds)으로 사용합니다. 0.01이라는 값은 $H_0$이 참이라고 가정할 때 우리가 본 $\delta$가 관찰될 확률(P-값)이 0.01 미만이면, 귀무 가설을 기각하고 A가 실제로 B보다 낫다고 가정한다는 뜻입니다. 우리가 관찰한 $\delta$가 문턱값보다 낮은 확률을 가져서 귀무 가설을 기각하게 되었을 때, 그 결과(예: "A가 B보다 낫다")를 통계적으로 유의미하다(statistically significant)고 말합니다.

How do we compute this probability we need for the p-value? In NLP we generally don't use simple parametric tests like t-tests or ANOVAs that you might be familiar with. Parametric tests make assumptions about the distributions of the test statistic (such as normality) that don't generally hold in our cases. So in NLP we usually use non-parametric tests based on sampling: we artificially create many versions of the experimental setup. For example, if we had lots of different test sets x′ we could just measure all the δ(x′) for all the x′. That gives us a distribution. Now we set a threshold (like .01) and if we see in this distribution that 99% or more of those deltas are smaller than the delta we observed, i.e. that p-value(x)—the probability of seeing a δ(x) as big as the one we saw, is less than .01, then we can reject the null hypothesis and agree that δ(x) was a sufficiently surprising difference and A is really a better algorithm than B.
Korean Translation
P-값을 위해 필요한 이 확률을 어떻게 계산할까요? NLP에서는 대개 보통 익숙할 수도 있는 T-검정(t-tests)이나 분산 분석(ANOVAs) 같은 단순한 모수적 검정(parametric tests)을 사용하지 않습니다. 모수적 검정은 테스트 통계량의 분포(예: 정규성)에 대한 가정들을 전제로 하는데, NLP 사례에서는 일반적으로 이러한 가정들이 성립하지 않기 때문입니다. 그래서 NLP에서는 대개 샘플링에 기반한 비모수적 검정(non-parametric tests)을 사용합니다. 즉, 실험 환경의 다양한 버전들을 인위적으로 만들어냅니다. 예를 들어, 수많은 서로 다른 테스트 세트 $x'$이 있다면 모든 $x'$에 대해 $\delta(x')$를 측정할 수 있을 것입니다. 이를 통해 분포를 얻게 됩니다. 이제 문턱값(예: 0.01)을 설정하고, 이 분포에서 해당 델타들의 99% 이상이 우리가 관찰한 델타보다 작다면—즉 우리가 본 것만큼 큰 $\delta(x)$를 보게 될 확률인 $P\text{-value}(x)$가 0.01보다 작다면—우리는 귀무 가설을 기각하고 $\delta(x)$가 충분히 놀라운 차이였으며 A가 실제로 B보다 더 나은 알고리즘이라는 점에 동의할 수 있습니다.
Study Note

There are two common non-parametric tests used in NLP: approximate randomization (Noreen, 1989) and the bootstrap test. We will describe bootstrap below, showing the paired version of the test, which again is most common in NLP. Paired tests are those in which we compare two sets of observations that are aligned: each observation in one set can be paired with an observation in another. This happens naturally when we are comparing the performance of two systems on the same test set; we can pair the performance of system A on an individual observation xi with the performance of system B on the same xi.

The bootstrap test (Efron and Tibshirani, 1993) can apply to any metric; from precision, recall, or F1 to the BLEU metric used in machine translation. The word bootstrapping refers to repeatedly drawing large numbers of smaller samples with replacement (called bootstrap samples) from an original larger sample. The intuition of the bootstrap test is that we can create many virtual test sets from an observed test set by repeatedly sampling from it. The method only makes the assumption that the sample is representative of the population.
Korean Translation
부트스트랩 검정(Efron and Tibshirani, 1993)은 정밀도, 재현율, $F_1$에서부터 기계 번역에서 사용되는 BLEU 지표에 이르기까지 어떠한 지표에도 적용될 수 있습니다. 부트스트랩(bootstrapping)이라는 단어는 원래의 큰 샘플로부터 중복을 허용한(replacement) 작은 샘플들(부트스트랩 샘플이라고 함)을 대량으로 반복 추출하는 행위를 의미합니다. 부트스트랩 검정의 직관은 관찰된 하나의 테스트 세트로부터 반복 샘플링을 통해 수많은 '가상 테스트 세트'를 만들어낼 수 있다는 점입니다. 이 방법은 해당 샘플이 모집단을 잘 대변한다는 점만을 유일한 가설로 삼습니다.
