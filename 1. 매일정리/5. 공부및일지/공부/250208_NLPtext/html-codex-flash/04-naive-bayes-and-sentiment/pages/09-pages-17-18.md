## Block 5
- type: paragraph
- source_text: Now that we have the b test sets, providing a sampling distribution, we can do statistics on how often A has an accidental advantage. There are various ways to compute this advantage; here we follow the version laid out in Berg-Kirkpatrick et al. (2012). Assuming H0 (A isn’t better than B), we would expect that δ(X), estimated over many test sets, would be zero; a much higher value would be surprising, since H0 specifically assumes A isn’t better than B. To measure exactly how surprising is our observed δ(x) we would in other circumstances compute the p-value by counting over many test sets how often δ(x(i)) exceeds the expected zero value by δ(x) or more:
- korean_translation: 이제 샘플링 분포를 제공하는 $b$개의 테스트 세트가 확보되었으므로, A가 우연한 우위를 점하는 빈도에 대해 통계 처리를 할 수 있습니다. 이 우위를 계산하는 방법은 다양하지만, 여기서는 Berg-Kirkpatrick et al. (2012)이 제시한 버전을 따릅니다. $H_0$(A가 B보다 낫지 않음)를 가정할 때, 수많은 테스트 세트에 대해 추정된 $\delta(X)$의 기대값은 0일 것입니다. 따라서 성능 차이가 0보다 훨씬 크다면 놀라운 일이 될 것입니다. $H_0$은 구체적으로 A가 B보다 낫지 않다고 가정하기 때문입니다. 우리가 관찰한 $\delta(x)$가 정확히 얼마나 놀라운지 측정하기 위해 다른 상황이라면 수많은 테스트 세트 중 $\delta(x^{(i)})$가 기대값 0을 $\delta(x)$ 이상으로 초과하는 횟수를 세어 다음과 같이 P-값을 계산할 것입니다.
- study_note: 부트스트랩 샘플링 분포를 사용하여 관찰된 성능 차이의 유의성을 평가하는 논리를 설명합니다.

## Block 6

## Block 7
- type: paragraph
- source_text: (We use the notation 1(x) to mean “1 if x is true, and 0 otherwise”.) However, although it’s generally true that the expected value of δ(X) over many test sets, (again assuming A isn’t better than B) is 0, this isn’t true for the bootstrapped test sets we created. That’s because we didn’t draw these samples from a distribution with 0 mean; we happened to create them from the original test set x, which happens to be biased (by .20) in favor of A. So to measure how surprising is our observed δ(x), we actually compute the p-value by counting over many test sets how often δ(x(i)) exceeds the expected value of δ(x) by δ(x) or more:
- korean_translation: 하지만 수많은 테스트 세트에 걸친 $\delta(X)$의 기대값이 (A가 B보다 낫지 않다는 가정 하에) 0이라는 것이 일반적으로는 사실일지라도, 우리가 생성한 부트스트랩 테스트 세트들에 대해서는 이것이 성립하지 않습니다. 그 이유는 우리가 이 샘플들을 평균이 0인 분포에서 추출한 것이 아니기 때문입니다. 우리는 A에게 유리하게 편향된(0.20만큼) 원본 테스트 세트 $x$로부터 이들을 생성했습니다. 따라서 우리가 관찰한 $\delta(x)$가 얼마나 놀라운지 측정하기 위해, 실제로 우리는 수많은 테스트 세트에서 $\delta(x^{(i)})$가 $\delta(x)$의 기대값을 $\delta(x)$ 이상으로 초과하는 빈도를 세어 P-값을 계산합니다.
- study_note: 원본 샘플 자체가 이미 A에 편향되어 있으므로, 부트스트랩 샘플의 기준점(null point)을 0이 아닌 원본의 차이값 $\delta(x)$로 조정해야 함을 설명합니다.

- type: code
- source_text:
  function BOOTSTRAP(test set x, num of samples b) returns p-value(x)
    Calculate δ(x) # how much better does algorithm A do than B on x
    s = 0
    for i = 1 to b do
      for j = 1 to n do # Draw a bootstrap sample x(i) of size n
        Select a member of x at random and add it to x(i)
      Calculate δ(x(i)) # how much better does algorithm A do than B on x(i)
      if δ(x(i)) ≥ 2δ(x) then s ← s + 1
    p-value(x) ≈ s/b # on what % of the b samples did algorithm A beat expectations?
    return p-value(x) # if very few did, our observed δ is probably not accidental
