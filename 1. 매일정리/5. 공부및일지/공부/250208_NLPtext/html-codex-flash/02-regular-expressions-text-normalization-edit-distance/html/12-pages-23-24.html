Let's first define the minimum edit distance between two strings. Given two strings, the source string X of length n, and target string Y of length m, we'll define $D[i,j]$ as the edit distance between $X[1..i]$ and $Y[1..j]$, i.e., the first i characters of X and the first j characters of Y. The edit distance between X and Y is thus $D[n,m]$.
Korean Translation
먼저 두 문자열 사이의 최소 편집 거리를 정의해 봅시다. 두 문자열, 즉 길이가 n인 원본 문자열 X와 길이가 m인 대상 문자열 Y가 주어졌을 때, $D[i,j]$를 $X[1..i]$와 $Y[1..j]$ 사이의 편집 거리, 즉 X의 처음 i개 문자와 Y의 처음 j개 문자 사이의 거리로 정의하겠습니다. 따라서 X와 Y 사이의 편집 거리는 $D[n,m]$이 됩니다.
Study Note
동적 계획법을 위한 상태 공간 $D[i,j]$를 수학적으로 정의합니다. 행렬의 각 셀이 부분 문자열 간의 거리를 나타내게 됩니다.

Having computed $D[i,j]$ for small i, j we then compute larger $D[i,j]$ based on previously computed smaller values. 우리는 동적 계획법을 사용하여 하위 문제들에 대한 해답을 결합하며 상향식(bottom up)으로 $D[n,m]$을 계산할 것입니다. 기저 사례(base case)의 경우, 길이가 i인 원본 부분 문자열에서 빈 대상 문자열로 가는 데는 i개의 삭제가 필요합니다. 길이가 j인 대상 부분 문자열에 대해 빈 원본 문자열에서 시작하여 j개 문자로 가는 데는 j개의 삽입이 필요합니다. 작은 i, j에 대해 $D[i,j]$를 계산한 후, 이전에 계산된 작은 값들을 바탕으로 더 큰 $D[i,j]$를 계산합니다. $D[i,j]$의 값은 행렬을 통해 해당 위치에 도달할 수 있는 세 가지 가능한 경로 중 최솟값을 취하여 계산됩니다.

function MIN-EDIT-DISTANCE(source, target) returns min-distance
n ← LENGTH(source)
m ← LENGTH(target)
Create a distance matrix distance[n+1,m+1]

# Initialization: the zeroth row and column is the distance from the empty string
D[0,0] = 0
for each row i from 1 to n do
D[i,0] ← D[i-1,0] + del-cost(source[i])
for each column j from 1 to m do
D[0,j] ← D[0,j-1] + ins-cost(target[j])

# Recurrence relation:
for each row i from 1 to n do
for each column j from 1 to m do
D[i,j] ← MIN( D[i-1,j] + del-cost(source[i]),
D[i-1,j-1] + sub-cost(source[i], target[j]),
D[i,j-1] + ins-cost(target[j]) )

# Termination
return D[n,m]
Korean Translation
함수 MIN-EDIT-DISTANCE(source, target) 반환 최소-거리
n ← LENGTH(source)
m ← LENGTH(target)
거리 행렬 distance[n+1,m+1] 생성

# 초기화: 0번째 행과 열은 빈 문자열로부터의 거리임
D[0,0] = 0
1부터 n까지 각 행 i에 대해:
D[i,0] ← D[i-1,0] + del-cost(source[i])
1부터 m까지 각 열 j에 대해:
D[0,j] ← D[0,j-1] + ins-cost(target[j])

# 점화식(Recurrence relation):
1부터 n까지 각 행 i에 대해:
1부터 m까지 각 열 j에 대해:
D[i,j] ← MIN( D[i-1,j] + del-cost(source[i]),
D[i-1,j-1] + sub-cost(source[i], target[j]),
D[i,j-1] + ins-cost(target[j]) )

# 종료
return D[n,m]
