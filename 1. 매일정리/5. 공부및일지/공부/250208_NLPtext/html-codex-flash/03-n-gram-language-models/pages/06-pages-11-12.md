## Block 11
- type: paragraph
- source_text: Our model will incorrectly estimate that the P(offer|denied the) is 0! These zeros— things that don’t ever occur in the training set but do occur in the test set—are a problem for two reasons. First, their presence means we are underestimating the probability of all sorts of words that might occur, which will hurt the performance of any application we want to run on this data. Second, if the probability of any word in the test set is 0, the entire probability of the test set is 0. By definition, perplexity is based on the inverse probability of the test set. Thus if some words have zero probability, we can’t compute perplexity at all, since we can’t divide by 0!
- korean_translation: 우리의 모델은 $P(offer|denied the)$의 확률을 0으로 잘못 추정하게 될 것입니다! 학습 세트에는 등장하지 않지만 테스트 세트에는 나타나는 이러한 '제로(zeros)' 문제에는 두 가지 심각한 이유가 있습니다. 첫째, 제로 확률은 나타날 법한 온갖 단어들의 확률을 과소평가하게 만들며, 이는 해당 데이터를 바탕으로 실행되는 모든 애플리케이션의 성능을 저해합니다. 둘째, 테스트 세트의 단어 중 단 하나라도 확률이 0이 되면 전체 테스트 세트의 확률 곱이 0이 되어 버립니다. 정의상 펄플렉서티는 테스트 세트 확률의 역수를 기반으로 하므로, 확률이 0인 단어가 있으면 0으로 나눌 수 없게 되어 펄플렉서티 자체를 아예 계산할 수 없게 됩니다.
