The inputs x and outputs y of transformers, as well as the intermediate vectors after the various layers, all have the same dimensionality 1 × d. For now let's assume the dimensionalities of the transform matrices are W^Q ∈ R^{d×d}, W^K ∈ R^{d×d}, and W^V ∈ R^{d×d}. Later we'll need separate dimensions for these matrices when we introduce multi-headed attention, so let's just make a note that we'll have a dimension d_k for the key and query vectors, and a dimension d_v for the value vectors, both of which for now we'll set to d. In the original transformer work (Vaswani et al., 2017), d was 1024.
Korean Translation
트랜스포머의 입력 x와 출력 y, 그리고 다양한 층을 거친 후의 중간 벡터들은 모두 1 × d의 동일한 차원을 갖습니다. 일단 변환 행렬들의 차원이 W^Q ∈ ℝ^{d×d}, W^K ∈ ℝ^{d×d}, W^V ∈ ℝ^{d×d}라고 가정해 봅시다. 나중에 멀티 헤드 주의 집중(multi-headed attention)을 도입할 때 이 행렬들에 대해 별도의 차원이 필요할 것이므로, 키와 쿼리 벡터를 위한 차원 d_k와 값 벡터를 위한 차원 d_v를 가질 것이며 지금은 둘 다 d로 설정한다는 점만 메모해 둡시다. 원래의 트랜스포머 논문(Vaswani et al., 2017)에서 d는 1024였습니다.

The result of a dot product can be an arbitrarily large (positive or negative) value. Exponentiating such large values can lead to numerical issues and to an effective loss of gradients during training. To avoid this, the dot product needs to be scaled in a suitable fashion. A scaled dot-product approach divides the result of the dot product by a factor related to the size of the embeddings before passing them through the softmax. A typical approach is to divide the dot product by the square root of the dimensionality of the query and key vectors (d_k), leading us to update our scoring function one more time, replacing Eq. 9.27 and Eq. 9.32 with Eq. 9.34:
Korean Translation
내적의 결과는 임의로 큰 (양수 또는 음수) 값일 수 있습니다. 이러한 큰 값들에 지수 함수를 적용하면 수치적 문제(numerical issues)가 발생할 수 있으며 학습 중에 실질적인 그래디언트 손실로 이어질 수 있습니다. 이를 방지하기 위해 내적은 적절한 방식으로 스케일링(scaling)되어야 합니다. '스케일드 닷 프로덕트(scaled dot-product)' 접근 방식은 내적 결과를 softmax에 통과시키기 전에 임베딩 크기와 관련된 인수로 나눕니다. 일반적인 방법은 내적 값을 쿼리와 키 벡터의 차원(d_k)의 제곱근으로 나누는 것이며, 이를 통해 우리의 점수 함수를 다시 한 번 업데이트하여 식 9.27과 식 9.32를 식 9.34로 대체하게 됩니다.

Source Text
This description of the self-attention process has been from the perspective of computing a single output at a single time step i. However, since each output, y_i, is computed independently this entire process can be parallelized by taking advantage of efficient matrix multiplication routines by packing the input embeddings of the N tokens of the input sequence into a single matrix X ∈ R^{N×d}. That is, each row of X is the embedding of one token of the input. We then multiply X by the key, query, and value matrices (all of dimensionality d × d) to produce matrices Q ∈ R^{N×d}, K ∈ R^{N×d}, and V ∈ R^{N×d}, containing all the key, query, and value vectors:
Korean Translation
지금까지의 자기 주의 집중 프로세스 설명은 단일 시간 단계 i에서 단일 출력을 계산하는 관점이었습니다. 그러나 각 출력 y_i는 서로 독립적으로 계산되므로, 입력 시퀀스의 N개 토큰에 대한 입력 임베딩들을 하나의 행렬 X ∈ ℝ^{N×d}로 묶음으로써 효율적인 행렬 곱셈 루틴을 활용하여 이 전체 프로세스를 병렬화할 수 있습니다. 즉, X의 각 행은 입력 토큰 하나하나의 임베딩입니다. 그런 다음 X에 키, 쿼리, 값 행렬(모두 d × d 차원)을 곱하여 모든 키, 쿼리, 값 벡터를 포함하는 행렬 Q ∈ ℝ^{N×d}, K ∈ ℝ^{N×d}, V ∈ ℝ^{N×d}를 생성합니다.
