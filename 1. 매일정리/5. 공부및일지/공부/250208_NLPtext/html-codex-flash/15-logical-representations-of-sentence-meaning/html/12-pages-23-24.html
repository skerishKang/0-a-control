Instance checking is the task of determining whether a particular individual can be classified as a member of a particular category. This process takes what is known about a given individual, in the form of relations and explicit categorical statements, and then compares that information with what is known about the current terminology. It then returns a list of the most specific categories to which the individual can belong.
Korean Translation
instance checking은 특정 개체가 특정 범주의 구성원으로 분류될 수 있는지를 결정하는 과제입니다. 이 과정은 관계와 명시적 범주 진술의 형태로 주어진 개체에 대한 정보를 가져와, 현재 용어 체계에 대해 알려진 내용과 비교합니다. 그리고 그 개체가 속할 수 있는 가장 구체적인 범주들의 목록을 반환합니다.

Source Text
The highest-profile role for Description Logics to date, has been as part of the development of the Semantic Web. The Semantic Web is an ongoing effort to provide a formally specified semantics of the contents of the Web (Fensel et al., 2003). A key component of this effort involves the creation and deployment of ontologies for various application areas of interest. The meaning representation language used to represent this knowledge is the Web Ontology Language (OWL) (McGuinness and van Harmelen, 2004). OWL embodies a Description Logic that corresponds roughly to the one we've been describing here.
Korean Translation
지금까지 Description Logic이 가장 두드러지게 사용된 분야는 시맨틱 웹의 발전 과정입니다. 시맨틱 웹은 웹 콘텐츠에 대해 형식적으로 명시된 의미론을 제공하려는 지속적인 노력입니다(Fensel et al., 2003). 이 노력의 핵심 구성 요소 중 하나는 다양한 응용 영역을 위한 온톨로지를 만들고 배포하는 일입니다. 이 지식을 표현하는 의미 표현 언어가 바로 Web Ontology Language, 즉 OWL입니다(McGuinness and van Harmelen, 2004). OWL은 여기서 설명해 온 것과 대략 대응되는 Description Logic을 구현합니다.
