- source_text: Figure 8.7 shows the same excerpt represented with BIO tagging, as well as variants called IO tagging and BIOES tagging. In BIO tagging we label any token that begins a span of interest with the label B, tokens that occur inside a span are tagged with an I, and any tokens outside of any span of interest are labeled O. While there is only one O tag, we’ll have distinct B and I tags for each named entity class. The number of tags is thus 2n + 1 tags, where n is the number of entity types. BIO tagging can represent exactly the same information as the bracketed notation, but has the advantage that we can represent the task in the same simple sequence modeling way as part-of-speech tagging: assigning a single label y_i to each input word x_i.

## Block 7
- type: paragraph
- source_text: The HMM is based on augmenting the Markov chain. A Markov chain is a model that tells us something about the probabilities of sequences of random variables, states, each of which can take on values from some set. These sets can be words, or tags, or symbols representing anything, for example the weather. A Markov chain makes a very strong assumption that if we want to predict the future in the sequence, all that matters is the current state. All the states before the current state have no impact on the future except via the current state. It’s as if to predict tomorrow’s weather you could examine today’s weather but you weren’t allowed to look at yesterday’s weather.
- korean_translation: HMM은 마르코프 체인을 확장한 것에 기반합니다. 마르코프 체인은 어떤 집합에서 값을 가질 수 있는 상태(states)인 확률 변수 시퀀스의 확률에 대해 알려 주는 모델입니다. 이 상태들은 단어, 태그, 혹은 날씨 같은 무엇이든 나타내는 기호가 될 수 있습니다. 마르코프 체인은 시퀀스에서 미래를 예측할 때 오직 현재 상태만이 중요하다는 매우 강력한 가정을 합니다. 현재 상태 이전의 모든 상태들은 현재 상태를 통하는 경로를 제외하고는 미래에 아무런 영향을 주지 않습니다. 이는 마치 내일의 날씨를 예보하기 위해 오늘의 날씨는 볼 수 있지만 어제의 날씨는 볼 수 없는 것과 같습니다.
- study_note: 마르코프 체인의 핵심 아이디어인 '미래는 오직 현재에만 의존한다'는 속성을 날씨 예보 비유와 함께 설명합니다.
