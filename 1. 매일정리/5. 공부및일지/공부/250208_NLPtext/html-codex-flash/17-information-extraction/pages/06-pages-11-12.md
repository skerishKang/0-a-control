- study_note: 이렇게 다양한 방법(지도 학습, 부트스트래핑, 원거리 지도 등)으로 구성된 관계 추출 모델이 얼마나 우수한지 측정하는 기준을 다룹니다.

## Block 3
- type: paragraph
- source_text: Supervised relation extraction systems are evaluated using precision, recall, and F1 over the extracted relations. However, evaluating semi-supervised methods is more challenging because we usually lack a complete test set of all true relations.
- korean_translation: 지도 모델 관계 추출 시스템들은 추출된 관계 전체에 대한 정밀도, 재현율, 그리고 F1 점수를 기반으로 평가합니다. 하지만, 반지도의(semi-supervised) 방법론들을 평가하는 것은 더욱 어려운데, 그 이유는 진정한 모든 관계 정답을 다루는 완벽한 테스트 데이터 세트가 빈번하게 부족하기 때문입니다.
- study_note: 풍부한 정답을 갖춘 지도 학습 평가 대비, 정답군 구비가 어려운 반지도/비지도 학습의 평가가 갖는 한계를 지적합니다.

## Block 4
- type: paragraph
- source_text: For distant supervision, researchers often use held-out datasets or perform manual evaluation on a sampled subset of extracted relations to estimate precision.
- korean_translation: 원거리 지도의 경우, 연구자들은 주로 예측 정밀도를 추정하기 위해 홀드아웃(held-out; 별도로 떼어 놓은) 데이터세트를 활용하거나, 추출된 일부 하위 관계 집합을 바탕으로 수동 평가(manual evaluation)를 진행합니다.
- study_note: 불완전한 정답셋 상황 하에서 실무 연구자들이 채택하는 우회적인 모델 성능 산정 기법을 소개합니다.

## Block 2
- type: section_title
- source_text: 17.7 Event Extraction
- korean_translation: 17.7 사건 추출(Event Extraction)
- study_note: 단절된 두 개체 간 관계를 넘어, 누가, 언제, 어디서, 무엇을 했는지를 포함한 복잡 구조의 사건(Event) 단위로 정보를 파악하는 차상위 추출 기술을 다룹니다.

## Block 3
- type: paragraph
- source_text: Extracting events goes beyond identifying simple binary relations. An event typically involves a trigger word and a set of arguments filling specific roles (e.g., perpetrator, victim, weapon, time).
- korean_translation: 사건 추출은 단순한 이항 관계를 찾아내는 수준을 뛰어넘습니다. 보통 사건은 하나의 트리거 단어(trigger word)와 함께, 특정한 역할을 채우는 논항 집합(예: 가해자, 희생자, 범행 도구, 시간 등)을 동반하게 됩니다.
- study_note: 이항 구조를 띄는 일반적인 '관계'와, 중심축어(trigger)를 바탕으로 여러 행위자/대상물이 연관되는 다자간 '사건' 간의 명확한 구조적 차이를 서술합니다.

## Block 4
- type: margin_note
- source_text: event trigger / arguments
- korean_translation: 사건 트리거(event trigger) / 논항(arguments)
- study_note: 사건을 일으키거나 대변하는 핵심 어휘(트리거)와 그 사건에 여러 역할로 참여하는 객체들(논항)을 의미하며, 의미론적 역할 할당 모델과 밀접하게 연관됩니다.

## Block 5
- type: paragraph
- source_text: The standard approach to event extraction frames it as a multi-stage classification process: first detecting the event trigger and event type, then identifying the arguments, and finally classifying the semantic role of each argument.
- korean_translation: 사건 추출에 대한 표준 접근 방식은 이를 다단계 분류 절차로 설계하는 것입니다. 처음에는 사건 트리거와 사건 유형을 감지하고, 이어서 논항들을 식별하며, 마지막으로 각 논항의 의미적 역할(semantic role)을 별도로 분류합니다.
- study_note: 여러 세부 단계(트리거 식별 -> 논항 추출 -> 역할 결정) 순서로 해결하는 파이프라인(pipeline) 형태의 사건 추출 프레임워크를 설명합니다.
