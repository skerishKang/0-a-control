17.8 템플릿 채우기(Template Filling)
Study Note
여러 문장이나 문서 전체에 걸쳐 있는 정보들을 취합하여, 사전 정의된 정형 슬롯(빈칸)들을 채워 완결된 형태의 정보 구조를 구축하는 태스크를 다릅니다.
Block 3
paragraph
Source Text
The ultimate goal for many information extraction tasks is template filling. A template is a fixed set of slots, each corresponding to a specific piece of information (e.g., a corporate acquisition template might have slots for Buyer, Seller, Price, and Date).
Korean Translation
수많은 정보 추출 모델들이 향하는 최종 목적지는 바로 템플릿 채우기(template filling)입니다. 여기서 템플릿이란 특정한 정보 단위들을 각각 매핑하고 있는 일련의 고정된 슬롯 집합을 의미합니다(예를 들어, 기업 인수 템플릿은 구매자, 판매자, 거래 금액, 그리고 날짜 등을 위한 슬롯을 가질 수 있습니다).
Study Note
특정 도메인에서의 목표나 활용도를 구체적으로 설정하고, 문서를 온전히 이해해야만 풀 수 있는 상위 수준 정보 추출 태스크임을 설명합니다.

Block 5
paragraph
Source Text
Template filling requires coreference resolution to link different mentions of the same entity across a text, and relation/event extraction to understand how those entities fill the template slots.
Korean Translation
이런 템플릿 채우기 작업이 성공하려면, 텍스트 전반에 걸쳐 기재된 동일 개체의 다른 언급 지점들을 이어주는 상호 참조 해결(coreference resolution)이 요구되며, 나아가 이런 개체들이 템플릿 내 슬롯들을 어떤 양상으로 채우는지 파악할 수 있는 관계/사건 추출 기능이 동반되어야만 합니다.
Study Note
템플릿 채우기가 NER, 상호 참조, 관계 추출 등을 모두 통합적으로 활용해야 하는 파이프라인의 종착역임을 강조합니다.

Block 3
paragraph
Source Text
This chapter introduced the core tasks of information extraction. We explored how to move from unstructured text to structured knowledge representations, focusing on entities, relations, and events.
Korean Translation
이번 장은 정보 추출 분야의 핵심 과제들을 소개했습니다. 우리는 비정형 텍스트 처리에서 시작하여, 구조화된 지식 표현물로 나아가는 과정에 대해 집중 탐구했으며, 주로 개체들, 관계들, 그리고 사건들에 초점을 맞추었습니다.
Study Note
17장의 전체적인 학습 목표(비정형 -> 정형 구조 변환)와 세 가지 중심 주제를 재확인합니다.
Block 4
list
Source Text
- Named Entity Recognition (NER): Finding and classifying entities, often using BIO tagging.
- Relation Extraction: Finding binary semantic relations between entities, using rule-based (Hearst patterns), supervised, semi-supervised (bootstrapping), or distant supervision methods.
- Event Extraction: Identifying event triggers and their arguments with specific multi-stage semantic roles.
- Template Filling: Populating predefined slots in a template with extracted entities and events, relying on an integration of various NLP systems.
Korean Translation
- 개체명 인식(NER): 주로 BIO 태깅을 사용하여 텍스트 상 개체들을 찾고 분류함.
- 관계 추출(Relation Extraction): 규칙 기반(허스트 패턴), 지도 모델, 반지도(부트스트래핑), 혹은 원거리 지도 등 방법을 활용해 개체들 간의 이항 의미 관계들을 탐색함.
- 사건 추출(Event Extraction): 사건 트리거(촉발어) 및 특정적 다층 구조의 의미 역할을 지니고 있는 그들의 논항(arguments)들을 함께 식별해냄.
- 템플릿 채우기(Template Filling): 다양하게 개별 구성된 NLP 시스템들과의 통합 의존을 토대로, 추출된 개체나 사건을 템플릿 내부의 사전에 구비된 빈 슬롯들에 채움.
Study Note
17장에서 학습한 4가지 정보 추출 핵심 작업들을 묶어 요약 정리합니다.
