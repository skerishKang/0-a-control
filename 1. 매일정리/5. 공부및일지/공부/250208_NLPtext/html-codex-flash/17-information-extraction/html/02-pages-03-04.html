paragraph
Source Text
Once entities have been identified in a text, the next step is often to identify the relations that hold between them. For example, given the entities "Microsoft" and "Bill Gates", a system might extract the relation FounderOf(Bill Gates, Microsoft).
Korean Translation
텍스트에서 개체들이 식별되고 나면, 다음 단계는 흔히 그들 사이에 성립하는 관계들을 찾아내는 것입니다. 예를 들어, "Microsoft"와 "Bill Gates"라는 개체가 주어졌을 때, 시스템은 FounderOf(Bill Gates, Microsoft)와 같은 관계를 추출할 수 있습니다.
Study Note
개체명 인식의 다음 단계로서 논리적 연결 고리를 짓는 모듈임을 예시와 함께 보여줍니다.

Block 5
paragraph
Source Text
Many approaches to relation extraction treat the problem as classification: given a pair of entities in a sentence, classify the relation between them into a predefined set of relation types (e.g., Organization-Affiliation, Part-Whole).
Korean Translation
관계 추출에 대한 많은 접근법들은 이 문제를 분류 작업으로 취급합니다. 즉, 문장 내에 있는 개체 쌍이 주어지면 그들 사이의 관계를 사전에 정의된 관계 유형 세트(예: 조직-소속 체계, 부분-전체 체계) 중 하나로 분류하는 것입니다.
Study Note
관계 추출을 지도 학습 기반의 다중 클래스 분류 태스크로 모델링하는 기본적인 접근 방식을 설명합니다.

17.2.1 규칙 및 패턴 기반 관계 추출
Study Note
머신러닝이 도입되기 전, 혹은 레이블 데이터가 부족할 때 사용되는 특정 언어적 패턴 매칭에 의한 관계 추출 방식을 다룹니다.
Block 3
paragraph
Source Text
Early relation extraction systems relied heavily on hand-crafted rules or lexico-syntactic patterns. The most famous example is Hearst patterns for extracting hypernym (is-a) relations.
Korean Translation
초기 관계 추출 시스템들은 수작업으로 만든 규칙들이나 어휘-통사적 패턴들에 크게 의존했습니다. 가장 유명한 예는 상의어(hypernym, is-a) 관계를 추출하기 위한 허스트(Hearst) 패턴입니다.
Study Note

margin_note
Source Text
Hearst patterns
Korean Translation
허스트 패턴(Hearst patterns)
Study Note
특정 어휘적 구조(예: 'X such as Y')를 바탕으로 Y가 X의 하위 범주임을 찾아내는 정밀도 높은 패턴 세트입니다.
Block 5
list
Source Text
- NP_x such as NP_y (e.g., "cities such as Chicago") -> is-a(Chicago, city)
- NP_y and other NP_x (e.g., "Chicago and other cities") -> is-a(Chicago, city)
Korean Translation
- NP_x 등의 NP_y (예: "Chicago와 같은 도시들(cities such as Chicago)") -> is-a(Chicago, city)
- NP_y 및 다른 NP_x (예: "Chicago 및 다른 도시들(Chicago and other cities)") -> is-a(Chicago, city)
Study Note
상위어/하위어 관계를 확실하게 보장하는 전형적인 허스트 패턴 두 가지를 보여줍니다.
