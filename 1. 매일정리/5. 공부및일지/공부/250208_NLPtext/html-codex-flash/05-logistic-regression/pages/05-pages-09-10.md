## Block 2
- type: paragraph
- source_text: How shall we find the minimum of this (or any) loss function? Gradient descent is a method that finds a minimum of a function by figuring out in which direction (in the space of the parameters θ) the function’s slope is rising the most steeply, and moving in the opposite direction. The intuition is that if you are hiking in a canyon and trying to descend most quickly down to the river at the bottom, you might look around yourself 360 degrees, find the direction where the ground is sloping the steepest, and walk downhill in that direction.
- korean_translation: 이 손실 함수(또는 어떤 함수든)의 최솟값을 어떻게 찾을까요? 경사 하강법(gradient descent)은 파라미터 $\theta$ 공간에서 함수의 기울기가 가장 가파르게 상승하는 방향을 파악한 뒤, 그 반대 방향으로 이동하여 함수의 최솟값을 찾는 방법입니다. 이에 대한 직관적인 비유는 이렇습니다. 여러분이 협곡에서 하이킹을 하며 바닥에 있는 강으로 가장 빠르게 내려가려 한다면, 주위를 360도 둘러보고 땅이 가장 가파르게 기울어진 방향을 찾아 그 내리막길로 걸어 내려가는 것과 같습니다.
- study_note: 경사 하강법의 기본 원리를 산에서 내려오는 물리적 상황에 비유하여 설명합니다.

## Block 4
- type: paragraph
- source_text: For logistic regression, this loss function is conveniently convex. A convex function has just one minimum; there are no local minima to get stuck in, so gradient descent starting from any point is guaranteed to find the minimum. (By contrast, the loss for multi-layer neural networks is non-convex, and gradient descent may get stuck in local minima for neural network training and never find the global optimum.)
- korean_translation: 로지스틱 회귀의 경우, 다행히도 이 손실 함수는 볼록(convex) 함수입니다. 볼록 함수는 단 하나의 최솟값만을 가집니다. 빠져나오지 못할 지역 최솟값(local minima)이 없으므로, 어떤 지점에서 경사 하강법을 시작하든 최솟값을 찾는 것이 보장됩니다. (대조적으로, 다층 신경망의 손실 함수는 비볼록(non-convex) 함수이며, 신경망 학습 시 경사 하강법은 지역 최솟값에 갇혀 전역 최적점(global optimum)을 영영 찾지 못할 수도 있습니다.)

## Block 9
- type: paragraph
- source_text: The gradient descent algorithm answers this question by finding the gradient of the loss function at the current point and moving in the opposite direction. The gradient of a function of many variables is a vector pointing in the direction of the greatest increase in a function. The gradient is a multi-variable generalization of the slope, so for a function of one variable like the one in Fig. 5.3, we can informally think of the gradient as the slope. The dotted line in Fig. 5.3 shows the slope of this hypothetical loss function at point w = w^1. You can see that the slope of this dotted line is negative. Thus to find the minimum, gradient descent tells us to go in the opposite direction: moving w in a positive direction.
- korean_translation: 경사 하강법 알고리즘은 현재 지점에서 손실 함수의 경사(gradient)를 찾고 그 반대 방향으로 이동함으로써 이 질문에 답합니다. 다변수 함수의 경사는 함수가 가장 가파르게 증가하는 방향을 가리키는 벡터입니다. 경사는 기울기(slope)를 다변수로 일반화한 것이므로, 그림 5.3과 같은 일변수 함수의 경우 경사를 비공식적으로 기울기라고 생각할 수 있습니다. 그림 5.3의 점선은 지점 $w = w^1$에서 이 가상의 손실 함수의 기울기를 보여줍니다. 이 점선의 기울기가 음수임을 확인할 수 있습니다. 따라서 최솟값을 찾기 위해 경사 하강법은 반대 방향, 즉 $w$를 양의 방향으로 이동시키라고 알려줍니다.
- study_note: 경사가 함수의 최대 증가 방향을 가리킨다는 정의와, 최솟값을 찾기 위해 그 반대 방향으로 가야 하는 논리적 이유를 설명합니다.

## Block 4
- type: paragraph
- source_text: Now let’s extend the intuition from a function of one scalar variable w to many variables, because we don’t just want to move left or right, we want to know where in the N-dimensional space (of the N parameters that make up θ) we should move. The gradient is just such a vector; it expresses the directional components of the sharpest slope along each of those N dimensions. If we’re just imagining two weight dimensions (say for one weight w and one bias b), the gradient might be a vector with two orthogonal components, each of which tells us how much the ground slopes in the w dimension and in the b dimension. Fig. 5.4 shows a visualization of the value of a 2-dimensional gradient vector taken at the red point.
- korean_translation: 이제 직관을 하나의 스칼라 변수 $w$에서 다변수 함수로 확장해 봅시다. 우리는 단순히 왼쪽이나 오른쪽으로 움직이려 하는 것이 아니라, $\theta$를 구성하는 $N$개의 파라미터로 이루어진 $N$차원 공간의 어디로 이동해야 할지 알고 싶기 때문입니다. 경사는 바로 그러한 벡터로, $N$개의 각 차원을 따라 가장 가파른 기울기의 방향 성분들을 표현합니다. 만약 우리가 두 개의 가중치 차원(예를 들어 하나의 가중치 $w$와 하나의 편향 $b$)만 상상한다면, 경사는 두 개의 직교하는 성분을 가진 벡터가 될 것이며, 각 성분은 $w$ 차원과 $b$ 차원에서 지면이 얼마나 기울어져 있는지를 알려줍니다. 그림 5.4는 빨간색 지점에서 측정된 2차원 경사 벡터의 시각화 자료를 보여줍니다.

## Block 6
- type: paragraph
- source_text: In an actual logistic regression, the parameter vector w is much longer than 1 or 2, since the input feature vector x can be quite long, and we need a weight w_i for each x_i. For each dimension/variable w_i in w (plus the bias b), the gradient will have a component that tells us the slope with respect to that variable. Essentially we’re asking: “How much would a small change in that variable w_i influence the total loss function L?”
- korean_translation: 실제 로지스틱 회귀에서 파라미터 벡터 $w$는 1이나 2보다 훨씬 깁니다. 입력 특징 벡터 $x$가 상당히 길 수 있고, 각 $x_i$에 대해 가중치 $w_i$가 필요하기 때문입니다. $w$의 각 차원/변수 $w_i$ (그리고 편향 $b$)에 대해, 경사는 해당 변수에 대한 기울기를 알려주는 성분을 가질 것입니다. 본질적으로 우리는 이렇게 묻는 셈입니다. “해당 변수 $w_i$의 아주 미세한 변화가 전체 손실 함수 $L$에 얼마나 영향을 미치는가?”
