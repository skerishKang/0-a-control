Note the differences between training (Fig. 11.7) and inference (Fig. 11.4) with respect to the outputs at each time step. The decoder during inference uses its own estimated output yˆt as the input for the next time step xt+1 . Thus the decoder will tend to deviate more and more from the gold target sentence as it keeps generating more tokens. In training, therefore, it is more common to use teacher forcing in the decoder. Teacher forcing means that we force the system to use the gold target token from training as the next input xt+1 , rather than allowing it to rely on the (possibly erroneous) decoder output yˆt . This speeds up training.
Korean Translation
각 시점의 출력과 관련하여 학습(그림 11.7)과 추론(그림 11.4) 사이의 차이점에 주목하십시오. 추론 시의 디코더는 자신의 추정 출력인 $\hat{y}_t$를 다음 시점의 입력 $x_{t+1}$로 사용합니다. 따라서 디코더는 토큰을 계속 생성함에 따라 실제 정답 문장으로부터 점점 더 벗어나는 경향을 보이게 됩니다. 그러므로 학습 시에는 디코더에서 티처 포싱(teacher forcing)을 사용하는 것이 더 일반적입니다. 티처 포싱은 시스템이 (잠재적으로 오류가 있을 수 있는) 디코더 출력 $\hat{y}_t$에 의존하게 하는 대신, 학습 데이터의 실제 정답 토큰을 다음 입력 $x_{t+1}$로 사용하도록 강제하는 것을 의미합니다. 이는 학습 속도를 높여줍니다.

Source Text
The simplicity of the encoder-decoder model is its clean separation of the encoder — which builds a representation of the source text — from the decoder, which uses this context to generate a target text. In the model as we’ve described it so far, this context vector is hn , the hidden state of the last (nth) time step of the source text. This final hidden state is thus acting as a bottleneck: it must represent absolutely everything about the meaning of the source text, since the only thing the decoder knows about the source text is what’s in this context vector. Information at the beginning of the source text, having been processed first, may be long-forgotten by the time we have processed all the n tokens of the input, making hn a poor representation for the early parts of the sentence.
Korean Translation
인코더-디코더 모델의 단순함은 소스 텍스트의 표현을 구축하는 인코더와, 이 문맥을 사용하여 타깃 텍스트를 생성하는 디코더가 깔끔하게 분리되어 있다는 점에 있습니다. 지금까지 설명한 모델에서 이 문맥 벡터는 소스 텍스트의 마지막($n$번째) 시점의 은닉 상태인 $h_n$입니다. 따라서 이 마지막 은닉 상태는 병목(bottleneck) 역할을 하게 됩니다. 디코더가 소스 텍스트에 대해 알 수 있는 유일한 것은 이 문맥 벡터에 담긴 내용뿐이므로, 이 벡터는 소스 텍스트의 의미에 관한 절대적으로 모든 것을 표현해야만 합니다. 가장 먼저 처리된 소스 텍스트의 앞부분 정보는 입력 토큰 $n$개를 모두 처리했을 때쯤이면 오래되어 잊혀질 수 있으며, 이는 $h_n$을 문장의 앞부분을 표현하기에 부족한 표현으로 만듭니다.
Study Note
고정 길이 문맥 벡터가 인코더-디코더 구조에서 정보 전달의 '병목'이 되는 문제를 지적하며 어텐션의 필요성을 시사합니다.
