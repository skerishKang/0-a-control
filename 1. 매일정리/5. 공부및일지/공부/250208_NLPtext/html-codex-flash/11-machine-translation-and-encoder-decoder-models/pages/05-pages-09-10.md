- type: paragraph
- source_text: The simplicity of the encoder-decoder model is its clean separation of the encoder — which builds a representation of the source text — from the decoder, which uses this context to generate a target text. In the model as we’ve described it so far, this context vector is hn , the hidden state of the last (nth) time step of the source text. This final hidden state is thus acting as a bottleneck: it must represent absolutely everything about the meaning of the source text, since the only thing the decoder knows about the source text is what’s in this context vector. Information at the beginning of the source text, having been processed first, may be long-forgotten by the time we have processed all the n tokens of the input, making hn a poor representation for the early parts of the sentence.
- korean_translation: 인코더-디코더 모델의 단순함은 소스 텍스트의 표현을 구축하는 인코더와, 이 문맥을 사용하여 타깃 텍스트를 생성하는 디코더가 깔끔하게 분리되어 있다는 점에 있습니다. 지금까지 설명한 모델에서 이 문맥 벡터는 소스 텍스트의 마지막($n$번째) 시점의 은닉 상태인 $h_n$입니다. 따라서 이 마지막 은닉 상태는 병목(bottleneck) 역할을 하게 됩니다. 디코더가 소스 텍스트에 대해 알 수 있는 유일한 것은 이 문맥 벡터에 담긴 내용뿐이므로, 이 벡터는 소스 텍스트의 의미에 관한 절대적으로 모든 것을 표현해야만 합니다. 가장 먼저 처리된 소스 텍스트의 앞부분 정보는 입력 토큰 $n$개를 모두 처리했을 때쯤이면 오래되어 잊혀질 수 있으며, 이는 $h_n$을 문장의 앞부분을 표현하기에 부족한 표현으로 만듭니다.
