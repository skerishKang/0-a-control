## Block 6
- type: paragraph
- source_text:
  Machine translation is one of the most widely used applications of NLP, and the encoder-decoder model, first developed for MT is a key tool that has applications throughout NLP.
  • Languages have divergences, both structural and lexical, that make translation difficult.
  • The linguistic field of typology investigates some of these differences; languages can be classified by their position along typological dimensions like whether verbs precede their objects.
  • Encoder-decoder networks are composed of an encoder network that takes an input sequence and creates a contextualized representation of it, the context. This context representation is then passed to a decoder which generates a task-specific output sequence.
  • The attention mechanism enriches the context vector to allowing the decoder to view information from all the hidden states of the encoder, not just the last hidden state.
  • The encoder-decoder architecture can be implemented by RNNs or by Transformers.
  • For the decoder, choosing the single most probable token to generate at each step is called greedy decoding.
  • In beam search, instead of choosing the best token to generate at each timestep, we keep k possible tokens at each step. This fixed-size memory footprint k is called the beam width.
  • Machine translation models are trained on a parallel corpus, sometimes called a bitext, a text that appears in two (or more) languages.
  • Backtranslation is a way of making use of monolingual corpora in the target language by running a pilot MT engine backwards to create synthetic bitexts.
  • MT is evaluated by measuring a translation’s adequacy (how well it captures the meaning of the source sentence) and fluency (how fluent or natural it is in the target language). Human evaluation is the gold standard, but automatic evaluation metrics like BLEU, which measure word or n-gram overlap with human translations, or more recent metrics based on embedding similarity, are also commonly used.
- korean_translation:
  기계 번역은 NLP에서 가장 널리 사용되는 애플리케이션 중 하나이며, MT를 위해 처음 개발된 인코더-디코더 모델은 NLP 전반에 걸쳐 응용되는 핵심 도구입니다.
  • 언어는 구조적 및 어휘적 차이(divergences)를 가지고 있어 번역을 어렵게 만듭니다.
  • 언어 유형론(typology) 분야에서는 이러한 차이점들을 조사합니다. 언어는 동사가 목적어 앞에 오는지와 같은 유형론적 차원에 따라 분류될 수 있습니다.
  • 인코더-디코더 네트워크는 입력 시퀀스를 받아 문맥화된 표현인 문맥(context)을 생성하는 인코더 네트워크로 구성됩니다. 이 문맥 표현은 특정 태스크의 출력 시퀀스를 생성하는 디코더로 전달됩니다.
  • 어텐션 메커니즘(attention mechanism)은 디코더가 마지막 은닉 상태뿐만 아니라 인코더의 모든 은닉 상태의 정보를 볼 수 있게 함으로써 문맥 벡터를 풍부하게 합니다.
  • 인코더-디코더 아키텍처는 RNN 또는 트랜프포머(Transformers)로 구현될 수 있습니다.
  • 디코더에서 매 단계마다 생성할 가장 확률이 높은 단일 토큰을 선택하는 것을 탐욕적(greedy) 디코딩이라고 합니다.
  • 빔 서치(beam search)에서는 매 시점마다 최적의 토큰 하나를 선택하는 대신, 각 단계에서 $k$개의 가능한 토큰을 유지합니다. 이 고정된 메모리 사용량 $k$를 빔 폭(beam width)이라고 부릅니다.
  • 기계 번역 모델은 두 개(또는 그 이상)의 언어로 나타나는 텍스트인 병렬 코퍼스(parallel corpus, 때로는 bitext라고 함)로 학습됩니다.
  • 역번역(Backtranslation)은 예비 MT 엔진을 반대 방향으로 실행하여 합성 병렬 데이터를 생성함으로써 타깃 언어의 단일 언어 코퍼스를 활용하는 방법입니다.
  • MT는 번역의 적절성(adequacy, 소스 문장의 의미를 얼마나 잘 포착하는가)과 유창성(fluency, 타깃 언어로서 얼마나 유창하고 자연스러운가)을 측정하여 평가됩니다. 인간 평가가 표준이지만, 인간 번역과의 단어 또는 n-그램 중복을 측정하는 BLEU와 같은 자동 평가 지표나 임베딩 유사도에 기반한 최신 지표들도 흔히 사용됩니다.

- source_text: In early years, the space of MT architectures spanned three general models. In perhaps the earliest developed method, direct translation, the system proceeds word-by-word through the source-language text, translating each word incrementally. Direct translation uses a large bilingual dictionary, each of whose entries is a small program with the job of translating one word. In transfer approaches, we first parse the input text and then apply rules to transform the source-language parse into a target language parse. we then generate the target language sentence from the parse tree. In interlingua approaches, we analyze the source language text into some abstract meaning representation, called an interlingua. we then generate into the target language from this interlingual representation. A common way to visualize these three early approaches was the Vauquois triangle shown in Fig. 11.20.
