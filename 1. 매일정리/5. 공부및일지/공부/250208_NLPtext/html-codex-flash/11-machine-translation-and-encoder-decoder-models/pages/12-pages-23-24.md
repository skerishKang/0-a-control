## Block 5
- type: paragraph
- source_text: While automatic metrics like BLEU are useful, they have important limitations. BLEU is very local: a large phrase that is moved around might not change the BLEU score at all, and BLEU can’t evaluate cross-sentence properties of a document like its discourse coherence (Chapter 22). BLEU and similar automatic metrics also do poorly at comparing very different kinds of systems, such as comparing human-aided translation against machine translation, or different machine translation architectures against each other (Callison-Burch et al., 2006). Such automatic metrics are probably most appropriate when evaluating changes to a single system.

- source_text: The BLEU metric is based on measuring the exact word or n-grams a human reference and candidate machine translation have in common. However, this criterion is overly strict, since a good translation may use alternate words or paraphrases. A solution pioneered in early metrics like METEOR (Banerjee and Lavie, 2005) was to allow synonyms to match between the reference x and candidate x˜. More recent metrics use BERT or other embeddings to implement this intuition.
- korean_translation: BLEU 지표는 인간 참조 번역과 후보 기계 번역이 공통으로 가진 정확한 단어나 n-그램을 측정하는 데 기반합니다. 하지만 좋은 번역이라도 다른 단어나 패러프레이즈(paraphrase)를 사용할 수 있기 때문에, 이 기준은 지나치게 엄격합니다. ## Block 8
- type: paragraph
- source_text: For example, in some situations we might have datasets that have human assessments of translation quality. Such datasets consists of tuples (x, x˜, r), where x = (x1 , ..., xn ) is a reference translation, x˜ = (xˆ1 , ..., xˆm ) is a candidate machine translation, and r ∈ R is a human rating that expresses the quality of x˜ with respect to x. The output of such models correlates highly with human labels.
- korean_translation: 예를 들어, 인간이 번역 품질을 평가한 데이터셋을 보유한 상황이 있을 수 있습니다. 이러한 데이터셋은 튜플 $(x, \tilde{x}, r)$로 구성되는데, 여기서 $x$는 참조 번역이고 $\tilde{x}$는 후보 기계 번역이며, $r$은 $x$에 대한 $\tilde{x}$의 품질을 표현하는 인간의 평점입니다. 이러한 데이터가 주어지면 BLEURT(Sellam et al., 2020)와 같은 알고리즘은 인간이 라벨링한 데이터셋으로 예측 모델을 학습시킵니다. 이 방식은 $x$와 $\tilde{x}$를 (추가적인 사전 학습을 거친 후 인간 라벨 문장들로 미세 조정된) BERT 버전에 통과시킨 뒤, $r$을 예측하도록 학습된 선형 레이어를 연결합니다. 이러한 모델의 출력은 인간의 라벨과 높은 상관관계를 보입니다.

- type: paragraph
- source_text: In other cases, however, we don’t have such human-labeled datasets. In that case we can measure the similarity of x and x˜ by the similarity of their embeddings. Each pair of tokens (xi , x˜j ) is scored by its cosine (xi · x˜j) / (|xi||x˜j|). Each token in x is matched to a token in x˜ to compute recall, and each token in x˜ is matched to a token in x to compute precision (with each token greedily matched to the most similar token in the corresponding sentence). BERTSCORE provides precision and recall (and hence F1):
- korean_translation: 하지만 인간이 라벨링한 데이터셋이 없는 경우도 있습니다. 그럴 때는 $x$와 $\tilde{x}$의 임베딩 유사도를 통해 유사성을 측정할 수 있습니다. 예를 들어 그림 11.18에 나타난 BERTSCORE 알고리즘(Zhang et al., 2020)은 참조 $x$와 후보 $\tilde{x}$를 BERT에 통과시켜 각 토큰 $x_i$와 $\tilde{x}_j$에 대한 BERT 임베딩을 계산합니다. 각 토큰 쌍 $(x_i, \tilde{x}_j)$은 코사인 유사도인 $(x_i \cdot \tilde{x}_j) / (|x_i||\tilde{x}_j|)$로 점수가 매겨집니다. 재현율(recall)을 계산하기 위해 $x$의 각 토큰을 $\tilde{x}$의 토큰과 매칭시키고, 정밀도(precision)를 계산하기 위해 $\tilde{x}$의 각 토큰을 $x$의 토큰과 매칭시킵니다(각 토큰은 대응하는 문장에서 가장 유사한 토큰과 탐욕적으로 매칭됨). BERTSCORE는 정밀도와 재현율(그리고 이를 통한 F1 점수)을 제공합니다.
- study_note: 별도의 라벨링 없이도 임베딩 유사도만으로 정밀도와 재현율을 산출하는 BERTSCORE의 메커니즘을 설명합니다.
