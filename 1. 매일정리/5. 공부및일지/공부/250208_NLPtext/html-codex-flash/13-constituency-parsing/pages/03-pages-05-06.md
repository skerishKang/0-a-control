- source_text: The main loop of the CKY algorithm iterates over the length of the subproblems, starting from length 1 and going up to length n. For each length, we iterate over the starting index i of the subproblem. Then we try all possible split points k such that i < k < j.
- korean_translation: CKY 알고리즘의 메인 루프는 길이가 1인 것부터 시작하여 문장의 길이인 n에 도달할 때까지 하위 문제의 길이를 반복합니다. 각 길이에 대해 하위 문제의 시작 인덱스 i를 반복하고, i < k < j를 만족하는 모든 가능한 분할 지점 k를 시도합니다.
- study_note: CKY 알고리즘의 동적인 하위 문제 분할과 반복 절차를 설명합니다.

## Block 3
- type: code
- source_text:
    function CKY-PARSE(words, grammar) returns table
      for j = 1 to n do
        table[j-1, j] = {A | A → words[j] is in grammar}
        for i = j-2 down to 0 do
          for k = i+1 to j-1 do
            if A → B C is in grammar and B ∈ table[i, k] and C ∈ table[k, j] then
              table[i, j] = table[i, j] ∪ {A}
      return table
- korean_translation:
    함수 CKY-PARSE(words, grammar) 반환 table
      j = 1부터 n까지 반복
        table[j-1, j] = {A | A → words[j]가 문법에 있음}
        i = j-2부터 0까지 역순 반복
          k = i+1부터 j-1까지 반복
            만약 A → B C가 문법에 있고 B ∈ table[i, k]이며 C ∈ table[k, j]이면
              table[i, j] = table[i, j] ∪ {A}
      표(table) 반환
- study_note: CKY 알고리즘의 의사 코드를 보여줍니다. 단어 기반의 기저 사례와 동적 프로그래밍을 통한 성분 조합 과정을 나타냅니다.

## Block 4
- type: paragraph
- source_text: This is a recognizer. If the start symbol S is in table[0, n], it means the sentence is valid in the language. To make it a parser, we need to store backpointers in each entry table[i, j] to remember which B and C from which split point k were used to create A.
- korean_translation: 이것은 인식기(recognizer)입니다. 시작 기호 S가 table[0, n]에 포함되어 있다면, 해당 문장은 그 언어에서 유효하다는 뜻입니다. 이를 파서(parser)로 만들려면 각 칸 table[i, j]에 어떤 분할 지점 k로부터 어떤 B와 C가 사용되어 A가 만들어졌는지를 기억하도록 역추적 포인터(backpointers)를 저장해야 합니다.
- study_note: 단순한 인식기 구조에서 구조를 복원하기 위한 파서로 확장하는 방법을 설명합니다.

## Block 3
- type: paragraph
- source_text: Conversion to CNF involves three main steps. Any grammar can be converted into an equivalent CNF grammar while preserving the strings generated by the grammar.
- korean_translation: CNF로의 변환에는 세 가지 주요 단계가 수반됩니다. 모든 문법은 그에 상응하는 CNF 문법으로 변환될 수 있으며, 이때 문법이 생성하는 문자열들은 그대로 유지됩니다.
- study_note: CNF 변환의 기본적인 보장과 주요 단계를 도입합니다.

## Block 4
- type: list
- source_text:
    - 1. Replace all terminal symbols that occur on the RHS of rules that also contain other symbols with new non-terminals and new rules that derive those terminals.
    - 2. Break down rules with more than two non-terminals on the RHS by introducing intermediate non-terminals.
    - 3. Remove all unit productions (A → B) and epsilon productions (A → ε).
- korean_translation:
    - 1. 하나이상의 기호를 포함하고 있는 규칙의 오른쪽면(RHS)에 단말 기호가 나타날 경우, 새로운 비단말 기호와 해당 단말 기호를 유도하는 규칙으로 대체합니다.
    - 2. 오른쪽면(RHS)에 세 개 이상의 비단말 기호를 가진 규칙을 중간 비단말 기호를 도입하여 더 작은 규칙들로 쪼갭니다.
    - 3. 모든 단위 규칙(A → B)과 엡실론 규칙(A → ε)을 제거합니다.
- study_note: CNF로 변환하기 위해 필요한 구체적인 세 가지 절차를 명시합니다.
