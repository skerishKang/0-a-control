## Block 1
- type: paragraph
- source_text: To use the CKY algorithm, the grammar must be in Chomsky Normal Form (CNF). A grammar is in CNF if every rule is of the form A → B C or A → a.
- korean_translation: CKY 알고리즘을 사용하려면 문법이 반드시 촘스키 정규형(Chomsky Normal Form, CNF)이어야 합니다. 모든 규칙이 A → B C 또는 A → a 형식을 취할 때, 해당 문법은 CNF라고 합니다.
- study_note: CKY 알고리즘의 전제 조건인 촘스키 정규형의 정의를 설명합니다.

## Block 3
- type: paragraph
- source_text: Any context-free grammar can be converted into an equivalent grammar in CNF. This involves several steps, including removing unit productions (rules of the form A → B) and epsilon productions (rules of the form A → ε).
- korean_translation: 모든 문맥 자유 문법은 그와 동등한 CNF 문법으로 변환될 수 있습니다. 여기에는 단위 분석 규칙(A → B 형태의 규칙)과 엡실론 분석 규칙(A → ε 형태의 규칙)을 제거하는 등의 여러 단계가 포함됩니다.
- study_note: 임의의 CFG를 CNF로 변환할 수 있는 가능성과 그 과정을 간략히 설명합니다.

- korean_translation: 13.2.1 CKY 알고리즘

## Block 2
- type: paragraph
- source_text: The CKY algorithm works by filling in a triangular table. For a sentence of length n, the table has (n+1) rows and (n+1) columns.
- korean_translation: CKY 알고리즘은 삼각형 모양의 표를 채우는 방식으로 작동합니다. 길이가 n인 문장에 대해 표는 (n+1)개의 행과 (n+1)개의 열을 가집니다.
- study_note: CKY 알고리즘에서 사용되는 표의 구조와 크기를 설명합니다.

## Block 3
- type: margin_note
- source_text: recognize / parse
- korean_translation: 인식(recognize) / 파싱(parse)
- study_note: 주어진 문자열이 문법에 부합하는지 확인하는 것과 구조를 찾아내는 것을 구분합니다.

## Block 4
- type: paragraph
- source_text: The entries in the table represent the non-terminals that can derive a portion of the sentence. For example, the entry table[i, j] contains the set of all non-terminals A such that A derives the segment of the sentence from index i to index j.
- korean_translation: 표의 각 칸은 문장의 일부를 유도할 수 있는 비단말 기호들을 나타냅니다. 예를 들어, 표의 칸[i, j]에는 인덱스 i부터 인덱스 j까지의 문장 부분을 유도하는 모든 비단말 기호 A의 집합이 포함됩니다.
- study_note: CKY 표의 각 칸이 갖는 의미를 설명합니다.
