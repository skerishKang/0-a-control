Block 1

13.6 Parsing Evaluation: PARSEVAL
Korean Translation
13.6 파싱 평가: PARSEVAL
Study Note
생성된 파스 트리의 정확도를 골드 표준 트리와 비교하여 측정하는 표준 평가 방식인 PARSEVAL을 다릅니다.
Block 2
paragraph
Source Text
To evaluate a parser, we need a way to compare the parse tree it produces to a "gold standard" parse tree, usually from a treebank. The standard metric for this is the PARSEVAL metric, which was originally proposed by Abney et al. (1991).
Korean Translation
파서를 평가하기 위해, 파서가 생성한 파스 트리를 트리뱅크 등에 있는 "골드 표준(gold standard)" 파스 트리와 비교하는 방법이 필요합니다. 이를 위한 표준 척도는 원래 Abney 등에 의해 제안된 PARSEVAL입니다.
Study Note
파싱 평가의 기본 개념과 척도인 PARSEVAL의 기원과 용도를 설명합니다.

Block 5
paragraph
Source Text
PARSEVAL measures the similarity between two trees by comparing the constituents they contain. A constituent is defined by its label and its span (the start and end indices of the segment of the sentence it covers).
Korean Translation
PARSEVAL은 트리들이 포함하는 성분들을 비교하여 두 트리 간의 유사성을 측정합니다. 성분은 그 레이블과 범주(span, 문장에서 해당 부분이 시작되고 끝나는 인덱스)에 의해 정의됩니다.
Study Note
PARSEVAL이 트리를 비교할 때 사용하는 '범주'와 '레이블'의 개념을 설명합니다.
Block 6
formula
Source Text
Labeled Precision = # of correct constituents in T_p / # of constituents in T_p
Labeled Recall = # of correct constituents in T_p / # of constituents in T_g
F1 = (2 * Precision * Recall) / (Precision + Recall)
Korean Translation
레이블된 정밀도 = 생성 트리(T_p)의 올바른 성분 개수 / 생성 트리(T_p)의 전체 성분 개수
레이블된 재현율 = 생성 트리(T_p)의 올바른 성분 개수 / 정답 트리(T_g)의 전체 성분 개수
F1 = (2 * 정밀도 * 재현율) / (정밀도 + 재현율)
Study Note
PARSEVAL에서 사용되는 구체적인 계산 공식들입니다.

Study Note

Block 2
paragraph
Source Text
While PARSEVAL is the standard metric, it has some limitations. One issue is that it treats all errors as equally important, even if some errors are more semantically significant than others.
Korean Translation
PARSEVAL이 표준 척도이기는 하지만 몇 가지 한계가 있습니다. 한 가지 문제는 모든 오류를 동일하게 중요하다는 점입니다. 실제로는 어떤 오류가 다른 오류보다 더 중요한 의미를 지닐 수 있음에도 불구하고 말입니다.
Study Note
PARSEVAL의 단순 수치적 평가 방식이 실제 의미적 차이를 모두 반영하지 못함을 지적합니다.
