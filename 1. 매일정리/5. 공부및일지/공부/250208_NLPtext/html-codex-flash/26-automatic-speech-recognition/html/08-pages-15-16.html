The standard evaluation metric for speech recognition systems is the word error rate. The word error rate is based on how much the word string returned by the recognizer (the hypothesized word string) differs from a reference transcription. The first step in computing word error is to compute the minimum edit distance in words between the hypothesized and correct strings, giving us the minimum number of word substitutions, word insertions, and word deletions necessary to map between the correct and hypothesized strings. The word error rate (WER) is then defined as follows (note that because the equation includes insertions, the error rate can be greater than 100%):

음성 인식 시스템의 표준 평가 지표는 단어 오류율(word error rate, WER)입니다. 단어 오류율은 인식기가 반환한 단어 문자열(가설 문자열)이 참조 전사문(정답 문자열)과 얼마나 다른지에 기초합니다. 단어 오류 계산의 첫 번째 단계는 가설 문자열과 정답 문자열 사이의 최소 편집 거리(minimum edit distance)를 단어 단위로 계산하여, 두 문자열을 일치시키는 데 필요한 최소 단어 교체(substitution), 삽입(insertion), 삭제(deletion) 횟수를 구하는 것입니다. 단어 오류율(WER)은 다음과 같이 정의됩니다(이 식에는 삽입이 포함되어 있어 오류율이 100%를 넘을 수 있음에 유의하십시오).

Source Text
The standard method for computing word error rates is a free script called sclite, available from the National Institute of Standards and Technologies (NIST) (NIST, 2005). Sclite is given a series of reference (hand-transcribed, gold-standard) sentences and a matching set of hypothesis sentences. Besides performing alignments, and computing word error rate, sclite performs a number of other useful tasks. For example, for error analysis it gives useful information such as confusion matrices showing which words are often misrecognized for others, and summarizes statistics of words that are often inserted or deleted. sclite also gives error rates by speaker (if sentences are labeled for speaker ID), as well as useful statistics like the sentence error rate, the percentage of sentences with at least one word error.
Korean Translation
단어 오류율을 계산하는 표준 방법은 NIST(미국 국립표준기술연구소)에서 제공하는 sclite라는 무료 스크립트를 사용하는 것입니다(NIST, 2005). sclite는 일련의 참조(수동 전사된 골드 스탠다드) 문장과 그에 대응하는 가설 문장 세트를 입력받습니다. sclite는 정렬 수행 및 단어 오류율 계산 외에도 여러 유용한 작업을 수행합니다. 예를 들어, 어떤 단어가 다른 단어로 자주 오인되는지를 보여주는 혼동 행렬(confusion matrices)과 같이 오류 분석에 유용한 정보를 제공하며, 자주 삽입되거나 삭제되는 단어의 통계를 요약해 줍니다. 또한 (화자 ID가 레이블링된 경우) 화자별 오류율뿐만 아니라, 최소 하나 이상의 단어 오류가 있는 문장의 비율인 문장 오류율(sentence error rate)과 같은 유용한 통계도 제공합니다.

As with other language processing algorithms, we need to know whether a particular improvement in word error rate is significant or not. The standard statistical tests for determining if two word error rates are different is the Matched-Pair Sentence Segment Word Error (MAPSSWE) test, introduced in Gillick and Cox (1989).
Korean Translation
다른 언어 처리 알고리즘과 마찬가지로, 단어 오류율의 특정 개선이 유의미한지 여부를 알아야 합니다. 두 단어 오류율이 서로 다른지 결정하기 위한 표준 통계 테스트는 Gillick과 Cox(1989)가 도입한 MAPSSWE(Matched-Pair Sentence Segment Word Error) 테스트입니다.
Study Note
ASR 성능 비교 시 통계적 유의성 검정의 필요성과 대표적인 방법인 MAPSSWE를 언급합니다.
Block 12
paragraph
Source Text
The MAPSSWE test is a parametric test that looks at the difference between the number of word errors the two systems produce, averaged across a number of segments. The segments may be quite short or as long as an entire utterance; in general, we want to have the largest number of (short) segments in order to justify the normality assumption and to maximize power. The test requires that the errors in one segment be statistically independent of the errors in another segment. Since ASR systems tend to use trigram LMs, we can approximate this requirement by defining a segment as a region bounded on both sides by words that both recognizers get correct (or by turn/utterance boundaries).
Korean Translation
MAPSSWE 테스트는 두 시스템이 생성한 단어 오류 수의 차이를 여러 세그먼트에 걸쳐 평균하여 살펴보는 모수적 테스트(parametric test)입니다. 세그먼트는 매우 짧을 수도 있고 전체 발화만큼 길 수도 있습니다. 일반적으로 정규성 가정을 정당화하고 검정력을 극대화하기 위해 가능한 많은 수의 (짧은) 세그먼트를 확보하는 것이 좋습니다. 이 테스트는 한 세그먼트의 오류가 다른 세그먼트의 오류와 통계적으로 독립적일 것을 요구합니다. ASR 시스템은 트라이그램(trigram) LM을 사용하는 경향이 있으므로, 두 인식기 모두 정답을 맞힌 단어(또는 차례/발화 경계)로 양쪽이 둘러싸인 영역을 세그먼트로 정의함으로써 이 요구 사항을 근사할 수 있습니다.
Study Note
MAPSSWE 테스트의 방식과 세그먼트 정의를 통한 독립성 확보 방안을 설명합니다.

Source Text
In region I, system A has two errors (a deletion and an insertion) and system B has zero; in region III, system A has one error (a substitution) and system B has two. Let’s define a sequence of variables Z representing the difference between the errors in the two systems as follows: Ni_A: the number of errors made on segment i by system A, Ni_B: the number of errors made on segment i by system B, Zi = Ni_A - Ni_B ; i = 1, 2, ..., n where n is the number of segments. Intuitively, if the two systems are identical, we would expect the average difference, that is, the average of the Z values, to be zero.
Korean Translation
영역 I에서 시스템 A는 두 개의 오류(삭제 및 삽입)가 있고 시스템 B는 오류가 없습니다. 영역 III에서 시스템 A는 한 개의 오류(교체)가 있고 시스템 B는 두 개가 있습니다. 두 시스템의 오류 차이를 나타내는 변수 시퀀스 Z를 다음과 같이 정의해 봅시다. Ni_A: 시스템 A가 세그먼트 i에서 범한 오류 수, Ni_B: 시스템 B가 세그먼트 i에서 범한 오류 수, Zi = Ni_A - Ni_B (i = 1, 2, ..., n, 여기서 n은 세그먼트 수). 직관적으로 두 시스템이 동일하다면 평균 차이, 즉 Z 값들의 평균은 0이 될 것으로 기대합니다.
Study Note

Block 9
paragraph
Source Text
The goal of text-to-speech (TTS) systems is to map from strings of letters to waveforms, a technology that’s important for a variety of applications from dialogue systems to games to education. Like ASR systems, TTS systems are generally based on the encoder-decoder architecture, either using LSTMs or Transformers. There is a general difference in training. The default condition for ASR systems is to be speaker-independent: they are trained on large corpora with thousands of hours of speech from many speakers because they must generalize well to an unseen test speaker. By contrast, in TTS, it’s less crucial to use multiple voices, and so basic TTS systems are speaker-dependent: trained to have a consistent voice, on much less data, but all from one speaker. For example, one commonly used public domain dataset, the LJ speech corpus, consists of 24 hours of one speaker, Linda Johnson, reading audio books in the LibriVox project (Ito and Johnson, 2017), much smaller than standard ASR corpora which are hundreds or thousands of hours.
Korean Translation
텍스트 음성 변환(text-to-speech, TTS) 시스템의 목표는 문자열을 파형(waveform)으로 매핑하는 것이며, 이는 대화 시스템부터 게임, 교육에 이르기까지 다양한 응용 분야에서 중요한 기술입니다. ASR 시스템과 마찬가지로 TTS 시스템도 일반적으로 LSTM 또는 트랜스포머를 사용하는 인코더-디코더 아키텍처를 기반으로 합니다. 학습 방식에는 일반적인 차이가 있습니다. ASR 시스템의 기본 조건은 화자 독립적(speaker-independent)인 것입니다. 보지 못한 테스트 화자에게 잘 일반화되어야 하므로 수많은 화자의 수천 시간 분량의 음성이 포함된 대규모 코퍼스로 학습됩니다. 반면 TTS에서는 여러 목소리를 사용하는 것이 덜 중요하므로, 기본적인 TTS 시스템은 화자 의존적(speaker-dependent)입니다. 즉, 단 한 명의 화자로부터 얻은 훨씬 적은 양의 데이터로 일관된 목소리를 갖도록 학습됩니다. 예를 들어, 흔히 사용되는 공개 데이터셋인 LJ speech 코퍼스는 LibriVox 프로젝트에서 오디오북을 읽어주는 한 명의 화자 Linda Johnson의 24시간 분량 음성으로 구성되어 있는데, 이는 수백 또는 수천 시간 분량인 표준 ASR 코퍼스보다 훨씬 작습니다.
Study Note

We generally break up the TTS task into two components. The first component is an encoder-decoder model for spectrogram prediction: it maps from strings of letters to mel spectrographs: sequences of mel spectral values over time. The second component maps from mel spectrograms to waveforms. Generating waveforms from intermediate representations like spectrograms is called vocoding and this second component is called a vocoder.
Korean Translation
우리는 일반적으로 TTS 작업을 두 개의 구성 요소로 나눕니다. 첫 번째 구성 요소는 스펙트로그램 예측을 위한 인코더-디코더 모델입니다. 이는 문자열을 시간에 따른 멜 스펙트럼 값의 시퀀스인 멜 스펙트로그램(mel spectrographs)으로 매핑합니다. 두 번째 구성 요소는 멜 스펙트로그램을 파형으로 매핑합니다. 스펙트로그램과 같은 중간 표현으로부터 파형을 생성하는 것을 보코딩(vocoding)이라고 하며, 이 두 번째 구성 요소를 보코더(vocoder)라고 부릅니다.
Study Note
