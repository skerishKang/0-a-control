The goal of the subsampling is to produce a shorter sequence X = x1, ..., xn that will be the input to the encoder. The simplest algorithm is a method sometimes called low frame rate (Pundak and Sainath, 2016): for time i we stack (concatenate) the acoustic feature vector fi with the prior two vectors fi-1 and fi-2 to make a new vector three times longer. Then we simply delete fi-1 and fi-2. Thus instead of (say) a 40-dimensional acoustic feature vector every 10 ms, we have a longer vector (say 120-dimensional) every 30 ms, with a shorter sequence length n = t/3.
Korean Translation
서브샘플링(subsampling)의 목표는 인코더의 입력이 될 더 짧은 시퀀스 X = x1, ..., xn을 만드는 것입니다. 가장 간단한 알고리즘은 때때로 낮은 프레임 레이트(low frame rate)라고 불리는 방법입니다(Pundak and Sainath, 2016). 시간 i에 대해 음향 특징 벡터 fi를 이전의 두 벡터 fi-1, fi-2와 쌓아서(결합하여) 세 배 더 긴 새로운 벡터를 만듭니다. 그런 다음 fi-1과 fi-2를 단순히 삭제합니다. 이렇게 하면 예를 들어 10ms마다 40차원 음향 특징 벡터가 있는 대신, 30ms마다 120차원의 더 긴 벡터가 있게 되어 시퀀스 길이가 n = t/3로 짧아집니다.

Since an encoder-decoder model is essentially a conditional language model, encoder-decoders implicitly learn a language model for the output domain of letters from their training data. However, the training data (speech paired with text transcriptions) may not include sufficient text to train a good language model. After all, it’s easier to find enormous amounts of pure text training data than it is to find text paired with speech. Thus we can usually improve a model at least slightly by incorporating a very large language model.
Korean Translation
인코더-디코더 모델은 본질적으로 조건부 언어 모델이므로, 학습 데이터를 통해 출력 영역인 문자에 대한 언어 모델을 암시적으로 학습합니다. 그러나 학습 데이터(텍스트 전사문과 쌍을 이룬 음성)에는 좋은 언어 모델을 학습시키기에 충분한 텍스트가 포함되어 있지 않을 수 있습니다. 결국, 음성과 쌍을 이룬 텍스트보다 방대한 양의 순수 텍스트 학습 데이터를 찾는 것이 훨씬 쉽습니다. 따라서 매우 큰 규모의 언어 모델을 통합함으로써 보통 모델의 성능을 조금이라도 향상시킬 수 있습니다.

We pointed out in the previous section that speech recognition has two particular properties that make it very appropriate for the encoder-decoder architecture, where the encoder produces an encoding of the input that the decoder uses attention to explore. First, in speech we have a very long acoustic input sequence X mapping to a much shorter sequence of letters Y, and second, it’s hard to know exactly which part of X maps to which part of Y.

In this section we briefly introduce an alternative to encoder-decoder: an algorithm and loss function called CTC, short for Connectionist Temporal Classification (Graves et al., 2006), that deals with these problems in a very different way. The intuition of CTC is to output a single character for every frame of the input, so that the output is the same length as the input, and then to apply a collapsing function that combines sequences of identical letters, resulting in a shorter sequence.
Korean Translation
이 섹션에서는 인코더-디코더의 대안으로, 이러한 문제들을 매우 다른 방식으로 다루는 CTC(Connectionist Temporal Classification의 약자, Graves et al., 2006)라는 알고리즘과 손실 함수를 간략히 소개합니다. CTC의 직관은 입력의 모든 프레임에 대해 하나의 문자를 출력하여 출력이 입력과 동일한 길이가 되게 한 다음, 동일한 문자의 시퀀스를 결합하는 축소 함수(collapsing function)를 적용하여 더 짧은 시퀀스를 만드는 것입니다.
Study Note
모든 프레임에 대해 출력을 내고 나중에 중복을 합치는 CTC의 독특한 방식을 설명합니다.
