The ASR component can also be made dependent on the dialogue state. For example, if the system has just asked the user "What city are you departing from?", the ASR language model component can be constrained to assign high probability to answers with city names. This can be done by training a language model on answers to this question, or by hand-writing finite-state or context-free grammars for such sentences. A language model that is completely dependent on dialogue state is called a restrictive grammar; usually real systems mix restrictive grammars with more general language models.
Korean Translation
ASR 구성 요소는 대화 상태에 따라 달라질 수 있습니다. 예를 들어, 시스템이 사용자에게 "어느 도시에서 출발하시나요?"라고 질문했다면, ASR 언어 모델 구성 요소는 도시 이름이 포함된 답변에 높은 확률을 할당하도록 제한될 수 있습니다. 이는 해당 질문에 대한 답변으로 언어 모델을 훈련시키거나, 그러한 문장에 대한 유한 상태 또는 문맥 자유 문법을 직접 작성함으로써 가능합니다. 대화 상태에 완전히 의존하는 언어 모델을 제한적 문법(restrictive grammar)이라고 하며, 보통 실제 시스템은 제한적 문법과 더 일반적인 언어 모델을 혼합하여 사용합니다.

dialogue state tracker maintains the current state of the dialogue (which include the user's most recent dialogue act, plus the entire set of slot-filler constraints the user has expressed so far). The dialogue policy decides what the system should do or say next. The dialogue policy in GUS was simple: ask questions until the frame was full and then report back the results of some database query. But a more sophisticated dialogue policy can help a system decide when to answer the user's questions, when to instead ask the user a clarification question, when to make a suggestion, and so on. Finally, dialogue state systems have a natural language generation component. In GUS, the sentences that the generator produced were all from pre-written templates. But a more sophisticated generation component can condition on the exact context to produce turns that seem much more natural.
Korean Translation
대화 상태 추적기(dialogue state tracker)는 대화의 현재 상태(사용자의 가장 최근 대화 행위와 지금까지 사용자가 표현한 모든 슬롯 채우기 제약 조건을 포함)를 유지합니다. 대화 정책(dialogue policy)은 시스템이 다음에 무엇을 하거나 말해야 할지 결정합니다. GUS의 대화 정책은 간단했습니다: 프레임이 찰 때까지 질문을 던지고 데이터베이스 쿼리 결과를 보고하는 것이었습니다. 그러나 더 정교한 대화 정책은 시스템이 사용자의 질문에 언제 답할지, 언제 명확화 질문(clarification question)을 던질지, 언제 제안을 할지 등을 결정하는 데 도움을 줄 수 있습니다. 마지막으로, 대화 상태 시스템에는 자연어 생성 구성 요소가 있습니다. Dialogue-state systems make use of dialogue acts. Dialogue acts represent the interactive function of the turn or sentence, combining the idea of speech acts and grounding into a single representation. Different types of dialogue systems require labeling different kinds of acts, and so the tagset—defining what a dialogue act is exactly— tends to be designed for particular tasks.
Korean Translation
대화 상태 시스템은 대화 행위(dialogue acts)를 사용합니다. 대화 행위는 발언이나 문장의 상호작용적 기능을 나타내며, 화행(speech acts)과 접지(grounding)의 개념을 하나의 표현으로 결합합니다. 대화 시스템의 종류에 따라 서로 다른 종류의 행위를 레이블링해야 하므로, 대화 행위가 정확히 무엇인지 정의하는 태그 세트(tagset)는 특정 과업에 맞게 설계되는 경향이 있습니다.
