Source Text
Automatic evaluations are generally not used for chatbots. That's because computational measures of generation performance like BLEU or ROUGE or embedding dot products between a chatbot's response and a human response correlate very poorly with human judgments (Liu et al., 2016). These methods perform poorly because there are so many possible responses to any given turn; simple word-overlap or semantic similarity metrics work best when the space of responses is small and lexically overlapping, which is true of generation tasks like machine translation or possibly summarization, but definitely not dialogue.
Korean Translation
자동 평가는 일반적으로 챗봇에 사용되지 않습니다. 이는 BLEU나 ROUGE와 같은 생성 성능의 계산 지표나 챗봇의 응답과 인간의 응답 간의 임베딩 내적이 인간의 판단과 매우 낮은 상관관계를 보이기 때문입니다(Liu 등, 2016). 이러한 방법들은 주어진 턴에 대해 가능한 응답이 너무 많기 때문에 성능이 저하됩니다. 단순한 단어 중복이나 의미적 유사성 지표는 응답 공간이 작고 어휘적으로 중복될 때 가장 잘 작동하며, 이는 기계 번역이나 요약과 같은 생성 작업에는 해당되지만 대화에는 확실히 해당되지 않습니다.

For task-based dialogue, if the task is unambiguous, we can simply measure absolute task success (did the system book the right plane flight, or put the right event on the calendar).
To get a more fine-grained idea of user happiness, we can compute a user satisfaction rating, having users interact with a dialogue system to perform a task and then having them complete a questionnaire. For example, Fig. 24.20 shows sample multiple-choice questions (Walker et al., 2001); responses are mapped into the range of 1 to 5, and then averaged over all questions to get a total user satisfaction rating.
