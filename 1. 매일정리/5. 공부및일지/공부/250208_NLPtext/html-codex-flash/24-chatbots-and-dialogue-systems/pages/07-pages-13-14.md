## Block 11
- type: paragraph
- source_text: All modern task-based dialogue systems, whether the simple GUS architecture we describe here, or the more sophisticated dialogue state architectures we turn to in the following section, are based around frames. A frame is a kind of knowledge structure representing the kinds of intentions the system can extract from user sentences, and consists of a collection of slots, each of which can take a set of possible values. Together this set of frames is sometimes called a domain ontology.
- korean_translation: 여기서 설명하는 단순한 GUS 아키텍처든, 다음 섹션에서 살펴볼 더 정교한 대화 상태 아키텍처든, 모든 현대적인 태스크 기반 대화 시스템은 프레임을 기반으로 합니다. 프레임은 시스템이 사용자 문장에서 추출할 수 있는 의도의 종류를 나타내는 일종의 지식 구조이며, 각각 가능한 값 세트를 가질 수 있는 슬롯들의 모음으로 구성됩니다. 이러한 프레임 세트를 도메인 온톨로지(domain ontology)라고 부르기도 합니다.

## Block 6
- type: paragraph
- source_text: The control architecture for frame-based dialogue systems, used in various forms in modern systems like Apple's Siri, Amazon's Alexa, and the Google Assistant, is designed around the frame. The system's goal is to fill the slots in the frame with the fillers the user intends, and then perform the relevant action for the user (answering a question, or booking a flight).
- korean_translation: Apple의 Siri, Amazon의 Alexa, Google 어시스턴트와 같은 현대 시스템에서 다양한 형태로 사용되는 프레임 기반 대화 시스템의 제어 아키텍처는 프레임을 중심으로 설계되었습니다. 시스템의 목표는 사용자가 의도한 값으로 프레임의 슬롯을 채운 다음, 사용자에게 관련 작업(질문에 답하거나 항공편 예약)을 수행하는 것입니다.
- study_note: 프레임 기반 시스템의 핵심 목적(슬롯 채우기 및 작업 수행)을 설명합니다.
