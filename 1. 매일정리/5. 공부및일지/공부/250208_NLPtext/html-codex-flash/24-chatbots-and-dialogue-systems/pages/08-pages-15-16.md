## Block 5
- type: paragraph
- source_text: Because of this need to dynamically switch control, the GUS architecture is a production rule system. Different types of inputs cause different productions to fire, each of which can flexibly fill in different frames. The production rules can then switch control according to factors such as the user's input and some simple dialogue history like the last question that the system asked.
- korean_translation: 이러한 동적 제어 전환의 필요성 때문에 GUS 아키텍처는 프로덕션 규칙 시스템(production rule system)입니다. 서로 다른 유형의 입력은 서로 다른 프로덕션 규칙을 실행시키며, 각각은 서로 다른 프레임을 유연하게 채울 수 있습니다. 그러면 프로덕션 규칙은 사용자의 입력과 시스템이 마지막으로 던진 질문과 같은 간단한 대화 이력 등의 요인에 따라 제어권을 전환할 수 있습니다.
