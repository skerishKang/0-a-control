## Block 4
- type: paragraph
- source_text: In industrial contexts, machine learning-based systems for slot-filling are often bootstrapped from GUS-style rule-based systems in a semi-supervised learning manner. A rule-based system is first built for the domain, and a test set is carefully labeled. As new user utterances come in, they are paired with the labeling provided by the rule-based system to create training tuples. A classifier can then be trained on these tuples, using the test set to test the performance of the classifier against the rule-based system. Some heuristics can be used to eliminate errorful training tuples, with the goal of increasing precision. As sufficient training samples become available the resulting classifier can often outperform the original rule-based system (Suendermann et al., 2009), although rule-based systems may still remain higher-precision for dealing with complex cases like negation.
- korean_translation: 산업 현장에서 슬롯 채우기를 위한 기계 학습 기반 시스템은 종종 준지도 학습(semi-supervised learning) 방식으로 GUS 스타일의 규칙 기반 시스템으로부터 부트스트랩(bootstrap)됩니다. 먼저 해당 도메인에 대한 규칙 기반 시스템을 구축하고 테스트 세트를 정교하게 레이블링합니다. 새로운 사용자 발언이 들어오면 규칙 기반 시스템이 제공하는 레이블과 쌍을 이루어 훈련 튜플을 생성합니다. 그런 다음 이러한 튜플에 대해 분류기를 훈련시키고, 테스트 세트를 사용하여 규칙 기반 시스템 대비 분류기의 성능을 테스트할 수 있습니다. 정밀도를 높이기 위해 오류가 있는 훈련 튜플을 제거하는 몇 가지 휴리스틱을 사용할 수 있습니다. 충분한 훈련 샘플이 확보되면 결과적으로 얻은 분류기는 종종 원래의 규칙 기반 시스템보다 성능이 뛰어나게 됩니다(Suendermann 등, 2009). 하지만 규칙 기반 시스템은 부정(negation)과 같은 복잡한 경우를 처리하는 데 여전히 더 높은 정밀도를 유지할 수 있습니다.
- study_note: 규칙 기반 시스템에서 기계 학습 시스템으로 전환하는 실무적인 전략(부트스트래핑)과 각각의 장단점을 설명합니다.
