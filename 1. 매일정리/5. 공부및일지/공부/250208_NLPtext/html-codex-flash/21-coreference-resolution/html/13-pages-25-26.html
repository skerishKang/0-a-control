Source Text
Before evaluating a coreference system, we need to extract a test set containing the gold standard coreference links. The coreference links in a document are annotated by marking all the mentions in the document, and then linking each mention to its antecedent. This process clusters the mentions in a document into a set of equivalence classes (the coreference chains) as defined in Section 21.1. We thus evaluate a coreference system by comparing the coreference chains it produced with the gold coreference chains.
Korean Translation
상호참조 시스템을 평가하기 전에, 우리는 골드 표준 상호참조 링크를 포함하는 테스트 세트를 추출해야 합니다. 문서 내의 상호참조 링크는 문서의 모든 언급을 표시한 다음, 각 언급을 선행어에 연결하여 주석이 달립니다. 이 과정은 21.1절에 정의된 바와 같이 문서 내의 언급들을 동치류(상호참조 사슬)의 집합으로 군집화합니다. 따라서 우리는 생성된 상호참조 사슬과 골드 상호참조 사슬을 비교함으로써 상호참조 시스템을 평가합니다.
Study Note
상호참조 시스템 평가를 위해 골드 표준 링크가 필요하며, 시스템이 생성한 상호참조 사슬과 정답 사슬을 비교하여 평가합니다.
Block 3
paragraph
Source Text
As with most systems, we evaluate them by computing precision, recall, and F1. We can compute these over the links between mentions, or over the resulting equivalence classes. Since it turns out there are multiple ways to define precision and recall, the standard way to evaluate coreference is to use 3 metrics (MUC, B3, and CEAF) and to report the average of the 3. We'll summarize B3 and MUC here and give the details of the third, as well as the script used to compute them, at the end of the chapter.
Korean Translation
대부분의 시스템과 마찬가지로, 정밀도, 재현율, 그리고 F1을 계산하여 이들을 평가합니다. 우리는 언급들 사이의 링크에 대해, 혹은 그 결과인 동치류에 대해 이들을 계산할 수 있습니다. 정밀도와 재현율을 정의하는 여러 방법이 있기 때문에, 상호참조를 평가하는 표준적인 방법은 3가지 지표(MUC, B3, 그리고 CEAF)를 사용하고 이 3가지의 평균을 보고하는 것입니다. 여기서는 B3와 MUC를 요약하고, 세 번째의 세부 내용과 이들을 계산하는 데 사용되는 스크립트는 장의 끝에서 제공할 것입니다.
Study Note
평가 지표로 정밀도, 재현율, F1을 사용하며, 상호참조의 표준 평가 방법은 MUC, B3, CEAF의 3가지 지표를 평균하여 보고하는 것입니다.

The B3 (often pronounced B-cubed) algorithm metric (Bagga and Baldwin, 1998) B3 computes a precision and recall for each mention in the document, and then averages them to get overall precision and recall. Let's look at how to compute the B3 precision and recall for a single mention. Recall that a coreference system clusters mentions into equivalence classes (chains). The B3 precision for a given mention i is the percentage of mentions in the system chain for i that are also in the gold chain for i. The B3 recall for a mention is the percentage of mentions in the gold chain for i that are also in the system chain for i:

Block 7
paragraph
Source Text
The widely used older MUC metric (Vilain et al., 1995) evaluates the system based MUC on the number of links that need to be added or deleted to map the system-generated chains to the gold standard chains. A system receives a precision/recall penalty for each incorrectly posited or missed link. However, this definition of a link is ambiguous. Suppose a text has the entities A, B, C, and D, and all 4 are coreferent. Does the gold chain have 3 links (A-B, B-C, C-D) or 6 links (A-B, A-C, A-D, etc.)? The MUC algorithm assumes the minimum number of links are used: so a class with N mentions will have N - 1 links.
Korean Translation
널리 사용된 구형 MUC 지표(Vilain et al., 1995)는 시스템이 생성한 사슬을 골드 표준 사슬로 매핑하기 위해 추가하거나 삭제해야 하는 링크의 수를 기반으로 시스템을 평가합니다. 시스템은 잘못 상정되거나 누락된 각 링크에 대해 정밀도/재현율 페널티를 받습니다. 그러나 링크에 대한 이러한 정의는 모호합니다. 텍스트에 A, B, C, D라는 개체가 있고 4개 모두 상호참조된다고 가정해 봅시다. 골드 사슬은 3개의 링크(A-B, B-C, C-D)를 가지나요, 아니면 6개의 링크(A-B, A-C, A-D 등)를 가지나요? MUC 알고리즘은 최소 링크 수가 사용된다고 가정합니다. 따라서 N개의 언급을 가진 클래스는 N - 1개의 링크를 가집니다.
Study Note
MUC 평가 지표는 사슬간 변환에 필요한 최소 링크(N-1)의 추가/삭제 수를 기준으로 패널티를 부여하는 방식입니다.
