## Block 2
- type: paragraph
- source_text: Before evaluating a coreference system, we need to extract a test set containing the gold standard coreference links. The coreference links in a document are annotated by marking all the mentions in the document, and then linking each mention to its antecedent. This process clusters the mentions in a document into a set of equivalence classes (the coreference chains) as defined in Section 21.1. We thus evaluate a coreference system by comparing the coreference chains it produced with the gold coreference chains.

## Block 3
- type: paragraph
- source_text: As with most systems, we evaluate them by computing precision, recall, and F1. We can compute these over the links between mentions, or over the resulting equivalence classes. Since it turns out there are multiple ways to define precision and recall, the standard way to evaluate coreference is to use 3 metrics (MUC, B3, and CEAF) and to report the average of the 3. We'll summarize B3 and MUC here and give the details of the third, as well as the script used to compute them, at the end of the chapter.

## Block 7
- type: paragraph
- source_text: The widely used older MUC metric (Vilain et al., 1995) evaluates the system based MUC on the number of links that need to be added or deleted to map the system-generated chains to the gold standard chains. A system receives a precision/recall penalty for each incorrectly posited or missed link. However, this definition of a link is ambiguous. Suppose a text has the entities A, B, C, and D, and all 4 are coreferent. Does the gold chain have 3 links (A-B, B-C, C-D) or 6 links (A-B, A-C, A-D, etc.)? The MUC algorithm assumes the minimum number of links are used: so a class with N mentions will have N - 1 links.
- korean_translation: 널리 사용된 구형 MUC 지표(Vilain et al., 1995)는 시스템이 생성한 사슬을 골드 표준 사슬로 매핑하기 위해 추가하거나 삭제해야 하는 링크의 수를 기반으로 시스템을 평가합니다. 시스템은 잘못 상정되거나 누락된 각 링크에 대해 정밀도/재현율 페널티를 받습니다. 그러나 링크에 대한 이러한 정의는 모호합니다. 텍스트에 A, B, C, D라는 개체가 있고 4개 모두 상호참조된다고 가정해 봅시다. 골드 사슬은 3개의 링크(A-B, B-C, C-D)를 가지나요, 아니면 6개의 링크(A-B, A-C, A-D 등)를 가지나요? MUC 알고리즘은 최소 링크 수가 사용된다고 가정합니다. 따라서 N개의 언급을 가진 클래스는 N - 1개의 링크를 가집니다.
- study_note: MUC 평가 지표는 사슬간 변환에 필요한 최소 링크(N-1)의 추가/삭제 수를 기준으로 패널티를 부여하는 방식입니다.
