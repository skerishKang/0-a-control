## Block 2
- type: paragraph
- source_text: The B-cubed metric calculates precision and recall for each mention and then averages across all mentions in the text.
- korean_translation: B-cubed 지표는 각 언급에 대해 정밀도와 재현율을 계산한 다음, 텍스트 내의 모든 언급들에 대해 평균을 냅니다.
- study_note: 언급별 정밀도(Precision)와 재현율(Recall)을 산출하여 평균을 내는 B-cubed의 기초 계산 원리입니다.

## Block 3
- type: margin_note
- source_text: mention-based evaluation
- korean_translation: 언급 기반 평가(mention-based evaluation)
- study_note: 클러스터 내 각 언급의 정답 포함 여부를 판단하여 평가의 정밀도를 높이는 방식입니다.

## Block 4
- type: paragraph
- source_text: B-cubed addresses some of the limitations of MUC by penalizing over-clustering and under-clustering, making it a more robust evaluation metric.
- korean_translation: B-cubed는 과도한 클러스터링(over-clustering)과 미흡한 클러스터링(under-clustering)을 모두 처벌함으로써 MUC의 일부 한계점을 해결하며, 더 견고한 평가 지표가 됩니다.
- study_note: B-cubed가 MUC보다 성능 평가에 있어 더 공정하게 작동하는 이유를 설명합니다.

- korean_translation: 21.6.3 CEAF (제약된 개체 정렬 F-measure)
- study_note: 예측된 클러스터와 골드 클러스터 사이의 1대1 매핑을 통해 유사도를 측정하는 CEAF 지표를 다룹니다.

## Block 3
- type: paragraph
- source_text: The CEAF metric uses an alignment approach between the gold clusters and the predicted clusters. It finds the best one-to-one mapping between them based on their similarity.
- korean_translation: CEAF 지표는 골드 클러스터와 예측된 클러스터 사이의 정렬(alignment) 방식을 사용합니다. 유사도에 기반하여 두 분류 사이의 최적의 일대일 매핑을 찾습니다.
- study_note: 클러스터 간의 일대일 대응(Matching)을 통해 평가의 엄밀성을 확보하는 방식을 설명합니다.

## Block 4
- type: margin_note
- source_text: cluster-based evaluation
- korean_translation: 클러스터 기반 평가(cluster-based evaluation)
- study_note: 개별 링크나 언급이 아닌, 클러스터 덩어리 자체의 일치 정도를 평가하는 방식입니다.

## Block 5
- type: paragraph
- source_text: CEAF provides a more global view of the coreference clusters but can be more computationally expensive to evaluate than MUC or B-cubed.
- korean_translation: CEAF는 상호 참조 클러스터에 대해 더 전역적인 관점을 제공하지만, MUC나 B-cubed보다 평가를 위한 계산 비용이 더 많이 들 수 있습니다.
- study_note: CEAF 지표의 전역적 정확성과 계산 복잡도라는 트레이드오프를 설명합니다.
