## Block 3
- type: paragraph
- source_text: A perfect model of the probability that a word will be mistyped would condition on all sorts of factors: who the typist was, whether the typist was left-handed or right-handed, and so on. Luckily, we can get a pretty reasonable estimate of P(x|w) just by looking at local context: the identity of the correct letter itself, the misspelling, and the surrounding letters. For example, the letters m and n are often substituted for each other; this is partly a fact about their identity and partly a fact about context.
- korean_translation: 어떤 단어가 오타로 입력될 확률의 완벽한 모델은 입력한 사람이 누구인지, 왼손잡이인지 오른손잡이인지 등 온갖 요인에 조건화되어야 할 것입니다. 하지만 다행히도 P(x|w)는 지역적 문맥만 보아도 꽤 괜찮게 추정할 수 있습니다. 즉, 올바른 글자 자체, 실제 오타, 그리고 주변 글자들입니다. 예를 들어 m과 n은 자주 서로 대체되는데, 이것은 두 글자의 성질 때문이기도 하고 비슷한 문맥에 나타나기 때문이기도 합니다.
- study_note: 현실의 오류 모델은 복잡하지만, 실제 시스템에서는 글자와 주변 문맥만으로도 충분히 유용한 추정이 가능합니다.

## Block 4
- type: paragraph
- source_text: A simple model might estimate p(acress|across) using the number of times that the letter e was substituted for the letter o in some large corpus of errors. To compute the probability for each edit in this way we need a confusion matrix that contains counts of errors. In general, a confusion matrix lists the number of times one thing was confused with another.
- korean_translation: 단순한 모델은 큰 오류 말뭉치에서 e가 o로 잘못 입력된 횟수를 사용해 p(acress|across)를 추정할 수 있습니다. 이렇게 각 편집의 확률을 계산하려면 오류 빈도가 담긴 혼동 행렬(confusion matrix)이 필요합니다. 일반적으로 혼동 행렬은 어떤 항목이 다른 항목과 몇 번 혼동되었는지를 기록합니다.
- study_note: 채널 모델의 핵심 자료구조는 혼동 행렬이며, 실제 편집 빈도를 확률로 바꾸는 데 사용됩니다.

## Block 5
- type: code
- source_text: |
    del[x, y]: count(xy typed as x)
    ins[x, y]: count(x typed as xy)
    sub[x, y]: count(x typed as y)
    trans[x, y]: count(xy typed as yx)
- korean_translation: |
    del[x, y]: xy가 x로 입력된 횟수
    ins[x, y]: x가 xy로 입력된 횟수
    sub[x, y]: x가 y로 입력된 횟수
    trans[x, y]: xy가 yx로 입력된 횟수
- study_note: 삭제, 삽입, 치환, 전치에 대한 네 종류의 혼동 행렬 정의입니다.

## Block 6
- type: paragraph
- source_text: One way to get these confusion matrices is to extract them from lists of misspellings such as additional/addional, additonal; environments/enviornments, enviorments, enviroments; preceded/preceeded. There are lists available on Wikipedia and from Roger Mitton and Peter Norvig. Norvig also gives counts for each single-character edit that can be used directly to create the error model probabilities.
- korean_translation: 이러한 혼동 행렬을 얻는 한 가지 방법은 additional/addional, additonal, environments/enviornments, enviorments, enviroments, preceded/preceeded 같은 오타 목록에서 추출하는 것입니다. 위키피디아와 Roger Mitton, Peter Norvig가 제공한 목록들을 활용할 수 있습니다. Norvig는 각 단일 문자 편집의 빈도도 제공하며, 이것을 곧바로 오류 모델 확률을 만드는 데 사용할 수 있습니다.
- study_note: 공개된 오타 목록과 편집 빈도 자료가 채널 모델 학습의 실질적 자원이 됩니다.

## Block 7
- type: paragraph
- source_text: An alternative approach used by Kernighan et al. (1990) is to compute the matrices by iteratively using the spelling correction algorithm itself. The algorithm first initializes the matrices with equal values. Next, the spelling correction algorithm is run on a set of spelling errors. Given the typos paired with their predicted corrections, the confusion matrices can be recomputed, the spelling algorithm run again, and so on. This iterative algorithm is an instance of the EM algorithm.
- korean_translation: Kernighan 등(1990)이 사용한 다른 접근은 철자 교정 알고리즘 자체를 반복적으로 사용하여 행렬을 계산하는 것입니다. 먼저 행렬을 동일한 값으로 초기화합니다. 다음으로 철자 교정 알고리즘을 오류 집합에 실행합니다. 오타와 예측된 교정 결과가 쌍으로 모이면 혼동 행렬을 다시 계산하고, 철자 교정 알고리즘을 다시 실행하는 식으로 반복합니다. 이 반복 알고리즘은 EM 알고리즘의 한 사례입니다.
- study_note: 오류 모델을 고정된 데이터에서 한 번에 구하는 대신, 교정 결과를 이용해 반복적으로 개선할 수 있습니다.

## Block 1
- type: formula
- source_text: |
    P(x|w) =
      del[x_{i-1}, w_i] / count[x_{i-1} w_i]        if deletion
      ins[w_{i-1}, w_i] / count[w_{i-1}]            if insertion
      sub[x_i, w_i] / count[w_i]                    if substitution
      trans[w_i w_{i+1}] / count[w_i w_{i+1}]       if transposition
    (B.6)
- korean_translation: |
    P(x|w) =
      del[x_{i-1}, w_i] / count[x_{i-1} w_i]        삭제의 경우
      ins[w_{i-1}, w_i] / count[w_{i-1}]            삽입의 경우
      sub[x_i, w_i] / count[w_i]                    치환의 경우
      trans[w_i w_{i+1}] / count[w_i w_{i+1}]       전치의 경우
    (B.6)
- study_note: confusion matrix의 카운트를 이용해 각 편집 연산의 확률을 추정하는 식입니다.
