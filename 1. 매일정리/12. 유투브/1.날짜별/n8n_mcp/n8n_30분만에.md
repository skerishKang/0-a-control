## 유튜브 영상 요약: n8n 노코드 자동화 툴 소개 및 워크플로우 구축 가이드

이 영상은 n8n이라는 강력한 노코드/로우코드 워크플로우 자동화 툴을 소개하고, 간단한 예시를 통해 기본적인 사용법과 워크플로우 구축 과정을 설명합니다.

**1. n8n 이란? (0:00 - 0:44)**
*   n8n은 반복적인 업무를 자동화하여 시간과 노력을 절감할 수 있는 워크플로우 자동화 플랫폼입니다.
*   Zapier나 Make(Integromat)와 같은 기존 자동화 툴과 유사하지만, n8n은 "셀프 호스팅(Self-hosted)" 옵션을 제공하여 비용 부담을 줄이고 더 많은 유연성을 가질 수 있다는 장점이 있습니다.
*   공식 홈페이지(n8n.io)에서 Secure, AI-native workflow automation 이라는 문구를 통해 그 특징을 확인할 수 있습니다.

**2. n8n의 활용 예시 - 신규 입사자 온보딩 (0:45 - 1:41)**
*   회사에 신규 입사자가 들어왔을 때, 인사팀에서 Google Workspace, Jira, Slack 등 다양한 시스템의 계정을 만들고 권한을 부여하는 작업은 반복적이고 시간이 많이 소요될 수 있습니다.
*   n8n을 활용하면 이러한 과정을 자동화하여 한 명당 10분씩 걸리던 작업을 훨씬 단축할 수 있습니다.
*   예시 워크플로우: 신규 입사자가 설문지 제출 -> Google Workspace 사용자 생성 -> (매니저 여부에 따라) Jira Admin/Member 권한 부여 -> Slack 사용자 생성/프로필 업데이트 등의 과정을 자동화할 수 있습니다.
*   이러한 워크플로우는 퇴사자의 계정 회수 과정에도 동일하게 적용하여 업무를 자동화할 수 있습니다.

**3. n8n 사용의 장점 (1:42 - 4:22)**
*   개발 지식이 없는 사람도 노드(Node) 기반의 시각적인 인터페이스를 통해 쉽게 워크플로우를 구축할 수 있습니다.
*   영상 제작자는 약 한 달간 사용해 본 결과, 3~4시간 정도만 공부해도 툴의 전반적인 기능을 이해하고 활용할 수 있었다고 합니다.
*   어떻게 자동화할 것인지보다, *무엇을* 자동화할 것인지에 집중할 수 있게 됩니다.

**4. n8n 가격 정책 (4:23 - 6:24)**
*   n8n은 클라우드 기반 서비스의 경우 유료입니다.
    *   Starter 플랜: 월 20 유로 (약 3~4만원), 월 2,500 워크플로우 실행, 5개 활성 워크플로우 제한
    *   Pro 플랜: 월 50 유로 (약 7~8만원), 월 10,000 워크플로우 실행, 15개 활성 워크플로우 제한
    *   Enterprise 플랜: 문의 필요, 무제한 실행/활성 워크플로우
*   **셀프 호스팅 (Self-hosted)** 옵션은 무료로 제공됩니다. Docker Compose 등을 이용하여 직접 서버에 설치해야 하지만, 워크플로우 실행 및 활성 개수 제한 없이 자유롭게 사용할 수 있습니다. (영상 제작자는 이 셀프 호스팅 환경으로 시연 진행)

**5. n8n 워크플로우 구축 실습 (6:51 - 33:15)**
*   **UI 설명 (6:51 - 9:58):**
    *   왼쪽 메뉴: 홈, 워크플로우 목록, 인증 정보, 템플릿, 변수, 실행 기록, 도움말 등
    *   메인 화면: 워크플로우를 설계하는 캔버스. [+] 버튼을 눌러 첫 번째 스텝(노드) 추가.
    *   트리거 (Trigger): 워크플로우의 시작점 (수동 실행, 앱 이벤트, 스케줄, 웹훅, 폼 제출 등)
    *   워크플로우 실행 시 각 노드의 상태(성공/실패) 및 처리된 아이템(데이터) 개수 표시.
*   **데이터 가져오기 (HTTP Request 노드) (9:59 - 14:12):**
    *   첫 번째 노드로 수동 트리거 선택.
    *   다음 노드로 HTTP Request 추가.
    *   가져올 데이터의 URL 입력.
    *   인증 정보 (Authentication): Header Auth 방식으로 API 키와 값을 입력 (Airtable에서 발급 필요).
    *   `Send Headers` 활성화 및 필요한 헤더 파라미터 입력 (예: `unique_id`와 그 값).
    *   Test Step 실행하여 30개 아이템이 정상적으로 불러와지는지 확인.
*   **Airtable 연동 및 레코드 생성 (14:13 - 19:19):**
    *   Airtable 계정 및 연동할 베이스/테이블 생성 필요. (영상 제작자는 미리 생성된 'beginner course' 베이스의 'orders' 테이블 사용 - orderID, customerID, orderPrice, employeeName, orderStatus 컬럼)
    *   Airtable Builder Hub에서 Personal Access Token 발급 (data.records:read, data.records:write, schema.bases:read 스코프 필요). 발급 시 토큰 값을 즉시 복사하여 안전하게 보관해야 함.
    *   n8n에 Airtable 노드 추가, 작업(Operation)을 'Create a record'로 설정.
    *   Airtable Credential 선택 (방금 발급받은 토큰 정보).
    *   Base ('beginner course')와 Table ('orders') 선택.
    *   Mapping Column Mode를 'Map Automatically'로 설정하여 들어온 데이터와 Airtable 컬럼 이름을 자동으로 매핑.
    *   Test Step 실행하여 Airtable 테이블에 데이터가 정상적으로 입력되는지 확인. (Airtable 화면과 n8n 화면을 분할하여 시각적으로 확인)
*   **데이터 필터링 (If 노드) (19:13 - 21:10):**
    *   Airtable 노드 뒤에 If 노드 추가.
    *   조건 (Conditions) 설정: `{{ $json.fields.orderStatus }}` (들어온 데이터의 orderStatus 값)이 "processing"과 같은지 확인.
    *   Test Step 실행하여 조건에 맞는 데이터는 True 브랜치, 아닌 데이터는 False 브랜치로 분기되는지 확인.
*   **데이터 변환/선택 (Edit Fields 노드) (21:10 - 23:04):**
    *   If 노드의 False 브랜치 뒤에 Edit Fields 노드 추가.
    *   모드(Mode)를 'Manual Mapping'으로 설정.
    *   Fields to Set에 원하는 필드만 추가하고, Expression을 이용하여 해당 필드의 값 매핑. (예: orderID, employeeName 필드만 선택)
    *   Test Step 실행하여 필터링된 데이터에서 원하는 속성만 추출되었는지 확인. (JSON 데이터를 다루는 Expression 문법 설명)
*   **데이터 집계 (Code 노드) (23:05 - 27:01):**
    *   If 노드의 False 브랜치 뒤에 Code 노드 추가.
    *   Language는 JavaScript 또는 Python 선택 가능.
    *   입력받은 데이터(booked 상태의 주문 목록)를 순회하며 총 주문 수 (`totalBooked`)와 총 가격 합계 (`bookedSum`)를 계산하는 JavaScript 코드 작성.
    *   Test Step 실행하여 계산 결과가 정상적으로 출력되는지 확인.
*   **알림 발송 (Discord 노드) (27:02 - 29:02):**
    *   Code 노드 뒤에 Discord 노드 추가.
    *   작업(Operation)을 'Send a Message'로 설정.
    *   Discord Webhook Credential 생성/선택. (Discord에서 웹훅 URL 발급 필요)
    *   메시지 내용 작성 시 Expression을 이용하여 앞선 Code 노드에서 계산된 `totalBooked` 및 `bookedSum` 값과 HTTP Request에서 가져온 `unique_id` 값을 삽입.
    *   Test Step 실행하여 Discord 채널에 메시지가 정상적으로 발송되는지 확인.
*   **노드 이름 변경 (29:02 - 30:38):** 각 노드의 역할을 명확히 알 수 있도록 한글 이름으로 변경 (데이터 가져오기, Processing 여부 분기, 필요한 속성만 가져오기, 집계하기, 알림 발송).
*   **스케줄 트리거 설정 및 활성화 (30:39 - 32:50):**
    *   처음의 수동 트리거를 삭제하고 Schedule Trigger 노드 추가.
    *   Trigger Rules 설정: Interval을 Days로, Days Between Triggers를 1로, Trigger at Hour를 9am (오전 9시)로 설정.
    *   워크플로우 설정(Settings)에서 Timezone을 Asia/Seoul로 변경.
    *   워크플로우를 Active 상태로 변경하여 설정된 스케줄대로 자동 실행되도록 함.

**6. 워크플로우 공유 및 확인 (32:51 - 34:34)**
*   구축한 워크플로우는 JSON 파일 형태로 다운로드하거나 URL로 공유하여 다른 사람이 임포트하여 사용할 수 있습니다.
*   영상 제작자는 다른 예시 워크플로우 (API 테스트, 일기 예보 알림 등)를 보여주며 n8n의 다양한 활용 가능성을 시사합니다.

**결론 (34:35 - 34:51)**
n8n은 데이터 가져오기, 필터링, 가공, 집계, 알림 발송 등 다양한 자동화 작업을 코딩 없이(또는 적은 코딩으로) 시각적인 워크플로우 형태로 구축할 수 있는 강력한 툴입니다. 셀프 호스팅 옵션을 통해 무료로도 활용 가능하므로, 직접 체험해 보면서 자신만의 자동화 워크플로우를 만들어 볼 것을 추천합니다.
