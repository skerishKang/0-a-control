## YouTube 영상 요약: n8n 자동화와 MCP (모델 컨텍스트 프로토콜)

**세미나 개요 및 소개 (이태경 님 진행)**

*   이번 세미나는 커뮤니티 스페이스 패스트캠퍼스, n8n Korea, 더 배러 커뮤니티가 공동 주최했습니다.
*   최근 뜨거운 관심사인 AI 에이전트 기능 확장에 필수적인 **모델 컨텍스트 프로토콜(MCP)** 과 워크플로우 자동화 도구인 **n8n**을 함께 사용하는 방법에 대해 다룹니다.
*   RAG(Retrieval Augmented Generation) 개념과 코드 없이 RAG를 구축하는 방법, 최신 AI 트렌드에 대해서도 소개할 예정입니다.
*   강연자가 외롭지 않도록 비디오를 켜고 적극적으로 참여해주시면 감사하겠습니다!

**MCP(모델 컨텍스트 프로토콜)란? (박정기 님 발표)**

*   앤트로픽(Anthropic)에서 제안한 프로토콜로, AI 모델(LLM)이 다양한 외부 툴(Tool) 및 데이터 소스와 표준화된 방식으로 상호 작용할 수 있게 해주는 핵심적인 역할을 합니다.
*   어떤 개발자가 MCP 서버를 잘 만들어 놓으면, 다른 누구나 그 기능을 쉽게 가져다 AI 에이전트에 연결하여 사용할 수 있습니다.
*   MCP 생태계는 `mcp.so`, `pulsemcp.com` 등 다양한 사이트에서 확인 가능하며, 김민규 님께서 모아주신 MCP 관련 링크 모음도 공유되었습니다.
*   MCP에 대한 자세한 설명은 테디노트, 코드를까는노인 등 여러 유튜버들의 영상을 참고하시는 것을 추천합니다. (특히 코드를까는노인님 영상은 빠르고 쉽게 설명, 테디노트님 영상은 생산자 관점 설명이 좋음)

**클로드 데스크톱에서 MCP 사용하기**

*   클로드 데스크톱의 `claude_desktop_config.json` 파일에 MCP 서버 정보를 추가하면 클로드에서 해당 MCP 툴을 사용할 수 있습니다.
*   예시로 Airbnb MCP 서버를 클로드에 추가하여 도쿄의 숙소를 추천받는 시연을 보여주었습니다. (4월 3일 ~ 4월 7일, 4박 5일 기준)
*   Claude는 `airbnb_search` 툴과 `airbnb_listing_details` 툴을 사용하여 정보를 검색하고 상세 정보를 가져와 보고서를 작성했습니다.
*   이 과정에서 robots.txt 제한으로 인한 에러가 발생했으나, Claude가 `ignoreRobotsText: true` 옵션을 사용하여 스스로 해결하는 모습도 보여주었습니다.
*   Claude Desktop은 MCP 툴을 사용해 데이터를 얻거나 보고서를 만드는 데는 유용하지만, 이를 기반으로 웹 서비스 등 애플리케이션을 구축하고 배포하는 데는 한계가 있습니다.

**n8n에서 MCP 사용하기**

*   n8n은 셀프 호스팅(Self-hosting) 방식을 사용하면 MCP를 활용하여 코드 없이 강력한 AI 서비스를 만들 수 있습니다. (n8n 클라우드 버전은 커뮤니티 노드 설치 불가)
*   `n8n-nodes-mcp-client` 커뮤니티 노드를 사용하면 n8n 워크플로우에서 MCP 서버와 상호작용할 수 있습니다.
*   n8n 설정의 `Community nodes`에서 npm 패키지 이름을 입력하여 커뮤니티 노드를 설치할 수 있습니다.
*   n8n 워크플로우는 **웹훅(Webhook)** 으로 외부 서비스에서 요청을 받고, **Edit Fields** 노드로 데이터를 가공한 후, **AI Agent** 노드를 통해 작업을 처리합니다.
*   **AI Agent** 노드는 내부적으로 LLM(GPT-4 등)과 메모리, 그리고 **MCP Client** 노드를 '툴(Tool)'로 사용하여 동작합니다.
*   Agent는 MCP Client 노드의 `List Tools` 기능을 통해 어떤 MCP 서버가 어떤 툴을 제공하는지 먼저 알아낸 후, 사용자의 요청에 적합한 툴(예: `airbnb_search`)을 선택하고, 필요한 파라미터를 자동으로 채워 `Execute Tool` 기능을 실행합니다.
*   실행 결과는 Agent로 돌아오고, Agent는 이 정보를 바탕으로 최종 응답을 생성합니다.
*   이렇게 구성한 n8n 워크플로우를 웹훅 엔드포인트로 노출하면, Lovable과 같은 프론트엔드에서 사용자 입력을 받아 n8n으로 보내고, n8n이 MCP를 통해 데이터를 처리하여 다시 프론트엔드로 보내주는 형태로 AI 서비스를 구축할 수 있습니다.
*   n8n을 사용하면 코드 작성 없이도 다양한 외부 툴(MCP 서버)을 LLM과 연결하여 복잡한 AI 에이전트 워크플로우를 만들고 서비스화할 수 있습니다.

**핵심 요약**

*   **MCP**는 LLM이 외부 툴을 사용할 수 있게 하는 **프로토콜**입니다.
*   **Claude Desktop**은 MCP 툴을 사용하기는 쉽지만, 서비스 개발 및 배포에는 한계가 있습니다.
*   **n8n**은 셀프 호스팅 환경에서 **MCP 커스텀 노드**를 통해 MCP 서버와 연결하여, **코드 없이** AI 에이전트 백엔드 워크플로우를 구축하고 **웹 서비스로 배포**할 수 있습니다.
*   n8n의 **AI Agent** 노드는 스스로 사용 가능한 툴을 파악하고, 사용자 요청에 맞춰 최적의 툴을 실행하는 등 지능적인 역할을 수행합니다.
*   n8n + MCP 조합은 문과생(비개발자)도 AI 자동화/서비스를 만들 수 있는 강력한 가능성을 제공합니다.
