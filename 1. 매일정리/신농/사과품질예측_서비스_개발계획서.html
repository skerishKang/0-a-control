사과 품질 예측 AI 서비스 개발 계획서

RTX 5080 기반 로컬 추론 및 보안 터널링 시스템
작성일: 2025. 12. 15
버전: 2.1 (상세 설계 포함)

1. 프로젝트 개요

본 프로젝트는 사과의 RGB(가시광선) 및 NIR(근적외선) 이미지를 활용하여 비파괴적으로 당도, 산도, 경도를 예측하는 AI 시스템을 구축합니다. RTX 5080 GPU를 탑재한 로컬 서버를 기반으로, Cloudflare Tunnel을 통해 외부(웹/모바일)에서 안전하게 접속 가능한 서비스를 구현합니다.

2. 상세 시스템 아키텍처

2.1 기술 스택 (Tech Stack)

  • Frontend: Next.js (React) + NextAuth (Google Login) + Netlify 배포
  • Backend: FastAPI (Python) + Docker Container
  • Infrastructure: Local Workstation (RTX 5080) + Local SSD/HDD
  • Network: Cloudflare Tunnel (Inbound 포트 개방 없는 안전한 외부 연결)
  • Database: SQLite (초기) → PostgreSQL (확장 고려)

2.2 데이터 파이프라인 흐름

1
촬영/업로드: 사용자가 웹/모바일에서 Google 로그인 후 사과 사진(RGB/NIR) 업로드
2
보안 터널링: Cloudflare Edge가 'api.domain.com' 요청을 암호화하여 로컬 서버로 전달
3
전처리 & 추론: FastAPI가 이미지 검증(100MB 제한) → 전처리 → RTX 5080 추론
4
저장 & 반환: 원본 이미지(Local Disk) 및 예측 결과(DB) 저장 후 클라이언트에 응답

2.3 보안 및 운영 정책

  • 업로드 제한: Cloudflare 프록시 안정성을 위해 파일당 100MB로 soft-limit 설정 (추후 분할 업로드 고려).
  • 권한 관리 (RBAC): Google 계정 기반 3단계 권한 시스템 적용 (OPERATOR / REVIEWER / USER).

3. 데이터 구축 및 DB 설계

3.1 데이터 입력 요구사항

학습 및 추론의 정확도를 위해 아래 기준을 준수합니다.

  • RGB 이미지: 스마트폰 또는 일반 카메라 (JPG/PNG), 사과 전체가 나오도록 촬영.
  • NIR 이미지: 전용 장비 측정 불가시 RGB 전용 모델(Single-mode)로 대응, 장비 가용 시 Dual-mode 지원.
  • 데이터 Split: 동일 사과의 여러 각도 사진은 반드시 학습/검증/테스트 중 하나의 세트로만 분류 (Data Leakage 방지).

3.2 데이터베이스 스키마 (설계안)

효율적인 메타데이터 관리와 감사 로그(Audit)를 위한 테이블 구조입니다.

(1) 요청 및 결과 테이블 (requests)

컬럼명 타입 설명
request_id UUID 요청 고유 식별자 (PK)
files JSON 원본 이미지 저장 경로 (RGB, NIR)
brix FLOAT 예측 당도 (Brix)
acidity FLOAT 예측 산도 (%)
firmness FLOAT 예측 경도 (kg/cm²)
strength FLOAT 강도 (선택적 지표, Nullable)
status VARCHAR 검수 상태 (PENDING, APPROVED, RETAKE)

(2) 사용자 및 권한 테이블 (users)

id (PK), google_sub (Unique), email, role (ENUM: OPERATOR, REVIEWER, USER), created_at

(3) 수정 이력 테이블 (label_audit)

audit_id, request_id, field_name, old_value, new_value, changed_by, changed_at

* 검수자나 관리자가 라벨(정답)을 수정할 경우 반드시 이력을 남겨 데이터 신뢰성을 보장합니다.

4. 관리자 기능 및 WBS

4.1 관리자 웹 (Mobile Responsive)

별도의 앱 설치 없이 모바일 브라우저에서도 최적화된 관리자 기능을 제공합니다.

  • 대시보드: 금일 요청 건수, 모델 평균 응답 속도, 에러 로그 모니터링
  • 데이터 검수: 업로드된 사진 확인 및 라벨(실측값) 입력/수정 인터페이스
  • 사용자 관리: 가입 승인 및 권한 등급(User ↔ Reviewer) 변경
  • 다운로드: 원본 데이터셋 및 CSV 형태의 결과 리포트 추출

4.2 주차별 상세 일정 (WBS)

주차 주요 목표 상세 활동 및 산출물
1주차 기획 및 설계 - 데이터 폴더 구조 분석 및 Manifest 파일 생성
- DB 스키마 확정 (SQLite) 및 검증 리포트 작성
- 100MB 업로드 정책 수립 및 API 명세서 작성
2주차 기능 구현
(Back/Front)		 - FastAPI 구현 (인증, 업로드, DB연동)
- NextAuth 구글 로그인 연동
- 전처리 모듈 공용화 및 베이스라인 모델 학습
3주차 모델 최적화 &
관리자 개발		 - 모델 추론 최적화: RTX 5080 가속 및 워밍업 적용
- 듀얼 모델(RGB/NIR) 튜닝 및 성능 평가 (RMSE 등)
- 관리자 페이지 UI (반응형) 구현
4주차 배포 및 시연 - Cloudflare Tunnel 도메인 연결 및 보안 설정
- 통합 테스트 (모바일 촬영 → 서버 → 결과)
- 운영 매뉴얼 및 시연 시나리오 확정
