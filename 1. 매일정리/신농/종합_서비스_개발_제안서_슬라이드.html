Project Proposal
1. 프로젝트 요약 (Executive Summary)
02
프로젝트 목표
고성능 로컬 GPU(RTX 5080) 자원을 활용하여, 웹/모바일에서 실시간으로 사과 품질(당도/산도/경도)을 예측하고 데이터를 자산화하는 End-to-End AI 서비스 MVP를 1개월 내 구축

핵심 추진 전략

  • 고성능 로컬 추론: 클라우드 GPU 임대 대신 보유 중인 RTX 5080을 활용하여 비용 절감 및 추론 성능 극대화
  • 안전한 외부 접속: 포트포워딩의 보안 취약점을 해결하기 위해 Cloudflare Tunnel 도입 (Zero Trust 기반)
  • 데이터 자산화: 원본 이미지(RGB/NIR) 영구 보관 및 사과 ID 기반의 엄격한 데이터셋 관리체계 수립
  • 사용자 중심 서비스: Google 로그인 기반의 간편 인증 및 역할 기반 접근 제어(RBAC) 적용

3. 네트워크 구성 분석 (Network Security)
04

로컬 서버를 외부에 노출하는 3가지 방안을 분석하여 최적의 솔루션을 선정했습니다.

구분 방안 1: 포트포워딩 방안 2: 클라우드 이전 방안 3: Cloudflare Tunnel (채택)
보안성 취약 (포트 전면 개방) 우수 (보안 도구 완비) 우수 (인바운드 포트 차단)
비용/자원 저비용 (로컬 활용) 고비용 (GPU 임대료) 저비용 (로컬 RTX 5080 활용)
안정성 낮음 (ISP NAT/IP변동) 높음 (SLA 보장) 높음 (고정 도메인 제공)
결론 운영 부적합 예산 초과 최적 솔루션
Why Cloudflare Tunnel?
공유기 설정을 건드리지 않고, 로컬 서버가 외부로 나가는(Outbound) 터널을 뚫어 연결하므로 '방화벽을 열지 않고도' 안전하게 고정 도메인 서비스를 제공할 수 있습니다.
4. MVP 개발 범위 (Scope of Work)
05

(1) 사용자 기능

  • 인증: Google 계정 연동 (NextAuth)
  • 입력: 이미지 업로드 및 카메라 촬영 지원
  • 결과: 당도/산도/경도 예측값 및 등급 표시
  • 이력: 내 요청 기록 조회 (마이페이지)

(2) 관리자 기능

  • 모니터링: 전체 요청/사용자 현황 대시보드
  • RBAC: 운영자/검수자/사용자 3단계 권한 관리
  • 데이터 검수: 원본 보기 및 라벨(실측값) 수정
  • 다운로드: 학습용 데이터셋 추출
제외 범위 (Out-of-Scope)
  • NIR(근적외선) 모델 실시간 추론: 1차 MVP는 RGB-Only 모델 서빙에 집중
  • 고정된 등급 정책: 등급 컷오프는 설정값으로 관리하여 유동적으로 운영
5. 데이터 전략 및 모델링
06

데이터 구축 계획

  • 목표량: 기존 1,600장 + 추가 촬영 8,400장 = 총 10,000장 확보
  • 촬영 전략: 다양한 조명/배경/각도 환경 반영 (일일 600장 목표)
  • Split 원칙: '사과 ID 단위' 분할 (동일 사과가 Train/Test에 섞이지 않도록 방지)

모델링 전략 (RTX 5080 활용)

  • Model Backcbone: EfficientNet 또는 ResNet 계열 (속도/성능 균형)
  • Input: 224x224 ~ 512x512 RGB 이미지
  • Output: Multi-output Regression (Brix, Acidity, Firmness)
  • Performance: 로컬 RTX 5080의 고성능을 활용하여 고해상도 배치 추론 최적화

07
주차 핵심 과업 산출물 및 목표
1주차 분석 및 설계 - 데이터/DB 스키마 정의 (SQLite)
- 사과 ID 단위 Split 규칙 확정
- API 명세서 작성

- 파일 업로드 제한(100MB) 정책 적용
3주차 모델 연동 및 고도화 - RTX 5080 최적화 모델 서빙
- 관리자 페이지(RBAC, 검수) 구현
- 자동화된 평가 리포트 생성
4주차 통합 및 배포 - Cloudflare Tunnel 도메인 연결
- End-to-End 시연 시나리오 리허설
- 최종 운영 매뉴얼 작성
