# 대화 폴더 안내

이 폴더는 `0-a-control` 작업 중 실제로 오간 대화 원문과, 그 대화에서 추린 짧은 참조 요약을 두는 곳이다.

## 왜 `docs/`와 따로 두는가
- `docs/`는 설계, 운영 규칙, 스킬, 구현 문서처럼 비교적 정제된 문서를 둔다.
- `대화/`는 세션 중간의 판단 흐름, 맥락 변화, 프롬프트 초안, 임시 결론 같은 "대화 기반 증거"를 둔다.
- 즉 `docs/`가 정리된 문서라면, `대화/`는 나중에 맥락 복원에 쓰는 원문 아카이브에 가깝다.

## 이 폴더에 두는 것
- 실제 대화 로그 `.txt`
- 이후 세션에서 빠르게 참조하려고 만든 요약 메모

## 이 폴더에 두지 않는 것
- 공식 운영 규칙
- 장기 유지할 설계 문서
- 구현 가이드
- 프로젝트 README 성격 문서

그런 내용은 `docs/` 또는 루트 문서에 둔다.

## 사용 원칙
- 원문 대화는 가능하면 그대로 보존한다.
- 요약 파일은 원문을 대체하지 않는다.
- 이후 세션에서 "왜 이런 결정을 했는지"를 복원할 때 먼저 참고할 수 있다.

## 현재 역할
- session recovery 보조 근거
- board-v2 같은 세부 이슈의 결정 맥락 보존
- handoff 없이도 흐름을 다시 잡기 위한 참고 자료
