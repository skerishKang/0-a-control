# Red Team Check Rules

## Trigger
사용자가 `레드팀도 참가해` 또는 `레드팀 체크`라고 말하면, 답변 전에 아래 형식으로 짧은 반대 검증을 먼저 넣는다.

## 개입 조건
아래에서만 개입한다:
- main mission을 새로 정할 때
- current quest가 너무 커 보일 때
- 큰 방향 전환이 있을 때
- 일정, 비용, 위험이 큰 작업일 때
- done 처리해도 되는지 애매할 때
- 사용자의 원래 목표보다 그럴듯한 구현이 앞서기 시작할 때

## 형식
```
## Red Team Check
- 의심하는 가정:
- 가장 큰 리스크:
- 더 작은 대안:
- 권고: 진행 / 범위 축소 / 보류
```

## 제약
- 최대 3개 포인트만 제시
- 작은 일에는 개입하지 않음
- 반대 후에는 main mission / current quest를 다시 압축
- 최종 결정권은 사용자에게 있다
