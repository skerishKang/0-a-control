# 빠른 입력(Bridge) 검증 시나리오

## 개요
이 문서는 `0-a-control`의 빠른 입력(Bridge) 기능 검증을 위한 샘플 입력과 기대 결과를 정리합니다.

---

## 샘플 입력 5개 이상

### 1. 오늘 일정형 입력
```
오늘 오후 2시에 김철수와 미팅
```

### 2. 날짜 포함 입력
```
3월 25일에 사업계획서 제출 마감
```

### 3. 보류 입력
```
정부 지원사업 아이디어는 다음 주에 다시 검토
```

### 4. 여러 사람 연락/통화 일정
```
내일 오전 10時に박영희님과 통화, 오후 3시에 이민수님 미팅
```

### 5. 메인 미션 자동 선정이 필요한 입력
```
올해 성장지원사업 참여申请해야 하는데 아직 아이디어가 안 떠올라
```

### 6. 복합 입력 (여러 항목)
```
来週、月曜日に회의, 水曜일에 보고서 제출, 金曜일에 미팅 2개
```

### 7. 반복 일정
```
매주 월요일 아침 9시에 주간 회의
```

## 수동 테스트 체크리스트

### 테스트 순서

1. **브라우저에서 UI 열기**
   - `0-a-control` 폴더에서 서버 실행
   - `http://localhost:3000` 접근

2. **빠른 입력 패널 열기**
   - 좌측 하단 "빠른 입력" 또는 "Bridge" 버튼 클릭
   - 입력 필드可见確認

3. **샘플 입력 테스트**

   | 순서 | 입력문 | 작업 |
   |------|--------|------|
   | 1 | 오늘 오후 2시에 김철수와 미팅 | "파싱" 버튼 클릭 |
   | 2 | 3월 25일에 사업계획서 제출 마감 | "파싱" 버튼 클릭 |
   | 3 | 정부 지원사업 아이디어는 다음 주에 다시 검토 | "파싱" 버튼 클릭 |
   | 4 | 내일 오전 10時に박영희님과 통화 | "파싱" 버튼 클릭 |
   | 5 | 今年の成長支援事業に参加申請したい | "파싱" 버튼 클릭 |

4. **파싱 결과 확인**
   - 각 입력별 `suggested_plans` 확인
   - bucket 분류가 올바른지 확인
   - 날짜/시간 파싱이 정확한지 확인

5. **플랜 생성 테스트**
   - "플랜 생성" 버튼 클릭
   - plans에 추가되는지 확인
   - 상태가 정확한지 확인

6. **에지 케이스 테스트**
   - 한국어/일본어/영어 혼합 입력
   - 날짜 형식 다양한 경우 (3/25, 3월 25일, 25일)
   - 시간 형식 다양한 경우 (2시, 14시, 2:00, 14:00)

---

## 검증 기준

### 성공 조건
- [ ] 모든 샘플 입력 대해 파싱이 오류 없이 수행됨
- [ ] bucket 분류가 예상과 일치 (80% 이상)
- [ ] 날짜/시간 파싱이 정확한 입력에 대해 올바름
- [ ] 플랜 생성 후 plans에 정상 추가됨
- [ ] UI에 에러 메시지가 표시되지 않음

### 실패 조건
- [ ] 파싱 중 JavaScript 에러 발생
- [ ] 분류 결과가 완전히 잘못됨
- [ ] 플랜 생성 후 데이터에 반영 안됨
- [ ] 입력 필드가 렌더링 안 됨
