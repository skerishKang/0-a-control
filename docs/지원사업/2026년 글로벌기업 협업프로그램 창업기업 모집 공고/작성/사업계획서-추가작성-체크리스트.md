# 사업계획서 추가 작성 체크리스트

## 목적

이 문서는 `OpenAI-열림-사업계획서-초안-v19.hwpx`를 기준으로, 앞으로 더 채워야 할 내용을 정리한 실행용 메모다.  
핵심은 `아이템 설명`을 끝내는 것이 아니라, 평가자가 `시장성`, `사업화 가능성`, `OpenAI 협업 명분`, `숫자 목표`까지 한 번에 이해하도록 만드는 것이다.

## 현재 판단

현재 초안은 아이템 방향과 OpenAI 협업 논리는 어느 정도 잡혔다.  
하지만 아래 4개는 아직 더 보강해야 한다.

- 숫자
- 고객 근거
- 파일럿 계획
- 사업화 설득력

## 제일 먼저 보강할 것

### 1. 아이템 한 줄 소개 최종 확정

지금 문장도 괜찮지만, 지원서 전반에서 반복해서 쓸 공식 문장을 하나 고정해야 한다.

추천안:

> 기업 업무데이터를 외부 생성형 AI에 안전하게 연결하기 위한  
> 보안 필터 및 오케스트레이션 계층

이 문장은 아래 모든 곳에 일관되게 써야 한다.

- 창업아이템명 설명
- 개요(요약)
- 1-1 문제인식
- 5-1 OpenAI 협업 가능성

### 2. 목표 고객 더 좁히기

지금은 `문서 중심 B2B 조직`, `공공·민간 업무팀` 정도로 써두었는데, 평가자는 더 구체적인 고객을 보고 싶어할 가능성이 크다.

추천 우선순위:

1. 계약서·제안서·보고서 등 문서형 업무가 많은 중소·중견기업
2. 민감정보 포함 상담 기록/문서 검토가 많은 B2B 서비스 조직
3. 생성형 AI 도입이 필요하지만 정보보호 이슈로 막혀 있는 공공·준공공 업무팀

즉 다음 버전에는 아래를 넣는 게 좋다.

- 1차 고객군
- 고객이 실제로 넣는 데이터 유형
- 왜 지금 도입이 필요한지

### 3. 고객 문제를 더 현실적으로 쓰기

문제는 “AI를 쓰고 싶은데 보안이 걱정된다” 수준으로 끝내면 약하다.  
조금 더 업무 장면으로 내려와야 한다.

예시:

- 계약서 초안 검토를 AI에 맡기고 싶지만 고객명, 금액, 조항 정보가 포함돼 업로드가 부담됨
- 제안서/보고서 요약을 자동화하고 싶지만 대외비 문서라 외부 AI 투입이 막힘
- 상담 녹취를 분석하고 싶지만 개인정보가 포함돼 전사 후 활용이 어려움

## 반드시 숫자로 바꿔야 할 항목

### 1. KPI

현재 5-2는 방향은 맞지만 숫자가 없다.  
최소 아래는 숫자로 바꾸는 게 좋다.

- 파일럿 수
- PoC 또는 도입 논의 수
- MVP 완성 기준
- 처리 가능한 입력 범위
- 내부 평가 지표

예시 형태:

- 협약기간 내 파일럿 1~2건 추진
- 고객 시연 가능한 MVP 1식 완성
- 문서형 입력 기준 핵심 시나리오 3종 검증
- 탐지/치환/복원 시나리오 테스트셋 운영
- OpenAI 연계 workflow 데모 1종 완성

### 2. 사업화 목표

지원서는 결국 사업화 가능성을 본다.  
그래서 아래를 숫자 또는 단계 목표로 적어야 한다.

- 협약기간 내 유료 전환 목표
- 도입 논의 기업 수
- 파일럿 이후 전환 가능 고객 수
- 목표 단가 또는 패키지 구조

아직 매출 숫자를 확정 못 하더라도 아래 정도는 정리해야 한다.

- 기본 패키지
- 구축형/컨설팅형
- 고도화 옵션

### 3. 일정

3-3 일정표는 아직 예시 더미가 남아 있을 가능성이 높다.  
여기는 실제 일정으로 바꿔야 한다.

권장 흐름:

1. 고객 시나리오 확정
2. OpenAI 연계 핵심 기능 고도화
3. 평가/로그/운영 체계 정리
4. 파일럿 적용
5. 개선
6. 세일즈 자료/레퍼런스화

## 사업계획서에서 더 설득력 있게 써야 하는 내용

### 1. 왜 OpenAI 협업이 필요한가

이건 단순히 `OpenAI 모델을 쓰기 때문`으로 쓰면 약하다.  
아래 식으로 써야 한다.

- 최신 모델을 고객 업무에 안전하게 연결하는 구조가 핵심이다
- agent/workflow/best practice가 제품 완성도에 직접 영향을 준다
- 평가, 비용·성능 최적화, 운영 가드레일이 상용화에 중요하다
- 따라서 OpenAI 협업은 단순 크레딧 지원이 아니라 제품 고도화에 직접 연결된다

### 2. 왜 지금 이 아이템이 시장에서 필요한가

다음 논리를 더 선명하게 잡아야 한다.

- 생성형 AI 도입은 늘고 있다
- 그러나 기업은 보안 때문에 실제 연결을 못 한다
- 그래서 “모델 성능”보다 “안전한 연결 계층” 수요가 생긴다
- IPU는 바로 그 병목을 푼다

### 3. 경쟁 서비스와 무엇이 다른가

지금은 차별성을 말하고 있지만, 다음 버전에는 비교 구조가 있으면 더 좋다.

비교축 예시:

- 범용 챗봇
- 문서 요약 서비스
- 단순 비식별화 도구
- IPU AI Security Filter

IPU 쪽 차별점:

- 탐지 + 치환 + 복원
- 정책 preset
- 로그/운영 메타
- OpenAI workflow 연결 전 단계 통제
- 문서/텍스트/음성 확장 가능성

## 다음에 실제로 준비해야 할 자료

### 1. 이미지/도식

초안에는 이미지 칸이 있다. 이건 비워두지 말고 최소 1~2개는 넣는 게 좋다.

추천:

- 제품 흐름도
- manual-preview 화면 캡처
- 입력 -> 탐지 -> 치환 -> OpenAI 연계 -> 복원 구조도

### 2. 고객 시나리오 2~3개

문장으로만 말하지 말고 간단 시나리오로 정리해야 한다.

예시:

- 계약서 검토 전 AI 업로드 보안 처리
- 제안서/보고서 요약 전 민감정보 치환
- 상담 음성 전사 후 AI 분석 전 보안 필터 적용

### 3. 파일럿 대상 후보

실제 회사명을 못 써도 아래는 정리해두는 게 좋다.

- 어떤 업종
- 어떤 업무
- 어떤 데이터 유형
- 어떤 성공 기준

## 다음 버전에서 내가 같이 채우면 좋은 항목

### 바로 작성 가능한 것

- 개요(요약) 문안 다듬기
- 1-1 / 1-2 문장 더 세게 만들기
- 5-1 / 5-2 / 5-3 완성도 높이기
- 경쟁 비교표 초안
- 고객 시나리오 2~3개 문안

### 사용자 판단이 필요한 것

- 실제 파일럿 가능 고객/산업
- 목표 단가
- 협약기간 내 매출 목표
- 투자/자금 조달 계획
- 이미지로 넣을 실제 화면/도식 선택

## 추천 다음 작업 순서

1. 아이템 한 줄 소개 최종 확정
2. 목표 고객 1차 타깃 2개로 좁히기
3. 고객 시나리오 2~3개 작성
4. KPI 숫자 넣기
5. 3-3 일정표 실제 값으로 교체
6. 이미지/도식 넣기
7. 매출/사업화 수치 정리

## 한 줄 결론

지금 초안은 “아이템 방향”은 잡혔지만,  
다음 단계에서는 반드시 `고객`, `숫자`, `파일럿`, `사업화`를 더 선명하게 써야 한다.  
그 4개가 들어가야 평가자가 “기술 설명”이 아니라 “실제 될 사업”으로 받아들인다.
