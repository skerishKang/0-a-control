# 16. 설정 및 안전 가드레일 체크리스트

이 문서는 `0-a-control`을 개인 컨트롤 타워로 운영할 때 확인해야 할 설정값과 위험 작업 가드레일을 정리한다. 현재 구현은 로컬 단일 사용자 운영을 기본으로 하며, 설정 화면 또는 자동 점검 UI를 만들기 전까지 이 문서를 기준으로 삼는다.

## 목적

- 현재 운영에 필요한 설정을 한 곳에 모은다.
- 설정 누락과 위험 노출을 빠르게 식별한다.
- UI에서 나중에 표시해야 할 guardrail 항목을 명확히 한다.
- 로컬 전용 작업과 원격 접근 작업의 위험 수준을 구분한다.

## 설정 범주

### 1. 서버 실행 설정

| 항목 | 기본값 | 권장 | 위험 신호 |
|---|---:|---|---|
| `HOST` | `127.0.0.1` | 로컬 운영은 `127.0.0.1` 유지 | `0.0.0.0`인데 접근 제어가 없음 |
| `PORT` | `4310` | 충돌 없는 로컬 포트 | 공용 서비스 포트와 충돌 |
| `DEBUG` | unset | 필요할 때만 임시 사용 | 운영 중 상세 로그 상시 출력 |

운영 기준:

- 로컬 개인 사용은 `HOST=127.0.0.1`을 기본값으로 유지한다.
- `HOST=0.0.0.0`은 LAN 접근이 필요한 경우에만 사용한다.
- 인터넷 원격 접근은 Python 서버 직접 공개가 아니라 앞단 접근 제어 계층을 사용한다.

### 2. 데이터 및 런타임 경로

| 항목 | 위치 | 주의 |
|---|---|---|
| 운영 DB | `data/` 하위 | 백업 대상 |
| 런타임 상태 | `data/runtime/` | 일부 민감 파일 포함 가능 |
| 세션 파일 | `data/runtime/` 또는 env override | 공개 저장소 커밋 금지 |
| 큐 파일 | `data/queue/` | 실패/중복/보관 정책 필요 |

운영 기준:

- 민감 세션 파일과 토큰은 공개 저장소에 커밋하지 않는다.
- 백업 대상과 제외 대상을 분리한다.
- 손상된 JSON이나 실패 파일은 삭제보다 격리/보관을 우선한다.

### 3. Telegram 설정

| 항목 | 필수 여부 | 설명 |
|---|---|---|
| `TELEGRAM_API_ID` | Telegram sync 사용 시 필수 | Telegram API ID |
| `TELEGRAM_API_HASH` | Telegram sync 사용 시 필수 | Telegram API hash |
| `CONTROL_TOWER_TELEGRAM_SESSION_PATH` | 선택 | 세션 파일 경로 override |

운영 기준:

- Telegram 기능은 선택 기능이다.
- 설정이 없을 때는 sync 기능만 제한하고 대시보드 전체를 실패 처리하지 않는다.
- UI는 Telegram 설정 부족을 경고로 표시하되, 핵심 대시보드 사용을 막지 않는다.
- 세션 파일은 민감 파일로 취급한다.

### 4. 에이전트 실행 설정

| 항목 | 확인 내용 |
|---|---|
| `AGENTS.md` | 운영 역할과 원칙이 현재 방식과 맞는지 확인 |
| `launchers/` | Windows/WSL 의존 여부 확인 |
| `scripts/agent_registry.py` | 등록된 에이전트와 실제 실행 파일 일치 확인 |
| `scripts/agent-work.sh` | 공통 세션 기록 경로 확인 |

운영 기준:

- 루트의 `open-*.bat`는 사용자 진입점으로 유지한다.
- 실제 구현은 `launchers/`와 `scripts/`를 기준으로 확인한다.
- 에이전트 실행 경로 변경 시 README와 관련 문서를 함께 갱신한다.

## 위험 작업 가드레일

### 금지 또는 주의 작업

| 작업 | 기본 정책 | 필요 조건 |
|---|---|---|
| 데이터 삭제 | 기본 금지 | 백업/대상/복구 방식 확인 |
| 세션 파일 출력 | 금지 | 없음 |
| 토큰/환경변수 출력 | 금지 | 없음 |
| DB 마이그레이션 | 주의 | 백업 후 실행 |
| `HOST=0.0.0.0` 공개 | 주의 | 방화벽/접근 제어 확인 |
| 인터넷 직접 공개 | 금지 | reverse proxy/VPN/Zero Trust 필요 |
| 대규모 리팩터링 | 주의 | 이슈 범위와 롤백 계획 필요 |

### UI에 표시해야 할 guardrail 상태

향후 Settings/Guardrails 화면에서 아래 상태를 계산해 표시한다.

- `LOCAL_ONLY_SAFE`: `HOST=127.0.0.1`로 로컬 전용 실행 중
- `LAN_EXPOSED`: `HOST=0.0.0.0` 또는 사설망 접근 가능 상태
- `REMOTE_REQUIRES_FRONTDOOR`: 원격 접근을 원하지만 접근 제어 문서/설정이 없는 상태
- `TELEGRAM_NOT_CONFIGURED`: Telegram env 미설정
- `TELEGRAM_SESSION_PRESENT`: Telegram 세션 파일 존재
- `DEBUG_ENABLED`: DEBUG 로그 활성화
- `BACKUP_NOT_CONFIRMED`: 백업 기준 미확인
- `VALIDATION_REQUIRED`: 변경 후 실행/브라우저 검증 필요

## 설정 화면 요구사항

Settings/Guardrails 화면은 다음 정보를 표시해야 한다.

### 기본 설정

- 현재 `HOST`
- 현재 `PORT`
- `DEBUG` 활성 여부
- Python 버전
- 프로젝트 루트 경로
- 데이터 디렉터리 경로

### 통합 설정

- Telegram 설정 상태
- Telegram 세션 파일 경로
- 등록된 에이전트 목록
- 실행 가능 여부

### 안전 상태

- 로컬 전용 여부
- LAN 노출 여부
- 원격 접근 시 앞단 보호 필요 여부
- 민감 파일 존재 여부
- 백업 확인 상태

## 오류 표시 기준

| 유형 | UI 표현 | 예시 |
|---|---|---|
| 정보 | 회색/중립 배너 | Telegram 미사용 상태 |
| 경고 | 노란색 배너 | `HOST=0.0.0.0` |
| 차단 | 빨간색 배너 | 인터넷 직접 공개 감지 또는 민감 파일 출력 시도 |
| 성공 | 녹색 배너 | 로컬 전용 안전 상태 |

## 로컬 필요 작업과 원격 가능 작업

### 원격에서 처리 가능

- 문서 작성/수정
- 이슈/PR 상태 정리
- 안전 기준 추가
- README 링크 정리
- 정적 코드 리뷰

### 로컬 확인 필요

- 서버 실행
- 브라우저 렌더링
- DB 상태 확인
- Telegram 세션 생성/동기화
- 로컬 모델 또는 에이전트 실행
- 환경변수 적용 여부 확인
