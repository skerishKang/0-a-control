# Red Team Example

이 문서는 `red-team-trigger`를 실제 대화에서 어떻게 끼워 넣을지 보여주는 짧은 예시다.

## 목적
- 새 세션 없이도 바로 시험할 수 있는 대화 패턴 제공
- 레드팀이 "항상 말하는 존재"가 아니라 "조건부로 튀어나오는 검증 단계"임을 명확히 함

## 예시 1: 미션 선정

### 사용자
`오늘은 0-a-control에 멀티에이전트 구조를 넣고 codex/anti/kilo까지 다 연결하고 싶어`

### 주 에이전트
`오늘 main mission 후보는 "멀티에이전트 운영 틀 확정"입니다. 구현까지 같이 가면 범위가 커질 수 있습니다.`

### Red Team Check
```md
## Red Team Check
- 의심하는 가정: 운영 설계와 실제 연결 작업을 한 세션에서 모두 끝낼 수 있다는 가정
- 가장 큰 리스크: 설계 없는 구현으로 스크립트만 늘고 운영 원칙이 흐려질 수 있음
- 더 작은 대안: 오늘은 역할표, 호출 규칙, 레드팀 발동 조건만 확정
- 권고: 범위 축소
```

### 주 에이전트 최종 정리
`오늘 current quest는 "역할표와 레드팀 호출 규칙 문서화"로 줄이는 것이 맞습니다.`

## 예시 2: 완료 판정 직전

### 사용자
`문서도 만들었고 대충 방향도 있으니 done 해도 되지 않을까`

### Red Team Check
```md
## Red Team Check
- 의심하는 가정: 문서 초안이 곧 운영 가능한 설계라는 가정
- 가장 큰 리스크: 실제 호출 조건과 책임 구분이 빠진 채 완료 처리될 가능성
- 더 작은 대안: 오늘은 partial로 기록하고 "호출 규칙 10개"까지 적은 뒤 done 검토
- 권고: 범위 축소
```

### 주 에이전트 최종 정리
`이번 퀘스트는 partial이 맞습니다. restart point는 "호출 규칙 명문화"입니다.`

## 새 세션 필요 여부
- 이 문서형 스킬은 새 세션 없이도 바로 참고 가능하다.
- 다만 "항상 자동으로 발동하는 Codex 글로벌 skill"처럼 쓰려면 별도 글로벌 설치와 세션 재시작이 필요할 수 있다.
