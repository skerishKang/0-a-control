# 0-a-control 매일 시작 루틴 (Startup Routine)

## 개요
이 문서는 매일 아침 `0-a-control` 시스템을 켜고, 활용하고, 안전하게 종료하기 위한 최소 실행 루프를 정의합니다.

## 1. 시스템 시작하기
1. Windows 탐색기에서 저장소 루트(예: `0-a-control/`)로 이동합니다.
2. **`auto.bat`** 파일을 더블 클릭하거나 터미널에서 실행합니다.

**실행 시 발생하는 일:**
- **메인 서버 창**: `.bat` 파일을 실행한 원본 콘솔 창은 웹 서버 구동을 담당합니다.
- **대시보드 서버 창**: `localhost:4310` 웹 UI를 제공하는 로컬 서버가 시작됩니다.
- **Telegram 연결**: 텔레그램 동기화는 `0-a-control` 내부 서비스가 직접 처리합니다.

## 2. 인터페이스 접속
메인 서버 창에 시작 안내 문구가 표시되면, 브라우저를 열고 다음 주소로 접속합니다:
**[http://localhost:4310](http://localhost:4310)**

## 3. 아침 루틴 (1분)

### 3-1. current state 확인 (10초)
UI에서 오늘의 주 임무와 현재 퀘스트를 확인합니다.

### 3-2. 관련 archive 선택적 읽기 (30초)
오늘 작업할 주제와 관련된 최근 세션을 확인합니다:

```bash
# 최근 세션 목록 확인
ls -lt sessions/ | head -3

# 또는 sessions_html/index.html에서 빠르게 탐색
```

**읽기 기준:**
- 오늘 주제와 관련된 세션이 있으면 → 해당 세션의 Dialogue를 읽음
- 관련 세션이 없으면 → 건너뜀
- 전체를 읽을 필요 없음

### 3-3. main mission / current quest 확정 (20초)
```text
"오늘 주 임무를 정하자. [주제] 관련 어제 archive를 보니 [상태]. 오늘은 [임무]로 가자."
```

## 4. 작업 중 루틴 (30초)

퀘스트 수행 중 필요할 때만 archive를 확인합니다:

| 상황 | 행동 |
|------|------|
| 맥락이 기억 안 남 | 관련 세션 Dialogue 일부 재확인 |
| 새 세션 시작 | `codex 세션 복원해줘` 또는 관련 archive 읽기 |
| 퀘스트 완료 | 보고/판정 후 다음 quest 확정 |

## 5. 마감 루틴 (2분)

### 5-1. 오늘 세션 archive 확인 (30초)
```bash
# 오늘 생성된 세션 확인
ls sessions/$(date +%Y-%m-%d)/
```

오늘 작업한 세션이 archive에 잘 남았는지 확인합니다.

### 5-2. 내일 첫 시작점 정리 (1분)
```
"오늘 마감하자. 한 일, 남은 일, 내일 첫 퀘스트 정리해줘."
```

### 5-3. session recovery용 압축 문장 (30초)
필요한 경우 내일 아침을 위한 압축 문장을 남깁니다:
```
내일 아침 시작점: [주제]의 [구체적 상태]
```

## 6. 안전한 시스템 종료
하루 일과를 완전히 마쳤을 때:
1. **메인 서버 창**에서 `Ctrl + C`를 누른 뒤 `Y`를 입력하여 서버를 종료합니다.
2. **큐 워커 창**은 창의 `X` 버튼을 눌러 닫거나, `exit`를 타이핑하여 종료합니다.

*참고: 상태 관리는 로컬 SQLite 기반이므로, 트랜잭션 도중 전원이 나가지 않는 이상 강제 종료하더라도 데이터는 안전하게 보존됩니다.*
