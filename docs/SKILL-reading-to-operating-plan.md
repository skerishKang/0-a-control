# SKILL: reading-to-operating-plan

## skill name
`reading-to-operating-plan`

## description
입력으로 받은 책, 사상가, 철학, 개념メモ를 단순 독서 계획이 아닌 0-a-control 운영 체계(통제·위임·판단·협업)를 구축하는 플랜으로 변환한다.

## trigger 조건
다음 입력이 주어졌을 때 이 skill을 사용:
- 책 읽기 요청
- 독서 메모/노트
- 사상가/철학자 논의
- 개념 정리 요청
- "책 4권 읽기" 같은 단순 독서 플랜

## 변환 규칙

### 1. 출력 구조 (고정)
모든 입력은 반드시 아래 4개 섹션으로 변환:
- `long_term`: 장기 운영 원칙
- `short_term`:近期 실행 플랜
- `recurring`: 정기 루틴
- `today`:今日 시작 행동

### 2. 운영 4축 연결 (필수)
모든 플랜은 최소 1개 이상의 축에 연결:
- **통제**: 한비자 - AI 평가와 통제
- **위임**: 드러커 - AI에게 위임하는 영역
- **판단**: 폴라니 - AI와 인간의 판단 분담
- **협업**: 부버 - AI와의 관계적 협력

### 3. 필수 요소 (각 항목)
- **제목**: 구체적 실행 항목
- **왜 지금**: 현재 운영体系中 필요 이유
- **완료 기준** 또는 **산출물**: 결과물 형태 명확히

### 4. 실패 기준
다음 중 하나라도 해당하면 실패:
- "책 4권 읽기" 같은 단순 독서 TODO
- 운영 원칙 산출물 없음
- 4축 연결 없음

## 예시

### 입력
"한비자, 드러커, 폴라니, 부버 읽고 싶다"

### 출력 (올바른 예)
```
### long_term
**AI 통제·위임·판단·협업 운영 체계 구축**
- 왜: 4사상 통합 → 운영 원칙 10조 완성
- 완료 기준: "AI와 함께하는 운영 원칙 10조" 문서

### short_term
1. **한비자 통제 핵심 정리**
   - 왜: AI 평가의 적정성 판단틀 확보
   - 완료 기준: "적합한 AI 평가 영역 vs 인간 판단 영역" 노트

2. **드러커 위임 원칙 추출**
   - 왜: AI 위임 가능 작업 분류
   - 완료 기준: "AI 위임 가능 작업 분류표"

### recurring
**주 1회 독서 → 운영 원칙 통합**
- 산출물: 주간 통합 노트 300자

### today
**4本书 핵심 키워드 추출** (각 1개씩)
- 산출물: 메모 4개
```

## 주의
- 독서 자체가 목적ではない
- 운영 원칙 구축이 목적
- 산출물 형태가 명확해야 함
