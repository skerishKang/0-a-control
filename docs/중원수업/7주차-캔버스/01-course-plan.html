중원로지스 7주차 ChatGPT Gemini Canvas

7주차 수업 계획

ChatGPT Canvas와 Gemini Canvas의 강점 비교 및 상호 보완적 조합(Pipeline)을 통한 실무 문서 완성

수업 목표
  • ChatGPT Canvas(문장 품질)와 Gemini Canvas(구조 설계)의 강점 차이를 명확히 설명할 수 있다.
  • 두 도구에 동일 프롬프트를 입력하여 결과물의 차이를 직접 비교하고 분석할 수 있다.
  • 실무 문서의 성격에 따라 '권장 사용 흐름(Pipeline)'을 설계하고 적용할 수 있다.
  • 프롬프트를 4단계(초안→구조→톤→표)로 확장하며 문서의 완성도를 높이는 과정을 체득한다.
  • 두 도구의 조합을 통해 '내일 당장 사용 가능한' 수준의 실무 문서를 최종 완성한다.

수업 운영 원칙

  1. 설명 최소화: 5분 설명 후 바로 데모 또는 실습. 이론 강의는 10분 넘지 않기.
  2. 같은 프롬프트, 두 도구: 비교 실습에서는 반드시 동일 프롬프트를 두 도구에 넣게 한다.
  3. 한 번에 완성 금지: "초안 → 수정 → 비교 → 개선" 4단계 흐름을 반복 강조.
  4. 내일 바로 쓸 문서: 결과물은 반드시 학생 자신의 업무와 연결된 것이어야 한다.
  5. 돌아다니며 피드백: 실습 시간에는 반드시 순환하며 개별 피드백.
1교시 (50분)

Canvas 개념 이해와 도구 소개

목표

  • 학생이 "Canvas가 뭔지" 한 문장으로 설명할 수 있어야 한다.
  • ChatGPT Canvas를 열어 초안을 하나 만들어본다.
  • Gemini Canvas를 열어 같은 주제로 초안을 하나 만들어본다.

시간 배분

구간시간활동
1-1 도입5분인사 및 오늘 목표 안내 (비교와 조합의 중요성)
1-2 Canvas 개념10분일반 채팅 vs Canvas 비교 시연, 작업 효율성 강조
1-3 병렬 비교 시연15분동일 주제 병렬 비교 (ChatGPT: 문장 vs Gemini: 구조)
1-4 미니 실습15분동절기 안전 운행 공지문 생성 및 수정 실습 (10분), 공유 (5분)

데모 내용

  • 시연 1 (배송 지연 안내문): ChatGPT의 매끄러운 문장 표현력과 문단 단위 수정 기능 강조
  • 시연 2 (물류 운영 개선 메모): Gemini의 체계적인 구조 설계와 표/체크리스트 확장 기능 강조

학생 실습

  • 주제: "차주 기사님 대상 동절기 결빙 구간 안전 운행 및 상차 확인 요청 공지" 또는 "신규 배차 노선 효율화 방안 메모"
  • ChatGPT Canvas 또는 Gemini Canvas 중 하나를 골라 초안 생성
  • 생성된 문서에서 한 부분을 수정해보기 (예: "결빙 구간 리스트 추가해줘")

2교시 (50분)

공통점과 차이점 비교 실습

목표

  • 동일 프롬프트로 두 도구의 '결과물'을 병렬로 두고 무엇이 다른지 직접 체감한다.
  • 핵심 원칙: "어떤 도구가 더 좋냐"가 아니라 "어떤 작업에 더 맞냐"를 판단하는 안목을 기른다.
  • 비교표를 통해 자신의 업무에 맞는 '최애 도구'를 찾아본다. (이후 단계의 조합을 위한 기초 과정)

시간 배분

구간시간활동
2-1 공통점10분강사 설명 + 학생 경험 공유
2-2 차이점10분비교표 설명 + 시연
2-3 비교 실습20분학생 각자 두 도구로 실습 (15분) + 비교표 작성 (5분)
2-4 토론10분결과 공유 + 강사 정리

강사 정리 메시지

  • ChatGPT는 "글쓰기 코치" — 문장을 다듬고, 표현을 개선하는 데 강하다.
  • Gemini는 "구조 설계자" — 큰 틀을 잡고, 아이디어를 확장하는 데 강하다.
  • 실무에서 가장 좋은 조합: 초안은 Gemini로, 다듬기는 ChatGPT로.
3교시 (60분)

실무 문서 제작 실습

목표

  • 자신의 업무에 바로 쓸 수 있는 문서 1개를 두 도구를 이어 쓰는 'Pipeline 방법'으로 완성한다.
  • 핵심 원칙: 2교시의 비교를 넘어, 두 도구의 강점을 순서대로 연결하는 '조합(Combination)' 흐름을 체득한다.
  • "한 번에 완성"이 아니라 "수정 반복과 도구 전환"이 핵심이라는 것을 경험한다.

시간 배분

구간시간활동
3-1 주제 선정5분4개 주제 중 1개 선택
3-2 1단계: 초안 생성15분프롬프트 구성 + Canvas에 입력 + 초안 확인
3-3 2단계: 구조 변경15분초안을 항목형/문단형으로 변환 요청
3-4 3단계: 톤 조정10분대상 독자에 맞춰 어조 변경 요청
3-5 4단계: 표 변환15분최종 결과를 표/체크리스트로 재구성

실습 주제 (4개)

번호주제권장 사용 흐름
1신규 대형 화주사 대상 최적 배차 노선 제안서Gemini(구조) → ChatGPT(문장 다듬기)
2동절기 차량 소모품 점검 및 안전 수칙 안내문ChatGPT(초안) → Gemini(체크리스트화)
3물류 터미널 상하차 작업 효율 개선 아이디어Gemini(아이디어 확장) → ChatGPT(보고서화)
4주간 정산 지연 방지를 위한 서류 수거 실행 계획ChatGPT(현황 정리) → Gemini(운영표 변환)

각 단계별 프롬프트 예시

1단계 - 초안 생성:

당신은 물류 제안서 작성 전문가입니다.
목적: [주제]에 대한 제안서 목차 작성 (신규 노선 및 배차 효율 중심)
대상: [고객사명] 물류 담당자
형식: 아래 섹션 순서
1. 현재 물류 흐름 진단
2. 중원로지스 솔루션 (배차 최적화)
3. 기대 효과 (비용 및 시간 절감)
4. 비상 상황 대응 체계 (CS)

2단계 - 구조 변경:

생성된 내용을 항목 형식으로 다시 정리해 주세요.
각 항목은 2~3문장 이내로 짧게 해주세요.

3단계 - 톤 조정:

이 문서를 현장 기사님들 대상 안내문으로 다시 써 주세요.
어려운 한자어는 빼고, 실제로 현장에서 바로 인지하고 따라할 수 있는 수준으로 쉽게 고쳐 주세요.

4단계 - 표 변환:

생성된 내용을 실행 계획표로 변환해 주세요.
형식: | 순번 | 할 일 | 담당자 | 기한 | 진행 상황 |

강사 체크 포인트

  • 학생이 한 번의 프롬프트로 끝내려 하면 "수정해보세요"라고 유도.
  • "추가", "정리", "요약", "재작성"을 구분해서 쓰게 한다.
  • 문서 목적과 대상 독자를 먼저 적게 한다.
4교시 (60분)

프롬프트 실전과 결과물 피드백

목표

  • 잘 되는 프롬프트의 4요소(역할, 목적, 대상, 형식)를 구조적으로 이해한다.
  • 두 도구를 조합하는 흐름(Gemini 뼈대 → ChatGPT 살)을 마스터한다.
  • 오늘 만든 문서를 최종 완성하고 발표한다.

시간 배분

구간시간활동
4-1 프롬프트 구조10분4요소 설명 + 좋은/나쁜 프롬프트 비교
4-2 도구 조합10분4단계 조합 흐름 시연
4-3 마무리 실습25분학생 최종 문서 완성 (20분) + 개별 피드백 (5분)
4-4 발표15분2~3명 발표 + 한 줄 평가

프롬프트 4요소

요소역할좋은 예시나쁜 예시
역할AI에게 입장 설정"당신은 실무 문서 작성 전문가입니다"(없음)
목적무엇을 만들 것인지"고객사 제안서의 목차를 만들어주세요""이거 해줘"
대상누가 읽을 것인지"AI에 익숙하지 않은 직원 대상"(없음)
형식어떤 모양으로"항목형, 5개 섹션, 각 3문장""문서 좀 써줘"

도구 조합 4단계 시연

4단계 조합 시연 (Pipeline):

1단계 (Gemini - 설계): "중원로지스 2분기 배차 효율화 계획서 목차 설계"
2단계 (ChatGPT - 서술): "특정 섹션 상세 내용 작성 및 문장 보강"
3단계 (Gemini - 재구성): "작성된 계획을 월간 실행 체크리스트(표)로 변환"
4단계 (ChatGPT - 다듬기): "임원 보고용으로 격식 있고 간결하게 최종 다듬기"

발표 질문 유도

  • "오늘 두 도구 중, 내 업무에는 무엇이 더 맞을 것 같나요?"
  • "어떤 부분에서 차이를 느꼈나요?"
  • "내일 바로 적용해볼 수 있는 작업이 있나요?"
