|------|------|

## [1교시] Canvas 개념 이해와 도구 소개 (50분)

### 강의 목표
- Canvas가 일반 채팅과 어떻게 다른지 이해
- ChatGPT Canvas와 Gemini Canvas의 기본 사용법을 습득

### 1-1. 도입 (5분)

**강사 대사:**
> "안녕하세요, 7주차 수업입니다. 오늘은 ChatGPT Canvas와 Gemini Canvas 두 가지를 배울 건데, 이름은 비슷하지만 강점이 다릅니다. 같이 비교해 보면서 내 업무에 맞는 도구를 고르는법을 알아보겠습니다."

**핵심 포인트:**
- 일반 채팅: 한 번에 하나 질문하고 답변 받음
- Canvas: 문서 전체를 workspace에서 편집하듯이 작업

### 1-2. Canvas란 무엇인가 (10분)

**강사 대사:**
> "Canvas는 '작업 공간'이라는 뜻입니다. 일반 채팅은 말하면 사라지는 대화이지만, Canvas는 문서를 뜨겁게 유지하면서 수정이 가능합니다."
>
> "예를 들어, 보고서 초안을 받고 '이 부분 더 자세하게', '표로 바꿔줘', '어조는 격식 있게'라고 하나씩 요청할 수 있습니다."

**시연 내용:**
1. ChatGPT에서 Canvas 모드 시작하는 방법 화면 보여주기
2. Gemini에서 Canvas/Compose 시작하는 방법 화면 보여주기

**학생에게 던질 질문:**
> "평소 업무에서 문서를 어떻게 작성하세요? 처음부터 한 번에 끝내시나요, 여러 번 수정하시나요?"

### 1-3. 두 도구 간단 소개 (15분)

**ChatGPT Canvas 시연:**
- 화면 구성 설명 (메뉴, 수정 옵션)
- "사내-announcement 만들기" 예시로 간단히 보여주기

**Gemini Canvas 시연:**
- 화면 구성 설명 ( Compose, Refine 모드)
- 같은 내용 Gemini에서 생성하는 것 보여주기

**핵심 포인트 (강사 메모):**
- ChatGPT: 문장 다듬기에 강점, 글쓰기 코치 역할
- Gemini: 구조 설계, 큰 틀 잡기에 강점, Google 생태계 연결

### 1-4. 미니 실습 (15분)

**실습 지시:**
> "지금 간단한 과제를 내겠습니다. 주제는 '업무 시작 전 팀에 보내는 간단한 공지 메일'입니다. 3분간 각자 ChatGPT Canvas 또는 Gemini Canvas에서 아무거나 하나 골라서 초안을 만들어 보세요."

**학생이 작업하는 동안:**
- 순환하며 개별 질문 받기
- 화면이 안 보이는 학생 있으면 도움

**3분 후:**
> "어떤 도구를 쓰셨나요? 어떤 느낌이었는지 한마디씩만share해 보세요."

## [2교시] 공통점과 차이점 비교 실습 (50분)

### 강의 목표
- 두 도구의 차이를 체감
- 어떤 상황에 어떤 도구가 더 맞는지를 판단

### 2-1. 공통점 정리 (10분)

**강사 대사:**
> "먼저 두 도구의 같은 부분을 정리해 보겠습니다. 다들 공통적으로 이런 경험이 있을 거예요."

**화면 공유 또는 노트:**
| 공통점 | 설명 |
|--------|------|
| 긴 글 초안 작성 | 한 번에 여러 문단 생성 가능 |
| 부분 수정 | 특정 문장/문단만 골라서 수정 가능 |
| 반복 개선 | "수정 -> 확인 -> 다시 수정" 흐름이 자연스러움 |
| 문서 작업 시간 단축 | 업무 문서 초안 만들기에 적합 |

### 2-2. 차이점 비교 (15분)

**강사 대사:**
> "이제 차이점을 볼게요. 같은 과제를 줬을 때 어떤 결과가 나오는지 비교해 보겠습니다."

**비교표 설명:**

| 구분 | ChatGPT Canvas | Gemini Canvas |
|------|-----------------|----------------|
| **강점** | 문장 다듬기, 표현 개선 | 구조 설계, 아이디어 확장 |
| **수정 흐름** | 기존 문서에 붙여넣고 손보기 좋음 | 큰 틀 먼저 잡고 세부 채우기 좋음 |
| **한국어** | 매우 자연스러운 문장 | 정확한데 sometimes 딱딱함 |
| **표/목차** | 기본적 지원 | 구조화 도구 강함 |
| **적합 상황** | 보고서 다듬기, 메일 수정 | 제안서 뼈대, 교육자료 구조 |

> "표만 보고 바로 'ChatGPT가 더좋다'고 판단하시면 안 됩니다. 목적에 따라 다릅니다. 문장을 매끄럽게 하고 싶으면 ChatGPT, 전체 틀을 짜고 싶으면 Gemini라고 생각하시면 됩니다."

### 2-3. 비교 실습 진행 (20분)

**실습 과제:**
> "이제 같은 주제를 가지고 두 도구를 각각 써 볼게요. 주제는 '중원로지스 내부 AI 활용 가이드'입니다."
>
> "각 도구에서 아래 프롬프트를 그대로 넣고 생성해 보세요."

**프롬프트 (화면에 공유):**
```
당신은 실무 문서 작성 도우미입니다.
주제: 중원로지스에서 AI 도구를 활용하는 기본 가이드
대상: AI를 처음 사용하는 실무 직원
형식: 아래 순서
1. 도입 (왜 필요한가)
2. 실제 사용 예시 3가지
3. 바로 해볼 일 3가지
```

**학생 작업 시간 10분:**
- 순환하며 질문 받기
- "ChatGPT에서 Gemini로 넘어갈 때 느낌이 어떻게 달라지는지" 관찰

### 2-4. 결과 비교 및 토론 (5분)

**질문:**
> "ChatGPT로 만든 것 vs Gemini로 만든 것, 어떤 차이가 났나요?"
> "각 도구의 좋은 점과 아쉬운 점이 뭔가요?"

**강사 정리:**
> "결론: 한 번에 완성하려고 하지 마세요. 초안은 Gemini, 다듬기는 ChatGPT. 이것이 실무에서 가장 많이 쓰는 조합입니다."

## [3교시] 실무 문서 제작 실습 (60분)

### 강의 목표
- 실제 업무 문서를 Canvas로 직접 제작
- 프롬프트를 단계별로 써서 초안→수정→최종 정리 흐름 체득

### 3-1. 실습 주제 선정 (5분)

**강사 대사:**
> "이제 실제 업무에 바로 쓸 수 있는 문서를 만들어 보겠습니다. 아래 주제 중 하나를 골라주세요."

**주제 선택지 (화면 또는 인쇄물):**
1. 고객사 제안서 목차 초안
2. 사내 AI 활용 교육 안내문
3. 물류/운영 개선 아이디어 정리
4. 팀 회의록 실행 계획서
5. 거래처에 보내는 설명 메일

> "직접 만들어서 내일에 바로 쓸 수 있는 걸 만드는 게 목표입니다."

### 3-2. 1단계: 초안 생성 (15분)

**지시:**
> "먼저 주제를 정했다면, 아래 순서로 프롬프트를 구성해 보세요."
>
> "- 역할: (예) 실무 문서 작성 도우미"
> "- 목적: (예) 제안서 목차 만들기"
> "- 대상 독자: (예) 고객사 담당자"
> "- 원하는 형식: (예) 항목 형태, 5개 섹션"

**샘플 프롬프트 (화면 공유):**
```
당신은 제안서 작성 전문가입니다.
목적: [주제]에 대한 제안서 목차 작성
대상: [고객사/팀] 담당자
형식: 아래 섹션 순서
1. 현재 문제 상황
2. 해결 방안
3. 기대 효과
4. 추진 일정
5. 비용 개요
```

**학생 작업:**
- 각자 프롬프트 작성 → Canvas에 입력 → 초안 생성
- 10분간 진행

### 3-3. 2단계: 구조 변경 (15분)

**지시:**
> "초안이できました? 이제 구조를 바꿔보겠습니다."
>
> "예를 들어, '문단형을 항목형으로', 또는 '요약 부분을 상세하게'라고 요청해 보세요."

**프롬프트 예시:**
```
생성된 내용을 항목 형식으로 다시 정리해 주세요.
각 항목은 2~3문장 이내로 짧게 해주세요.
```

**학생 작업:**
- 개인별로 수정 프롬프트 입력
- 결과 확인

### 3-4. 3단계: 톤 조정 (10분)

**지시:**
> "이제 대상 독자에 맞춰 어조를 바꿔보겠습니다."
>
> "예를 들어, '초보자용으로 쉽게', '격식 있는 보고서 톤으로', '가벼운 메일 형식으로'라고 요청해 보세요."

**체크포인트:**
- 문장 길이
- 용어 수준
- 정중 표현

### 3-5. 4단계: 표/체크리스트로 변환 (15분)

**지시:**
> "마지막으로, 내용을 표나 체크리스트로 바꿔보겠습니다. 실무에서는 표나 리스트 형태가 훨씬 유용해요."

**프롬프트 예시:**
```
생성된 내용을 실행 계획표로 변환해 주세요.
형식: | 순번 | 할 일 | 담당자 | 기한 | 진행 상황 |
```

## [4교시] 프롬프트 실전과 결과물 피드백 (60분)

### 강의 목표
- 실무에서 바로 쓰는 프롬프트 패턴 체득
- 두 도구를 조합하는 방식 마스터

### 4-1. 잘 되는 프롬프트 구조 (15분)

**강사 대사:**
> " 프롬프트를 잘 쓰면 결과가 전혀 달라집니다. 잘 되는 프롬프트 4가지 요소를 설명하겠습니다."

**4가지 요소 (화면 또는 노트):**

| 요소 | 역할 | 예시 |
|------|------|------|
| **역할** | AI에게 입장 설정 | "당신은 실무 문서 작성 전문가입니다" |
| **목적** | 무엇을 만들 것인지 | "고객사 제안서의 목차를 만들어주세요" |
| **대상** | 누가 읽을 것인지 | "AI에 익숙하지 않은 직원들을 대상으로" |
| **형식** | 어떤 모양으로 | "항목 형태, 5개 섹션, 각 3문장" |

**실패하는 프롬프트 예시:**
> "이거 해줘" → 목적도, 대상도, 형식도 없어서 결과가 막됨
> "문서 좀 써줘" → 무엇을 쓸지 모름

### 4-2. 도구 조합 예시 (10분)

**강사 대사:**
> "실무에서는 둘 중 하나만 쓰지 않습니다. 조합해서쓰는 게 최고입니다."

**조합 예시:**

```
1단계: Gemini Canvas → 전체 구조/목차 설계
2단계: ChatGPT Canvas → 각 섹션 상세 내용 작성
3단계: Gemini Canvas → 표, 체크리스트로 변환
4단계: ChatGPT Canvas → 최종 문장 다듬기
```

> "이 흐름을 기억하시면 됩니다. Gemini는 뼈대, ChatGPT는 살이라고 생각하면 쉬워요."

### 4-3. 마무리 실습 (25분)

**지시:**
> "오늘 배운 내용을 다 합해서, 처음에 뽑은 주제의 최종 결과물을 완성해 보세요."
>
> "- 최소 1개 이상의 수정을 거친 문서"
> "- 어떤 도구를 조합했는지 메모"
> "- 최종 결과물은 내일 바로 쓸 수 있는 수준"

**진행:**
- 개인 작업 20분
- 순환しながら 개별 피드백

### 4-4. 발표 및 한 줄 평가 (10분)

**진행 방식:**
- 2~3명 간단 발표 (1분 each)
- "가장 좋았던 점 / 개선하면 좋은 점" 피드백

**마지막 질문:**
> "오늘 두 도구 중, 내 업무에는 무엇이 더 맞을 것 같나요? 한마디로 정리해 보세요."

**강사 마무리:**
> "오늘 배운 건 도구입니다. 중요한 건 내일 아침에 바로 하나くらい 적용해보는 거예요. 다름 아니라 바로 써보세요!"
