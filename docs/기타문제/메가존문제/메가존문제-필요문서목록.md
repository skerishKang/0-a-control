# 메가존 문제 필요 문서 목록

작성일: 2026-03-26  
작성 기준: 파디엠 강철원 이사 관점

## 1. 목적

이 문서는 메가존/클라우드그램 AWS 청구 분쟁과 지급명령 대응을 위해 파디엠이 준비해야 할 문서를 정리한 목록이다.

문서는 아래 4개 묶음으로 본다.

1. 법원 대응 문서
2. 상대방 사실조회/자료요구 문서
3. 부산광역시·부산정보산업진흥원 대응 문서
4. 증거 확보 및 내부 정리 문서

---

## 2. 최우선 문서

### 2.1 법원 제출용 지급명령 이의신청서
- 목적: 지급명령 확정을 막고 본안 다툼으로 넘기기
- 필요 시점: 최우선
- 필수 확인 사항:
  - 사건번호
  - 법원명
  - 원고/채권자 명칭
  - 송달일
  - 이의신청 기한
- 핵심 취지:
  - 2021년 이후 실사용 부인
  - 종료/정리 시도 존재
  - 후행 청구 원인과 사용주체 특정 부족
  - 지원사업 구조와 통합빌링 구조의 책임선 불명확

### 2.2 지급명령 대응용 쟁점 1장 요약
- 목적: 법원 대응 논리를 내부적으로 빠르게 공유
- 사용처:
  - 법률대리인 상담
  - 대표 보고
  - 내부 의사결정
- 포함할 것:
  - 사건 개요
  - 핵심 쟁점 5개
  - 파디엠 유리 포인트
  - 상대방 입증 부족 포인트

## 3. 메가존/클라우드그램 대응 문서

### 3.1 메가존/클라우드그램 사실조회 요구서
- 목적: 상대방 청구 근거를 문서로 특정하게 만들기
- 수신 대상:
  - 메가존 채권담당
  - 클라우드그램 담당
- 반드시 요구할 자료:
  1. AWS Account ID
  2. 월별 과금 서비스명
  3. 리전별 사용 내역
  4. 2024년 과금 급증 원인 서비스
  5. IAM 사용자 또는 접근 주체 확인 가능 자료
  6. 통합빌링 승인 자료
  7. 세금계산서 발행 이력 전체
  8. 2021년 종료/삭제/해지 관련 처리 기록
  9. 388원 이후 미수 누적 경위

### 3.2 메가존 채권추심 대응 공문 또는 이메일
- 목적: 단순 독촉이 아니라 분쟁 상태임을 명확히 통지
- 핵심 취지:
  - 이 사건은 단순 미납이 아니라 사용주체·책임귀속 다툼이 있는 사안임
  - 자료 제출 전 일방적 채권추심만으로는 해결될 수 없음
  - 법원 대응과 별개로 사실관계 문서 제출을 요구함

### 3.3 메가존/클라우드그램에 대한 증거목록 송부문
- 목적: 파디엠이 이미 보유한 주요 정황 자료를 상대방에게 구조적으로 제시
- 포함 후보:
  - 테스트베드 신청 메일
  - 통합빌링 승인 요청 메일
  - 2021-06-10 KMS 문의 메일
  - 388원 미납 정황
  - 2024-12-06 부산진흥원 설명 메일
  - 주요 통화기록 요약

## 4. AWS 대응 문서

### 4.1 AWS Billing / Support 문의서
- 목적: 원장 사실관계 확인
- 요구할 것:
  1. 실제 과금 서비스 내역
  2. 리전별 사용 내역
  3. KMS 관련 과금 구조
  4. seller of record
  5. billing contact
  6. 계정 종료/정지/해지 관련 기록 여부
- 주의:
  - 감정적 분쟁 서술보다 사실조회 중심으로 작성

### 4.2 AWS 계정 사용 내역 확인 요청서
- 목적: 2021년 이후 무엇이 남아 있었는지 구체화
- 구체 질문:
  - 어느 서비스가 언제부터 언제까지 비용을 발생시켰는가
  - 버지니아 리전 사용이 실제 있었는가
  - KMS 키 또는 리소스가 잔존했는가

## 5. 부산광역시 / 부산정보산업진흥원 대응 문서

### 5.1 부산광역시·부산정보산업진흥원 민원서
- 목적: 지원사업 종료 후 계정 정리 책임과 인수인계 문제 확인
- 핵심 질문:
  1. 테스트베드 사업 종료 시 계정/리소스 정리 책임이 누구에게 있었는가
  2. 당시 메가존/클라우드그램과 파디엠의 역할 구분을 어떻게 안내했는가
  3. 2024-12-06 사건 설명 메일을 누가 받고 무엇을 했는가
  4. 2025년 1월 인수인계가 어디까지 되었는가
  5. 왜 파디엠에게 명확한 구조 설명을 하지 못했는가

### 5.2 부산진흥원 사실확인 요청서
- 목적: 민원보다 더 구조적으로 당시 사업 운영자료를 확보
- 요구할 자료:
  - 당시 사업 안내자료
  - 테스트베드 종료 가이드
  - 클라우드 계정 종료/반납 절차 문서
  - 참여기업 안내 문서

### 5.3 부산진흥원 면담/통화 질문서
- 목적: 실제 통화나 면담 전에 질문을 구조화
- 대상:
  - 김지혜 담당
  - 이승희
  - 관련 팀장/과장

## 6. 내부 준비 문서

### 6.1 내부 사실관계 정리서
- 목적: 파디엠 내부 입장을 한 문서로 정리
- 포함할 것:
  - 2020년 사업 시작
  - 2021년 종료 인식
  - 2021년 KMS 문의 및 정리 시도
  - 388원 정산 인식
  - 2024년 이후 독촉 경과

### 6.2 내부 증거목록표
- 목적: 증거를 번호 붙여 관리
- 구분:
  - 메일
  - PDF/이미지
  - 거래원장
  - 통화기록
  - 내부 메모

### 6.3 퇴사자/접근권한 정리표
- 목적: 2021년 이후 실제 사용 가능자를 특정
- 포함 항목:
  - 이름
  - 역할
  - 퇴사일
  - AWS 접근 가능 여부
  - 인수인계 여부

### 6.4 2021년 종료/정리 정황 정리서
- 목적: `우리는 끝난 줄 알았다`를 자료 구조로 입증
- 넣을 것:
  - KMS 문의 메일
  - 키 삭제/정리 정황
  - 388원 미납 안내
  - 종료 인식 근거

## 7. 보조 대응 문서

### 7.1 KISA 또는 사이버 신고 검토 메모

  - 리전/서비스 흔적은 있는데 파디엠 내부 사용 흔적이 없을 때

### 7.2 대외 설명용 1장 브리프

- 포함:
  - 사건 개요
  - 핵심 쟁점
  - 왜 단순 미납이 아닌지
  - 지금 필요한 조치

## 9. 결론

지금 파디엠에게 필요한 것은 문서를 많이 만드는 것이 아니라, **법원 대응 문서 + 상대방 자료요구 문서 + 공공기관 확인 문서 + 내부 증거 정리 문서**를 우선순위대로 갖추는 것이다.

핵심은 아래 4개다.

1. 법원 대응
2. 메가존/클라우드그램 사실조회
3. AWS 원장 확인
4. 부산진흥원/부산시 책임선 확인

이 4개가 정리되면, 이후 협상·법적 대응·추가 민원 모두 훨씬 유리해진다.
