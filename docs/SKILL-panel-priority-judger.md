# SKILL: panel-priority-judger

## skill name
`panel-priority-judger`

## description
웹 UI 패널 순서나 우선순위를 논의할 때, 패널을 역할(성격) 기준으로 분류하고 그에 따른 배치 순위를 일관되게 판단한다.

## trigger 조건
다음 상황에서 이 skill을 사용:
- 패널 순서 재배치 논의
- 새 패널 추가 논의
- 패널 우선순위 결정
- "어떤 패널이 위에 와야 하는가" 질문

## 패널 분류 기준 (고정)

| 성격 | 설명 | 예시 |
|------|------|------|
| **경보** | 즉각조치 필요, 날짜/시간 기반 알림 | 기한 임박, 마감 경고 |
| **판단** | 우선순위 결정 돕는 분석/추천 | AI 추천 퀘스트, 우선 검토 후보 |
| **계획** | 사용자가 설정한 운영 목표 | 단기/장기/반복 플랜 |
| **기록** | 작업 이력, 히스토리 | 최근 세션, 미완료 플랜 |
| **참고** | 맥락 자료, 외부 입력 | 텔레그램, 브리프, workdiary 폴더 |

## 기본 정렬 규칙

### 지원영역 (Support Zone)
항상 아래 순서:
1. **경보** (기한 임박)
2. **판단** (AI 추천 → 우선 검토 후보)
3. **계획** (단기 → 장기 → 반복)
4. **기록** (최근 세션, 미완료)
5. **참고** (브리프, 텔레그램, 폴더)

### 예외 규칙
- 사용자가 명시적으로 "AI 추천 우선" 철학 선택 → 판단 패널을 경보 다음에 배치 가능
- 단, 판단 패널도 계획 패널보다는 뒤에 배치
- 기록과 참고는 항상 마지막

## 실패 기준
다음 중 하나라도 해당하면 실패:
- 패널 역할(성격) 분류 없이 순서만 제안
- "사용자 결정이므로"理由로 분류 없이 순서만 정함
- 모델/AI 추천을 사용자 확정 영역(계획) 위에 배치
- 단순 "중요한 것 먼저" 같은 추상적 기준만 언급

## 적용 예시

### 상황
"패널 순서를 어떻게 해야 할까?"

### 분석 (올바른 예)
1. "기한 임박" → 경보 → 1순위 ✓
2. "프로젝트에서 온 추천 퀘스트" → 판단 → 2순위 ✓
3. "우선 검토 후보" → 판단 → 3순위 ✓
4. "단기 플랜" → 계획 → 4순위 ✓
5. "최근 세션" → 기록 → 뒤쪽 ✓

### 실패 예시
- "사용자가 중요하다고 생각하는 것을 위에" (분류 없음)
- "AI 추천을 맨 위에" (사용자 계획 위에 배치)
