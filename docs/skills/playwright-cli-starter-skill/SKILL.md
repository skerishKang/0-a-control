# Playwright-CLI Starter Skill

작성일: 2026-03-27  
목적: 반복 웹업무를 자연어 -> Playwright-CLI 실행 순서로 바꾸는 시작용 스킬

## 이 스킬이 맞는 경우

이 스킬은 아래 작업에 맞다.

1. 같은 웹페이지를 반복 방문한다.
2. 입력 필드와 버튼 위치가 대체로 고정돼 있다.
3. 결과를 텍스트나 숫자로 읽어오면 충분하다.
4. 사람이 매번 같은 순서로 하는 웹 작업이다.

예:
- 계산 페이지에 값 입력 후 결과 읽기
- 정기 조회 페이지에서 특정 수치 확인
- 테스트 폼 반복 입력
- 업무용 웹 앱에 정해진 값 넣고 결과 확인

## 이 스킬이 안 맞는 경우

아래 경우는 MCP나 브라우저 탐색형 도구가 더 낫다.

1. 처음 보는 사이트
2. 구조가 자주 바뀌는 사이트
3. 예외 팝업이 자주 뜨는 사이트
4. 디버깅이 많이 필요한 사이트

정리:
- 탐색은 MCP
- 반복 운영은 Playwright-CLI 스킬

## 핵심 원칙

1. 먼저 페이지를 열고 현재 상태를 확인한다.
2. 구조를 모르면 바로 입력하지 않는다.
3. 입력 -> 실행 -> 결과 확인 순서를 지킨다.
4. 결과는 사람이 읽는 한국어 요약으로 정리한다.
5. 중요한 작업은 screenshot 또는 video를 남긴다.

## 입력 형식

아래 형식으로 작업 내용을 정리해서 받는다고 가정한다.

```yaml
site_name: 예시 계산기
url: https://example.com
goal: 철근량 계산
inputs:
  - label: 두께
    value: 35
  - label: 폭
    value: 400
  - label: 높이
    value: 800
expected_output:
  - 철근량
  - 검토 결과
artifacts: screenshot
```

## 내부 실행 순서

1. URL 접속
2. 현재 페이지 snapshot 또는 screenshot
3. 입력 요소 확인
4. 값 입력
5. 실행 버튼 클릭
6. 결과 영역 확인
7. 결과 요약
8. 필요 시 screenshot/video 저장

## 출력 형식

```text
실행 요약
- 사이트:
- 목적:

입력값
- ...

결과
- ...

산출물
- screenshot:
- video:
```

## 실패 시 규칙

1. 요소를 못 찾으면 현재 페이지 상태를 먼저 요약한다.
2. 잘못된 추측으로 강행하지 않는다.
3. 막힌 단계와 이유를 분명히 쓴다.
4. 필요하면 MCP로 전환하라고 제안한다.

## 공통 실행 프롬프트

```text
너는 반복 웹업무를 Playwright-CLI로 수행하는 에이전트다.

목표:
- 주어진 URL에 접속
- 필요한 입력 수행
- 결과를 읽어 요약
- 필요하면 screenshot 또는 video 저장

행동 규칙:
1. 먼저 페이지를 열고 현재 상태를 확인하라.
2. 구조를 모르면 바로 입력하지 마라.
3. 입력 -> 실행 -> 결과 확인 순서를 지켜라.
4. 결과는 한국어로 간결하게 정리하라.
5. 실패 시 어느 단계에서 막혔는지 보고하라.
```

## 우리 팀 기준 운영 방식

1. 처음 보는 사이트는 MCP로 탐색
2. 반복되는 루틴만 이 스킬로 고정
3. 운영 단계에서는 이 스킬을 기본으로 사용
4. 실패하거나 구조가 바뀌면 다시 MCP로 복귀

한 줄 결론:
- 탐색은 MCP
- 반복 운영은 Playwright-CLI 스킬
