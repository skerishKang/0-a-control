# Site Profile: 철근량 산정 웹앱

작성일: 2026-03-27  
목적: Playwright-CLI Starter Skill의 첫 검증 대상으로 사용할 계산형 웹앱 프로파일

## 1. 기본 정보

- site_name: 철근량 산정 웹앱
- url: `[실제 URL 기입 필요]`
- 목적: 입력값을 넣고 철근량 및 검토 결과를 자동으로 읽어오기
- 로그인 필요 여부: 없음(가정)
- 결과 저장 필요 여부: screenshot 선택 가능

## 2. 왜 이 사이트를 첫 테스트 대상으로 잡는가

이 사이트는 첫 스킬 검증 대상으로 적합하다.

1. 입력 필드가 단순하다.
2. 계산 버튼과 결과 영역이 분리되어 있다.
3. 반복 테스트가 쉽다.
4. 결과가 숫자/텍스트 형태라 자동 요약하기 좋다.
5. 영상 데모와 유사한 흐름이라 스킬 검증이 쉽다.

즉, `열기 -> 값 입력 -> 계산 -> 결과 읽기` 흐름을 가장 작게 검증하기 좋다.

## 3. 반복 작업 설명

- 사용자가 주로 시키는 말:
  - 철근 계산 페이지에 접속해서 값들을 넣고 결과를 알려줘
  - 같은 조건으로 다시 계산하고 screenshot도 저장해줘
  - 입력값을 바꿔서 비교해줘

- 실제로 하는 순서:
  1. 페이지 열기
  2. 현재 화면 확인
  3. 입력 필드 3~5개 식별
  4. 값 입력
  5. 계산 버튼 클릭
  6. 결과 숫자와 검토 문구 읽기
  7. 결과 요약

- 결과로 보고 싶은 것:
  - 철근량
  - 철근비 또는 검토 결과
  - 필요 시 입력값 요약

## 4. 입력 필드

실제 셀렉터는 사이트를 보면서 채워야 한다. 지금은 개념 프로파일로 둔다.

1. label: 두께
   - selector 또는 식별 단서: 두께 입력창
   - 예시 값: 35

2. label: 폭
   - selector 또는 식별 단서: 폭 입력창
   - 예시 값: 400

3. label: 높이
   - selector 또는 식별 단서: 높이 입력창
   - 예시 값: 800

4. label: 길이 또는 추가 계산 변수
   - selector 또는 식별 단서: 네 번째 숫자 입력창
   - 예시 값: 1000

## 5. 버튼

1. 버튼명: 계산
   - selector 또는 식별 단서: 계산/실행 버튼

## 6. 결과 영역

1. 결과명: 철근량
   - selector 또는 식별 단서: 결과 숫자 영역

2. 결과명: 검토 결과
   - selector 또는 식별 단서: 여유율/적정성/검토 문구 영역

## 7. 안정성 메모

- 계산형 사이트라 구조가 크게 흔들리지 않으면 반복 자동화에 적합하다.
- 텍스트 입력과 숫자 입력 위주라 Playwright-CLI 루틴 검증이 쉽다.
- 첫 테스트에서는 꼭 screenshot을 남기는 것이 좋다.

## 8. 자연어 명령 예시

1. 철근량 계산 페이지에 접속해서 두께 35, 폭 400, 높이 800, 길이 1000을 입력하고 결과를 알려줘.
2. 같은 계산을 다시 하고 screenshot도 저장해줘.
3. 폭만 450으로 바꿔서 결과 차이를 알려줘.

## 9. 복붙용 작업 입력 예시

```yaml
site_name: 철근량 산정 웹앱
url: https://example.com/rebar
goal: 철근량 계산
inputs:
  - label: 두께
    value: 35
  - label: 폭
    value: 400
  - label: 높이
    value: 800
  - label: 길이
    value: 1000
expected_output:
  - 철근량
  - 검토 결과
artifacts: screenshot
```

## 10. 이 사이트로 첫 검증 때 확인할 것

1. 페이지를 문제없이 열 수 있는가
2. 입력 필드를 안정적으로 찾는가
3. 계산 버튼을 안정적으로 누르는가
4. 결과 텍스트를 읽을 수 있는가
5. 결과를 한국어로 요약할 수 있는가
6. screenshot 저장이 되는가

## 11. 다음 단계

이 프로파일로 첫 검증이 통과하면 다음으로 가면 된다.

1. 실제 URL 입력
2. 실제 selector/식별 단서 보강
3. 플랫폼별 스킬 초안 생성
4. 두 번째 사이트(조회형 또는 폼형) 추가

한 줄 결론:
- 첫 실험은 복잡한 업무 페이지보다 `철근량 산정 같은 계산형 웹앱`이 가장 안전하다.
