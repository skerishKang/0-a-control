# Session Architecture Prompt

아래 원칙에 맞춰 세션 시스템을 설계하거나 정리해라.

## 목표
세션은 단순 로그 저장이 아니라, 다음 작업을 이어갈 수 있는 복원 구조여야 한다.

## 반드시 지켜야 할 3층 구조

### 1. 원문층
- raw logs
- 원래 대화/원래 기록
- 최대한 가공하지 않은 형태
- 나중에 요약이 틀렸을 때 원문으로 되돌아갈 수 있어야 한다

### 2. 복원층
- 원문을 바탕으로 주제별 요약
- 현재 상태
- 아직 남은 것
- 다음 액션
- 즉, 그대로 복사가 아니라 운영 복원용 정리

### 3. 표시층
- HTML 또는 대시보드
- 원문 보기
- 요약 보기
- 분류 보기
- 사람이 빠르게 읽을 수 있는 형태

## 핵심 원칙
1. 원문은 반드시 남아 있어야 한다
2. 요약/분류는 원문과 분리되어야 한다
3. HTML은 보기용이지 원문 대체물이 아니다
4. 요약이 틀릴 수 있으므로 원문으로 언제든 돌아갈 수 있어야 한다
5. 세션 복원의 목적은 "지금 다시 이어서 일할 수 있게 만드는 것"이다

## 출력 시 항상 포함할 것
- 원문이 어디에 있는지
- 요약이 어디에 있는지
- HTML/보기 파일이 어디에 있는지
- 각 층의 역할이 무엇인지

## 금지
- 원문 없이 요약만 남기기
- HTML만 남기고 원문을 버리기
- 세션 복원을 단순 로그 나열로 끝내기
