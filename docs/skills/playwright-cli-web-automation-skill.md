# Playwright-CLI Web Automation Skill

작성일: 2026-03-27  
목적: Codex / Claude Code / Kilo Code / Anthropic 계열 에이전트가 **반복 웹 업무**를 Playwright-CLI 기반으로 수행하도록 만드는 공통 스킬 초안

## 1. 이 스킬이 맞는 경우

이 스킬은 아래 같은 작업에 맞다.

1. 정해진 사이트에 반복적으로 접속한다.
2. 입력 필드, 버튼, 결과 위치가 대체로 고정돼 있다.
3. 사람이 매번 같은 순서로 수행하는 웹 업무다.
4. 자연어 지시를 최종적으로 CLI 실행 루틴으로 바꾸고 싶다.

예:
- 계산기/견적기/산정기 입력 자동화
- 사내 대시보드 정기 조회
- 테스트 환경 반복 폼 입력
- 결과 캡처/녹화
- 고정된 웹 운영 루틴

## 2. 이 스킬이 아닌 것이 더 나은 경우

이 스킬보다 MCP 또는 브라우저 탐색형 도구가 더 나은 경우:

1. 처음 보는 사이트다.
2. 페이지 구조가 자주 바뀐다.
3. 요소 탐색과 디버깅이 많이 필요하다.
4. 팝업/로그인/예외 상태가 자주 다르다.

정리:
- 탐색/디버깅: MCP
- 반복 운영: Skill + CLI

## 3. 핵심 원칙

1. 브라우저를 직접 추론하기보다 **명시적인 단계**로 쪼갠다.
2. 자연어 지시는 내부적으로 **명령 시퀀스**로 변환한다.
3. 처음 실행 전에는 가능하면 `snapshot` 또는 `screenshot` 단계로 현재 페이지를 확인한다.
4. 결과는 반드시 텍스트로 요약한다.
5. 중요한 작업은 옵션으로 녹화 또는 스크린샷을 남긴다.

## 4. 입력 계약

이 스킬은 최소 아래 입력을 기대한다.

- `site_name`: 사이트 이름
- `url`: 접속 URL
- `goal`: 하려는 목적
- `inputs`: 입력할 값 목록
- `expected_output`: 읽어와야 할 결과
- `artifacts`: screenshot / video / none

예시 입력 구조:

```yaml
site_name: 철근 산정기
url: https://example.com/rebar
goal: 철근량 계산
inputs:
  - label: 두께
    value: 35
  - label: 폭
    value: 400
  - label: 높이
    value: 800
expected_output:
  - 철근량
  - 철근비 검토 결과
artifacts: screenshot
```

## 5. 내부 작업 순서

이 스킬은 내부적으로 아래 순서로 행동한다.

1. URL 접속
2. 현재 페이지 확인
   - snapshot 또는 screenshot
3. 입력 요소 식별
4. 값 입력
5. 제출/계산 버튼 클릭
6. 결과 영역 확인
7. 결과 텍스트 요약
8. 필요 시 캡처/녹화 저장

## 6. 출력 계약

출력은 아래 형식을 지향한다.

1. 실행 요약
2. 입력값 요약
3. 읽은 결과
4. 저장한 산출물 경로
5. 실패 시 어디서 막혔는지

예시:

```text
실행 요약
- 사이트: 철근 산정기
- 목적: 철근량 계산

입력값
- 두께: 35
- 폭: 400
- 높이: 800

결과
- 철근량: 123.4 kg
- 철근비 검토: 여유율 20%

산출물
- screenshot: outputs/rebar-2026-03-27.png
```

## 7. 에이전트가 따라야 할 규칙

1. 페이지 구조를 모르면 먼저 확인 단계부터 한다.
2. 요소 식별이 불확실하면 강제 입력하지 않는다.
3. 민감한 계정/결제/전송 작업은 사용자 확인 없이는 실행하지 않는다.
4. 실패 시 재시도보다 현재 상태를 먼저 보고한다.
5. 결과는 항상 사람이 읽는 요약으로 정리한다.

## 8. 공통 프롬프트 템플릿

```text
너는 Playwright-CLI를 이용해 반복 웹 업무를 수행하는 에이전트다.

목표:
- 주어진 URL에 접속
- 필요한 입력을 수행
- 계산/조회/제출 결과를 읽어 요약
- 필요하면 screenshot 또는 video를 남긴다

행동 규칙:
1. 먼저 페이지를 열고 현재 상태를 확인하라.
2. 입력 전에 필드/버튼을 식별하라.
3. 값 입력 -> 버튼 클릭 -> 결과 확인 순서를 지켜라.
4. 결과는 한국어로 간결하게 요약하라.
5. 실패 시 어느 단계에서 막혔는지 보고하라.
```

## 9. 사용 예시

### 예시 A. 계산기
- URL 접속
- 3개 숫자 입력
- 계산 버튼 클릭
- 결과 텍스트 읽기

### 예시 B. 반복 조회
- 로그인된 상태에서 특정 대시보드 접속
- 날짜 필터 변경
- 특정 카드 값 읽기
- 결과를 텍스트로 요약

### 예시 C. 테스트 루틴
- 테스트 폼 열기
- 샘플 데이터 입력
- 제출
- 성공 메시지 확인
- screenshot 저장

## 10. 확장 포인트

이 스킬은 나중에 아래로 확장할 수 있다.

1. 사이트별 프로파일
2. 로그인 상태 저장

6. 실패 시 진단 모드

## 11. 우리 팀 기준 운영 원칙

우리에게 맞는 기본 전략:

1. 처음 가는 사이트는 MCP로 탐색
2. 반복되는 루틴은 Playwright-CLI 스킬로 고정
3. 운영 단계에서는 스킬을 주력으로 사용
4. 구조가 바뀌거나 실패하면 다시 MCP로 복귀
