# Board-v2 실전 운영 점검 로그

본 문서는 `board-v2`의 독립 운영 가능성을 확인하고, 실사용 중 발생하는 마찰 지점(Friction)을 수집하여 차기 고도화 우선순위를 정하기 위한 기록 양식입니다.

---

## 📋 일일 운영 로그 템플릿 (Daily Ops Log)

매일 작업 마무리 시점에 아래 양식을 복사하여 새로운 로그를 작성하세요.

```markdown
## 📅 [2026-03-XX] board-v2 실전 점검

### 1. 운영 완결성
- **v2 단독 운영 비율**: [ ]% (시작 ~ 보고 ~ 마감 중 v2에서 처리한 비중)
- **Classic/CLI 전환 포인트**:
  - [ ] 시점: (예: 퀘스트 시작 전 상세 로그 확인 시)
  - [ ] 이유: (예: 세션 대화 전문이 안 보여서)

### 2. 기능별 마찰 (Friction)
- **Polling**: (예: 30초가 너무 길어서 F5를 3번 눌렀음 / 갱신 시 깜빡임 여부 등)
- **Draft**: (예: 입력 중 Phase 탭을 옮겼을 때 값 보존 상태 / 유실 상황)
- **Pending**: (예: AI 판정 지연 시 수동 종료 탈출구 사용 여부 및 편의성)
- **Quick Input**: (예: 할 일을 던졌는데 분류 결과가 의도와 달라 수동 수정함)

### 3. 오늘의 한 줄 평
- [ ] 

### 4. 내일의 보강 포인트 (Priority 1)
- [ ] (예: 최근 세션 클릭 시 모달에 대화 전문 노출 필요)
```

---

## 📝 1줄 요약 템플릿 (Daily Summary)

작업 일지(`workdiary`) 등에 매일 짧게 기록할 때 사용합니다.

- `[v2 Ops] Ratio: X% | Exit: [CLI/Classic 이동이유] | Friction: [주요마찰기능] | Next: [고도화필요기능]`

---

## 📊 주간 회고용 집계 항목 (Weekly Retrospective)

1주일 운영 후 고도화 전략 수립 시 점검할 통계 항목입니다.

1. **평균 완결률**: 7일간의 v2 단독 운영 비율 평균 (목표 90% 이상)
2. **최다 이탈 경로**: Classic/CLI로 가장 많이 돌아가게 만든 결핍 요소 (Top 1)
3. **데이터 무결성 사고**: 폴링이나 드래프트 로직 오류로 인해 입력 내용이 날아간 횟수
4. **입력 효율성**: 퀵 인풋 사용 시 분류 수정 없이 그대로 확정된 비율
5. **2차 고도화 확정 기능**: 수집된 마찰 지점 데이터에 기반한 실제 구현 순위 1, 2, 3
