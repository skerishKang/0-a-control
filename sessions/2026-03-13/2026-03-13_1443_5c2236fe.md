  4. 이동 시 충돌 우려 파일

  - docs/가 이미 존재하므로, 현재 docs/*.md를 그대
    로 두면 새 공개 문서 docs/와 성격이 섞입니다.
  - README.md의 기존 링크가 ./최종보고서/...를 직접
    가리켜서, 이동 후 전부 깨집니다.
  - 최종보고서/연구노트/README.md와 루트 README.md
    는 이름이 같아 단순 평면 이동 시 충돌합니다.
  - 최종제출_보고서_연구노트/와 연구노트/는 거의 동
    일 파일을 중복 보유해, 둘 다 공개 쪽으로 보내면
    중복/불일치 문제가 생깁니다.
  - 최종보고서/제출본_자체시험성적서/자체시험성적서
    _완성본.*과 자체시험성적서_통합편철본.*은 둘 다
    최종처럼 보이므로 공개본 1세트만 남기지 않으면
    혼선이 납니다.
  - frontend/와 frontend_unified/는 둘 다 UI 진입점
    이라 공개용 app/ 설명서에서 어떤 것을 기본 실행
    경로로 둘지 정해야 합니다.
  - finalv2/, backend/, modules/, orchestrator/ 사

    장소에서는 통합본만으로도 심사위원 검토 목적을
    충분히 충족합니다. 월별/일별 파일은 보존 가치가
    있어 archive가 적절합니다.
  - TODO, PRD, 계획, 점검표, 메모, 후보목록, 감점방
    지는 전형적인 내부 작업 흔적이라 공개 비권장입
    니다.
  - 이전보고서, 본문1224_*, 수정본, 양식,
    hwtx_assets, 이미지 소스 폴더는 최종 산출물이
    아니라 편집 중간물이라 archive가 적절합니다.
  - 원시 CSV/JSON, 평가자 목록, 샘플 목록, 내부 자
    산 경로표는 공개 시 불필요하게 상세하고 재식별/
    내부자산 노출 위험이 있어 공개 제외가 맞습니다.
  - 회계기관 감사보고서, 평가위원회 안내, 전북대 초
    안 문서, 세금계산 관련 자료는 외부기관 문서 또
    는 행정 내부자료 성격이 강해 공개 제외가 안전합
    니다.
  - 최종제출_보고서_연구노트/는 최종 제출 패키지 복
    제본이지만 내부 메모와 중복 파일이 섞여 있어,
    공개 루트에는 두지 않고 archive로 빼는 편이 깔

  4. 이동 시 충돌 우려 파일

  - docs/가 이미 존재하므로, 현재 docs/*.md를 그대
    로 두면 새 공개 문서 docs/와 성격이 섞입니다.
  - README.md의 기존 링크가 ./최종보고서/...를 직접
    가리켜서, 이동 후 전부 깨집니다.
  - 최종보고서/연구노트/README.md와 루트 README.md
    는 이름이 같아 단순 평면 이동 시 충돌합니다.
  - 최종제출_보고서_연구노트/와 연구노트/는 거의 동
    일 파일을 중복 보유해, 둘 다 공개 쪽으로 보내면
    중복/불일치 문제가 생깁니다.
  - 최종보고서/제출본_자체시험성적서/자체시험성적서
    _완성본.*과 자체시험성적서_통합편철본.*은 둘 다
    최종처럼 보이므로 공개본 1세트만 남기지 않으면
    혼선이 납니다.
  - frontend/와 frontend_unified/는 둘 다 UI 진입점
    이라 공개용 app/ 설명서에서 어떤 것을 기본 실행
    경로로 둘지 정해야 합니다.
  - finalv2/, backend/, modules/, orchestrator/ 사
    이에 상대경로 import나 문서 경로 참조가 있으면
    app/ 하위 이동 후 깨질 수 있습니다.
  - docs/demo/와 실제 앱 UI 캡처 자산이 최종보고서/
    최종보고서_이미지/프로그램UI/에 흩어져 있어, 데
    모 설명만 옮기면 링크 자산 누락이 생길 수 있습
    니다.

  4. 이동 시 충돌 우려 파일

  - docs/가 이미 존재하므로, 현재 docs/*.md를 그대
    로 두면 새 공개 문서 docs/와 성격이 섞입니다.
  - README.md의 기존 링크가 ./최종보고서/...를 직접
    가리켜서, 이동 후 전부 깨집니다.
  - 최종보고서/연구노트/README.md와 루트 README.md
    는 이름이 같아 단순 평면 이동 시 충돌합니다.
  - 최종제출_보고서_연구노트/와 연구노트/는 거의 동
    일 파일을 중복 보유해, 둘 다 공개 쪽으로 보내면
    중복/불일치 문제가 생깁니다.
  - 최종보고서/제출본_자체시험성적서/자체시험성적서
    _완성본.*과 자체시험성적서_통합편철본.*은 둘 다
    최종처럼 보이므로 공개본 1세트만 남기지 않으면
    혼선이 납니다.
  - frontend/와 frontend_unified/는 둘 다 UI 진입점
    이라 공개용 app/ 설명서에서 어떤 것을 기본 실행
    경로로 둘지 정해야 합니다.
  - finalv2/, backend/, modules/, orchestrator/ 사
    이에 상대경로 import나 문서 경로 참조가 있으면
    app/ 하위 이동 후 깨질 수 있습니다.
